# 0.51 Changelog

## 0.51.0 - 2026-05-31 - evidence envelope foundation

### Added

- Added `canic-host::evidence_envelope` with the stable outer envelope model:
  `EvidenceEnvelopeV1`, `ExitClassV1`, `EvidenceTargetV1`,
  `PayloadSchemaRefV1`, `InputFingerprintV1`, `CommandProvenanceV1`,
  `EvidenceSummaryV1`, and `EvidenceMessageV1`.
- Added stable envelope JSON output for passive adoption reports:

  ```text
  canic fleet adoption report <fleet> --profile <profile> --format envelope-json
  ```

- `--format envelope-json` wraps the raw adoption payload with fleet/profile
  target identity, command provenance, source config/input fingerprints,
  experimental payload schema identity, payload hash, structured
  warnings/blockers/evidence gaps/conflicts, and an automation-friendly exit
  class.
- Added stable envelope JSON output for passive deployment checks:

  ```text
  canic deploy check <deployment> --format envelope-json
  ```

- `--format envelope-json` wraps the raw `DeploymentCheckV1` payload in
  `EvidenceEnvelopeV1` with deployment/fleet target identity, command
  provenance, config fingerprint metadata, internal payload schema identity,
  payload hash, structured safety warnings/blockers/evidence gaps/conflicts,
  and an automation-friendly exit class.
- Added the maintained architecture note
  `docs/architecture/evidence-envelopes.md`.

### Changed

- `canic fleet adoption report ... --format json` remains the raw experimental
  `AdoptionReportV1` payload. It is not silently wrapped.
- Existing deployment-check JSON remains raw `DeploymentCheckV1`.
- Added an explicit attached `payload_limit_probe` test fleet config so the
  PocketIC payload-limit fixture no longer depends on generated declared-only
  standalone config.
- Updated root-suite fixture configs to declare the current `[fleet]` and
  `[roles.<role>]` shape required by the 0.49 role lifecycle hard cut.
- Internal PocketIC Wasm artifact builders now invoke Cargo with `--locked` so
  release validation cannot silently drift `Cargo.lock`.

### Safety

- Envelope output does not make adoption reporting active. It does not mutate
  fleet config, package manifests, topology, deployments, controllers,
  artifacts, adoption state, or canister lifecycle, and it does not perform
  additional live discovery.
- Envelope output does not change the deployment-check mutation boundary. The
  command remains passive and does not install Wasm, update controllers, write
  deployment state, attach topology, or register artifacts.
