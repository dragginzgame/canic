# 0.52 Changelog

## 0.52.3 - 2026-05-31 - build provenance CI policy

### Added

- Added CI/GitOps policy guidance for stable build provenance:

  ```text
  docs/architecture/build-provenance-ci-policy.md
  ```

- The guide documents recommended checks over:
  - dirty source state;
  - `Cargo.lock` drift;
  - package metadata `fleet.role`;
  - raw Wasm vs gzip Wasm artifact hashes;
  - saved build-provenance linkage from adoption/deployment-check envelopes.

### Safety

- This is documentation-only. It does not add signing, CI locks, provider
  wrappers, artifact registry import, controller mutation, topology mutation,
  deployment install authority, or active adoption/import.

## 0.52.2 - 2026-05-31 - build provenance evidence inputs

### Added

- Adoption-report and deployment-check evidence envelopes can now fingerprint a
  saved build provenance envelope as supplied evidence:

  ```text
  canic fleet adoption report <fleet> --profile <profile> --format envelope-json --build-provenance <path>
  canic deploy check <deployment> --format envelope-json --build-provenance <path>
  ```

- The supplied file is recorded as a stable input fingerprint with payload
  schema:

  ```text
  canic.build_provenance.v1
  ```

- Adapted Canic memory-ledger diagnostics to the locked `ic-memory 0.7.0` API
  while preserving the public memory-ledger response version fields.

### Safety

- `--build-provenance` is accepted only with `--format envelope-json` on these
  report surfaces, so raw/text output does not gain a silent no-op evidence
  argument.
- The report commands only fingerprint the saved provenance file. They do not
  re-run builds, import artifacts, validate deployment truth from provenance,
  attach topology, mutate controllers, or turn provenance into authority.

## 0.52.1 - 2026-05-31 - build provenance emitter

### Added

- Added explicit build provenance output:

  ```text
  canic build <fleet> <role> --provenance <path>
  ```

- The provenance file is an `EvidenceEnvelopeV1` whose stable payload schema is:

  ```text
  canic.build_provenance.v1
  ```

- Added stable build provenance payload records:

  ```text
  SourceProvenanceV1
  CargoProvenanceV1
  ArtifactProvenanceV1
  BuildProvenanceV1
  ```

- Build provenance records Git source state when available, dirty-source
  status, Cargo lock/package manifest fingerprints, package metadata
  `fleet.role`, Cargo/Rust versions, build profile, target, and produced
  Wasm/gzip Wasm SHA-256 hashes.

### Safety

- Ordinary `canic build <fleet> <role>` behavior is unchanged and writes no
  provenance file.
- Provenance is written only after successful artifact generation.
- The provenance path is the only new requested output. The command does not
  mutate deployment truth, controllers, topology, `wasm_store`, artifact
  registries, adoption state, install state, or canister lifecycle.

## 0.52.0 - 2026-05-31 - source/build/artifact provenance design

### Added

- Added the proposed 0.52 source, build, and artifact provenance design:

  ```text
  docs/design/0.52-source-build-artifact-provenance/0.52-design.md
  ```

- The design keeps 0.51's `EvidenceEnvelopeV1` as the stable automation
  wrapper and proposes stable source, Cargo, artifact, and build provenance
  records for the build path:

  ```text
  SourceProvenanceV1
  CargoProvenanceV1
  ArtifactProvenanceV1
  BuildProvenanceV1
  ```

- The preferred first emitter is:

  ```text
  canic build <fleet> <role> --provenance <path>
  ```

- The provenance output is specified as an `EvidenceEnvelopeV1` containing a
  stable payload schema:

  ```text
  canic.build_provenance.v1
  ```

### Safety

- This is design-only. It does not add runtime behavior.
- The design explicitly defers signing, CI locks, project manifests, provider
  wrappers, artifact registry import, controller mutation, topology mutation,
  and deployment/install authority.
