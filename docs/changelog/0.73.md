# 0.73 Changelog

## 0.73.1 - 2026-06-26 - Host shared query plumbing

### Changed

- Root-subnet evidence checks now use the `ic-query` 0.5.11 shared library
  with the `host` feature instead of spawning the `icq` CLI. The required
  developer-tool pin and repo-managed `icq` installer were removed, while
  maintainers who want standalone NNS inspection can install the upstream
  `ic-query-cli` package directly.

- Active Canic docs and helper scripts no longer describe `icq` as a Canic
  workflow dependency. Historical changelog and audit references remain
  unchanged.

## 0.73.0 - 2026-06-26 - Post-hardening recovery polish

### 🩹 Fixed

- Delegated-token prepare now maps missing/stale issuer active auth-proof state
  to `AuthMaterialStale` and expired issuer active auth-proof state to
  `AuthProofExpired` instead of returning a generic internal error. This
  preserves the delegated-token hardening while giving operators and callers
  actionable recovery signals.

- Wasm-store publication client calls and management `clear_chunk_store` now use
  bounded waits, matching the rest of the lifecycle management surface and
  avoiding indefinite pending calls during stop or upgrade windows.

- Runtime-managed wasm-store inventory upserts now distinguish inserted,
  already-present, and binding/principal conflict outcomes. Bootstrap,
  inventory sync, and store creation convert conflicts into recoverable workflow
  errors instead of trapping on inventory drift.

### 🔧 Changed

- Updated the maintainer ICP CLI toolchain pin and install guidance to
  `icp-cli 1.0.1`.

### 🧪 Testing

- Added focused coverage for delegated-token auth-proof error mapping and
  wasm-store inventory idempotency/conflict reporting, and retained the
  publication client endpoint-table guard for root-owned wasm-store calls.
