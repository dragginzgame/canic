# 0.23.x Detailed Changelog

Technical release notes for the `0.23.x` minor line.

## 0.23.2 - 2026-04-04 - Wasm Delta Tracking

### 📝 Summary

- `0.23.2` removes the checked-in wasm budget layer from the recurring footprint audit so `0.23` follow-through stays focused on making artifacts smaller and recording dated deltas each time.

### 🔧 Changed

- [wasm-footprint.md](/home/adam/projects/canic/docs/audits/recurring/system/wasm-footprint.md) no longer treats checked-in per-canister budgets as part of the recurring audit contract.
- [wasm-audit-report.sh](/home/adam/projects/canic/scripts/ci/wasm-audit-report.sh) now emits current size snapshots, shrink deltas, and baseline deltas without any budget bytes, budget statuses, or over-budget warnings.
- [wasm-budgets.tsv](/home/adam/projects/canic/docs/audits/recurring/system/wasm-budgets.tsv) is removed, so the live wasm-size story is now pure measured output plus comparable drift.

### 📊 Audit

- The current wasm audit now records the useful question directly: what each canister weighs today, how much `dfx` shrink changed it, and how that compares to the selected dated baseline.
- `root` remains the main control-plane outlier to reduce, but it is now tracked as a size/drift problem instead of a threshold violation.

### 🧪 Testing

- `bash -n scripts/ci/wasm-audit-report.sh`
- `bash scripts/ci/wasm-audit-report.sh`

## 0.23.0 - 2026-04-03 - Deferred Follow-Through

### 📝 Summary

- `0.23.0` starts the follow-through line with checked-in wasm budgets, a dated wasm-footprint rerun, a clearer split between the public `canic-testkit` PocketIC wrapper and the unpublished `canic-testing-internal` self-test crate, a removal of the unused `*cycles_accept` compatibility endpoint so management-canister cycle deposit stays the only Canic-managed funding path, and a fix for the curlable setup script so its default `canic-installer` version stays aligned with the current Canic release.

### 🔧 Changed

- [wasm-budgets.tsv](/home/adam/projects/canic/docs/audits/recurring/system/wasm-budgets.tsv) now defines the first checked-in shrunk-wasm budgets for the current reference roles.
- [wasm-footprint.md](/home/adam/projects/canic/docs/audits/recurring/system/wasm-footprint.md) now treats the budget surface as part of the recurring audit contract, so each run records the current shrunk-wasm budget bytes, budget deltas, and whether each canister is still within its checked-in budget.
- [wasm-audit-report.sh](/home/adam/projects/canic/scripts/ci/wasm-audit-report.sh) now loads the checked-in budget table and emits budget fields in the Markdown summary, per-canister detail pages, machine-readable JSON artifacts, and the top-level verification/checklist output.
- [pic_role_attestation.rs](/home/adam/projects/canic/crates/canic-core/tests/pic_role_attestation.rs) now gets its cached `root + signer` and `root + signer + verifier` fixture installs from [attestation.rs](/home/adam/projects/canic/crates/canic-testing-internal/src/pic/attestation.rs) instead of carrying a second suite-local snapshot capture/restore implementation.
- [mod.rs](/home/adam/projects/canic/crates/canic-testkit/src/pic/mod.rs) keeps the generic PocketIC primitives that downstream projects can reasonably reuse, including cached baselines, root-role lookup, and `canic_ready` polling.
- [canic-testing-internal](/home/adam/projects/canic/crates/canic-testing-internal/Cargo.toml) is now the unpublished home for Canic’s own root-baseline and auth-fixture harness code, and [canic-testkit](/home/adam/projects/canic/crates/canic-testkit/Cargo.toml) is trimmed back to the public PocketIC-oriented helper surface that downstream projects are actually meant to use.
- The old `ic_cycles_accept` / `canic_ic_cycles_accept` compatibility surface is removed from [endpoints.rs](/home/adam/projects/canic/crates/canic/src/macros/endpoints.rs), [protocol.rs](/home/adam/projects/canic/crates/canic/src/protocol.rs), and [wasm_store.did](/home/adam/projects/canic/crates/canic-wasm-store/wasm_store.did), so Canic’s own managed cycles path is unambiguously the management-canister `deposit_cycles` flow.
- [install.sh](/home/adam/projects/canic/scripts/install.sh) now defaults to `canic-installer 0.23.0` instead of the stale `0.22.10` pin, and [install_script_surface.rs](/home/adam/projects/canic/crates/canic/tests/install_script_surface.rs) now fails CI if that default ever drifts from `workspace.package.version` again.

### 📊 Audit

- The first dated post-cleanup wasm-footprint rerun is now recorded at [wasm-footprint.md](/home/adam/projects/canic/docs/audits/reports/2026-04/2026-04-03/wasm-footprint.md).
- The new budget surface held for every reference leaf canister in scope: `app`, `minimal`, `user_hub`, `user_shard`, `scale_hub`, `scale`, and `test` all stayed under their checked-in shrunk-wasm budgets in that rerun.
- `root` remains the one current budget outlier at `+616745` bytes over its checked-in shrunk-wasm budget, so the next `0.23` wasm-size decision is whether to reduce `root` or explicitly move its budget with rationale.

### 📚 Documentation

- [CHANGELOG.md](/home/adam/projects/canic/CHANGELOG.md) now opens the `0.23.x` minor line.
- [0.23-design.md](/home/adam/projects/canic/docs/design/0.23-deferred-follow-through/0.23-design.md) now records the deferred follow-through work that was intentionally left out of the closed `0.22` line.
- [0.22-status.md](/home/adam/projects/canic/docs/design/0.22-audits-wasm-perf/0.22-status.md) now treats the `0.22` line as concluded and points its remaining optional work at the new `0.23` design lane.
- [README.md](/home/adam/projects/canic/README.md) now distinguishes the public `canic-testkit` crate from the unpublished `canic-testing-internal` self-test crate in the workspace layout.
- [README.md](/home/adam/projects/canic/README.md) and [README.md](/home/adam/projects/canic/crates/canic-installer/README.md) now point the tagged setup example at `v0.23.0` instead of the old `v0.22.10` curl path.

### 🧭 Migration Notes

- `0.23.0` is still a narrow follow-through release, but it does change the recurring wasm audit surface by introducing checked-in shrunk-wasm budgets and budget-aware report output.
- The first dated wasm-footprint rerun is now in place, so the next wasm-size work should be budget discipline and deliberate follow-up on the `root` outlier rather than another untracked baseline pass.
