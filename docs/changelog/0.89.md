# Canic 0.89 Changelog

## [0.89.0] - 2026-07-13 - Typed Deployment Evidence

### 📝 Summary

0.89.0 makes deployment install state and local replica-query failures typed
and inspectable until each command finishes its own exit, hint, and finding
decisions. Filesystem, JSON, validation, network-mismatch, and replica evidence
no longer disappears into boxed errors or strings before classification.

Successful command output, JSON, exit codes, operator guidance, install-state
paths and schema, serialized bytes, and stable state are unchanged.

### ⚠️ Breaking

- Public host install-state APIs now return `InstallStateError` instead of
  boxed dynamic errors, and `InstalledDeploymentError` carries concrete
  `InstallStateError` and `ReplicaQueryError` values instead of strings.
- Downstream Rust callers that match or name the previous error payloads must
  adopt the concrete types. CLI usage, configuration, persisted data, and
  deployment state require no migration.

### 🔧 Changed

- Install-state failures distinguish invalid names, network mismatch, ICP-root
  resolution, reads, JSON decoding and encoding, and durable writes while
  retaining the relevant path and original source.
- Installed-deployment resolution preserves those sources through all command
  projections; command-specific missing-install and lost-root actions remain
  locally owned.
- ICP-root discovery exposes its actual I/O contract instead of erasing it
  behind a boxed error.

### 🩹 Fixed

- Filesystem failures other than a genuine missing state file are reported as
  read failures rather than being mistaken for an absent deployment.
- Local replica transport, decoding, and rejection evidence remains available
  to callers after installed-deployment classification.

### 🧹 Cleanup

- Four unused build and development dependencies are removed. The facade's
  direct `ic-cdk` test dependency remains because endpoint proc macros generate
  a real reference that static dependency scanning cannot observe.
- Upgrade, recycle, and cycles request adapters are private to their dispatch
  owner, and the two unused adapter exports are removed.
- No global error hierarchy, generic command projector, dependency upgrade,
  compatibility path, or broad visibility cleanup is introduced.

### 📊 Audit

- All three bounded post-0.88 findings are closed: six boxed install-state
  signatures and two string-valued source variants are hard-cut, all ten
  command projectors retain typed evidence, and the audited dependency and RPC
  surfaces match their proven consumers.
- Cargo Machete reports no unused dependency after compilation distinguishes
  the four dead declarations from the generated proc-macro consumer.

### 🧪 Testing

- Focused coverage now proves install-state path/source retention, structured
  replica rejection, command guidance and classification, and private RPC
  request shape, metadata, and response-variant behavior.
- Targeted affected-package compilation and warning-denied Clippy, formatting,
  layering, changelog governance, dependency, and diff checks pass. Full
  workspace, PocketIC, deployment, and release suites were not run under the
  targeted testing policy.
