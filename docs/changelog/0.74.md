# 0.74 Changelog

## 0.74.1 - 2026-06-27 - Renewal status and retained runtime proof

### 📝 Summary

- Adds the first root-managed renewal status CLI surface so operators can
  inspect one issuer's root-owned renewal template, state, and active attempt
  without running the renewal bridge.

- Retains the full scheduled-renewal PocketIC loop by ensuring the test issues
  delegated tokens with fresh replay request ids after simulated time advances.
  This keeps replay protection intact while proving repeated renewal,
  install, and issuer-local token issuance.

### ➕ Added

- `canic auth renewal status <deployment> --issuer <principal>` now queries the
  root `canic_root_issuer_renewal_status` endpoint for one issuer.

- `--json` emits a stable `auth_renewal_status` payload with schema version,
  deployment/network/root target, issuer principal, high-level status, template
  presence/enabled state, last root outcome, consecutive failures, active
  attempt status, batch id, failure reason, and timing fields.

- Plain text output renders a compact operator summary for template status,
  last outcome, consecutive failures, installed expiry, refresh threshold, next
  attempt time, and active attempt state.

Command surface:

```bash
canic auth renewal status <deployment> --issuer <principal>
canic auth renewal status <deployment> --issuer <principal> --json
```

### 🩹 Fixed

- The scheduled-renewal PocketIC scenario now uses a nonce-aware delegated-token
  test helper for token issuance after simulated time advances. The previous
  deterministic request id was correct for a single issue path but became an
  expired replay receipt when the long scenario verified the second renewed
  proof.

### 🧪 Testing

- Added CLI parsing, scripted transport, JSON parsing, invalid-principal, and
  status endpoint routing coverage for `canic auth renewal status`.

- Re-ran the full scheduled-renewal PocketIC scenario outside the sandbox so
  PocketIC could bind its local server.

## 0.74.0 - 2026-06-27 - Root-managed delegation renewal

### 📝 Summary

- Starts root-managed renewal for delegated-auth issuer proofs. Root now owns
  renewal intent, scheduled issuer attempts, constrained proof retrieval, and
  install validation while preserving the existing direct-query certificate
  requirement.

- Keeps the bridge narrow. The bridge can courier only root-scheduled renewal
  work and cannot choose arbitrary issuers, audiences, grants, TTLs, or manual
  proof references.

### ➕ Added

- Root issuer renewal templates, issuer renewal status, scheduled renewal
  attempt/batch state, and stable storage records for root-managed renewal.

- A scheduled renewal work endpoint, a scheduled proof-retrieval endpoint, and
  install handling that records per-issuer renewal outcomes for scheduled
  proofs.

- Renewal provisioner registration and authorization for scheduled renewal
  work only.

- Root renewal sweep startup, expired-attempt cleanup, retry/backoff state, and
  bounded delegated-auth metrics plus auth logs for sweeps, retrieval,
  install, and provisioner operations.

- `canic auth renewal run-once <deployment>` as the first CLI bridge command,
  with JSON output available for automation.

- A 0.74 design note documenting the root-managed renewal model, the
  direct-query bridge requirement, threat-model invariants, and deferred
  issuer-drift/medic follow-up.

Command surface:

```bash
canic auth renewal run-once <deployment>
canic auth renewal run-once <deployment> --json
```

### 🔧 Changed

- Root delegated-auth protocol exports now include the renewal template,
  status, scheduled work, scheduled proof retrieval, provisioner, and batch
  install DTOs needed by the renewal bridge.

- Root install preflight now validates scheduled batch identity and pending
  proof metadata before accepting scheduled renewal proof installs.

- Replay-policy coverage now classifies the renewal template and provisioner
  upserts as snapshot-convergent root auth configuration endpoints.

### 🧪 Testing

- Added unit coverage for renewal template policy checks, state conversion,
  scheduled attempt lifecycle, expired-attempt cleanup, provisioner
  authorization, renewal metrics, protocol surface shape, and CLI bridge
  parsing/execution.

- Added a PocketIC scheduled-renewal bridge scenario and retained compile
  coverage for it. At 0.74.0 closeout, a clean local runtime execution had not
  yet been retained because the sandboxed PocketIC server could not bind to
  `127.0.0.1:0` in this workspace.
