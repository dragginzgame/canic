## 0.29.1 - 2026-04-28 - Delegated Auth V2 Primitives

### 📝 Summary

- `0.29.1` expands Delegated Auth V2 from the initial design and canonical data model into pure issuance, minting, verification, and trust-policy primitives.

### ➕ Added

- Added V2 audience and certificate policy helpers for role/principal subset checks, verifier membership, single-role hash binding, root principal binding, TTL enforcement, and shard public-key hash validation.
- Added V2 root-key trust-anchor resolution, including trusted local root-key lookup and embedded root-key certificate fallback through `RootKeyAuthorityV2`.
- Added a pure V2 verifier path that validates self-contained tokens without local proof lookup.
- Added pure root-side V2 delegation proof issuance with canonical cert-hash signing, root-key embedding, role-hash binding, and TTL policy enforcement.
- Added pure shard-side V2 token minting with embedded proof preservation, cert-hash binding, audience/scope attenuation, TTL checks, and injected shard signing.

### 📚 Documentation

- Updated the implementation status tracker so pure V2 issue, mint, and verify primitives are marked complete while runtime root issuance, signer minting, verifier guards, migration, and V1 proof-sync removal remain explicit planned work.

- Documented that automated agents may inspect Git state but must not run `git add`, `git commit`, or `git push`; staging, committing, and pushing are human-owned actions.

### 🧪 Testing

- Added coverage for claims-level canonical audience/scope normalization, exact token TTL boundary handling, `Roles` claims under `RolesOrPrincipals` certificates, nonce behavior, and signed scope tampering.

## 0.29.0 - 2026-04-28 - Delegated Auth V2 Design

### 📝 Summary

- `0.29.0` starts the Delegated Auth V2 line with a hard-cut design for self-validating delegated tokens that do not depend on verifier-local proof caches.

### ➕ Added

- Added V2 auth wire types for `DelegationCertV2`, `DelegationProofV2`, `DelegatedTokenClaimsV2`, `DelegatedTokenV2`, root key trust material, shard key binding, and role/principal audiences.
- Added `CanicAuthCanonicalV2` hashing for certificates, token claims, root-key certificate payloads, and role hashes with deterministic sorting, deduplication, domain separation, and ASCII role/scope validation.

### 📚 Documentation

- Added the Delegated Auth V2 design:
  [docs/design/0.29-delegated-auth-v2/0.29-design.md](../design/0.29-delegated-auth-v2/0.29-design.md)
- Added the matching implementation status tracker:
  [docs/design/0.29-delegated-auth-v2/0.29-status.md](../design/0.29-delegated-auth-v2/0.29-status.md)

### ⚠️ Breaking

- Defines V2 as a hard-cut replacement for proof-cache based delegated auth. The target model removes verifier proof replication, proof fanout, topology catch-up, and proof-cache equality from the authentication correctness path.

### 🧭 Migration Notes

- The planned rollout order is verifier dual validation first, then V2 root issuance, then V2 signer token minting, then V1 removal after the maximum V1 TTL window.
