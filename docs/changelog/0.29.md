## 0.29.2 - 2026-04-29 - Delegated Auth V2 Hard Cut

### 📝 Summary

- `0.29.2` completes the hard break from proof-cache based delegated auth to self-validating V2 tokens. V2 verification no longer depends on verifier-local proof state, proof fanout, or topology-time catch-up.

### 🔧 Changed

- Changed endpoint guards to V2-only delegated-token decoding. Authenticated endpoints now accept `DelegatedTokenV2` directly or use `Reserved` while the guard decodes the first argument as V2.
- Changed signer lifecycle prewarm to warm local shard signing key material only. Startup no longer requests or stores delegation proofs.
- Removed `auth.delegated_tokens.proof_cache` from config and bootstrap rendering because proof-cache sizing is no longer part of delegated-auth correctness.

### 🗑️ Removed

- Removed V1 delegated-token DTOs, root `IssueDelegation` capability routing, proof-cache stable storage, verifier proof-install endpoints, proof-fanout metrics, admin repair DTOs, and V1-only PocketIC suites.
- Removed creation-time verifier proof catch-up from topology state propagation. New canisters no longer receive cached proofs during placement/state propagation.
- Removed old local/dev proof-material endpoints and signer-proof status helpers from the demo/test canisters.

### ⚠️ Breaking

- V1 delegated tokens are no longer accepted. Callers must mint and send `DelegatedTokenV2`.
- The old proof distribution contract is gone: there is no verifier-local proof presence check, no proof-cache tuning config, no creation-time catch-up, and no repair fanout endpoint.

### 📚 Documentation

- Updated the implementation status tracker so V2-only guards, signer lifecycle prewarm, root-owned TTL policy, topology catch-up removal, and V1 API/proof-fanout removal are marked complete.

## 0.29.1 - 2026-04-28 - Delegated Auth V2 Primitives

### 📝 Summary

- `0.29.1` expands Delegated Auth V2 from the initial design and canonical data model into pure issuance, minting, verification, trust-policy primitives, internal root signing, internal shard signing, internal verifier validation, explicit V2 API helpers, an explicit root V2 delegation endpoint, signer-facing V2 mint helpers, V2 root-key pull-on-verify, dual V1/V2 guard validation, dual token endpoint surfaces, V2 signer lifecycle prewarm, root-owned V2 TTL policy enforcement, and the first proof-sync removal from topology propagation.

### ➕ Added

- Added V2 audience and certificate policy helpers for role/principal subset checks, verifier membership, single-role hash binding, root principal binding, TTL enforcement, and shard public-key hash validation.
- Added V2 root-key trust-anchor resolution, including trusted local root-key lookup and embedded root-key certificate fallback through `RootKeyAuthorityV2`.
- Added a pure V2 verifier path that validates self-contained tokens without local proof lookup.
- Added pure root-side V2 delegation proof issuance with canonical cert-hash signing, root-key embedding, role-hash binding, and TTL policy enforcement.
- Added an internal root-side V2 signing hook that prepares the canonical V2 certificate, binds the configured IC threshold-ECDSA key and shard derivation path, signs the prepared cert hash, and returns a self-contained `DelegationProofV2` without verifier fanout.
- Added pure shard-side V2 token minting with embedded proof preservation, cert-hash binding, audience/scope attenuation, TTL checks, and injected shard signing.
- Added an internal shard-side V2 signing hook that prepares canonical V2 token claims, signs the prepared claims hash with the local shard threshold-ECDSA derivation path, and returns a self-contained `DelegatedTokenV2`.
- Added an internal verifier-side V2 validation hook that checks local shard key binding, resolves the local root trust anchor from the key cache, applies local role membership, and delegates cryptographic validation to the pure V2 verifier without proof-cache lookup.
- Added explicit `DelegationApi` V2 helpers for local root proof issuance, signer token issuance from an explicit V2 proof, and verifier validation without changing the existing V1 endpoint behavior.
- Added the explicit root `canic_request_delegation_v2` endpoint and matching protocol constant so callers can request self-contained `DelegationProofV2` values without changing the V1 delegation endpoint.
- Added a remote `DelegationApi::request_delegation_v2` helper for signer-to-root V2 proof requests.
- Added `DelegatedTokenMintRequestV2` and `DelegationApi::mint_token_v2`, which request a V2 root proof and immediately mint a self-contained V2 token without using local proof-cache state.
- Added local/dev V2 signer endpoints on the user-shard demo and delegation signer stub for exercising the one-call V2 mint path.
- Added V2 verifier root-key pull-on-verify. When a V2 token reaches a guarded endpoint and the verifier has no matching root key cached, Canic fetches the expected root threshold-ECDSA public key by configured key name and root derivation path, then rejects the token if the fetched key hash does not match the certificate.
- Added dual V1/V2 delegated-token decoding in endpoint guards. V2 guard validation uses the self-contained verifier path and does not look up local delegation proofs; V1 guard validation remains unchanged for the migration window.
- Added macro support for `DelegatedTokenV2` and Candid `Reserved` as authenticated endpoint first-argument types, then moved repo-local protected token endpoints to `Reserved` so V1 and V2 token payloads can share the same guarded surface.

### 🔧 Changed

- Moved V2 certificate/token TTL policy ownership fully to root-side config. `DelegationProofIssueRequestV2` now carries the requested certificate TTL only; root derives policy ceilings from `auth.delegated_tokens.max_ttl_secs`.
- Changed signer lifecycle prewarm to warm local shard signing key material only. Startup no longer requests or stores a V1 delegation proof; legacy V1 token issuance can still provision V1 proof state on demand during migration.

### 🗑️ Removed

- Removed creation-time verifier proof catch-up from topology state propagation. New canisters no longer receive root-cached V1 proofs as part of placement/state propagation; V2 token acceptance is independent of creation order.

### 📚 Documentation

- Updated the implementation status tracker so pure V2 issue, mint, verify, internal root signing, internal shard signing, internal verifier validation, explicit V2 API helpers, the root V2 endpoint, remote proof request helper, signer-facing mint helper, local/dev signer endpoints, V2 root-key pull-on-verify, dual V1/V2 guard validation, repo-local endpoint surface migration, V2 signer lifecycle prewarm, root-owned TTL policy, and topology catch-up removal are marked complete while legacy V1 API/issuance fanout removal remains explicit planned work.

## 0.29.0 - 2026-04-28 - Delegated Auth V2 Design

### 📝 Summary

- `0.29.0` starts the Delegated Auth V2 line with a hard-cut design for self-validating delegated tokens that do not depend on verifier-local proof caches.

### ➕ Added

- Added V2 auth wire types for `DelegationCertV2`, `DelegationProofV2`, `DelegatedTokenClaimsV2`, `DelegatedTokenV2`, root key trust material, shard key binding, and role/principal audiences.
- Added `CanicAuthCanonicalV2` hashing for certificates, token claims, root-key certificate payloads, and role hashes with deterministic sorting, deduplication, domain separation, and ASCII role/scope validation.

### 📚 Documentation

- Added the Delegated Auth V2 design:
  [docs/design/0.29-delegated-auth-v2/0.29-design.md](../design/0.29-delegated-auth-v2/0.29-design.md)
- Added the matching implementation status tracker:
  [docs/design/0.29-delegated-auth-v2/0.29-status.md](../design/0.29-delegated-auth-v2/0.29-status.md)

### ⚠️ Breaking

- Defines V2 as a hard-cut replacement for proof-cache based delegated auth. The target model removes verifier proof replication, proof fanout, topology catch-up, and proof-cache equality from the authentication correctness path.

### 🧭 Migration Notes

- `0.29.2` completes the hard cut; there is no V1 compatibility window in this line.
