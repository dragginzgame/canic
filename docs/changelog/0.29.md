## 0.29.4 - 2026-04-29 - Delegated Auth Cohesion

### 📝 Summary

- `0.29.4` tightens the final delegated-auth shape so the implementation, trust model, and 0.29 design notes describe one coherent hard-cut system.

### 🔧 Changed

- Removed the misleading verifier-side `expected_shard_key_hash` input from the pure delegated-token verifier. Verification now reflects the actual hard-cut model: root signs the embedded shard public key, the certificate carries the public-key hash, and runtime checks deterministic `ShardKeyBinding` metadata.
- Updated the `0.29` delegated-auth design doc so shard-key validation is described as signed shard-key binding, not verifier-side shard-key derivation.

### 🩹 Fixed

- Fixed a trust-model ambiguity where runtime verification passed `cert.shard_key_hash` back as the expected shard key hash. That check was tautological and made the verifier look like it had shard-key authority it does not own.

### 📊 Audit

- Rechecked the delegated-auth surface for leftover proof-cache, V2 naming, root-key fallback, and verifier proof-miss concepts.
- Rechecked stable key-cache identity, role-attestation key refresh, canonical encoding, TTL boundaries, and session expiry behavior against the hard-cut design.

- Moved the self-validating auth implementation out of the temporary versioned module path and collapsed internal helper names to the unsuffixed delegated-auth model.

- Removed the remaining unused proof-cache validation errors from delegated-token ops.

### ⚠️ Breaking

## 0.29.2 - 2026-04-29 - Delegated Auth Hard Cut

### 📝 Summary

- `0.29.2` completes the hard break from proof-cache based delegated auth to self-validating delegated tokens. Delegated auth verification no longer depends on verifier-local proof state, proof fanout, or topology-time catch-up.

### 🔧 Changed

- Changed endpoint guards to current-shape delegated-token decoding. Authenticated endpoints now accept `DelegatedToken` directly or use `Reserved` while the guard decodes the first argument as delegated auth.
- Renamed the verifier lifecycle cfg from proof-acceptance wording to `canic_auth_attestation_cache`, matching the attestation-cache behavior it actually gates.
- Changed signer lifecycle prewarm to warm local shard signing key material only. Startup no longer requests or stores delegation proofs.
- Removed `auth.delegated_tokens.proof_cache` from config and bootstrap rendering because proof-cache sizing is no longer part of delegated-auth correctness.

### 🗑️ Removed

- Removed V1 delegated-token DTOs, root `IssueDelegation` capability routing, proof-cache stable storage, verifier proof-install endpoints, proof-fanout metrics, admin repair DTOs, and V1-only PocketIC suites.
- Removed creation-time verifier proof catch-up from topology state propagation. New canisters no longer receive cached proofs during placement/state propagation.
- Removed old local/dev proof-material endpoints and signer-proof status helpers from the demo/test canisters.

### ⚠️ Breaking

- V1 delegated tokens are no longer accepted. Callers must mint and send `DelegatedToken`.
- The old proof distribution contract is gone: there is no verifier-local proof presence check, no proof-cache tuning config, no creation-time catch-up, and no repair fanout endpoint.

### 📚 Documentation

- Updated the implementation status tracker so current-shape guards, signer lifecycle prewarm, root-owned TTL policy, topology catch-up removal, and V1 API/proof-fanout removal are marked complete.

## 0.29.1 - 2026-04-28 - Delegated Auth Primitives

### 📝 Summary

- `0.29.1` expands Delegated Auth from the initial design and canonical data model into pure issuance, minting, verification, trust-policy primitives, internal root signing, internal shard signing, internal verifier validation, explicit API helpers, an explicit root delegation endpoint, signer-facing mint helpers, root-key pull-on-verify, current-shape guard validation, delegated signer lifecycle prewarm, root-owned TTL policy enforcement, and the first proof-sync removal from topology propagation.

### ➕ Added

- Added delegated auth audience and certificate policy helpers for role/principal subset checks, verifier membership, single-role hash binding, root principal binding, TTL enforcement, and shard public-key hash validation.
- Added root-key trust-anchor resolution through one explicit trusted root public key.
- Added a pure delegated verifier path that validates self-contained tokens without local proof lookup.
- Added pure root-side delegated auth delegation proof issuance with canonical cert-hash signing, role-hash binding, and TTL policy enforcement.
- Added an internal root-side delegated auth signing hook that prepares the canonical certificate, binds the configured IC threshold-ECDSA key and shard derivation path, signs the prepared cert hash, and returns a self-contained `DelegationProof` without verifier fanout.
- Added pure shard-side delegated token minting with embedded proof preservation, cert-hash binding, audience/scope attenuation, TTL checks, and injected shard signing.
- Added an internal shard-side delegated auth signing hook that prepares canonical delegated token claims, signs the prepared claims hash with the local shard threshold-ECDSA derivation path, and returns a self-contained `DelegatedToken`.
- Added an internal verifier-side delegated auth validation hook that checks local shard key binding, resolves the local root trust anchor from the key cache, applies local role membership, and delegates cryptographic validation to the pure delegated verifier without proof-cache lookup.
- Added explicit `DelegationApi` helpers for local root proof issuance, signer token issuance from an explicit proof, and verifier validation.
- Added the explicit root `canic_request_delegation` endpoint and matching protocol constant so callers can request self-contained `DelegationProof` values.
- Added a remote `DelegationApi::request_delegation` helper for signer-to-root proof requests.
- Added `DelegatedTokenMintRequest` and `DelegationApi::mint_token`, which request a root proof and immediately mint a self-contained delegated token without using local proof-cache state.
- Added local/dev delegated signer endpoints on the user-shard demo and delegation signer stub for exercising the one-call delegated auth mint path.
- Added delegated verifier root-key pull-on-verify. When a delegated token reaches a guarded endpoint and the verifier has no matching root key cached, Canic fetches the expected root threshold-ECDSA public key by configured key name and root derivation path, then rejects the token if the fetched key hash does not match the certificate.
- Added current-shape delegated-token decoding in endpoint guards. Guard validation uses the self-contained verifier path and does not look up local delegation proofs.
- Added macro support for `DelegatedToken` and Candid `Reserved` as authenticated endpoint first-argument types, then moved repo-local protected token endpoints to `Reserved` so guarded methods can hide token wire details from endpoint signatures.

### 🔧 Changed

- Moved certificate/token TTL policy ownership fully to root-side config. `DelegationProofIssueRequest` now carries the requested certificate TTL only; root derives policy ceilings from `auth.delegated_tokens.max_ttl_secs`.
- Changed signer lifecycle prewarm to warm local shard signing key material only. Startup no longer requests or stores a delegation proof.

### 🗑️ Removed

- Removed creation-time verifier proof catch-up from topology state propagation. New canisters no longer receive root-cached V1 proofs as part of placement/state propagation; delegated token acceptance is independent of creation order.

### 📚 Documentation

- Updated the implementation status tracker so pure delegated auth issue, mint, verify, internal root signing, internal shard signing, internal verifier validation, explicit API helpers, the root delegation endpoint, remote proof request helper, signer-facing mint helper, local/dev signer endpoints, root-key pull-on-verify, current-shape guard validation, repo-local endpoint surface migration, delegated signer lifecycle prewarm, root-owned TTL policy, and topology catch-up removal are marked complete.

### 🥾 Governance

- Documented that automated agents may inspect Git state but must not run `git add`, `git commit`, or `git push`; staging, committing, and pushing are human-owned actions.

## 0.29.0 - 2026-04-28 - Delegated Auth Design

### 📝 Summary

- `0.29.0` starts the Delegated Auth line with a hard-cut design for self-validating delegated tokens that do not depend on verifier-local proof caches.

### ➕ Added

- Added auth wire types for `DelegationCert`, `DelegationProof`, `DelegatedTokenClaims`, `DelegatedToken`, root key trust material, shard key binding, and role/principal audiences.

### 📚 Documentation

- Added the Delegated Auth design:
  [docs/design/0.29-delegated-auth/0.29-design.md](../design/0.29-delegated-auth/0.29-design.md)

- Defines delegated auth as a hard-cut replacement for proof-cache based delegated auth. The target model removes verifier proof replication, proof fanout, topology catch-up, and proof-cache equality from the authentication correctness path.
