## 0.29.0 - 2026-04-28 - Delegated Auth V2 Design

### 📝 Summary

- `0.29.0` starts the Delegated Auth V2 line with a hard-cut design for self-validating delegated tokens that do not depend on verifier-local proof caches.

### ➕ Added

- Added V2 auth wire types for `DelegationCertV2`, `DelegationProofV2`, `DelegatedTokenClaimsV2`, `DelegatedTokenV2`, root key trust material, shard key binding, and role/principal audiences.
- Added `CanicAuthCanonicalV2` hashing for certificates, token claims, root-key certificate payloads, and role hashes with deterministic sorting, deduplication, domain separation, and ASCII role/scope validation.
- Added V2 audience and certificate policy helpers for role/principal subset checks, verifier membership, single-role hash binding, root principal binding, TTL enforcement, and shard public-key hash validation.

### ⚠️ Breaking

- Defines V2 as a hard-cut replacement for proof-cache based delegated auth. The target model removes verifier proof replication, proof fanout, topology catch-up, and proof-cache equality from the authentication correctness path.

### 🧭 Migration Notes

- The planned rollout order is verifier dual validation first, then V2 root issuance, then V2 signer token minting, then V1 removal after the maximum V1 TTL window.
