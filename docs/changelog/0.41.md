## 0.41.17 - 2026-05-23 - receipt and verifier duplicate hardening

### Added

- Receipt-aware deployment diffs now detect duplicate phase receipts.
  Conflicting postcondition evidence hard-fails resume, while exact duplicate
  phase receipts warn without changing the resumable phase set.
- Receipt-aware deployment diffs now detect duplicate role-phase receipts.
  Conflicting role-scoped phase evidence hard-fails resume, while exact
  duplicate role-phase receipts warn without changing the resumable phase set.
- Deployment diffs now detect duplicate planned verifier role-epoch
  expectations. Conflicting minimum epochs hard-fail, while exact duplicate
  planned epoch requirements warn.

### Changed

- Duplicate receipt phases are skipped from the resumable phase set when their
  evidence conflicts, so resume safety reports point at the conflicting receipt
  evidence instead of treating the phase as resumable.
- Exact duplicate planned verifier role-epoch expectations are compared only
  once after their warning is emitted.

## 0.41.16 - 2026-05-23 - duplicate evidence hardening

### Added

- Deployment diffs now detect duplicate observed artifact evidence by role.
  Conflicting artifact observations hard-fail, while exact duplicate artifact
  observations warn instead of being collapsed by role-indexed lookup.
- Deployment diffs now detect duplicate verifier role-epoch observations.
  Conflicting epoch evidence hard-fails, while exact duplicate epoch evidence
  warns instead of being collapsed by role-indexed lookup.
- Deployment diffs now detect duplicate planned artifact entries by role.
  Conflicting planned artifact evidence hard-fails, while exact duplicate
  planned entries warn.
- Deployment diffs now detect duplicate planned canister declarations.
  Conflicting role-to-ID assignments hard-fail, while exact duplicate planned
  canister entries warn.
- Deployment diffs now detect duplicate planned pool declarations. Conflicting
  pool identity-to-ID assignments hard-fail, while exact duplicate planned pool
  entries warn.

### Changed

- Exact duplicate planned artifacts, canisters, and pool entries are compared
  only once after their warning is emitted.
- Roles with conflicting observed artifact or verifier epoch evidence are
  skipped from the normal missing/stale comparison path so reports point at the
  underlying ambiguity instead of a secondary symptom.

## 0.41.15 - 2026-05-23 - topology contradiction hardening

### Added

- Deployment diffs now hard-fail when a concrete expected canister ID is
  observed with a different role assignment.
- Deployment diffs now detect duplicate observed canister IDs. Conflicting role
  assignments hard-fail, while exact duplicate observations warn as suspicious
  inventory evidence.
- Deployment diffs now apply the same duplicate-ID guard to pool canisters.
  Conflicting pool identities for one canister ID hard-fail, while exact
  duplicate pool observations warn.
- Deployment diffs now hard-fail when a canister appears in both non-pool and
  pool observations with conflicting role identities.
- Deployment diffs now hard-fail when an expected non-pool role has no concrete
  planned canister ID and multiple observed canisters claim that role.

- Observed pool canister control classes now reuse enriched child live-status
  evidence, so pool drift reflects live controller state when it was observed.
- Controller drift checks now treat `subnet_registry+icp_canister_status`
  observations as live status evidence instead of registry-only uncertainty.
- Installed module-hash comparison now targets the concrete planned canister ID
  when available, and hard-fails ambiguous role-only module-hash evidence
  instead of letting duplicate role observations decide the hash check.

## 0.41.2 - 2026-05-22 - local deployment truth check

### Added

- Added per-design-line `status.md` logs for the 0.41-0.50 design
  directories so implementation progress and design drift can be recorded next
  to each design without rewriting the design documents themselves.
- Clarified the 0.41-0.43 install safety ladder: 0.41 remains truth/report
  groundwork around the current installer, 0.42 is report-first dry-run
  authority reconciliation, and full plan-driven deploy-install execution
  belongs to 0.43 unless explicitly promoted earlier.
- Added a read-only local deployment plan builder that turns resolved fleet
  config plus the local role artifact manifest into `DeploymentPlanV1` without
  querying IC state or replacing the current installer.
- Added a read-only local deployment check wrapper that builds
  `DeploymentPlanV1`, collects `DeploymentInventoryV1`, derives
  `DeploymentDiffV1`, and returns `SafetyReportV1` without mutating deployment
  state.
- Added a read-only local role artifact manifest builder that maps configured
  fleet roles and materialized `.wasm.gz` files into `RoleArtifactManifestV1`,
  including release-set payload hashes when a root release-set manifest is
  available.
- Missing artifact roots, release-set manifests, or role artifacts are captured
  as artifact observation gaps instead of changing installer behavior.
- Local artifact inventory and role manifests now include observed `.wasm.gz`
  file SHA-256 evidence with explicit `ObservedFileDigest` source metadata,
  keeping those live file hashes separate from release-set payload hashes.
- Added a passive deployment truth evaluator that compares
  `DeploymentPlanV1` with `DeploymentInventoryV1` into `DeploymentDiffV1` and
  renders `SafetyReportV1` without gating or mutating installer behavior.
- The initial evaluator reports missing artifacts, unsafe control classes,
  root/network identity mismatches, canonical config mismatches, unobserved
  verifier readiness, and inventory observation gaps as machine-readable
  findings.
- Deployment diffs now surface observed local artifact file hashes as
  informational `artifact_file_sha256` evidence instead of comparing them as
  release-set payload hashes.
- Added a read-only local inventory collector for `canic-host::deployment_truth`
  that snapshots configured fleet roles, local install-state root identity, and
  materialized local `.wasm.gz` artifacts into `DeploymentInventoryV1` without
  querying or mutating IC state.
- Missing config, role, artifact-root, or artifact observations are recorded as
  inventory observation gaps instead of being treated as installer errors.

### Changed

- Split `canic-host::deployment_truth` into focused module files: public
  exports and the schema version remain in `mod.rs`, passive V1 DTOs live in
  `model.rs`, local observation and artifact-manifest collection live in
  `observe.rs`, diff/safety-report logic lives in `report.rs`, and focused
  unit coverage lives in `tests.rs`.

### Added

  `docs/design/0.41-deployment-truth-model/0.41-design.md`.

  user-owned lifecycle, or multi-deployment operations work.
