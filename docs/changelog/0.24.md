# 0.24.x Detailed Changelog

Technical release notes for the `0.24.x` minor line.

## 0.24.1 - 2026-04-04 - Root Auth Key Prewarm

### 📝 Summary

- `0.24.1` follows up the first delegation-path replay/storage reductions by moving root auth key warmup into setup, so the first live delegated-auth request no longer pays the cold key-fetch cost.

### 🔧 Changed

- Warmed the root delegation and attestation key caches during the reference root canister's `canic_setup` hook through the public auth facade, so the cold root key fetch cost moves out of the first live delegated-auth request path without changing runtime behavior.
- In the current instruction audit, that drops sampled `root::canic_request_delegation` from `4356980` local instructions in `instruction-footprint-17` to `3934327` in `instruction-footprint-18`.
- In the same path, `issue_proof` drops from `882945` to `504032`.

## 0.24.0 - 2026-04-04 - Shared Runtime Reduction and Test Boundary Cleanup

### 📝 Summary

- `0.24.0` opens the new minor line after the accidental version bump without changing the current engineering direction: the next work still centers on shrinking the shared wasm floor, keeping public `canic-testkit` support separate from Canic-only self-test harness code, and hardening release-facing surfaces such as the curlable installer path.

### 🔧 Changed

- [CHANGELOG.md](/home/adam/projects/canic/CHANGELOG.md) now opens the `0.24.x` minor line.
- `0.24.x` is the new home for the current follow-through work rather than a broad reset or topology redesign.
- Switched shared `CandidType` source comments from `///` to `//` across the shared DTO and spec files, so Candid runtime docs stop inflating release wasm without changing the exported endpoint surface.
- In the current `minimal` release artifact, that reduces the shrunk wasm from `1764427` to `1741732` bytes and the `.wasm.gz` artifact from `498796` to `486798` bytes.
- Expanded the recurring instruction audit so it now measures real scaling, sharding, replay/cycles, delegated-auth issuance, and verifier-confirmation update paths alongside the local-only query perf probes, with stage-level checkpoint deltas captured for the sampled update flows where internal checkpoints exist.
- Reworked staged chunk storage so chunk bytes now live in a stable payload vector behind a small key-to-slot index instead of sitting directly in the stable chunk BTreeMap value path.
- In the current instruction audit, that drops the sampled `root::canic_template_publish_chunk_admin` single-chunk path from `9713784` local instructions in `instruction-footprint-12` to `390788` in `instruction-footprint-13`, with the old `chunk_store_insert` stage falling from `9518067` to `194978`.
- Replaced replay-store `RootReplayRecord` CBOR persistence with a direct byte layout and swapped replay payload hashing from canonical Candid encode to a structured field hash, so the sampled `root::canic_request_delegation` path drops from `5516827` local instructions in `instruction-footprint-15` to `4356980` in `instruction-footprint-17`.
- In the same delegation path, `commit_replay` drops from `1151412` to `993079` local instructions and `prepare_replay_input` drops from `779044` to `137129`, which means replay prep is no longer the dominant stage inside delegated auth issuance.

### 🧭 Migration Notes

- This minor bump does not signal a new API model by itself.
- The practical focus remains shared-runtime wasm reduction, public/internal testing-boundary cleanup, and release-surface correctness.
