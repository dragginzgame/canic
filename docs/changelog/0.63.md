# 0.63 Changelog

## 0.63.0 - 2026-06-08 - NNS topology inspection

### Summary

- Adds `canic nns topology summary` as a cached mainnet NNS topology summary
  view.

- Adds filtered `canic nns node list` reads for subnet, subnet kind,
  data-center, node-provider, and node-operator selection.

- Keeps this as a CLI/host inspection expansion: no Cargo package version,
  release-script default version, Candid, runtime canister, package artifact,
  lockfile, or node-cache schema changes.

### Added

- `canic nns topology summary` composes the existing cached subnet, node,
  node-provider, node-operator, and data-center reports into aggregate counts,
  subnet-kind distribution, and per-source registry-version provenance.

- `canic nns node list --subnet <subnet|subnet-prefix>` filters cached nodes
  by assigned subnet principal or prefix.

- `canic nns node list --kind <application|system|unknown>` filters cached
  nodes by assigned subnet kind.

- `canic nns node list --data-center <data-center|data-center-prefix>`
  filters cached nodes by data-center id or prefix.

- `canic nns node list --node-provider <node-provider|node-provider-prefix>`
  filters cached nodes by node-provider principal or prefix.

- `canic nns node list --node-operator <node-operator|node-operator-prefix>`
  filters cached nodes by node-operator principal or prefix.

### Changed

- The node-list filters are applied to the report before text or JSON
  rendering, so automation receives the same filtered node set in both output
  formats.

- README and CLI README NNS inspection examples now advertise
  `canic nns topology summary` and representative filtered
  `canic nns node list` commands.
