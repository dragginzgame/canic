# 0.56 Changelog

## 0.56.1 - 2026-06-01 - installed CLI smoke hardening

### Added

- Added the installed CLI smoke runbook:

  ```text
  docs/operations/0.56-installed-cli-smoke.md
  ```

### Changed

- Hardened the installed CLI proof:

  ```text
  scripts/ci/verify-installed-canic-cli.sh
  ```

  The script now asserts the proof path uses the temporary installed binary
  instead of `target/debug/canic`, isolates `HOME`, `CARGO_HOME`,
  `CARGO_TARGET_DIR`, and `TMPDIR` under the proof root, and continues to run
  the maintained v1 readiness smoke through the installed binary.

## 0.56.2 - 2026-06-01 - packaged downstream CLI proof hardening

### Added

- Added the packaged downstream CLI proof runbook:

  ```text
  docs/operations/0.56-packaged-downstream-cli.md
  ```

### Changed

- Hardened the packaged downstream CLI proof:

  ```text
  scripts/ci/verify-packaged-downstream-cli.sh
  ```

  The script now rejects repository crate paths and `target/debug/canic` in the
  packaged tool root, isolates `HOME`, `CARGO_TARGET_DIR`, and `TMPDIR` during
  packaged CLI execution, and verifies current v1 read-only commands against a
  downstream project:

  ```text
  canic fleet list
  canic fleet role list downstream
  canic fleet role inspect downstream app
  canic deploy catalog list --format json --output <path>
  ```

  It also packages and patches `canic-control-plane` explicitly, so local
  pre-publication versions do not pass by resolving that dependency from
  crates.io.

## 0.56.0 - 2026-06-01 - tentative packaged downstream proof design and retained probe hard cut

### Added

- Added the tentative 0.56 design:

  ```text
  docs/design/0.56-v1-packaged-downstream-proofs/0.56-design.md
  ```

- Added the retained v1 release probe inventory:

  ```text
  docs/operations/0.56-v1-release-probes.md
  ```

### Changed

- Reworked the installed CLI probe so it installs `canic` into a temporary
  root and runs the maintained v1 readiness smoke through the installed binary:

  ```text
  scripts/ci/verify-installed-canic-cli.sh
  scripts/ci/v1-readiness-smoke.sh
  ```

- Hard-cut the packaged downstream CLI fixture to current fleet-scoped config
  shape:

  ```text
  fleets/downstream/canic.toml
  [roles.root]
  [roles.app]
  ```

- Clarified Makefile help for the retained packaged/installed release probes:

  ```text
  make test-installed-canic-cli
  make test-packaged-downstream-cli
  make test-packaged-downstream-wasm-store
  ```

### Removed

- Removed the stale installed-CLI probe behavior that built repo-local roles
  with the old role-only build shape and wrote repository `.icp` artifacts.

### Clarified

- Defined 0.56 as a v1 release-confidence line rather than a new product
  feature line. The line should prove installed CLI and packaged downstream
  behavior using current v1 command shapes:

  ```text
  canic fleet create <fleet>
  canic scaffold canister <fleet> <role>
  canic fleet role attach <fleet> <role> --subnet <subnet>
  canic build <fleet> <role> --provenance <path>
  canic deploy check <deployment> --format envelope-json
  canic evidence gate --policy <path> --manifest <path>
  canic deploy catalog list
  canic deploy catalog inspect <deployment>
  ```

- Kept deployment groups, signing, locks, registry import, teardown,
  controller mutation, active adoption/import, broad live verification, and
  new stable public DTO families out of scope.

- Defined "packaged" strictly for the line: after package archives are
  created, packaged downstream proofs must not pass by using repository path
  dependencies, `target/debug/canic`, unpublished local crates, hard-coded local
  paths, or repository `.canic` / `.icp` state.

- Added release-proof guardrails for retained-script inventory, installed CLI
  isolation, packaged crate isolation, special `wasm_store` treatment, and MSRV
  handling. A missing local MSRV toolchain may be recorded as a limitation; a
  packaged crate requiring a newer Rust version than declared is a release
  blocker.
