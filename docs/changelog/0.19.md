## 0.19.5 - 2026-03-30 - Downstream Sharding Facade Compatibility

### đź“ť Summary

- `0.19.5` completes the downstream facade follow-up by adding a feature-gated `sharding` lane on `canic`, so sharding coordinator canisters can keep using the facade-owned `ShardingApi` and generated placement endpoints instead of importing `canic-sharding-runtime` directly.

### đź”§ Changed

- Restored `canic::api::canister::placement::ShardingApi` behind that feature.
- Updated `start!()`'s sharding placement endpoint emitters to route through the facade-owned hidden sharding lane instead of assuming the downstream crate imported `canic_sharding_runtime` itself.

### đź©ą Fixed

- Fixed downstream sharding canister builds that failed inside `start!()` because the generated placement endpoints referenced `::canic_sharding_runtime` directly.
- Fixed downstream sharding coordinator imports that still expected `canic::api::canister::placement::ShardingApi` to exist on the public facade.

### đź§­ Migration Notes

- Downstream `root` and `wasm_store` crates should continue to enable `canic`'s `control-plane` feature if they want to use `start_root!()` and the facade-owned template/store DTO/API paths.
- Downstream sharding coordinator canisters should enable `canic`'s `sharding` feature if they want to use `start!()` with sharding placement endpoints or import `canic::api::canister::placement::ShardingApi`.
- Ordinary leaf canisters should stay on the default feature set so they continue to avoid control-plane and sharding baggage.

## 0.19.1 - 2026-03-30 - Library Lane Cleanup and Reference Install

### đź“ť Summary

- `0.19.1` finishes the optional-lane split that started from the `0.19.0` baseline: template/store control-plane code, sharding orchestration, and demo-only helper baggage no longer define the default `canic` / `canic-core` canister lane.
- `0.19.1` also compiles `canic.toml` into the canister at build time, removes runtime TOML parsing from the default path, standardizes debug-only Candid export on `canic::cdk::export_candid_debug!()`, and hardens the staged `root` -> `wasm_store` local install/bootstrap flow behind `make demo-install` once `dfx` is already running.

### đź”§ Changed

- Moved template/store runtime and sharding orchestration behind optional lanes so plain downstream canisters no longer pull those paths transitively through `canic`, and then collapsed the temporary template helper crates back into `canic-control-plane` once the split was proven.
- Removed root-owned `subnet_state` / `wasm_store` publication inventory from the non-root cascade path, so leaf canisters no longer receive root/control-plane state through ordinary sync snapshots.
- Compiled embedded config into generated Rust/static data so runtime bootstrap no longer ships the TOML parser in the default canister lane.
- Moved the debug-gated Candid convenience macro to `canic-cdk` as `export_candid_debug!()`, making `cdk` the only supported Candid-export macro owner.
- Removed `canic::utils` and `canic_core::utils` from the supported public API, then deleted the now-unused `canic-utils` crate from the workspace entirely once downstream users had moved off it.
- Moved the generic memory bootstrap helper to `canic-memory::runtime::MemoryRuntimeApi`, so generated code no longer needs the full `canic` facade just to register memory ranges.
- Consolidated the duplicate empty-shell reference canisters (`app`, `minimal`, `scale`, `shard`) onto one shared implementation while preserving distinct roles and packages.
- Standardized local reference installation on `make demo-install` and `scripts/app/install_reference_topology.sh`, with explicit fail-fast behavior if `dfx` is not already running and clearer staging logs that distinguish root-local stable-memory staging from bytes baked into `root.wasm`.

### đź©ą Fixed

- Restored explicit stable-memory ownership after the crate split so `root` reinstall no longer traps on moved template/control-plane memory ranges.
- Restored the first `wasm_store` install path to use rootâ€™s staged bootstrap chunk source instead of the ordinary published-template resolver.
- Fixed the root bootstrap/reference path so the full config-defined release set is staged before bootstrap resumes, which lets `root` create `wasm_store`, publish the staged roles, and finish `auto_create` without missing approved manifests.
- Fixed local reference-only helper endpoints to key off `build_network_local()` instead of `debug_assertions`, which keeps PocketIC/local topology flows aligned with the real build-network contract.
- Updated root-only test assertions for PocketICâ€™s current missing-method wording so transport failures continue to validate the intended boundary checks.
- Fixed host-target RPC/auth metadata generation so `canic-core` unit tests and embedded root test-canister builds no longer touch `ic0` during host builds.
- Removed default-lane baggage so non-sharding and non-control-plane canisters stop carrying sharding HRW placement code, template/store implementation lanes, public `utils` facade leakage, or runtime TOML parsing by default.

### đź—‘ď¸Ź Removed

- Removed the `canic::export_candid!()` forwarding macro in favor of `canic::cdk::export_candid_debug!()`.
- Removed the public `ShardingApi` re-export from the `canic` facade, so sharding coordinators depend on `canic-sharding-runtime` explicitly.
- Removed the dead `canic::dsl` facade namespace and deleted the unused `canic-dsl` crate; endpoint attributes continue to live in `canic-dsl-macros`.
- Removed the temporary `canic-template-runtime` and `canic-template-types` crates after their code was merged into `canic-control-plane`.
- Removed the orphaned `canic-utils` crate after the last in-repo and downstream users stopped depending on it.

### đź“Š Audit

- The default wasm-target lane no longer pulls `canic-control-plane`, `canic-sharding-runtime`, `canister_wasm_store`, `toml`, or `xxhash-rust` in `canister_minimal`â€™s normal dependency tree, and the removed helper crates no longer exist in the workspace graph at all.
- The `minimal` release line dropped from the `0.19.0` baseline of `2489858` raw bytes to a verified `1873454` raw bytes after the lane split, compiled-config cutover, and shared-runtime cleanup.
- The reference build flow now emits a shrunk `1752160` byte wasm and a `492943` byte gzip artifact for `minimal`.

### đź§­ Migration Notes

- Use `canic::cdk::export_candid_debug!()` instead of the removed `canic::export_candid!()` forwarding macro.
- Sharding coordinator canisters should now depend on `canic-sharding-runtime` directly and import `ShardingApi` from `canic_sharding_runtime::api::ShardingApi`; it is no longer re-exported from `canic::api::canister::placement`.
- Generated code that only needs memory bootstrap registration should use `canic_memory::runtime::MemoryRuntimeApi` instead of the broader `canic::api::runtime::MemoryRuntimeApi` facade path.
- Remove any lingering `canic::dsl` imports; the supported endpoint surface is `#[canic_query]` / `#[canic_update]` from `canic` or `canic-dsl-macros`.
- Remove any lingering `canic-utils` dependency pins; the helper crate is gone from the workspace.
- `make demo-install` and the reference bootstrap scripts assume `dfx` is already running in another terminal. They now fail fast instead of auto-starting a local replica.
