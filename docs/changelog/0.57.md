# 0.57 Changelog

## 0.57.2 - 2026-06-01 - wasm audit runner role-package cleanup

### Fixed

- Updated the recurring Wasm footprint audit runner to resolve Cargo package
  names from fleet role declarations instead of assuming
  `canister_<role>`. This keeps the audit compatible with hard-cut
  fleet-scoped role/package metadata such as `scale_replica` using the
  `canister_scale` package.

- Passed the selected fleet config into the Canic artifact-shrinking path with
  an absolute `CANIC_CONFIG_PATH`, so canister build scripts no longer fall
  back to stale default config locations.

- Removed stale `minimal = N/A` reporting from the Wasm footprint template and
  recurring audit definition. Missing baseline evidence is now called out as
  such instead of being treated as a size signal.

## 0.57.1 - 2026-06-01 - layer-boundary cleanup

### Changed

- Moved root-capability envelope validation, proof verification, replay
  projection, metrics, and dispatch orchestration out of the public RPC API
  module and into workflow ownership:

  ```text
  crates/canic-core/src/workflow/rpc/capability/
  ```

  The public API now delegates through `workflow::rpc::capability` and maps
  workflow/internal errors at the boundary.

- Moved role-attestation verification with single key-refresh retry into
  runtime auth workflow ownership so capability proof verification no longer
  calls back into `api::auth`.

- Changed the module-source resolver contract to return Canic's internal error
  type instead of public `dto::error::Error`, and removed the stale public-error
  support wrapper in the control-plane path.

- Tightened the recurring layer-boundary audit to explicitly scan for API
  proof-orchestration drift and public DTO error leakage in ops-facing
  contracts:

  ```text
  docs/audits/recurring/system/layer-violations.md
  ```

- Clarified packaged README posture for `canic-cli` and `canic-host` so the
  installed binary is described around the compact v1
  setup/build/evidence/catalog workflow, with install, backup, and restore
  documented as specific operator workflows.

## 0.57.0 - 2026-06-01 - audit rotation and feedback window

### Audit

- Started a maintenance line for rotating recurring audits while waiting for
  real user feedback on the compact v1 operator surface.

- Refreshed the recurring publish-surface audit definition:

  ```text
  docs/audits/recurring/system/publish-surface.md
  ```

  The audit now reflects the current eight published crates, the post-0.56
  installed/packaged proof story, the declared Rust `1.91.0` MSRV package
  contract, and the special `canic-wasm-store` bootstrap/runtime posture.

- Kept the line intentionally non-feature:

  ```text
  no new commands
  no new DTOs
  no deployment groups
  no signing or locks
  no registry import
  no teardown
  no controller mutation
  no active adoption/import
  ```
