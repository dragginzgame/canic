## 0.57.1 - 2026-06-01 - layer-boundary cleanup

### Changed

- Moved root-capability envelope validation, proof verification, replay
  projection, metrics, and dispatch orchestration out of the public RPC API
  module and into workflow ownership:

  ```text
  crates/canic-core/src/workflow/rpc/capability/
  ```

  The public API now delegates through `workflow::rpc::capability` and maps
  workflow/internal errors at the boundary.

- Moved role-attestation verification with single key-refresh retry into
  runtime auth workflow ownership so capability proof verification no longer
  calls back into `api::auth`.

- Changed the module-source resolver contract to return Canic's internal error
  type instead of public `dto::error::Error`, and removed the stale public-error
  support wrapper in the control-plane path.

- Tightened the recurring layer-boundary audit to explicitly scan for API
  proof-orchestration drift and public DTO error leakage in ops-facing
  contracts:

  ```text
  docs/audits/recurring/system/layer-violations.md
  ```

- Clarified packaged README posture for `canic-cli` and `canic-host` so the
  installed binary is described around the compact v1
  setup/build/evidence/catalog workflow, with install, backup, and restore
  documented as specific operator workflows.
