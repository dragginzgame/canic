# 0.57 Changelog

## 0.57.12 - 2026-06-02 - DRY deploy CLI cleanup continued

### 📝 Summary

- Continued the DRY cleanup of the large deploy CLI owner by extracting the
  authority dry-run, resume-report, and passive deployment-truth command
  families into:

  ```text
  crates/canic-cli/src/deploy/authority.rs
  crates/canic-cli/src/deploy/resume_report.rs
  crates/canic-cli/src/deploy/truth.rs
  ```

### 🧹 Cleanup

- Moved `deploy authority check`, `deploy authority evidence`,
  `deploy authority report`, and `deploy authority receipt` parsing, help
  rendering, JSON/text output selection, dry-run evidence/receipt builders, and
  command dispatch out of the top-level deploy module.

- Moved `deploy resume-report` parsing, help rendering, latest-receipt lookup,
  receipt decoding, resume-safety report construction, and command dispatch
  out of the top-level deploy module.

- Moved passive `deploy plan`, `deploy inventory`, `deploy diff`, and
  `deploy report` parsing, help rendering, deployment-truth loading, field
  selection, and command dispatch out of the top-level deploy module.

- Kept authority output-format parsing centralized through the existing
  deploy output-format module and kept resume-report receipt decoding on the
  shared deploy JSON-file reader.

- Updated deploy tests to assert the extracted authority command family still
  preserves dispatch, help text, invalid-format errors, local-id dry-run
  builders, and the no-controller-mutation boundary.

- Reduced the main deploy CLI owner from 6,452 lines after `0.57.11` to 5,950
  lines after the authority, resume-report, and passive truth extractions.

### 🧪 Testing

- Validated the cleanup with:

  ```text
  cargo fmt --all
  cargo test -p canic-cli deploy
  ```
