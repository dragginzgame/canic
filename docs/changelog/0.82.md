## 0.82.23 - 2026-07-06 - Control-plane support owner cleanup

### 📝 Summary

- Continues 0.82 support-boundary hardening by removing hidden
  `control_plane_support` facades that only mirrored public core owners.

  `canic_core::dto::validation`, and deployment workflow imports `CostClass`
  from `canic_core::replay_policy`.

  facades for crate-private `SubnetConfig`, `CanisterPoolStatus`, and

  `canic-control-plane`.

### 🧹 Cleanup

- Removed `control_plane_support::dto::validation`.

- Removed `control_plane_support::replay_policy`.

- Removed the unused `control_plane_support::ids` public-owner mirror.

- Updated root bootstrap and deployment workflow imports to use canonical
  public core owners where they are available.

- Added a docs-only boundary report:
  `docs/design/0.82-boundary-hardening/0.82-control-plane-core-owner-support-boundary-report.md`.

### ⚠️ Breaking

- None for operator/runtime behavior. This patch removes hidden Rust support
  facade paths that mirrored public owners.

### 🧭 Migration Notes

- This patch adds no new operator commands.

- This patch does not change command names, command options, endpoint surfaces,
  Candid, JSON, deployment truth, evidence/report schemas, stable-state layout,
  root bootstrap validation behavior, or deployment cost-guard behavior.

- Use canonical public owners for these values:

```rust

use canic_core::ids::{BuildNetwork, CanisterRole};
```

## 0.82.21 - 2026-07-06 - Inspect metadata and deployment fallback hard cuts

### 📝 Summary

- Continues the 0.82 pre-1.0 hard-cut cleanup by removing implicit
  inspect response metadata, deployment observation fallbacks, and install-state
  fallback behavior from host-owned report/state paths.

- `canic inspect` still requires typed `response_bytes` for
  `canic_runtime_status` decoding and still labels the response format as
  `candid`, but text and JSON inspect reports no longer expose response-wrapper
  presence fields.

- Deployment-truth and deploy-plan artifact observation now require artifacts
  under the selected network root. Non-local observations no longer fall back
  to `.icp/local/canisters` when `.icp/<network>/canisters` is missing.

- Local config observation no longer copies the deployment target name into
  fleet-template identity when the fleet name cannot be resolved. It reports
  the existing config gap and uses the explicit `unknown` marker for the
  required root-observation field.

- Deployment catalog and install-root state reads now ignore removed
  fleet-state paths and read only current deployment-target state under
  `.canic/<network>/deployments`.

### 🧹 Cleanup

- Removed `response_candid_present` and `response_bytes_present` from the
  inspect runtime-status report wrapper.

- Removed response-Candid presence detection from inspect response parsing.

- Simplified typed response-byte decoding so missing `response_bytes` fails
  directly before Candid decoding.

- Removed the cross-network artifact-root fallback and the
  `local_artifacts.network_fallback` production observation.

- Removed caller-provided fleet-name fallback from local config observation.

- Removed deployment catalog probing of removed `.canic/<network>/fleets`
  paths and the `catalog.legacy_fleet_state_ignored` warning.

- Removed install-root legacy fleet-state lookup and the old 0.46 recovery
  framing from deployment registration help/docs.

- Docs-only boundary reports record the slice contracts and validation:
  `docs/design/0.82-boundary-hardening/0.82-inspect-response-candid-metadata-hard-cut-report.md`,
  `docs/design/0.82-boundary-hardening/0.82-artifact-root-network-fallback-hard-cut-report.md`,
  `docs/design/0.82-boundary-hardening/0.82-local-config-fleet-name-fallback-hard-cut-report.md`,
  `docs/design/0.82-boundary-hardening/0.82-deployment-catalog-legacy-fleet-warning-hard-cut-report.md`,
  and
  `docs/design/0.82-boundary-hardening/0.82-install-root-legacy-fleet-state-hard-cut-report.md`.

### ⚠️ Breaking

- Automation consuming `canic inspect --json` must no longer read
  `runtime_status.response_candid_present` or
  `runtime_status.response_bytes_present`.

- Non-local deployment-truth/deploy-plan artifact observation must have
  artifacts under `.icp/<network>/canisters`; local artifacts are no longer
  used as a fallback for another selected network.

- Missing local fleet-template config no longer reports the deployment target
  as the fleet template in root observations.

- Deployment catalog reports no longer emit the legacy
  `catalog.legacy_fleet_state_ignored` warning.

- Install-state reads no longer inspect removed
  `.canic/<network>/fleets/<name>.json` paths when current deployment-target
  state is missing.

- These are intentional hard cuts; no compatibility fallback, alias,
  replacement shim, or old-surface anti-resurrection test was added.

### 🧭 Migration Notes

- This patch adds no new operator commands.

- This patch does not change command names, command options, Candid, stable
  state, deployment-truth schemas, evidence schemas, install-state schema, or
  runtime endpoint DTOs.

- Consumers should treat successful inspect runtime reports as already having
  decoded typed response bytes. Use `runtime_status.response_format` for the
  remaining response-format attribution.

- Missing or invalid `response_bytes` remains an invalid inspect response.

- Build or stage artifacts for the selected network before running
  deployment-truth/deploy-plan observation for that network.

- If an existing root should be represented locally, use the current explicit
  deployment-target registration surface rather than relying on removed
  fleet-state paths:

```bash
canic deploy register <deployment> --fleet-template <fleet> --root <principal> --allow-unverified
```

- Root bootstrap no longer has an implicit non-IC subnet fallback. Local/test
