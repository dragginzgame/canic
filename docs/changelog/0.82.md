# 0.82 Changelog

## 0.82.33 - 2026-07-07 - Backup and restore parser hard cuts

### 📝 Summary

- Hard-cuts backup create layout repair by rejecting existing plan-only layouts
  that are missing `backup-execution-journal.json`.

- Hard-cuts restore runner command-output parsing so upload snapshot IDs and
  stopped-canister preconditions require structured JSON output.

- Hard-cuts backup manifest and plan decoding so unknown JSON fields are
  rejected instead of ignored.

- Removes unused plan-facing backup receipt types now that execution journals
  own durable operation receipts.

### 🧹 Cleanup

- Removed backup create's fallback that synthesized a missing execution journal
  from an existing `backup-plan.json`.

- Removed restore upload parser support for alternate JSON keys and human text
  output; the maintained shape is now JSON with `snapshot_id`.

- Removed restore stopped-status parser support for human status text; the
  maintained precondition output is structured JSON.

- Updated restore fake `icp` scripts and receipt fixtures to use the current
  JSON upload-output shape.

- Added strict unknown-field rejection to backup manifest, backup plan, and
  backup execution preflight structs.

- Removed stale manifest fixture coverage for ignored removed fields.

- Removed `BackupReceipt` and `BackupReceiptStatus` from backup plan types.

- Added a docs-only hard-cut report:
  `docs/design/0.82-boundary-hardening/0.82-backup-restore-legacy-repair-hard-cut-report.md`.

### ⚠️ Breaking

- Existing backup create output layouts with `backup-plan.json` but no
  `backup-execution-journal.json` now fail closed instead of being repaired.

- Restore upload helper output must include a non-empty JSON `snapshot_id`
  field.

- Restore stopped-canister precondition output must be structured JSON.

- Backup manifests, backup plans, and backup execution preflight contracts now
  reject unknown JSON fields.

- Rust namespace only: `canic_backup::plan::BackupReceipt` and
  `canic_backup::plan::BackupReceiptStatus` no longer exist. Execution
  operation receipts are the maintained backup receipt path.

- No operator command names, flags, endpoint method names, Candid schemas, JSON
  report schemas, deployment truth, evidence/report schemas, restore receipt
  schema, or stable-state layout changed.

### 🧭 Migration Notes

- This patch adds no new operator commands.

- Current restore upload helper output should be JSON:

```bash
printf '%s\n' '{"snapshot_id":"<snapshot-id>"}'
```

- Current restore status helper output should be JSON:

```bash
printf '%s\n' '{"status":"Stopped"}'
```
