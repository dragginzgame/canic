## 0.82.11 - 2026-07-05 - Canister and management metric domain ownership

  paths.

  through re-exports.

- This patch does not change Candid method names, endpoint behavior, JSON
  report fields, stable state layout, management canister operation behavior,

  mirrors the architectural rule that policy is pure decision logic.

- Moves policy-owned input and decision shapes out of `view/`, keeping
  `view/` reserved for internal read-only projections rather than policy input
  contracts.

  but they now hand policy-owned inputs to policy rather than importing those

## 0.82.0 - 2026-07-05 - Boundary hardening foundation

### 📝 Summary

- Starts 0.82 as a controlled boundary-hardening line rather than a broad
  architecture cleanup pass.

- Defines the slice contract, hardening report model, severity rubric,
  behavior-impact labeling, and public/serialized surface freeze rules used to
  keep cleanup work reviewable and no-behavior-change by default.

- Completes the first hardening slices for diagnostic/report ownership and
  policy-purity import ownership.

### 🔧 Changed

- The 0.82 design now names diagnostic ownership hardening as the first slice
  and records cleanup safety rules for future slices: no incidental command
  changes, endpoint changes, JSON/schema changes, stable-state changes, or
  behavior changes unless a separate slice contract declares them.

- Diagnostic ownership is documented in a closeout report so report producers
  own diagnostic semantics while CLI renderers remain formatting and exit-code
  consumers.

- Policy helper modules now expose narrower dependency surfaces:
  deployment-truth policy helpers no longer rely on parent wildcard imports,
  core policy modules import shared pure value types through
  `crate::domain::value`, and policy decisions consume policy-owned input
  structs instead of serialized config schema structs.

- Workflow and test boundaries now map config into policy input shapes before
  calling log retention, cycle top-up, ICP refill, scaling, sharding, and
  topology registry policy.

### 🧹 Cleanup

- Added policy-owned input/rules/shape types for log retention, cycle top-up,
  ICP refill, scaling, sharding, and registry topology decisions.

- Split topology registry policy from `CanisterConfig` / `CanisterKind` by
  introducing pure registry canister shape inputs.

- Kept policy behavior, CLI behavior, endpoint surfaces, JSON shapes, stable
  memory layout, deployment truth, and evidence/report schemas unchanged.

### 🧭 Migration Notes

- This patch adds no new operator commands.

- This patch adds no backwards-compatibility aliases, shims, or alternate
  command surfaces.

- This patch does not change serialized DTOs, Candid methods, JSON report
  fields, stable state layout, deployment truth schema, or runtime endpoint
  behavior.

- The remaining 0.82 follow-up is to replace ad hoc import scans with a robust
  dependency-boundary check before making policy-purity import rules a release
  gate.
