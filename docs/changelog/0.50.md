# 0.50 Changelog

## 0.50.0 - 2026-05-30 - passive adoption report model

### Added

- Added host-side adoption profile and report model types:
  - `AdoptionProfileV1`
  - `AdoptionReportV1`
  - `AdoptionRoleFindingV1`
  - `AdoptionObservedCanisterFindingV1`
  - `AdoptionRecommendationV1`
- Added the first passive adoption report builder:
  ```rust
  adoption_report_from_config_source(...)
  ```
  The builder reads supplied config, package metadata, deployment inventory,
  and artifact manifest evidence, then returns a report. It does not read from
  or write to the filesystem.

### Safety

- Role classification preserves the 0.49 lifecycle split:
  declared-only roles remain visible, attached roles remain explicit, and
  report generation does not make declared-only roles deployable.
- Adoption reports distinguish managed configuration from live authority.
  User-controlled or externally controlled observed canisters are classified
  separately and do not become safely manageable merely because names or roles
  match.
- Recommendations are non-executed report items. Suggested mutating actions are
  marked as unsupported by the `0.50.0` adoption surface.
- The report model includes `attached-unobserved`, `observed-only`,
  `external-controller-required`, `imported-pool-candidate`, and
  `evidence-conflict` classifications for brownfield onboarding.
