# 0.50 Changelog

## 0.50.8 - 2026-05-30 - deployment-check adoption evidence

### Added

- Added a read-only adoption report evidence input for saved deployment checks:

  ```text
  canic fleet adoption report <fleet> --profile <profile> --deployment-check <path>
  ```

- `--deployment-check` reads inventory evidence from the `inventory` field in a
  `DeploymentCheckV1` JSON artifact.

### Safety

- Adoption reports reject `--deployment-check` and `--inventory` together so
  the inventory evidence source is explicit.
- Deployment-check evidence is a file read only. It does not query live
  inventory or mutate deployment-truth state.

### Testing

- `cargo test -p canic-cli fleets --lib --locked`

## 0.50.7 - 2026-05-30 - explicit adoption evidence inputs

### Added

- Added read-only adoption report evidence input options:

  ```text
  canic fleet adoption report <fleet> --profile <profile> --inventory <path>
  canic fleet adoption report <fleet> --profile <profile> --artifact-manifest <path>
  canic fleet adoption report <fleet> --profile <profile> --package-metadata <path>
  ```

- `--inventory` reads `DeploymentInventoryV1` JSON evidence.
- `--artifact-manifest` reads `RoleArtifactManifestV1` JSON evidence.
- `--package-metadata` reads a JSON array of `AdoptionPackageMetadataV1`
  entries.

### Safety

- Evidence inputs are file reads only. The report command still performs no live
  discovery, topology attachment, config edits, artifact import, install,
  upgrade, controller mutation, inventory rewrite, or deployment-truth write.

### Testing

- `cargo test -p canic-cli fleets --lib --locked`

## 0.50.0 - 2026-05-30 - passive adoption report model

### Added

- Added host-side adoption profile and report model types:
  - `AdoptionProfileV1`
  - `AdoptionReportV1`
  - `AdoptionRoleFindingV1`
  - `AdoptionObservedCanisterFindingV1`
  - `AdoptionRecommendationV1`
- Added the first passive adoption report builder:
  ```rust
  adoption_report_from_config_source(...)
  ```
  The builder reads supplied config, package metadata, deployment inventory,
  and artifact manifest evidence, then returns a report. It does not read from
  or write to the filesystem.

### Safety

- Role classification preserves the 0.49 lifecycle split:
  declared-only roles remain visible, attached roles remain explicit, and
  report generation does not make declared-only roles deployable.
- Adoption reports distinguish managed configuration from live authority.
  User-controlled or externally controlled observed canisters are classified
  separately and do not become safely manageable merely because names or roles
  match.
- Recommendations are non-executed report items. Suggested mutating actions are
  marked as unsupported by the `0.50.0` adoption surface.
- The report model includes `attached-unobserved`, `observed-only`,
  `external-controller-required`, `imported-pool-candidate`, and
  `evidence-conflict` classifications for brownfield onboarding.
