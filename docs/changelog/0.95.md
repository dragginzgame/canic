# Canic 0.95 Changelog

## [0.95.0] - 2026-07-20 - Timer Authority Slice A

### 📝 Summary

Closes the completed 0.94 backup/restore line at released `v0.94.14` and
activates the accepted 0.95 timer-authority design. The audit inventories every
production timer, lifecycle deferral, public timer form, retry scheduler, and
bounded host wait before scheduler implementation begins.

### 📊 Audit

- Confirms `ops::runtime::timer` is the sole production consumer of direct IC
  timer operations.
- Dispositions all 14 timer/process rows and both bounded host waits.
- Reproduces async interval overlap, stale guarded slots, false live status,
  unnecessary idle wake-ups, unrelated full scans, duplicate mechanical and
  lifecycle flows, and unreachable configured root self-refill.
- Measures the empty-root first-hour baseline at ten background callbacks and
  72,303 timer-callback instructions. Seven log/pool wakes had no
  authoritative work.
- Adds a source inventory guard so a new timer wrapper, bypass, or timed wait
  cannot enter silently.

### 🔧 Decisions

- Retains cancellable public one-shot and after-completion interval forms.
  Unused guarded macros, `TimerSlot`, fixed-rate async recurrence, and public
  raw-CDK timer access are hard-cut in the common-authority slice.
- Freezes event/deadline ownership for log age retention, pool resets, intent
  expiry, placement acknowledgement, and delegated-proof renewal.
- Keeps cycle sampling and automatic top-up independent so disabled top-up is
  unregistered and configured root ICP self-refill becomes reachable through
  its existing workflow.
- Freezes per-owner batch, retry, and cadence bounds without adding a global
  heartbeat or receipt-specific state.

### 🧪 Testing

- Uses the existing root PocketIC harness to measure current callback count and
  instruction cost from the released anchor.
- Runs the targeted source inventory and direct-owner guards.
- Adds no production scheduling, Candid, JSON, stable-state, Cargo-version,
  dependency, backup/restore, receipt-reclamation, or compatibility change.
