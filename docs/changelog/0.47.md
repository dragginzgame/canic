# 0.47 Changelog

## 0.47.0 - 2026-05-27 - passive root verification evidence

### Added

- Added explicit `DeploymentRootObservationV1` evidence to
  `DeploymentInventoryV1`, including deployment target, network, fleet
  template, root principal, observed canister ID, observation source,
  controller facts, module hash, status, and role-assignment source.
- Added passive `DeploymentRootVerificationRequestV1` and
  `DeploymentRootVerificationReportV1` artifacts. These reports can produce
  `EvidenceSatisfied` when source-check evidence is digest-bound and complete,
  but they do not write verified local state.
- Added validation for root-verification report digest stability, required
  source check/plan/inventory IDs, explicit observed-root evidence, and the
  exact `unverified_deployment_root` hard-failure exception.
- Added read-only CLI inspection for passive root-verification reports.

```bash
canic deploy root inspect --request root-verification.json
```

```bash
canic deploy root inspect --request root-verification.json --format text
```

### Changed

- Local deployment inventory now records the deployment target name from the
  request instead of reusing the fleet-template name.
- Deployment-truth registry observations now resolve installed deployment state
  by deployment target name, keeping fleet templates and live deployment
  targets separate at the root-evidence boundary.
- Root-verification reports treat `LocalDeploymentState` root observations as
  insufficient evidence; root verification must bind to explicit
  deployment-truth evidence instead of local-state echo.
- The passive report allows only the exact single
  `unverified_deployment_root` blocker for
  `local_state.unverified_root_canister_id`; unrelated hard failures keep the
  evidence status blocked.

### CLI

- `canic deploy root inspect --request <file>` reads a
  `DeploymentRootVerificationRequestV1` JSON file and prints a
  `DeploymentRootVerificationReportV1` JSON artifact by default.
- `--format text` renders host-owned passive text that calls out
  `mode: passive`, `execution: none`, and `local_state_write: none`.
- The root inspect path has source-guard coverage proving it does not call
  install, register, dfx, or controller-mutation primitives.

### Validation

- Added host tests for bound root evidence, missing explicit root evidence,
  local-state-only evidence, exact unverified-root blocker handling,
  unrelated hard failures, and digest drift.
- Added CLI tests for root inspect parsing, help wording, command dispatch,
  passive source guards, and host report delegation.

### Testing
