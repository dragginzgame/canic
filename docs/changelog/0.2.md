# 0.2.x Detailed Changelog

Technical release notes for the `0.2.x` minor line.

## 0.2.24 - 2025-11-10 - PocketIC Test Helpers

### 📝 Summary

- `0.2.24` added `cfg(test)`-gated PocketIC helper support under `test/` to reduce repeated integration-test setup code.

---

## 0.2.21 - 2025-10-24 - Nested Role Validation Fix

### 🩹 Fixed

- Configuration validation now detects invalid nested canister-role definitions instead of only checking shallow structures.

---

## 0.2.17 - 2025-10-20 - Ledger Dependency Cleanup

### 🗑️ Removed

- Removed the `icrc-ledger-types` dependency from the release line.

### 🔧 Changed

- Replaced dependency usage with a local implementation to keep the integration surface controlled.

---

## 0.2.10 - 2025-10-20 - HRW Scaling Standardization

### 🔧 Changed

- Sharding data structures now use `String` IDs instead of `Principal`, improving interoperability with non-principal identity sources.
- Scaling now consistently uses HRW placement and removes parallel legacy paths.

---

## 0.2.9 - 2025-10-18 - Recursive Config Validation Hardening

### 🩹 Fixed

- Recursive config validation now catches invalid subnet-directory entries and other nested configuration errors more reliably.

---

## 0.2.7 - 2025-10-16 - Shared Hashing Utilities

### 🔧 Changed

- Moved `xxhash` helpers into `canic` so sharding and external consumers can share one implementation path.

---

## 0.2.6 - 2025-10-16 - Ops/Memory Responsibility Split

### 🔧 Changed

- Continued moving `memory::*` responsibilities behind ops boundaries, including clearer `CycleTracker` versus `CycleTrackerOps` separation.
- Moved canister reserve configuration to per-subnet settings for better topology-level control.

---

## 0.2.3 - 2025-10-15 - Registry-Derived Directory State

### 🔧 Changed

- `app_directory` and `subnet_directory` are now projected from `SubnetCanisterRegistry` as the source of truth.
- Directory payloads were added to canister init flow alongside `Env`, so new canisters receive topology context at creation time.

---

## 0.2.2 - 2025-10-13 - Delegation Removal and Signature Ops

### 🗑️ Removed

- Removed legacy delegation code from this line.

### ➕ Added

- Added `ops::signature` as a wrapper for canister-signature creation and verification.

---

## 0.2.1 - 2025-10-13 - Post-Launch Stabilization

### 🩹 Fixed

- Applied post-launch bug fixes after the `0.2.0` topology baseline rollout.

---

## 0.2.0 - 2025-10-13 - PRIME Subnet Baseline

### 📝 Summary

- `0.2.0` established the PRIME subnet baseline for topology, directory management, and sync flow behavior.

### ➕ Added

- Added `SubnetRole` for prime-versus-secondary subnet modeling.
- Added an `Env` cell so canisters persist root, subnet, parent, and type identifiers.

### 🔧 Changed

- Split topology storage into dedicated directory modules and updated ops helpers to use them.
- `AppDirectory` became an app-level canister directory boundary.
- `SyncBundle` now synchronizes both state and directory data.
- Performed broad internal cleanup, including clearer `memory::*` and ops separation.
