# 0.17.x Detailed Changelog

Technical release notes for the `0.17.x` minor line.

## 0.17.3 - 2026-03-26 - Metrics, Log Paging, and Root Handoff Completion

### 📝 Summary

- `0.17.3` continues the wasm audit line by tightening the shared `canic_metrics` and `canic_log` runtime paths, completing the root decomposition and cutover-input handoff for `0.18`, and reducing the `minimal` raw release artifact to `2433930` bytes (`delta -26446` vs `0.17.2`) without removing intended operator-facing features.

### 🔧 Changed

- Collapsed the shared metrics transport further by using the direct `MetricsKind` plus `PageRequest` query path, replacing multi-field metric values with the explicit `MetricValue` enum, and simplifying the metrics read path to endpoint -> workflow page -> ops entries while keeping DTO shaping inside ops.
- Moved `canic_log` filtering, newest-first ordering, and page slicing into ops so log queries no longer build and sort a full filtered DTO list before pagination, while preserving the existing endpoint arguments, output shape, and ordering semantics.
- Continued small shared-runtime reductions in `ICRC` and Canic standards helper paths and in common formatting/logging helpers so the low-level wasm floor keeps moving down without feature removal.

### 📊 Audit

- Raw release wasm sizes are now `2433930` for `minimal`, `app`, `scale`, and `shard`; `2485213` for `test`; `2495757` for `scale_hub`; `2591211` for `user_hub`; `2596159` for `shard_hub`; `2890061` for `user_shard`; and `14517836` for `root`.
- Relative to `0.17.2`, this patch reduces raw release wasm by `26446` bytes on `minimal`/`app`/`scale`/`shard`, `27250` on `test`, `26460` on `scale_hub`, `28030` on `user_hub`, `28152` on `shard_hub`, `40325` on `user_shard`, and `932138` on `root`.
- Root decomposition is now recorded for the `0.18` handoff: `14517836` raw root bytes, `11393191` embedded child payload bytes, `3124645` synthetic runtime remainder, `78.48%` payload share, and `125.87%` utilization against the assumed `11 MiB` cap (`delta -2983500` bytes of headroom).
- The current `0.17` handoff inputs now also include the canonical deployable inventory, the empty irreducible bootstrap set for current ordinary embedded roles, the ordered extraction sequence, the required manifest fields for template-canister cutover, and the `wasm-debug` vs `wasm-release` comparison baseline.

### 📚 Documentation

- Updated the `0.17` status tracking so the wasm audit now closes with concrete `0.18` cutover inputs instead of only leaf-canister floor reductions.

## 0.17.2 - 2026-03-26 - Shared Runtime and Log Query Reduction

### 📝 Summary

- `0.17.2` continues the wasm audit line by slimming shared runtime, metrics, and observability paths, reducing the `minimal` raw release artifact to `2460376` bytes (`delta -100624` vs `0.17.1`) while keeping the intended `canic_metrics`, `canic_log`, and control-plane feature set intact.

### 🔧 Changed

- Split the shared `start!` endpoint surface into explicit capability bundles so lifecycle core, metrics, auth/attestation, sync/topology, standards, and observability can be measured and optimized as separate retained wasm families without changing the default Canic feature set.
- Simplified the shared metrics read path onto the unified `MetricEntry` transport shape, kept DTO shaping inside ops, flattened root-capability metric storage, and removed dead perf/metrics wrappers that no longer owned a real boundary.
- Tightened the shared logging/runtime path by keeping typed internal topics, simplifying log label formatting, and moving `canic_log` filtering ahead of DTO projection so filtered queries do less allocation work.

### 🧹 Cleanup

- Removed stale metrics mapper and snapshot wrapper plumbing that no longer matched the unified metrics query surface.
- Removed dead log/perf wrappers exposed by the observability cleanup so `clippy -D warnings` stays aligned with the current runtime path.

### 📊 Audit

- Raw release wasm sizes are now `2460376` for `minimal`, `app`, `scale`, and `shard`; `2512463` for `test`; `2522217` for `scale_hub`; `2619241` for `user_hub`; `2624311` for `shard_hub`; `2930386` for `user_shard`; and `15449974` for `root`.
- Relative to `0.17.1`, this patch reduces raw release wasm by `100624` bytes on `minimal`/`app`/`scale`/`shard`, `101260` on `test`, `100624` on `scale_hub`, `102213` on `user_hub`, `102216` on `shard_hub`, `114367` on `user_shard`, and `2116467` on `root`.
- Current bundle attribution on `minimal` still shows `metrics` as the dominant shared family, followed by `observability_log`, `topology_directory`, and `topology_state`, which keeps the next `0.17` optimization targets explicit instead of guess-driven.

### 📚 Documentation

- Updated the `0.17` release notes and audit trail to reflect the new low-water mark and the current retained-family ranking after the shared runtime and observability cleanup.

## 0.17.1 - 2026-03-25 - Capability Split and Canic Standards Status Removal

### 📝 Summary

- `0.17.1` lowers the shared Canic wasm floor again by separating root-only capability verification from the ordinary non-root cycles path and by removing the old Canic standards canister-status surface, reducing the `minimal` raw release artifact to `2561000` bytes without removing intended runtime features.

### 🔧 Changed

- Split the response-capability implementation into explicit root and non-root modules so ordinary leaf canisters no longer retain the full root verifier/orchestrator path behind `canic_response_capability_v1`.
- Added a dedicated non-root cycles capability workflow that preserves structural proof checks, replay behavior, cycles funding policy, and root-capability metrics while narrowing the code retained by non-root canisters.
- Removed the old `icts_canister_status` endpoint from the shared Canic standards bundle and deleted the now-unused canister-status wrapper plus the stale status protocol constant.

### 🩹 Fixed

- Corrected an internal refactor regression where the root and non-root capability bodies had been inverted during the earlier auth/capability split, restoring the intended root-only verifier path and the intended non-root structural cycles path.

### 📊 Audit

- Raw release wasm sizes are now `2561000` for `minimal`, `app`, `scale`, and `shard`; `2613723` for `test`; `2622841` for `scale_hub`; `2721454` for `user_hub`; `2726527` for `shard_hub`; `3044753` for `user_shard`; and `17566441` for `root`.
- Relative to the previous measured baseline, this patch reduces raw release wasm by `105209` bytes on `minimal`/`app`/`scale`/`shard`, `104321` on `test`, `105206` on `scale_hub`, `105186` on `user_hub`, `105203` on `shard_hub`, `18340` on `user_shard`, and `4708` on `root`.
- `canister_minimal.wasm` no longer exports `icts_canister_status`, and after the capability split it also no longer imports the root-only `cost_sign_with_ecdsa` or `cost_create_canister` IC functions. Its function count is now `4551` and its data section is `206016` bytes.

### 📚 Documentation

- Updated the `0.17` status log with the corrected capability-path split, the new floor measurements, and the confirmation that root-only auth/management imports are no longer leaking into ordinary leaf canisters.

### 🧹 Cleanup

- Removed dead Canic standards status plumbing that no longer had any exported caller path after the endpoint removal, while leaving the separate root/admin `canic_canister_status` management surface in place.

## 0.17.0 - 2026-03-25 - Wasm Audit and Endpoint Surface Reduction

### 📝 Summary

- `0.17.0` starts the wasm audit line with a reproducible per-canister footprint baseline, fixes the baseline role naming on `minimal`, and moves the first optional endpoint groups behind compile-time config so disabled features no longer inflate every Canic build by default.

### 🔧 Changed

- Renamed the canonical baseline canister from `blank` to `minimal` across the active code, topology, audit runner, and current design docs so wasm reporting uses the same role name the architecture now expects.
- Added compile-time endpoint gating for scaling and sharding in the shared non-root endpoint bundle, using validated `canic.toml` role config to export those methods only for canisters that actually declare those capabilities.
- Added compile-time endpoint gating for delegated-token proof-install endpoints and the `ICRC-21` consent endpoint, using `auth.delegated_tokens.enabled` and `[standards].icrc21` as the authoritative config-to-ABI boundary.

### 📊 Audit

- Added the recurring Canic wasm-footprint audit definition and runner for built/shrunk artifact capture, `ic-wasm` structure snapshots, and `twiggy` hotspot attribution.
- Recorded the first `0.17` wasm audit baseline and a same-day comparison rerun under `docs/audits/reports/2026-03/2026-03-25/`, establishing the drift-comparison path for later `0.17` work.
- The first optional-endpoint reduction lowered shrunk wasm size by `109882` bytes for `app`, `scale`, `shard`, and `test`; `109913` for `user_shard`; `76549` for `scale_hub`; `32655` for `user_hub`; `32527` for `shard_hub`; and `260852` for `root`.
- The new shared-runtime floor is now `3104849` shrunk bytes for `minimal`, `app`, `scale`, and `shard`, which gives `0.17` a clean baseline for later shared-runtime attribution work.

### 📚 Documentation

- Added the `0.17` design and status tracking for the wasm audit line and its handoff to the template-canister cutover work in `0.18`.
- Recorded the first endpoint-surface reduction slice in the `0.17` status document so later wasm work has a dated audit trail, not just code diffs.
