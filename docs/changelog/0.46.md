## 0.46.1 - 2026-05-27 - deployment-target state hard cut

### Added

- Added first-install documentation for a minimal managed fleet, including the
  root/child lifecycle shape, subnet-scoped `canic.toml`, package-role mapping,
  managed-fleet test path, Candid surface expectations, and common install
  troubleshooting.
- Added `canic deploy register <deployment> --fleet-template <fleet> --root
  <principal> --allow-unverified` as the explicit operator recovery path for
  known live roots after the 0.46 deployment-target state hard cut.
- Added host `RegisterDeploymentStateOptions` and
  `register_deployment_state(...)` to write minimal deployment-target state
  without querying live inventory, copying receipts, recording artifacts,
  recording controller truth, installing code, or mutating canisters.
- Added `RootVerificationStatus` to local install state so explicitly
  registered roots can be persisted as `not_verified` instead of silently
  becoming deployment authority.

```bash
canic deploy register demo-local --fleet-template demo --root uxrrr-q7777-77774-qaaaq-cai --allow-unverified
```

### Changed

- Moved local install state from fleet-template paths to deployment-target
  paths: `.canic/<network>/deployments/<deployment>.json`.
- Updated local deployment plan and inventory collection to read root identity
  by deployment target name instead of configured fleet template name.
- Updated supplied-plan install validation to require exact deployment identity
  rather than accepting the old `plan.fleet_template == fleet_name` fallback.
- Stopped deleting other local deployment states that share the same root
  canister. Deployment targets are now distinct live targets even when they
  reuse a template or root.
- Updated deployment truth assumptions so unverified registered roots do not
  populate `trust_domain.root_trust_anchor` or expected root canister authority.
- Made the missing `canic::finish!()` compiler marker more actionable so the
  unresolved symbol tells canister authors to add `canic::finish!()` after all
  endpoints.
- Fixed current-install execution preflight so a fresh local fleet install is
  not blocked merely because no prior local root authority observation exists.
- Updated status/current handoff and root changelog to record the hard-cut
  behavior and recovery command.

### Removed

- Removed the old behavior that treated
  `.canic/<network>/fleets/<fleet>.json` as live deployment truth.
- Removed the old current-install supplied-plan compatibility fallback that let
  a plan match by fleet template instead of exact deployment target identity.

### Scope

- This is a hard-cut state-path change, not automatic migration. Legacy
  `.canic/<network>/fleets/*.json` state fails closed with recovery guidance.
- `canic deploy register` writes minimal local state only. It does not verify
  the root, crawl live inventory, copy receipts, infer artifacts, infer
  controller state, install code, or mutate canisters.
- Registered roots remain `not_verified`; future verification work must record
  live evidence before those roots can authorize mutation.

### Validation

- Added host coverage for deployment-state pathing, legacy fleet-state
  rejection, distinct deployment targets sharing a root, minimal unverified
  registration, and preventing unverified registered roots from becoming plan
  authority.
- Added CLI coverage for `canic deploy register` dispatch and minimal
  registration option construction.

### Testing

## 0.46.0 - 2026-05-26 - passive deployment comparison start

### Added

- Added `DeploymentComparisonReportV1`,
  `DeploymentComparisonTargetV1`, `DeploymentComparisonDiffV1`, and
  `DeploymentComparisonCategoryV1` as the first passive 0.46 multi-deployment
  artifact surface.
- Added `deployment_comparison_report_from_checks(...)` to compare two existing
  `DeploymentCheckV1` artifacts without querying live state or mutating
  deployments.
- Added `validate_deployment_comparison_report(...)` so archived comparison
  reports reject stale digest drift, schema drift, required-field gaps, and
  status/findings mismatches.
- Added host-owned passive text rendering for deployment comparison reports.
- Added `canic deploy compare --left <file> --right <file>` as the first 0.46
  operator command. It reads two `DeploymentCheckV1` JSON artifacts and emits
  `DeploymentComparisonReportV1` JSON by default, or host-owned passive text
  with `--format text`.

```bash
canic deploy compare --left staging-check.json --right prod-check.json
canic deploy compare --left staging-check.json --right prod-check.json --format text
```

### Changed

- Removed repo-pinned `@icp-sdk/icp-cli` and `@icp-sdk/ic-wasm` npm versions
  from dev setup and CI. Canic now installs the current upstream npm packages
  instead of managing those tool versions itself.
- Updated the 0.46 status/current handoff to record comparison as the first
  0.46 implementation slice and to keep catalog/group orchestration out of the
  initial release bar.
- Updated the current minimal managed fleet guide to use the current scaffold
  schema and `ic-cdk = "0.20"` dependency shape, with no legacy config aliases.
- Removed active recurring-audit references to removed standalone support
  crates so current process docs match the hard-cut crate topology.
- Updated the 0.46 design/status handoff to make deployment-target local state
  the required hard-cut foundation before group/catalog work continues. The
  design now forbids silent fleet-state fallback, requires exact supplied-plan
  deployment identity, and allows only explicit operator registration as a
  recovery path. Registration is scoped to minimal deployment state plus root
  verification status, not receipts, artifacts, controller snapshots, or other
  deployment truth.
- Moved tentative post-46 topic docs out of numbered future design directories
  and into the historical post-46 backlog archive, so deferred ideas no longer
  read as approved follow-on release lines.

### Scope

- Comparison is artifact-only in 0.46.0. It consumes existing deployment-truth
  checks and does not crawl live inventory, install code, apply authority,
  mutate controllers, or execute deployment work.
- External lifecycle comparison preserves the 0.45 handoff boundary: supplied
  observations, consent evidence, reported external action, and passive
  completion reports are evidence, not live deployment truth.

### Validation

- Added host-side coverage for drift detection, digest validation, and passive
  text rendering.
- Added CLI parse/build/help/source-guard coverage for `canic deploy compare`.
- Added CLI source guards proving the compare path has no live deployment-truth
  lookup and no mutation primitives.
- Added scaffold coverage pinning the current config schema, `ic-cdk = "0.20"`,
  and required `canic::finish!()` output.
