# 0.69 Changelog

## 0.69.4 - 2026-06-20 - Blob storage status DTO cleanup

### ✨ Added

- Adds `BlobStorageLocalCounters`, a named passive DTO for local
  blob-storage status wrappers. Downstream canisters can now expose stored
  blob, pending-deletion, and gateway-principal counts with stable field names
  instead of tuple-shaped Candid.

- Adds `BlobStorageApi::local_counters()` as the single facade helper for that
  DTO. The existing individual count helpers remain available for internal
  use.

- Updates the blob-storage integration runbook with a guarded host-canister
  status wrapper example using `local_counters()`. This remains host-owned and
  is not emitted by `canic_emit_blob_storage_endpoints!`.

### 🔧 Changed

- Changes the standalone `blob_storage_probe` count query to return
  `BlobStorageLocalCounters`, making the local PocketIC probe shape match the
  recommended downstream status wrapper.

- Documents and pins the create-certificate compatibility rule: the returned
  `CreateCertificateResult.blob_hash` echoes the request hash for gateway
  compatibility, while Canic stores and checks the canonical normalized root
  internally.

### 🥾 Governance

- Keeps local status counters separate from the gateway protocol surface. This
  patch does not add a Cashier endpoint, billing endpoint, public admin
  surface, or production gateway-principal sync path.

### 🧪 Testing

- Adds Candid round-trip coverage for `BlobStorageLocalCounters`.

- Adds a unit regression proving create-certificate echoes the request hash but
  registers the canonical normalized root used by liveness checks.

- Revalidates the blob-storage unit, facade, probe, macro, protocol-surface,
  clippy, inventory-gate, changelog-governance, and PocketIC lifecycle
  coverage for the updated status DTO shape.

## 0.69.3 - 2026-06-20 - Blob storage developer readiness

### ✨ Added

- Adds a focused blob-storage integration runbook for downstream canisters. The
  runbook documents `blob-storage` feature wiring, endpoint macro use, the
  create-certificate guard split, gateway-principal authorization, lifecycle
  helpers, canonical root-hash handling, product/backend flow, validation
  commands, and the 0.70 Cashier/billing boundary.

- Adds local blob-storage status counters through `BlobStorageApi`:
  `stored_blob_count`, `pending_deletion_count`, and
  `gateway_principal_count`. These are internal helper APIs for host-owned
  guarded status endpoints, not new gateway, Cashier, or public admin
  endpoints.

- Extends the standalone `blob_storage_probe` canister with a controller-only
  gateway-principal removal helper and a local count query for integration
  tests and downstream wrapper examples.

### 🔧 Changed

- Aligns the 0.69 design and handoff with the completed non-billing M0-M5 MVP
  state. The active design now reflects the implemented `BlobStorageApi`
  helper names and the status-counter pattern.

- Keeps the 0.69 release boundary explicit: downstreams can validate the
  non-billing backend and local gateway-principal state, but production Cashier
  sync, funding, billing status, frontend provisioning, and public admin
  surfaces remain outside 0.69.

### 🧪 Testing

- Expands PocketIC lifecycle coverage to prove gateway-principal removal
  revokes pending-deletion listing and makes deletion confirmation a no-op
  until the principal is re-added.

- Extends the same PocketIC flow to assert local blob-storage counters across
  create-certificate registration, pending deletion, gateway revocation,
  re-add, deletion confirmation, and upgrade.

- Revalidates focused blob-storage unit, facade, macro, clippy,
  protocol-surface, inventory-gate, changelog-governance, and PocketIC
  coverage.

## 0.69.2 - 2026-06-20 - Blob storage gateway lifecycle

### ✨ Added

- Adds the off-by-default `blob-storage` feature through `canic-core` and the
  facade crate.

- Adds the four non-billing immutable object-storage gateway protocol constants,
  Candid fixture coverage, and passive `CreateCertificateResult` DTO.

- Adds canonical `sha256:<64-hex>` blob-root validation, conversion from
  32-byte gateway inputs, stable live-blob and pending-deletion state, and
  gateway-principal membership state.

- Adds `BlobStorageApi` helpers for create-certificate registration, liveness
  checks, pending-deletion marking, gateway-scoped pending-deletion reads, and
  gateway deletion confirmation.

- Adds `canic_emit_blob_storage_endpoints!(guard = ...)`, which emits only the
  four non-billing `_immutableObjectStorage*` methods with exact exported wire
  names while keeping Rust-facing Canic helpers snake_case.

- Adds a `blob_storage_probe` standalone test canister and PocketIC coverage for
  the non-billing lifecycle: create-certificate registers a live blob, malformed
  liveness inputs return `false`, pending-deletion reads are gateway-filtered,
  non-gateway delete confirmation is ignored, and gateway confirmation clears
  pending state.

- Extends the same PocketIC flow across a post-upgrade round trip, proving
  live blobs, pending-deletion rows, and gateway-principal authorization survive
  stable-memory restore while liveness batches preserve input ordering and
  duplicate entries.

- Expands gateway pending-deletion coverage to prove multi-row response shape,
  stable returned ordering, and repeat-return behavior until confirmation.

### 🥾 Governance

- Keeps the separate Cashier inventory gate blocking billing/sync work so 0.70
  surfaces do not leak into the 0.69 non-billing implementation.

- Pins the generated endpoint authorization split: create-certificate remains
  the only host-guarded blob-storage endpoint, while gateway scrubber endpoints
  continue to delegate through gateway-principal-aware API helpers.

### 🧪 Testing

- Revalidates focused blob-storage unit, protocol, macro, canister-check,
  clippy, inventory-gate, protocol-surface auth, and PocketIC
  lifecycle/upgrade coverage.

## 0.69.1 - 2026-06-20 - ICP CLI 1.0 compatibility

### 🩹 Fixed

- Updates the shared host/CLI ICP CLI compatibility gate from the previous
  pre-1.0 range to `>=1.0.0, <2.0.0`, so Canic-backed commands accept the
  current stable `icp` binary.

- Keeps unsupported or unparseable ICP CLI binaries rejected before Canic
  dispatches the requested command, preserving the existing fail-fast behavior.

### 📚 Documentation

- Documents the ICP CLI identity-session commands operators can use to reduce
  repeated password prompts for password-protected PEM identities:
  `icp settings session-length <duration>` and
  `icp identity reauth <identity-name> --duration <duration>`.

- Adds the same identity-session guidance to `canic info medic` as a non-failing
  operator hint. This affects only the local ICP CLI identity session and does
  not change Canic canister auth or delegated-token behavior.

- Renders `canic info medic` as wrapped per-check blocks instead of a
  whitespace table, so long local-state diagnostics and next-action commands
  remain readable in ordinary terminals.

### 📊 Audit

- Confirms the active Canic command surface does not depend on the removed ICP
  CLI `--set-controller` flag.

- Confirms the active Canic command surface does not hardcode the old
  `icp0.io` gateway domain, while `icp project show` resolves the current
  implicit mainnet gateway through ICP CLI.

### 🧪 Testing

- Revalidates the focused host ICP CLI version/command-guard tests and the CLI
  guard test for unsupported ICP CLI output.

- Revalidates the `canic info medic` unit tests, including the new
  identity-session hint and wrapped block rendering, plus changelog governance,
  formatting, whitespace, and a local `canic info medic` smoke run.

## 0.69.0 - 2026-06-19 - Blob storage protocol preflight

### 📚 Documentation

- Starts the 0.69 blob-storage line as a protocol preflight release, not a
  backend implementation release.

- Documents the 0.69 boundary: four non-billing gateway endpoints belong to
  the later 0.69 implementation work, while Cashier sync and funding remain
  deferred to 0.70.

- Records the 0.69 implementation milestones in order: inventory unlock,
  protocol DTOs and Candid snapshots, internal model/storage, ops/workflow,
  endpoint macro emission, and mocked/PocketIC lifecycle coverage.

- Records the 2026-06-19 local source search result: no upstream immutable
  object-storage gateway implementation, generated `.did`, deployed `.did`, or
  source-backed Candid signatures were found in the checked local inputs.

### 🥾 Governance

- Keeps blob-storage implementation blocked until
  `docs/contracts/BLOB_STORAGE_INVENTORY.md` is complete from source-backed
  protocol evidence.

- Hardens the gateway inventory guard so an incomplete inventory rejects
  feature metadata, source/module paths, gateway method literals, public
  blob-storage API/model names, and premature billing/Cashier surfaces.

- Hardens the `Complete` inventory path so every required gateway method must
  include common source/Candid/DTO/auth fields, method-specific behavior
  fields, non-placeholder values, and a valid source commit SHA shape.

- Requires the Toko compatibility section to include local source, source
  commit, blob-root mapping, and migration/read-through strategy evidence
  before the inventory can unlock implementation.

- Leaves the separate Cashier protocol inventory gate in force so 0.70 billing
  work cannot leak into the 0.69 preflight line.

### 🧪 Testing

- Expands `protocol_inventory_gate` coverage for incomplete-inventory blocking,
  skeletal `Complete` method sections, placeholder evidence, invalid method
  source SHAs, unresolved Toko fields, missing Toko evidence, and invalid Toko
  source SHAs.

- Revalidates the focused inventory-gate suite plus both blob-storage gateway
  and Cashier gate scripts in their expected blocked state.

### 🧭 Migration Notes

- This release adds no `blob-storage` Cargo feature, DTOs, Candid snapshots,
  stable records, endpoint macro output, workflow, runtime API, or production
  blob-storage behavior.

- The first backend implementation release in the 0.69 line remains blocked on
  upstream gateway protocol evidence and a completed Toko compatibility answer.
