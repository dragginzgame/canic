# Canic 0.97 Changelog

## [0.97.0] - 2026-07-22 - Surface Inventory And Canonical Role Evidence

### 📝 Summary

- Closes the 0.96 receipt line at immutable `v0.96.8` with a `pass` verdict
  and re-anchors the accepted 0.97 design to that exact release identity.
- Completes audit-only Slice A and bounded Slice B without changing a Cargo
  package version, Candid surface, stable record, CLI command, or runtime
  feature selection.

### 📊 Audit

- Classifies every export family behind the public `canic::cdk` facade, every
  active repository consumer, every macro-emitted path, and every public
  prelude dependency. The only justified Canic-owned semantic relocations are
  cycle values under `dto::cycles` and public ICRC-21 protocol DTOs under
  `dto::icrc21`; other items move upstream, remain internal, or are removed.
- Inventories all 26 role-shaped package manifests and their purpose-bearing
  `canic::build!` edges. It freezes the current workspace-default leak,
  omitted explicit empty feature lists, one feature-bearing build edge, and
  the direct protected `canic-control-plane` sibling edge that Slice B must
  reject and remove.
- Proves the all-feature Canic normal closure contains exactly `canic`,
  `canic-core`, `canic-control-plane`, and `canic-macros`, with no additional
  allowed Canic-owned implementation package.
- Inventories build, deploy/install, Medic, state, internal PocketIC,
  built-in Wasm-store, discovery, release-package, and adoption-report Cargo
  callers and freezes which may establish role support.
- Freezes deterministic dependency-path source/path normalization and the
  complete positive, rejection, diagnostic, macro, and packaged-downstream
  fixture matrix required before validator changes.

### 🔧 Canonical Cargo Evidence

- Demonstrates that `cargo metadata --manifest-path <member>` does not isolate
  one role's activated graph in the virtual workspace: a feature-empty demo
  role receives the workspace union in metadata while package-selected Cargo
  tree evidence reports only its actual `default,metrics` activation.
- Corrects the accepted design before implementation. Canonical role evidence
  now intersects a complete metadata identity catalog, target-filtered
  metadata edges, and one package-selected normal Cargo tree per role. This
  excludes target-inactive tree rows while preserving exact role activation.
- Routes build, passive resolution, state-manifest resolution, and Medic
  through that one producer. Strict bounded parsing and correlation fail
  closed; normalized protected-path diagnostics contain neither raw Cargo IDs
  nor absolute paths.

### ⚠️ Hard Cut

- Requires resolver 2, disabled workspace Canic defaults, no workspace-owned
  Canic features, the exact dependency key `canic`, and an explicit feature
  list on every direct normal role edge. Renamed dependencies, unreviewed
  resolver versions, omitted feature lists, and role feature forwarding are
  rejected without aliases or fallback interpretation.
- Restricts the separate Canic build dependency to a feature-empty,
  unconditional edge justified by exactly one `canic::build!` invocation.
- Materializes every previously implicit `metrics` selection in the owning
  role manifest and removes the direct `delegation_root_stub ->
  canic-control-plane` edge. The stub uses the semantic Canic API/DTO facade,
  so the maintained runtime behavior remains selected by its one direct Canic
  declaration.
- Makes the generated built-in Wasm-store wrapper emit the same resolver,
  disabled-default, explicit-runtime-feature, and feature-empty build-edge
  contract. Graph comparison accepts only the role's direct selection and
  public feature implications declared by the Canic facade; internal semantic
  implications are verified separately and are not misclassified as another
  external feature owner.
