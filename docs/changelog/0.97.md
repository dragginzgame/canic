# Canic 0.97 Changelog

## [0.97.3] - 2026-07-22 - Root-Owned Manual ICP Refill

### 📝 Summary

- Makes manual ICP-to-cycles conversion an inherent root capability. The
  standard root bundle exposes one controller-guarded endpoint, derives both
  transfer source and cycle target as the executing root, and retains the
  existing ledger, CMC, replay, cost, and typed recovery authorities.
- This is a pre-1.0 hard cut. No generic feature, explicit endpoint emitter,
  public extension API, non-root capability, alias, old request reader, or
  compatibility route remains.

### ⚠️ Root Contract Hard Cut

- Moves the optional policy from `topup.icp_refill` to the root canister's
  `icp_refill` table. Policy presence enables manual conversion; the redundant
  `enabled` key is removed, and refill configuration on a non-root role is
  rejected.
- Narrows `IcpRefillRequest` to operation ID, optional source subaccount, ICP
  amount, and dry-run intent. Source/target principals and mode are no longer
  caller-controlled, while durable source and target fields remain as exact
  transfer and replay evidence.
- Emits `canic_icp_refill` from the standard root admin bundle with controller
  authorization. An unconfigured root rejects before replay reservation or an
  external call; no maintained non-root endpoint bundle exposes the method.
- Projects policy refusals through existing stable public classifications:
  invalid input for malformed or over-cap amounts, conflict for concurrent
  work, and unavailable for disabled, unconfigured, or rate-gated conversion.
- Makes memory ID 33 an inherent root allocation and initializes its indexes
  only through root lifecycle. Non-root lifecycle and metrics never open or
  project that allocation.
- Rejects root upgrades while any refill operation remains resumable, keeping
  the installed binary available to settle the operation before adoption of a
  changed contract.

### 🗑️ Removed

- Removes the `icp-refill` facade feature, role feature/capability projection,
  explicit endpoint macro, and public facade API.
- Removes canister/fabrication modes, local cycle fabrication, arbitrary
  source and target selectors, selected-role/canister routing, and mode-only
  dry-run output from `canic cycles convert`.
- Removes refill participation in direct-child grant and cooldown accounting,
  along with unproduced fabrication, request-denial, and persisted rate-gate
  error variants. The maintained pure exchange-rate policy violation remains.

### 🔧 Operator Surface

- The maintained command is
  `canic cycles convert <deployment> --icp-e8s <amount>`, with optional source
  subaccount, operation ID, dry-run, and JSON controls. It always resolves and
  calls the installed deployment root.
- Project-local pending evidence is keyed by environment, deployment, root,
  source subaccount, and amount. Unknown fields in the previous pending-log
  shape are rejected instead of silently interpreted.
- Active recovery guidance now requires the same derived root and immutable
  input. Operators must settle resumable or RecoveryRequired work before a
  root upgrade and must not bypass an uncertain transfer with a new operation
  ID.

### 🧪 Testing

- Core library coverage passes configuration parsing and rejection, role/state
  selection, replay binding, same-operation recovery, old non-self rejection,
  stable-record indexing, overflow boundaries, root upgrade admission, and
  the absence of automatic refill scheduling.
- Focused facade protocol tests pin the controller guard and narrowed named
  Candid records. CLI conversion, host role/state-manifest, generated-root
  compilation, strict targeted lint, formatting, layering, changelog, and diff
  checks pass.

## [0.97.2] - 2026-07-22 - Public CDK Hard Cut And Syn 3 Macro Parser

### 📝 Summary

- Deletes the human-facing `canic::cdk` facade after moving every maintained
  consumer to a semantic Canic owner or its direct upstream crate.
- Migrates the procedural macro parser from Syn 2 to Syn 3.0.3. Neither change
  alters macro expansion, Candid, stable state, CLI behavior, or runtime
  behavior.

### 📊 Audit Correction

- Direct source inspection found grouped imports such as
  `use canic::{cdk::...}` in test fleets, stubs, and `canic-tests` that the
  published Slice A lexical inventory did not match. The implementation does
  not trust that incomplete report: all grouped and fully qualified consumers
  are included in this hard cut.
- The corrected live inventory contains role API/time/caller/cycle helpers,
  test-only `Principal`, raw IC call/attribute use, internal Wasm hashing, the
  Canic README, and the managed-fleet guide. No additional application-facing
  facade consumer remains.

### ⚠️ Public CDK Hard Cut

- Removes `pub use canic_core::cdk` from the `canic` crate without an alias,
  deprecated module, forwarding macro, feature fallback, or duplicate path.
- Canister roles and test support import `Principal`, Candid derives and
  values, IC system APIs, attributes, calls, and traps from `candid` or
  `ic-cdk`. Testing-only Wasm hashing uses its existing direct `canic-core`
  dependency; the internal core CDK substrate is not a replacement app API.
- Removes obsolete Cargo Machete exceptions wherever the new direct upstream
  imports make those dependencies visible. The affected role and test package
  dependency sets pass focused unused-dependency analysis.
- Public rustdoc exposes no `canic::cdk` module. The exact doc-hidden
  `canic::__internal::cdk` compiler boundary remains limited to macro
  expansion.

### 🔧 Semantic Type Ownership

- Moves the ICRC-21 consent request, response, display, and error wire types
  from the internal CDK specification tree to `canic::dto::icrc21`; generated
  endpoint signatures and runtime dispatch use that single public owner.
- Re-exports `Cycles`, its typed parse/conversion errors, and the established
  cycle-unit constants from `canic::dto::cycles`, where public cycle DTOs
  already live.
- Updates current facade and getting-started documentation to teach
  `canic::api`, `canic::dto`, `canic::memory`, and direct upstream IC crates.

### 🔧 Dependency Maintenance

- The endpoint parser, validator, visitor, and expansion code require no
  source adaptation because their maintained API use remains compatible with
  Syn 3.
- Reuses the Syn 3 line already compiled for upstream procedural macros. Syn 1
  remains required by `binread_derive` and Syn 2 remains required by upstream
  macro crates, so the locked graph retains three distinct Syn versions rather
  than adding or removing one.

### 🩹 Role Contract Diagnostics

- Validates deterministic role manifest declarations before collecting the
  package-selected Cargo catalog and dependency tree, so graph inspection
  failures cannot shadow a more specific local contract rejection.
- Role fixture failures now include the unexpected diagnostic in assertion
  output to preserve the evidence needed to investigate future CI regressions.

### 🧪 Testing

- Strict all-target Clippy passes for the facade, core, macros, internal test
  support, integration tests, and all eleven migrated role packages.
- All 36 procedural-macro tests and all affected consumers pass; 26 focused
  endpoint, protocol, and reference tests plus the cycle DTO test pass.
  Semantic ICRC-21 and cycle imports are compiled and Candid-roundtripped.
- Fresh isolated rustdoc contains the semantic ICRC-21 and cycle modules and
  no public `canic::cdk` module. Focused Cargo Machete, formatting, changelog
  governance, lockfile, and diff-hygiene checks pass.

## [0.97.1] - 2026-07-22 - Hidden Macro CDK Boundary

### 📝 Summary

- Gives declarative and procedural macro expansion one exact hidden CDK
  boundary under `canic::__internal`, without exposing compiler plumbing as a
  human-facing framework facade.
- Keeps the public `canic::cdk` removal bounded to the next consumer-migration
  slice. This patch does not change Candid, stable state, endpoint behavior,
  CLI output, or Cargo package versions.

### 🔧 Macro Authority

- Routes lifecycle attributes, endpoint attributes, caller/time/cycle/status
  reads, controller checks, local test principals, and debug Candid export
  through the hidden boundary.
- Restricts that boundary to the frozen compiler inventory: six CDK macros,
  `Principal`, and five exact API functions. It does not re-export generic
  call, trap, print, future, Candid, or management conveniences.
- Uses definition-owned `$crate` paths for declarative expansion. Procedural
  expansion and Candid-visible root parameter types use the canonical
  `::canic::__internal` path required by downstream parsing and the enforced
  unrenamed Canic dependency contract.

### ⚠️ Hard Cut

- Removes generic `canister_self`, `msg_caller`, and `CandidType` imports from
  `canic::prelude`. Maintained demo roles import the required upstream IC API
  directly; no alias, compatibility prelude, or fallback remains.
- Does not yet remove the public `canic::cdk` facade. Active human consumers
  and documentation move in the next bounded slice before that facade is
  deleted once.

### 🩹 Locked Evidence Integrity

- Separates locked online build evidence from unlocked development evidence
  and locked-offline passive inspection. Internal PocketIC validation now
  matches its subsequent `cargo build --locked` operation exactly.
- Prevents role-package validation and its real-workspace tests from silently
  refreshing the repository lockfile while retaining the intentionally
  unlocked normal `canic build` development flow.

### 🧪 Validation

- All 36 procedural-macro tests, 25 focused facade/protocol/reference tests,
  and 22 role-package tests pass.
- Strict all-target Clippy passes for `canic`, `canic-macros`, and
  `canic-host`. Representative root, blob-storage, demo hub/shard, and
  built-in Wasm-store packages compile with the hidden macro boundary.
- The locked role-validation regression proves `Cargo.lock` remains
  byte-for-byte unchanged. Formatting and diff hygiene pass.

## [0.97.0] - 2026-07-22 - Surface Inventory And Canonical Role Evidence

### 📝 Summary

- Closes the 0.96 receipt line at immutable `v0.96.8` with a `pass` verdict
  and re-anchors the accepted 0.97 design to that exact release identity.
- Completes audit-only Slice A and bounded Slice B without changing a Cargo
  package version, Candid surface, stable record, CLI command, or runtime
  feature selection.

### 📊 Audit

- Classifies every export family behind the public `canic::cdk` facade, every
  active repository consumer, every macro-emitted path, and every public
  prelude dependency. The only justified Canic-owned semantic relocations are
  cycle values under `dto::cycles` and public ICRC-21 protocol DTOs under
  `dto::icrc21`; other items move upstream, remain internal, or are removed.
- Inventories all 26 role-shaped package manifests and their purpose-bearing
  `canic::build!` edges. It freezes the current workspace-default leak,
  omitted explicit empty feature lists, one feature-bearing build edge, and
  the direct protected `canic-control-plane` sibling edge that Slice B must
  reject and remove.
- Proves the all-feature Canic normal closure contains exactly `canic`,
  `canic-core`, `canic-control-plane`, and `canic-macros`, with no additional
  allowed Canic-owned implementation package.
- Inventories build, deploy/install, Medic, state, internal PocketIC,
  built-in Wasm-store, discovery, release-package, and adoption-report Cargo
  callers and freezes which may establish role support.
- Freezes deterministic dependency-path source/path normalization and the
  complete positive, rejection, diagnostic, macro, and packaged-downstream
  fixture matrix required before validator changes.

### 🔧 Canonical Cargo Evidence

- Demonstrates that `cargo metadata --manifest-path <member>` does not isolate
  one role's activated graph in the virtual workspace: a feature-empty demo
  role receives the workspace union in metadata while package-selected Cargo
  tree evidence reports only its actual `default,metrics` activation.
- Corrects the accepted design before implementation. Canonical role evidence
  now intersects a complete metadata identity catalog, target-filtered
  metadata edges, and one package-selected normal Cargo tree per role. This
  excludes target-inactive tree rows while preserving exact role activation.
- Routes build, passive resolution, state-manifest resolution, and Medic
  through that one producer. Strict bounded parsing and correlation fail
  closed; normalized protected-path diagnostics contain neither raw Cargo IDs
  nor absolute paths.

### ⚠️ Hard Cut

- Requires resolver 2, disabled workspace Canic defaults, no workspace-owned
  Canic features, the exact dependency key `canic`, and an explicit feature
  list on every direct normal role edge. Renamed dependencies, unreviewed
  resolver versions, omitted feature lists, and role feature forwarding are
  rejected without aliases or fallback interpretation.
- Restricts the separate Canic build dependency to a feature-empty,
  unconditional edge justified by exactly one `canic::build!` invocation.
- Materializes every previously implicit `metrics` selection in the owning
  role manifest and removes the direct `delegation_root_stub ->
  canic-control-plane` edge. The stub uses the semantic Canic API/DTO facade,
  so the maintained runtime behavior remains selected by its one direct Canic
  declaration.
- Makes the generated built-in Wasm-store wrapper emit the same resolver,
  disabled-default, explicit-runtime-feature, and feature-empty build-edge
  contract. Graph comparison accepts only the role's direct selection and
  public feature implications declared by the Canic facade; internal semantic
  implications are verified separately and are not misclassified as another
  external feature owner.

### 🧹 Root Manual Refill Inventory

- Freezes the complete generic refill feature, capability, config, API,
  endpoint macro, request, CLI, workflow, replay, stable-state, lifecycle,
  metrics, timer, and child-funding footprint for the later root-only hard
  cut.
- Confirms no maintained in-repository role enables the generic feature, no
  refill-specific timer exists, and the three removable stable error variants
  have no maintained producer. Downstream repositories remain read-only and
  receive no compatibility path.

### 🧪 Validation

- Canonical graph tests cover exact protected paths, multiple Canic packages,

- Targeted host, core, CLI Medic, state-manifest, and semantic-facade checks
