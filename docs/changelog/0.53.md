# 0.53 Changelog

## 0.53.0 - 2026-05-31 - single-envelope policy gate design and CLI cleanup

### Added

- Added the tentative 0.53 CI policy gate and project evidence manifest design:

  ```text
  docs/design/0.53-ci-policy-gates-project-manifests/0.53-design.md
  ```

- The design proposes passive policy evaluation over existing stable evidence,
  starting with a single-envelope gate:

  ```text
  canic evidence gate --policy <path> --envelope <path>
  ```

- The proposed line consumes:
  - 0.51 `EvidenceEnvelopeV1`;
  - 0.51 `ExitClassV1`;
  - 0.52 `canic.build_provenance.v1`.

- The first implementation slice is narrowed to envelope-level semantics:
  strict `CiPolicyV1` parsing, envelope schema checks, payload schema
  identity/stability allow-lists, evaluated exit-class checks, summary
  conflict/blocker/missing-evidence policy, and stable `PolicyGateReportV1`
  output.

- `PolicyGateReportV1` must distinguish the evaluated envelope's exit class
  from the gate result's exit class. Policy failure should preserve the
  evaluated evidence and report the gate's `ExitClassV1` separately.

- `--format json` should emit raw `PolicyGateReportV1`.
  `--format envelope-json` should wrap that report in a new
  `EvidenceEnvelopeV1` with policy-file and evaluated-envelope fingerprints as
  inputs. `--output <path>` should write the requested report artifact before
  returning a non-zero status when practical.

- Build-provenance field rules, including clean source, `Cargo.lock`, package
  identity, gzip Wasm, and SHA-256 requirements, are deferred until after the
  single-envelope gate semantics are proven. Project evidence manifests remain
  a later 0.53.x slice.

### Changed

- Moved fleet config inspection under the fleet command family and removed the
  old top-level command:

  ```text
  canic fleet config <fleet>
  ```

- Moved backup manifest validation under the backup command family and removed
  the old generic top-level manifest command:

  ```text
  canic backup manifest validate --manifest <file>
  ```

- Global network forwarding now reaches every `canic deploy ...` leaf that
  consumes deployment network state, including nested authority, external,
  root-verification, register, install, and resume-report commands.

### Removed

- Removed the top-level fleet-config command family:

  ```text
  canic config <fleet>
  ```

- Removed the top-level backup-manifest command family:

  ```text
  canic manifest validate --manifest <file>
  ```

### Safety

- This is design-only for the policy gate. It does not add runtime policy-gate
  behavior.
- The design explicitly defers deployment locks, signing, provider wrappers,
  artifact registry import, `wasm_store` retention/GC, controller mutation,
  topology mutation, active adoption/import, install/upgrade authority, and
  broad live verification.
