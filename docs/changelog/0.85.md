# Canic 0.85 Changelog

## [0.85.0] - 2026-07-11 - Operational Safety Hardening

### 📝 Summary

0.85.0 completes the bounded operational-safety line with three hard cuts:
durable restore recovery persistence, explicit command-local build authority,
and maintained Canic-owned CBOR encoding backed by exact compatibility proof.
It also adopts the current `ic-memory 0.10` diagnostic model without retaining
an adapter for the former shape.

Restore plans and journals can no longer be truncated in place during a
mutating restore transition. Canister builds no longer depend on unsafe
process-global environment mutation. Canic's stable and IC-wire encoders no
longer directly use the unmaintained `serde_cbor` crate. Existing stable bytes
and local-replica wire bytes are unchanged.

### 🔧 Changed - Durable Recovery Persistence

- One backup-owned persistence path serializes complete recovery documents,
  writes a unique sibling staging file, synchronizes its bytes, renames it over
  the destination, and synchronizes the parent directory.
- Backup layouts, restore plans, restore journals, and every mutating runner
  transition use that same durable replacement path.
- Serialization and pre-rename failures preserve the previous valid recovery
  document, while abandoned staging files are cleaned up on failure.
- The CLI receives typed plan and journal persistence entrypoints instead of
  writing backup-owned recovery documents through generic output helpers.

### 🔧 Changed - Explicit Build Authority

- `WorkspaceBuildContext` is the single host-owned authority for the role,
  profile, workspace and ICP roots, config path, selected environment,
  resolved build network, and optional direct-local replica target.
- Cargo and ICP child commands receive the resolved context explicitly.
  Build display and provenance consume the same value, so sequential builds
  cannot inherit a previous invocation's config or network authority.
- Canonical callers construct and pass the build context directly; lower-level
  build and ICP target helpers no longer infer authority from process-global
  environment state.

### 🔧 Changed - CBOR Ownership

- The core stable serialization adapter, replay receipts, and the private host
  replica-query/status adapter now use `ciborium 0.2.2`.
- Exact differential proof across the maintained core and control-plane stable
  owners demonstrated byte-for-byte equality before the legacy encoder was
  removed. Rich serde-shape and replay-receipt byte goldens remain permanent.
- Exact anonymous-query, replied-response, rejected-response, status-root-key,
  and malformed-input fixtures preserve the existing local replica wire
  contract.
- Canic manifests and codec call sites no longer directly depend on or invoke
  `serde_cbor`. Published IC signature, agent, transport, and PocketIC crates
  still select it transitively, so the workspace RustSec warning remains an
  upstream dependency issue rather than a Canic-owned codec path.

### 🔧 Changed - Memory Diagnostics

- Canic now consumes `ic-memory 0.10` enum-based empty, valid, and invalid
  commit-slot diagnostics plus its combined recovery result.
- Retirement generations come from the current allocation state, and invalid
  commit slots are exposed as the exact `InvalidCommitSlots` recovery result
  instead of being collapsed to `Unknown`.

### ⚠️ Breaking

- The CLI and install build-environment guards, unsafe mutation/restoration,
  internal build override, and local-target environment fallback are removed
  without aliases or compatibility paths.
- Host callers of the build and ICP target support APIs must pass a
  `WorkspaceBuildContext` and any direct-local target explicitly.
- The stable serde macros and replay storage hard-cut to `ciborium`; there is no
  dual decoder or migration fallback because every maintained compatibility
  fixture produced identical bytes.
- Memory diagnostics hard-cut to the `ic-memory 0.10` shapes. The public Candid
  recovery enum adds `InvalidCommitSlots`, so generated clients should refresh
  their bindings.

### 🗑️ Removed

- Fixed `.tmp` recovery writes and direct truncating plan/journal writes.
- Both process-global build environment guards and their unsafe environment
  mutation paths.
- Direct Canic-owned `serde_cbor` dependencies and source call sites.
- Compatibility adapters for the previous `ic-memory` diagnostic model.

### 🧭 Migration Notes

- No stable-memory migration is required for the CBOR change: the maintained
  stable owner suites produced exact legacy/current bytes, and memory IDs,
  record bounds, and persisted schemas are unchanged.
- Host integrations that called lower-level build helpers must construct the
  explicit build context and pass local replica targeting as typed input.
- Candid consumers of memory recovery diagnostics should regenerate bindings
  for the new `InvalidCommitSlots` variant.
- `serde_cbor 0.11.2` remains in the workspace dependency graph through current
  published upstream IC and PocketIC releases. This release removes Canic's
  direct ownership; it does not claim lockfile-wide removal of the crate or its
  advisory.

### 🧪 Testing

- Focused durability coverage verifies complete replacement, preservation on
  serialization and rename failure, cleanup, backup layout round trips, and
  mutating restore journal transitions.
- Focused build coverage verifies explicit child environments, direct-local
  targeting, sequential no-leakage, install command wiring, and provenance
  parity; targeted host and CLI Clippy checks pass with warnings denied.
- Temporary dual-codec gates enforced byte equality across 62 default core,
  115 all-feature core, and 38 control-plane storage tests before removal of
  the legacy encoder. Permanent rich-shape and replay byte goldens pass.
- Exact host IC-wire fixtures, default and all-feature core checks, the Wasm
  core check, targeted control-plane checks, and targeted Clippy pass.
- Focused `ic-memory 0.10` projection, recovery classification, and Candid
  round-trip coverage passes for empty, valid, invalid, and retired state.
