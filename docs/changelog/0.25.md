## 0.25.7 - 2026-04-06 - Dynamic Stable-Memory API and PocketIC Hardening

### 📝 Summary

- `0.25.7` closes the gap between macro-only stable-memory wiring and runtime-selected slot usage by giving `canic-memory` a small supported public `MemoryApi`, while also hardening the shared `canic-testkit` PocketIC baseline path so dead cached instances rebuild cleanly instead of cascading into teardown failures.

### 🔧 Changed

- Added [MemoryApi](/home/adam/projects/canic/crates/canic-memory/src/api/mod.rs) under `canic_memory::api` with the minimum downstream contract:
  - `MemoryApi::bootstrap_registry(...)` to reserve the owned range and run memory bootstrap
  - `MemoryApi::register_memory(...)` to register one runtime-selected slot and get its `VirtualMemory`
- Kept `MEMORY_MANAGER` hidden while moving the supported dynamic-slot entry point into `canic-memory` itself, so downstream crates no longer need to import manager internals just to open one `VirtualMemory`.
- Made repeated `register_memory(...)` calls idempotent when the owner and label already match the canonical existing entry, while preserving the existing duplicate-ID error for conflicting registrations.
- Updated [README.md](/home/adam/projects/canic/crates/canic-memory/README.md) with a runtime-selected memory-slot example and the matching bootstrap path.
- Added shared cached-baseline recovery in [baseline.rs](/home/adam/projects/canic/crates/canic-testkit/src/pic/baseline.rs) so cached PocketIC restores now rebuild only for the known dead-instance transport failure class instead of blindly swallowing unrelated restore panics.
- Centralized dead-instance transport detection in [startup.rs](/home/adam/projects/canic/crates/canic-testkit/src/pic/startup.rs), and reused that shared path from the attestation and cached-root harnesses in [attestation.rs](/home/adam/projects/canic/crates/canic-testing-internal/src/pic/attestation.rs) and [harness.rs](/home/adam/projects/canic/crates/canic-tests/tests/root/harness.rs).
- Continued the `canic_testkit::pic` ownership split by moving snapshot, diagnostics, call, and lifecycle implementations out of [mod.rs](/home/adam/projects/canic/crates/canic-testkit/src/pic/mod.rs) into:
  - [snapshot.rs](/home/adam/projects/canic/crates/canic-testkit/src/pic/snapshot.rs)
  - [diagnostics.rs](/home/adam/projects/canic/crates/canic-testkit/src/pic/diagnostics.rs)
  - [calls.rs](/home/adam/projects/canic/crates/canic-testkit/src/pic/calls.rs)
  - [lifecycle.rs](/home/adam/projects/canic/crates/canic-testkit/src/pic/lifecycle.rs)
- Kept the public `canic_testkit::pic::*` surface stable while cutting the main [mod.rs](/home/adam/projects/canic/crates/canic-testkit/src/pic/mod.rs) hub down again and making the baseline/debug/snapshot responsibilities easier to reason about by module.

```rust
use canic_memory::api::MemoryApi;

MemoryApi::bootstrap_registry("my_crate", 10, 19)?;
let memory = MemoryApi::register_memory(12, "my_crate", "CommitMarker")?;
```

## 0.25.5 - 2026-04-06 - Shared Runtime and Surface Trim

### 📝 Summary

- `0.25.5` keeps the `0.25` follow-through focused on shared runtime and cleaner boundaries: the default demo/reference surface is slimmer, the internal test/audit build path is drier, and the retained instruction lane lands two more small measured cuts without adding a new transport or compatibility layer.

### 🔧 Changed

- Gated `canic_metrics` out of the default demo/reference bundle, so ordinary demo canisters no longer ship the shared metrics query surface unless internal test endpoints are explicitly enabled.
- Gave `wasm_store` a dedicated slimmer runtime bundle instead of inheriting the generic shared endpoint set, removing the old carryover endpoints like `canic_env`, `canic_memory_registry`, `canic_metrics`, directory queries, and generic log/cycle/children helpers from its default contract.
- Centralized the internal PocketIC/test/audit wasm-build environment in `canic-testkit` artifact helpers, removing the repeated `CANIC_INTERNAL_TEST_ENDPOINTS=1` build wiring from the standalone, lifecycle, attestation, root, and core PocketIC helper/test paths.
- Added a small raw-response equality helper to the IC call façade and used it to fast-path the fixed success payload for delegation proof installs, so verifier proof pushes no longer pay full Candid decode cost on the common `Ok(())` case.
- Simplified cycles replay payload hashing so the non-root/root cycles capability path now uses the shared field-hashing helpers directly instead of cloning a canonical request and re-encoding it through a special `RootCapabilityCommand` path just to compute the replay hash.

### 📊 Audit

- The refreshed capability-surface report at [capability-surface.md](../audits/reports/2026-04/2026-04-05/capability-surface.md) now reflects the slimmer default roster, including the reduced `wasm_store` contract and the ordinary leaf baseline falling to `11` exported `canic_*` methods.
- The retained instruction rerun at [instruction-footprint-17.md](../audits/reports/2026-04/2026-04-05/instruction-footprint-17.md) shows the proof-install response fast path cut sampled `root::canic_request_delegation:fresh-shard` from `1768507` to `1734579`, with `decode_response` dropping from `251735` to `214079`.

## 0.25.3 - 2026-04-05 - Delegation Hot-Path Follow-Through

### 📝 Summary

- `0.25.3` continues the `0.25` runtime follow-through with another measured cut on the delegated-auth issuance path, keeping the work focused on local auth/replay overhead instead of broader transport or demo-flow redesign.

### 🔧 Changed

- Replaced the compact successful `DelegationIssued` replay payload path with a small internal binary codec, so the root replay cache stops paying generic serializer overhead for the hottest delegation replay shape.
- Added the same style of compact internal replay encoding for cached `CyclesResponse`, so the root cycles replay path no longer stores and decodes the tiny success shape through the generic `Response` codec.
- Switched delegation cert hashing to direct deterministic field hashing instead of re-encoding the cert before hashing, which removes more avoidable auth-path work from both proof issuance and verifier-proof cache keying.
- Tightened the verifier-local proof cache path by avoiding redundant shard-public-key rewrites and unnecessary proof-record cloning when replacing an existing keyed proof entry.
- Removed the root public-key prewarm from `sign_delegation_cert(...)`, so the root signing path only resolves what it actually needs to sign and leaves verification-side key fetches to the verifier cache path.

### 📊 Audit

- The retained rerun at [instruction-footprint-11.md](../audits/reports/2026-04/2026-04-05/instruction-footprint-11.md) cut sampled `root::canic_request_delegation:fresh-shard` from `1993625` in `instruction-footprint-10` to `1770255`, and `commit_replay` dropped out of the delegation hotspot table after the replay compaction work.
- The follow-up retained rerun at [instruction-footprint-12.md](../audits/reports/2026-04/2026-04-05/instruction-footprint-12.md) cut sampled `root::canic_response_capability_v1` `cycles-request` from `1481137` to `601860`, and the old cycles `commit_replay` hotspot dropped out after the compact cached cycles response path landed.
- The remaining main delegation stages are now `sign_cert`, `encode_install_request`, `decode_response`, and the smaller `cache_root_verifier_proof` path, so the easy replay-side savings are largely exhausted.

## 0.25.1 - 2026-04-05 - Auth Decomposition and Demo/Test Split

### 📝 Summary

- `0.25.1` follows the audit refresh with the first structural reduction pass: the main auth/runtime hotspots are split into smaller modules, and the reference demo canisters are separated more strictly from internal test-only canisters and helper flows.

### 🔧 Changed

- Split the old monolithic `storage/stable/auth.rs` into focused modules for auth records, proofs, sessions, and key state while preserving the existing `DelegationState` facade.
- Split `access/auth.rs` into explicit auth-phase modules so identity/session resolution, token verification, and predicate evaluation are no longer concentrated in one file.
- Split `access/expr.rs` and `ops/runtime/metrics/auth.rs` into directory modules so auth expression evaluation and auth metrics families are easier to reason about and extend without accreting more unrelated logic in one place.
- Moved the `test` role out of `canisters/` and into `crates/canic-core/test-canisters/`, so the default demo/reference topology no longer auto-creates or advertises a test helper canister.
- Split internal correctness fixtures from audit probes by keeping test-only canisters under `crates/canic-core/test-canisters/` and moving audit-only probes under `crates/canic-core/audit-canisters/`.
- Removed the debug-only root helper endpoints from the demo `root` canister and aligned the demo configs, DFX surface, and reference-canister lists with the slimmer topology.
- Added fallible PocketIC setup paths to public `canic-testkit`: downstreams can now use `try_pic()`, `PicBuilder::try_build()`, and `try_install_prebuilt_canister_with_cycles(...)` to skip cleanly when PocketIC cannot start instead of preflighting `POCKET_IC_BIN` themselves.
- Hardened the PocketIC process-lock path so `std::env::temp_dir()` and its parent chain are created before `canic-pocket-ic.lock`, which keeps repo-local `TMPDIR` paths usable without downstream temp-root preflight glue.

### 🧪 Testing

- `make test` passed after the split, covering the workspace gates plus the root hierarchy, reconcile, and PocketIC auth suites against the new module layout and demo/test boundary.
- `cargo clippy -p canic-testkit --all-targets -- -D warnings` and `cargo test -p canic-testkit` passed after the new fallible testkit setup surface and temp-root lock-path coverage were added.

### 🧭 Migration Notes

- The default demo topology is now intentionally smaller: `test` is no longer part of the standard reference canister set and only appears through internal test configs and internal test canister crates.
- Downstream or local scripts that assumed a default root helper like `create_minimal` or a default `test` demo canister should now use explicit internal test fixtures or pass an explicit bench method instead.
