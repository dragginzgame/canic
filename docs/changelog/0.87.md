# Canic 0.87 Changelog

## [0.87.2] - 2026-07-13 - Typed Diagnostic Closeout

### 📝 Summary

0.87.2 closes the final local ICP diagnostic-classification leak found by the
post-release audit. Install-root keeps the original typed command failure
through root-canister resolution and consumes the same centralized diagnostic
classification as every other host consumer.

Creation, retry, rendered diagnostics, command exits, configuration, and
deployment behavior are unchanged. The separately documented 0.88 line begins
with three new audit findings rather than carrying unfinished 0.87 work.

### 🩹 Fixed

- Missing root-canister IDs are recognized through
  `IcpDiagnostic::CanisterIdMissing` instead of a command-local string helper.
- All four maintained external ICP wording forms for the condition are owned
  and tested by the host ICP adapter.
- Root-canister resolution retains `IcpCommandError`, including its original
  I/O, JSON, version, and process-output sources, through diagnostic
  classification.

### 🧹 Cleanup

- The install-root string classifier, boxed-error conversion, and duplicate
  classifier test are removed without aliases or compatibility paths.
- The 0.87 design and handoff now record all three planned slices as complete
  within the 0.87 line.

### 📊 Audit

- The fresh post-0.87 audit finds no remaining production ICP wording outside
  the centralized host adapter.
- It selects exactly three bounded 0.88 candidates: durable backup artifact
  commits, failure-atomic CLI file output, and typed fleet-config errors.
- Broad visibility churn, dependency forks, filesystem frameworks, and global
  error architectures are explicitly excluded from that next line.

### 🧪 Testing

- Focused host diagnostic and install-root parsing tests pass.
- Targeted host Clippy passes with warnings denied.
- Formatting, changelog governance, layering guards, and diff hygiene pass.
- Full workspace, PocketIC, deployment, and release suites were not run under
  the targeted testing policy.

## [0.87.1] - 2026-07-13 - Typed Diagnostics and Explicit Build Inputs

### 📝 Summary

0.87.1 completes the bounded operator-boundary hygiene line. External ICP CLI
diagnostics now have one typed host classifier, command errors retain their
original I/O, JSON, version, and process-output sources, and duplicated
command-side error reconstruction is removed.

The environment-input audit hard-cuts public shortcuts that duplicated typed
command inputs, Cargo metadata, or canonical project discovery. Required
Cargo/build-script handoffs remain private and have centrally owned names.

### 🔧 Changed

- ICP diagnostic interpretation is owned by the host ICP adapter. CLI commands
  retain typed failures and continue to own only their command-specific exit
  codes, context, and actionable hints.
- Canister build profiles and project paths are selected through typed command
  arguments or canonical discovery rather than parallel environment
  precedence.
- Advanced Cargo callers use standard `CARGO_TARGET_DIR`; the default Canic
  Wasm target remains bounded and install/workspace-test cleanup is
  deterministic.
- Maintainers refresh the canonical Wasm-store DID through the low-level
  builder's explicit `--refresh-wasm-store-did` argument. The argument is
  rejected for any role other than `wasm_store`.
- Config path, ICP root, role validation, and embedded release-artifact
  requirements cross Cargo boundaries only through private, core-owned
  constants.

### 🗑️ Removed

- The public `CANIC_WASM_PROFILE`, `CANIC_WORKSPACE_ROOT`, `CANIC_ICP_ROOT`,
  `CANIC_CONFIG_PATH`, `CANIC_CANISTERS_ROOT`,
  `CANIC_WORKSPACE_MANIFEST_PATH`, and `CANIC_ROOT_MANIFEST_PATH` inputs are
  removed without aliases or fallback precedence.
- `CANIC_KEEP_WASM_BUILD_CACHE`, `CANIC_WASM_TARGET_DIR`, and
  `CANIC_REFRESH_WASM_STORE_DID` are removed without compatibility readers.
- Three unread child-build values for the local replica URL, local root key,
  and duplicated build environment are deleted rather than documented as
  supported configuration.
- Host tests no longer mutate process environment or require module-local
  environment locks and restoration helpers.

### 📊 Audit

- Thirteen public or dead environment shortcuts are removed. Remaining
  product-level `CANIC_*` inputs are required compiler, Cargo/build-script, or
  test-only process-boundary values.
- Code and automation are 558 net lines smaller than `0.87.0`; the complete
  source and documentation batch is 473 net lines smaller.
- The closeout scan found no removed environment names, repeated consumer ICP
  classifiers, repeated command/string translators, or host environment
  mutation helpers in active code.

### 🧪 Testing

- Focused host cache, build-context, bootstrap-store, ICP adapter, and
  installed-deployment tests pass.
- Focused CLI command-family tests preserve authentication, blob-storage,
  deployment, backup, metrics, replica, snapshot, and Medic exit and hint
  behavior.
- Targeted checks and Clippy with warnings denied pass for the affected core,
  runtime, host, CLI, and internal testing crates. The maintainer artifact
  example, shell syntax, formatting, changelog governance, and diff hygiene
  also pass.
- Full workspace and PocketIC suites were not run under the targeted testing
  policy.

### ⚠️ Breaking

This is a pre-1.0 hard cut. Removed environment names have no aliases,
deprecation readers, hidden precedence, or compatibility behavior. Operators
must use the maintained command arguments, project discovery, or standard
Cargo target selection described above.

## [0.87.0] - 2026-07-13 - Failure-Atomic Scaffolding

### 📝 Summary

0.87.0 prevents failed fleet or canister scaffolds from leaving a partially
created local project that blocks a clean retry. Existing project documents
are replaced durably, and one bounded rollback path restores their original
bytes and removes only the new scaffold directory.

Successful generated files, workspace membership, fleet configuration,
package identity, command forms, and deployment behavior are unchanged. The
release does not change or remove `CANIC_*` environment variables.

### 🩹 Fixed

- A failed fleet scaffold removes the new fleet directory instead of leaving
  an incomplete target that the next invocation treats as intentional state.
- A failed canister scaffold restores the original workspace manifest and
  fleet configuration before removing the new role directory.
- Rollback attempts every required cleanup action. If cleanup also fails, the
  reported error retains both the original operation failure and the rollback
  failure.

### 🔧 Changed

- Scaffold workspace-manifest replacement uses the existing host-owned durable
  writer instead of truncating the destination file in place.
- Rollback may recursively remove only a directory that scaffold preflight
  proved did not exist before the command began.
- Existing workspace and fleet documents are restored from the exact bytes
  captured before the first mutation.

### 🧹 Cleanup

- The two affected canister manifests now suppress only dependency-scan false
  positives caused by macro-derived Serde and CDK use; direct Candid use is no
  longer incorrectly ignored.
- No filesystem transaction framework, journal, recovery command, fallback
  writer, compatibility alias, or resumable partial-scaffold state is added.
