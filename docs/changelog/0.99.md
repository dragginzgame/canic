# Canic 0.99 Changelog

## [0.99.0] - 2026-07-23 - Reinstall-Only Minimum App Config Hard Cut

### ⚠️ Breaking

- 0.99 is fresh-install/reinstall-only. It does not upgrade or migrate 0.98
  Canister stable state, installed-state trees, deployment catalogs, or
  authorization records.
- The top-level `app_index` configuration field is removed. App-declared
  Fleet service roles now use:

  ```toml
  [services.fleet]
  roles = ["project_hub"]
  ```
- `[fleet].name` and `FleetConfig` are removed. Every maintained config now
  declares its required immutable source identity as `[app].name`.
- The source-topology type and reserved slot are hard-cut from
  `SubnetRole::PRIME` / `subnets.prime` to
  `SubnetSlotId::DEFAULT` / `subnets.default`.
- Keyed placement config is `binding`; the old `directory` config key and
  `DirectoryConfig` type are removed.

### 🔧 Changed

- `AppId`, `AppRoleRef` and `AppConfigSnapshot` are the source-config identity
  surfaces. Parsing, validation, build support and auth provenance derive the
  exact App identity from one required field.
- Configuration parsing and validation consume one nested Fleet-service
  declaration and one keyed placement binding while preserving current
  one-member placement and runtime Directory behavior.
- Generated compiled configuration, checked-in canister/Fleet configs,
  scaffolding, host role mutation, verbose config diagnostics, CI fixtures,
  and active operator guidance emit the current App/default/binding shape.
- Config-derived diagnostic sources use `app_config`; no `fleet_config`
  source label remains in the changed planning and medic paths.
- The 0.99 and 0.100 contracts now derive future work from completed
  fresh-install evidence rather than a nonexistent 0.98 migration receipt.

### 🧪 Testing

- Focused Canic-core configuration/bootstrap tests, Canic-host release-set and
  ICP-config tests, Canic CLI scaffold tests, Canic config/reference-surface
  tests, targeted package checks, formatting and strict Clippy pass for the
  changed packages.
- No Cargo package version, dependency, Candid endpoint label, JSON, CLI
  command, or stable allocation changes. The Rust config API and source TOML
  schema intentionally break; reinstall-only means no old slot value or
  config shape is decoded.
