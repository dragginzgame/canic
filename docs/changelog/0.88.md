# Canic 0.88 Changelog

## [0.88.0] - 2026-07-13 - Durable Backup Artifact Publication

### 📝 Summary

0.88.0 makes snapshot artifact-directory publication genuinely durable. Both
direct snapshot capture and planned backup finalization use one backup-owned
commit operation that synchronizes the complete artifact tree, publishes it
without replacing an existing destination, and synchronizes the owning
directory before the journal reports durable state.

An interruption after publication but before the durable journal transition
can be resumed safely. Recovery accepts only the journal-bound canonical
artifact after reverifying its checksum and resynchronizing its tree; unrelated
or unverifiable destinations continue to fail closed.

### 🔧 Changed

- Artifact traversal opens entries without following symlinks and accepts only
  regular files and directories.
- Regular files are synchronized before nested directories, the temporary
  artifact root, and finally the owning parent directory.
- Publication uses an atomic no-replace operation rather than an
  existence-check followed by an ordinary rename.
- Platforms that cannot provide the required publication and directory-sync
  semantics fail explicitly instead of using a weaker fallback.
- The existing directory-checksum algorithm can consume already-open file
  descriptors, preserving its hashes and serialized contracts while recovery
  avoids reopening inspected entries by path.

### 🩹 Fixed

- A crash after the final rename no longer leaves the backup permanently
  blocked by its own complete artifact.
- A failed final-parent synchronization leaves a complete, verifiable artifact
  that can be safely resynchronized on retry.
- A failed durable journal write exposes neither `ArtifactState::Durable` nor
  an incremented artifact-finalization completion metric in memory.
- Planned finalization rejects any artifact state below `ChecksumVerified`
  before attempting publication.

### 🧹 Cleanup

- Both previous artifact-finalization writers are removed. No compatibility
  writer, filesystem framework, or alternate recovery path remains.
- Directory commit policy remains private to backup persistence and is not
  shared with the separate 0.88 single-file output slice.

### 🧪 Testing

- Focused commit tests cover successful publication, exact recovery, symlink
  and unsupported-entry rejection, atomic no-replace races, and failures at
  every file, directory, publication, and final-parent persistence boundary.
- Focused direct-capture tests preserve successful output and prove destination
  conflicts and journal-write failures cannot report durable completion.
- Focused runner tests prove exact post-publication recovery and checksum
  rejection without replaying completed snapshot commands.
- Targeted `canic-backup` Clippy passes with warnings denied. Formatting,
  changelog governance, dependency hygiene, layering guards, and diff hygiene
  pass.
- Full workspace, PocketIC, deployment, and release suites were not run under
  the targeted testing policy.
