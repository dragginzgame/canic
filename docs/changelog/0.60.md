# 0.60 Changelog

## 0.60.3 - 2026-06-04 - Catalog-derived estimate source

### 📝 Summary

- Wired the cached mainnet subnet catalog into instruction-audit
  `execution_cycle_estimate` decoration as an optional source. The audit still
  performs no live NNS lookup; operators refresh the catalog explicitly with
  `canic subnet catalog refresh` before using it for estimates.

- Preserved source precedence. Explicit
  `--cycles-per-billion-instructions` wins over everything, explicit
  `--estimate-node-count` wins over the catalog, and catalog-derived estimates
  are omitted when the cache is missing, stale by default, unresolved, missing
  a usable node count, or resolved to a non-application subnet.

### ➕ Added

- Added catalog-derived instruction-audit estimate command forms:

  ```text
  bash scripts/ci/instruction-audit-report.sh --estimate-execution-cycles --estimate-canister-principal <canister-principal>
  bash scripts/ci/instruction-audit-report.sh --estimate-execution-cycles --estimate-canister-principal <canister-principal> --allow-stale-subnet-catalog
  bash scripts/ci/instruction-audit-report.sh --estimate-execution-cycles --estimate-canister-principal <canister-principal> --subnet-catalog-stale-after <duration>
  ```

- Added optional catalog provenance fields to estimate artifacts when the
  catalog supplies the rate: registry canister ID, registry version, subnet
  principal, subnet kind/specialization/geography with classification sources,
  catalog schema version, stale flag, resolver backend, matched canister
  principal, and matched routing range.

### 🔧 Changed

- Catalog-derived rates now accept any positive cached application-subnet
  `node_count` and use integer ceil scaling from the 13-node baseline:

  ```text
  catalog_cycles_per_billion_instructions = ceil(1_000_000_000 * node_count / 13)
  ```

- Kept the older explicit `--estimate-node-count` table pinned to `13` and
  `34` unless an explicit rate is supplied. This means the explicit 34-node
  table value remains `2_615_384_615`, while the catalog-derived 34-node ceil
  formula records `2_615_384_616`.

### 🧭 Notes

- Catalog-derived estimates read
  `.canic/subnet-catalog/ic/catalog.json` through the existing host/catalog
  boundary. Protobuf transport and generated registry types remain quarantined
  inside `canic-ic-registry` and are not used by the instruction-audit report
  path.

- Stale catalog data is omitted by default. Operators must pass
  `--allow-stale-subnet-catalog` to use stale cached metadata, and the emitted
  estimate records `catalog_stale = true`.

### 🧪 Testing

- Focused validation for this patch includes:

  ```text
  cargo fmt --all
  cargo test -p canic-tests --test instruction_audit estimates -- --nocapture
  cargo test -p canic-tests --test instruction_audit -- --nocapture
  cargo clippy -p canic-tests --test instruction_audit
  cargo test -p canic --test changelog_governance -- --nocapture
  bash scripts/ci/instruction-audit-report.sh --help
  bash -n scripts/ci/instruction-audit-report.sh
  make fmt-check
  git diff --check
  ```

## 0.60.2 - 2026-06-04 - Live subnet catalog refresh

### 📝 Summary

- Added live mainnet subnet catalog refresh behind a shared
  `canic-ic-registry` crate. The adapter owns NNS registry transport,
  protobuf request/response encoding, protobuf value decoding, registry key
  construction, and conversion into `canic-subnet-catalog` domain structs.

- Kept protobuf quarantined. `canic-host` and `canic-cli` receive Canic-owned
  catalog structs and reports only; no protobuf transport type crosses the
  shared registry adapter boundary.

### ➕ Added

- Added the live refresh command:

  ```text
  canic subnet catalog refresh
  canic --network ic subnet catalog refresh
  canic subnet catalog refresh --format json
  canic subnet catalog refresh --dry-run --output <path>
  canic subnet catalog refresh --source-endpoint <url>
  canic subnet catalog refresh --lock-stale-after <duration>
  ```

- Added host refresh orchestration for the mainnet catalog:

  ```text
  catalog: <canic-cache-root>/subnet-catalog/ic/catalog.json
  lock:    <canic-cache-root>/subnet-catalog/ic/refresh.lock
  temp:    <canic-cache-root>/subnet-catalog/ic/catalog.json.tmp.<pid>
  ```

- Added deterministic refresh reports with registry canister ID, registry
  version, fetched-at timestamp, source endpoint, fetched-by label, dry-run
  status, cache-write status, subnet count, and routing-range count.

### 🔧 Changed

- Missing-cache errors now point operators at:

  ```text
  canic subnet catalog refresh
  ```

  instead of saying live refresh is still planned.

- Release and packaged-downstream verification scripts now package and patch
  `canic-subnet-catalog` and `canic-ic-registry` before `canic-host`, so the
  new public host dependency resolves from packaged sources during isolated
  proofs.

### 🧭 Notes

- Refresh remains mainnet-only in 0.60. Non-`ic` networks still fail with the
  pinned mainnet-only usage error.

- The initial live adapter fails clearly if the registry returns chunked
  high-capacity values. Chunk retrieval can be added inside `canic-ic-registry`
  without changing the host, CLI, or catalog public boundary.

- Instruction-audit estimate integration remains deferred to a later 0.60
  patch.

### 🧪 Testing

- Focused validation for this patch includes:

  ```text
  cargo test -p canic-ic-registry -- --nocapture

  cargo test -p canic-cli --lib subnet -- --nocapture

  cargo test -p canic --test workspace_manifest publishable_members_do_not_depend_on_unpublished_workspace_members -- --nocapture
  cargo clippy -p canic-ic-registry -p canic-host -p canic-cli --all-targets
  make fmt-check
  git diff --check

  cargo run -p canic-cli --bin canic -- subnet catalog refresh
  target/debug/canic subnet catalog list
  ```

## 0.60.1 - 2026-06-04 - Subnet catalog command wording

### 📝 Summary

- Renamed the cached subnet inspection command group from
  `canic subnet network ...` to `canic subnet catalog ...` so the subcommand
  name does not collide conceptually with the global `--network` selector.

- Kept the implementation cached-only. This patch does not add live NNS
  registry refresh, protobuf registry transport, or instruction-audit estimate
  integration.

### 🔧 Changed

- Renamed the read-only cached subnet inspection command surface:

  ```text
  canic subnet catalog list
  canic --network ic subnet catalog list
  canic subnet catalog list --kind application --specialization fiduciary --format json
  canic subnet catalog list --show-ranges --range-limit <n> --range-offset <n>
  canic subnet catalog info <subnet-principal|canister-principal|deployment-target>
  canic subnet catalog info <deployment>/<role-or-canister>
  canic subnet catalog info <principal> --as subnet
  canic subnet catalog info <principal> --as canister
  ```

- Preserved the 0.60 mainnet-only scope. Omitted `--network` still means `ic`
  for this command group; `--network local` and any other non-`ic` network
  still fail with the pinned mainnet-only usage error.

- Clarified the missing-cache error. 0.60.1 reads an existing local catalog
  only; it does not fetch the NNS registry live before the planned refresh
  follow-up.

### 🧭 Deferred

- `canic subnet catalog refresh` is not introduced in 0.60.1. Live mainnet
  refresh remains sequenced for the shared protobuf registry adapter follow-up.

## 0.60.0 - 2026-06-04 - Cached NNS subnet inspection

### 📝 Summary

- Started the 0.60 NNS subnet inspection line with cached mainnet IC subnet
  catalog loading, schema validation, stale-cache reporting, and principal-byte
  routing resolution.

- Kept the slice cached-only. It does not add live NNS registry refresh,
  protobuf registry transport, or instruction-audit estimate integration.

### ➕ Added

- Added `canic-subnet-catalog` as the pure catalog model/resolver crate. It
  owns catalog JSON schema validation, future-schema rejection, classification
  fields, routing ranges, and inclusive canonical-principal-byte lookup without
  depending on `ic-agent`, `prost`, or generated registry protobuf types.

- Added host-side cache loading and report preparation for:

  ```text
  <canic-cache-root>/subnet-catalog/ic/catalog.json
  ```

- Added the read-only cached subnet inspection command surface:

  ```text
  canic subnet network list
  canic --network ic subnet network list
  canic subnet network list --kind application --specialization fiduciary --format json
  canic subnet network list --show-ranges --range-limit <n> --range-offset <n>
  canic subnet network info <subnet-principal|canister-principal|deployment-target>
  canic subnet network info <deployment>/<role-or-canister>
  canic subnet network info <principal> --as subnet
  canic subnet network info <principal> --as canister
  ```

- Added text and JSON outputs for list/info, including catalog path, registry
  version, fetched-at timestamp, stale status, resolver backend, subnet kind,
  specialization, geography, node count, charge applicability, and matched
  routing range details.

### 🔧 Changed

- Added top-level `subnet` help/dispatch as an IC network inspection namespace.
  The command is separate from `canic cycles ...`, which remains reserved for
  real cycle balances, minting, transfer, top-up, and ICP conversion.

- Scoped `canic subnet network ...` to the mainnet `ic` catalog for 0.60.0.
  Omitted `--network` means `ic` for this command group; `--network local` and
  any other non-`ic` network fail with the pinned mainnet-only usage error.

### 🧭 Deferred

- `canic subnet network refresh` is not introduced in 0.60.0. Live mainnet
  refresh remains sequenced for the shared protobuf registry adapter follow-up.

- Instruction-audit estimate integration remains deferred until the cached
  inspection and live-refresh surfaces are stable.

### 🧪 Testing

- Validated the release slice with:

  ```text
  cargo fmt --all
  cargo test -p canic-subnet-catalog -- --nocapture
  cargo test -p canic-host subnet_catalog -- --nocapture
  cargo test -p canic-cli --lib -- --nocapture
  cargo test -p canic --test changelog_governance -- --nocapture
  cargo check -p canic-subnet-catalog -p canic-host -p canic-cli
  cargo clippy -p canic-subnet-catalog -p canic-host -p canic-cli --all-targets
  make fmt-check
  git diff --check
  ```
