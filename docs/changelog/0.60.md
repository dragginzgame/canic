## 0.60.10 - 2026-06-04 - NNS node-provider list

### đź“ť Summary

- Added a read-only NNS governance view for node providers under the existing
  `canic nns` namespace.

- `canic nns node-provider list` queries the mainnet NNS governance canister's
  `list_node_providers` Candid query, converts the response into Canic-owned
  report structs, and renders a narrow text table by default.

### đź”§ Changed

- Added the new command surface:

  ```text
  canic nns node-provider list
  canic nns node-provider list --format json
  ```

- Kept the command mainnet-only for 0.60, matching the existing NNS subnet
  inspection behavior:

  ```text
  canic --network ic nns node-provider list
  ```

- Added JSON output with governance canister provenance, fetch timestamp,
  source endpoint, node-provider principals, and optional reward-account hex.
  Text output intentionally omits reward-account detail so the default table
  stays within standard terminal widths.

### đź§­ Notes

- This view is governance-backed, not subnet-catalog-backed. It does not read
  or mutate `.canic/subnet-catalog/ic/catalog.json`.

- The shared IC adapter owns the live Candid governance call. No raw governance
  response type crosses into `canic-host` or `canic-cli`.

## 0.60.5 - 2026-06-04 - Registry chunks, compact list, and help cleanup

### đź“ť Summary

  protobuf record decoding.

- Made `canic nns subnet list` text output compact by default so it fits a
  standard terminal width. The default table displays five-character subnet
  prefixes and omits the less important registry-version, fetch-time,
  catalog-path, and stale-reason text fields from the list view.

  `canic-subnet-catalog` continue to see Canic-owned catalog structs only.
