# 0.11.x Detailed Changelog

Technical release notes for the `0.11.x` minor line.

## 0.11.1 - 2026-03-07 - Root Capability Replay & Dispatcher Hardening

### 📝 Summary

- `0.11.1` hardens root capability replay/dispatch behavior and improves subnet-auth diagnostics to reduce triage time.

### 🩹 Fixed

- Sharding assignment now ignores active shard IDs that are not direct children in local topology cache, so stale shard picks no longer route into root auth denials during create flows.

### 🔧 Changed

- Root privileged mutation handling is centralized under one capability dispatcher envelope (`Provision`, `Upgrade`, `MintCycles`, `IssueDelegation`) with per-capability policy checks.
- Replay protection now enforces deterministic canonical payload hashing with strict slot conflict rules.
- Replay slot identity is scoped by `(caller, subnet, request_id)` and bound to capability discriminants.
- Replay expiry handling now checks existing slot state before purge sweeps, making expired/conflict outcomes explicit.
- Subnet-registry access denials now include authentication context to speed up field triage.
- Subnet-registry predicates now fail fast on non-root canisters with explicit auth errors.
- Macro validation now rejects `requires(caller::is_registered_to_subnet())` on non-`internal` endpoints at compile time.
- Root capability metrics now emit centralized dispatcher envelope events for authorize/deny/replay/execute outcomes.

### 🗑️ Removed

- Removed internal root endpoint `canic_delegation_provision`; delegation issuance now goes through dispatcher path (`canic_request_delegation` -> `root:issue_delegation`) only.

### 🧪 Testing

- Added PocketIC coverage for replay conflicts (cross-variant, mutated payload), identical-request caching, TTL-bound reject, and expiry reject behavior.
- Added coverage for unauthorized root capability requests and per-capability policy denials.
- Added sharding regression coverage for stale/non-child assignment filtering.
- Added delegation replay assertions for signature-stable cached responses and no-commit-on-execution-error behavior.
- Delegation replay tests now auto-skip threshold-key-dependent assertions when keys are unavailable, without environment toggles.

```text
replay behavior:
same canonical payload digest -> cached response
different canonical payload digest -> replay conflict
```

## 0.11.0 - 2026-02-27 - Capabilities Arc: Kickoff & Cleanup

### 📝 Summary

- `0.11.0` starts the capability-focused auth arc with scope-aware guards, account-model alignment, safer numeric conversion paths, and dependency cleanup.

### ⚠️ Breaking

- `authenticated("scope")` (now `is_authenticated("scope")`) now enforces required-scope presence.
- Legacy `authenticated(...)` macro usage is removed; use `is_authenticated(...)`.

### 🩹 Fixed

- `canic-cdk::types::Account` now uses `icrc-ledger-types` (`Icrc1Account`) for textual encoding/decoding to align with ICRC-1 reference behavior.

### 🔧 Changed

- Replaced `Nat`/cycle numeric downcasts with standard-library `TryFrom` conversions in HTTP/cycles paths while preserving overflow fallback behavior.
- Added shared capability scope constants in `canic::ids::cap` (`READ`, `WRITE`, `VERIFY`, `ADMIN`).
- Updated `is_authenticated(...)` macro parsing to accept string literals or path constants (for example `cap::VERIFY`).

### 🗑️ Removed

- Removed `canic-utils::rand::random_hex()`; call sites should use `random_bytes()` and encode as needed.
- Removed unused dependencies from workspace/crates (`base32`, `crc32fast`, `hex`, `num-traits`).

### 🧪 Testing

- Added PocketIC coverage for scope-gated delegated auth allow/deny behavior.
- Added coverage for unscoped delegated auth guard behavior.
- Added macro parser coverage for constant-path scope arguments.

```rust

#[canic_query(requires(auth::is_authenticated()))]

    Ok(())

#[canic_update(requires(auth::is_authenticated(cap::VERIFY)))]
async fn profile_verify(token: DelegatedToken) -> Result<(), Error> {
    Ok(())
}
```
