## 0.67.20 - 2026-06-14 - Replica-query module decomposition

### 📝 Summary

- Decomposes canic-host direct local replica querying without changing CLI
  behavior, local-query behavior, fallback behavior, or public host APIs.

  and wire-decoding child modules while leaving the parent module as the public
  API and small readiness JSON fallback parser.

  current direct local query helper sends anonymous queries.

- Moves anonymous local HTTP/CBOR query mechanics and local endpoint resolution
  into `replica_query::transport`.

- Moves `/api/v2/status` reachability and local root-key parsing into
  `replica_query::status`.

  subnet-registry JSON adapter code into `replica_query::wire`.

- Converts the new child modules to directory modules and colocates focused
  tests beside their owning code.

- Reduces the parent `replica_query` module from the pre-split catch-all shape
  to public API wiring and the command-output compatibility parser still used
  by CLI fallback paths.

### 🧪 Testing

- Moved direct-query transport, status, and wire-decoding tests beside their new
  owning modules while leaving parent readiness JSON parser tests in the parent
  test module.

- Validated with focused replica-query tests, host clippy with warnings denied,
  the full host library test suite, canic-cli check, formatting, and whitespace
  checks.
