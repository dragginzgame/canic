## 0.67.0 - 2026-06-13 - IC query extraction

### Summary

- Moves the NNS query command surface from `canic nns ...` into the external
  `ic-query` crate with the `icq nns ...` executable path.

- Moves the live NNS registry adapter and query/report modules out of
  `canic-host`.

- Folds the standalone `canic-subnet-catalog` package into `ic-query`, leaving
  `ic-query` as the standalone publishable `0.0.1` crate for IC registry query
  tooling.

- Changes mainnet root-auth subnet evidence collection so Canic executes `icq`
  instead of linking subnet-catalog code into the host.

- Adds `icq` to the pinned external tooling surface: local setup and CI install
  and verify the `ic-query` CLI alongside `icp` before Canic checks run.

### Operator Surface

- `canic nns ...` no longer owns the public NNS metadata query surface.

- `icq nns ...` is the executable path for NNS registry, subnet, node,
  node-provider, node-operator, data-center, and topology query/report
  commands.

- Canic root-auth evidence workflows now depend on the installed `icq` binary
  for mainnet subnet evidence.
