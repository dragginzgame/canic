# 0.67 Changelog

## 0.67.1 - 2026-06-13 - Host audit and test layout cleanup

### 📝 Summary

- Audits `canic-host` after the `icq` split and removes stale host module
  wiring that no longer had active source ownership.

- Reorganizes large host unit-test files into focused sibling and directory
  modules so deployment truth, install root, release set, adoption, registry,
  configuration, and artifact-support coverage can be reviewed by topic.

- Leaves the CLI surface, runtime behavior, persisted data shapes, and external
  tooling contract unchanged.

### 📊 Audit

- Confirms `canic-host` no longer carries inline `mod tests { ... }` blocks
  under `src`.

- Keeps the release-set and adoption test groups below their previous
  monolithic size by splitting role, profile, authority, evidence, artifact,
  path, mutation, and serialization scenarios into focused modules.

- Removes the orphan cache-file module wiring left behind by earlier host
  cleanup.

### 🧪 Testing

- Maintains the existing host unit-test coverage while moving tests into
  smaller module groups; no tests were intentionally removed or weakened.

## 0.67.0 - 2026-06-13 - IC query extraction

### Summary

- Moves the NNS query command surface from `canic nns ...` into the external
  `ic-query` crate with the `icq nns ...` executable path.

- Moves the live NNS registry adapter and query/report modules out of
  `canic-host`.

- Folds the standalone `canic-subnet-catalog` package into `ic-query`, leaving
  `ic-query` as the standalone publishable `0.0.1` crate for IC registry query
  tooling.

- Changes mainnet root-auth subnet evidence collection so Canic executes `icq`
  instead of linking subnet-catalog code into the host.

- Adds `icq` to the pinned external tooling surface: local setup and CI install
  and verify the `ic-query` CLI alongside `icp` before Canic checks run.

### Operator Surface

- `canic nns ...` no longer owns the public NNS metadata query surface.

- `icq nns ...` is the executable path for NNS registry, subnet, node,
  node-provider, node-operator, data-center, and topology query/report
  commands.

- Canic root-auth evidence workflows now depend on the installed `icq` binary
  for mainnet subnet evidence.
