# 0.83 Changelog

## 0.83.5 - 2026-07-08 - Blob-storage report label typing

### Summary

- Fixes `CANIC-083-DEBT-008` by tightening `canic blob-storage` report
  ownership so closed report kind, Candid source, action, funding status, and
  readiness state labels are typed internally instead of owned as raw strings.

- Keeps free-form command strings, error messages, blocker/warning code arrays,
  and canister-derived text values as strings.

- Keeps CLI behavior, JSON fields, JSON labels, text output meaning, endpoint
  surfaces, Candid, deployment truth, evidence/report schemas, and
  stable-state layout unchanged.

### CLI

- `canic blob-storage` still emits the same JSON labels, including
  `blob_storage_status`, `blob_storage_error`,
  `blob_storage_sync_gateways_result`, `blob_storage_fund_result`,
  `installed_deployment`, `sync_gateways`, `fund`, `ready`, `warning`,
  `blocked`, `funding_needed`, `not_configured`, `not_needed`, and `unknown`.

- No command forms changed.

```bash
canic blob-storage status <deployment> <canister-or-role> --json
canic blob-storage sync-gateways <deployment> <canister-or-role> --json
canic blob-storage fund <deployment> <canister-or-role> --cycles <cycles> --json
```

### Testing

Validated with:

```bash
cargo test --locked -p canic-cli blob_storage
```

## 0.83.1 - 2026-07-08 - Advanced deploy text-output flag hard cut

### Summary

- Fixes `CANIC-083-DEBT-002` by hard-cutting default-JSON advanced deploy
  report families from `--format json|text` to JSON by default plus `--text`
  for human-readable output.

- Keeps JSON payload schemas, deployment truth, evidence/report schemas,
  endpoint surfaces, Candid, and stable-state layout unchanged.

- Removes an active `canic state manifest` help breadcrumb for the removed
  `--format json` spelling without changing state command behavior.

### CLI

- `canic deploy inspect compare` now uses `--text` for human-readable
  comparison summaries.

- `canic deploy inspect root` and `canic deploy root verify` now use `--text`
  for human-readable root verification output.

- `canic deploy authority check|evidence|report|receipt` now use `--text` for
  human-readable dry-run authority summaries.

- `canic deploy external` report builders now use `--text` for human-readable
  external lifecycle summaries.

- `canic deploy promote` report builders now use `--text` for human-readable
  promotion summaries.

- `canic state manifest` help now describes only the maintained `--json`
  surface.

```bash
canic deploy inspect compare --left <file> --right <file> --text
canic deploy inspect root --request <file> --text
canic deploy root verify <deployment> --from-check <file> --text
canic deploy authority report <deployment> --text
canic deploy external plan <deployment> --text
canic deploy external verify --request <file> --text
canic deploy promote plan --request <file> --text
canic deploy promote inspect readiness --request <file> --text
```

### Breaking

- Hard-cuts the old `--format json|text` forms for the affected advanced deploy
  report families. No aliases, shims, compatibility routes, or
  anti-resurrection tests were added.

### Testing

## 0.83.0 - 2026-07-08 - Technical debt audit and report JSON flag hard cut

### Summary

- Starts the 0.83 technical debt audit line with docs-only audit artifacts.

- Makes `ledger.md` the canonical first-pass debt ledger and records the first
  accepted finding.

- Fixes `CANIC-083-DEBT-001` by standardizing the affected active
  report-command families on `--json` for raw JSON output.

- Keeps stable CI/GitOps evidence-envelope output as a distinct current
  surface through `--evidence-envelope`.

### Audit

- Added `docs/audits/0.83-technical-debt/README.md`.

- Added `docs/audits/0.83-technical-debt/ledger.md`.

- Added `docs/audits/0.83-technical-debt/recommended-slices.md`.

- Added `docs/audits/0.83-technical-debt/deferred.md`.

- Added `docs/audits/0.83-technical-debt/rejected.md`.

- Recorded and fixed `CANIC-083-DEBT-001`, a P2 finding for mixed CLI report
  JSON flag conventions across active report families.

- Recorded `CANIC-083-DEBT-002` for remaining advanced deploy report families
  that needed a separate convention slice.

- Recorded rejected/non-finding candidates for pure-policy imports,
  TODO/FIXME/HACK markers, and remaining semantic helper type aliases.

### CLI

- `canic deploy inspect catalog list|inspect` now use `--json` for catalog
  JSON reports.

- `canic deploy check <deployment>` now prints text by default, uses `--json`
  for raw `DeploymentCheckV1`, and uses `--evidence-envelope` for
  `EvidenceEnvelopeV1` output.

- `canic fleet adoption report <fleet>` now prints text by default, uses
  `--json` for raw adoption-report JSON, and uses `--evidence-envelope` for
  `EvidenceEnvelopeV1` output.

- `canic evidence gate` now uses `--json` for raw gate reports and
  `--evidence-envelope` for envelope output.

- `canic evidence compare` now uses `--json` for structured comparison output.

```bash
canic deploy inspect catalog list --json
canic deploy inspect catalog inspect <deployment> --json
canic deploy check <deployment> --json
canic deploy check <deployment> --evidence-envelope
canic fleet adoption report <fleet> --profile <profile> --json
canic fleet adoption report <fleet> --profile <profile> --evidence-envelope
canic evidence gate --policy <path> --envelope <path> --json
canic evidence gate --policy <path> --envelope <path> --evidence-envelope
canic evidence compare --left <path> --right <path> --json
```

### Breaking

- Hard-cuts the old `--format json` / `--format envelope-json` report-selection
  forms for the affected command families. No aliases or compatibility routes
  were added.

### Docs

- Updated active architecture, operation, and CLI docs to use `--json` and
  `--evidence-envelope` for the affected report families.

### Migration Notes

- This patch adds no new command families.
