# 0.83 Changelog

## 0.83.0 - 2026-07-08 - Technical debt audit and report JSON flag hard cut

### Summary

- Starts the 0.83 technical debt audit line with docs-only audit artifacts.

- Makes `ledger.md` the canonical first-pass debt ledger and records the first
  accepted finding.

- Fixes `CANIC-083-DEBT-001` by standardizing the affected active
  report-command families on `--json` for raw JSON output.

- Keeps stable CI/GitOps evidence-envelope output as a distinct current
  surface through `--evidence-envelope`.

### Audit

- Added `docs/audits/0.83-technical-debt/README.md`.

- Added `docs/audits/0.83-technical-debt/ledger.md`.

- Added `docs/audits/0.83-technical-debt/recommended-slices.md`.

- Added `docs/audits/0.83-technical-debt/deferred.md`.

- Added `docs/audits/0.83-technical-debt/rejected.md`.

- Recorded and fixed `CANIC-083-DEBT-001`, a P2 finding for mixed CLI report
  JSON flag conventions across active report families.

- Recorded `CANIC-083-DEBT-002` for remaining advanced deploy report families
  that still use `--format json|text` and need a separate convention slice.

- Recorded rejected/non-finding candidates for pure-policy imports,
  TODO/FIXME/HACK markers, and remaining semantic helper type aliases.

### CLI

- `canic deploy inspect catalog list|inspect` now use `--json` for catalog
  JSON reports.

- `canic deploy check <deployment>` now prints text by default, uses `--json`
  for raw `DeploymentCheckV1`, and uses `--evidence-envelope` for
  `EvidenceEnvelopeV1` output.

- `canic fleet adoption report <fleet>` now prints text by default, uses
  `--json` for raw adoption-report JSON, and uses `--evidence-envelope` for
  `EvidenceEnvelopeV1` output.

- `canic evidence gate` now uses `--json` for raw gate reports and
  `--evidence-envelope` for envelope output.

- `canic evidence compare` now uses `--json` for structured comparison output.

```bash
canic deploy inspect catalog list --json
canic deploy inspect catalog inspect <deployment> --json
canic deploy check <deployment> --json
canic deploy check <deployment> --evidence-envelope
canic fleet adoption report <fleet> --profile <profile> --json
canic fleet adoption report <fleet> --profile <profile> --evidence-envelope
canic evidence gate --policy <path> --envelope <path> --json
canic evidence gate --policy <path> --envelope <path> --evidence-envelope
canic evidence compare --left <path> --right <path> --json
```

### Breaking

- Hard-cuts the old `--format json` / `--format envelope-json` report-selection
  forms for the affected command families. No aliases or compatibility routes
  were added.

### Docs

- Updated active architecture, operation, and CLI docs to use `--json` and
  `--evidence-envelope` for the affected report families.

### Migration Notes

- This patch adds no new command families.
