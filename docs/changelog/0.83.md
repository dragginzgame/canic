### Summary

  report families from `--format json|text` to JSON by default plus `--text`
  for human-readable output.

- Keeps JSON payload schemas, deployment truth, evidence/report schemas,

  `--format json` spelling without changing state command behavior.

  comparison summaries.

- `canic deploy inspect root` and `canic deploy root verify` now use `--text`

  human-readable dry-run authority summaries.

  external lifecycle summaries.

- `canic deploy promote` report builders now use `--text` for human-readable
  promotion summaries.

- `canic state manifest` help now describes only the maintained `--json`
  surface.

```bash

canic deploy inspect root --request <file> --text
```

- Makes `ledger.md` the canonical first-pass debt ledger and records the first
  accepted finding.

- Fixes `CANIC-083-DEBT-001` by standardizing the affected active

### Audit

- Added `docs/audits/0.83-technical-debt/README.md`.

  that needed a separate convention slice.

- Recorded rejected/non-finding candidates for pure-policy imports,
  TODO/FIXME/HACK markers, and remaining semantic helper type aliases.

### CLI

- `canic deploy inspect catalog list|inspect` now use `--json` for catalog
  JSON reports.

  `EvidenceEnvelopeV1` output.

  `--json` for raw adoption-report JSON, and uses `--evidence-envelope` for
  `EvidenceEnvelopeV1` output.

- `canic evidence gate` now uses `--json` for raw gate reports and
  `--evidence-envelope` for envelope output.

- `canic evidence compare` now uses `--json` for structured comparison output.

