# 0.59 Changelog

## 0.59.1 - 2026-06-03 - Estimate input and workflow lint cleanup

### 📝 Summary

- Tightened the 0.59 estimate input contract across both the shell wrapper and
  the Rust report path. Estimate source inputs now require estimate mode
  instead of being ignored when the report generator is invoked directly.

- Fixed the GitHub Actions workflow validation failure caused by using the
  top-level `env` context inside job-level `env` expressions, and made the
  local developer setup path explicitly carry the pinned actionlint version and
  install directory.

### 🩹 Fixed

- Removed invalid job-level `RUSTUP_TOOLCHAIN: ${{ env.* }}` assignments from
  the CI workflow. The wasm target install steps now use the top-level shell
  environment variables directly.

- Added Rust-side estimate option validation for direct environment-driven
  report generation, matching the shell wrapper rule that
  `--estimate-node-count` and `--cycles-per-billion-instructions` require
  `--estimate-execution-cycles`.

### 🔧 Changed

- `make install-dev` and `make update-dev` now pass the pinned
  `ACTIONLINT_VERSION` and `ACTIONLINT_INSTALL_DIR` through to the shared dev
  installer, keeping the local actionlint setup explicit at the Makefile
  boundary.

### 🧪 Testing

- Validated the cleanup with:

  ```text
  actionlint
  actionlint .github/workflows/ci.yml
  actionlint -version
  make -n install-dev update-dev
  bash -n scripts/dev/install_dev.sh
  cargo fmt --all
  cargo test -p canic-tests --test instruction_audit -- --nocapture
  cargo clippy -p canic-tests --test instruction_audit
  cargo check --workspace
  cargo test -p canic --test changelog_governance -- --nocapture
  git diff --check
  ```

## 0.59.0 - 2026-06-03 - Instruction accounting and offline estimates

### 📝 Summary

- Started the Slice 59 instruction-accounting line with a hard terminology
  boundary: measured runtime rows remain instruction rows, while cycle language
  is reserved for real cycle balances/funding/refill surfaces or explicit
  estimates.

- Added opt-in host/test-side execution-cycle estimate decoration for the
  instruction-footprint audit. Estimates require an explicit node-count or
  explicit rate source and do not perform NNS, catalog, or network lookup.

### ➕ Added

- Added the optional `execution_cycle_estimate` sibling object on normalized
  instruction-audit rows. The object records estimate schema version, charge
  model, local instructions, counter ID, sample origin, decimal-string cycle
  estimate/rate values, formula version, rate source, and omitted billed
  components.

- Added pinned offline estimate support for `13` and `34` node assumptions,
  plus the explicit-rate override path for custom or future pricing inputs.

- Added `scripts/ci/instruction-audit-report.sh` estimate flags:

  ```text
  --estimate-execution-cycles
  --estimate-node-count <13|34>
  --cycles-per-billion-instructions <cycles>
  ```

### 🔧 Changed

- Updated instruction-footprint report artifacts to record
  `performance_counter(1)`, `counter_id = 1`, measured unit
  `local_instructions`, and counter semantics in `method.json` and report
  preambles.

- Changed normalized row `sample_origin` from a generic derived label to the
  message-kind scope: `update`, `query`, or `composite_query`.

- Updated endpoint matrix artifacts and markdown output to include
  `sample_origin`, so update, query, and composite-query samples are not
  compared as if they shared identical counter semantics.

### 📚 Documentation

- Added the 0.59 design for instruction accounting and offline cost estimates.

- Updated the recurring instruction-footprint audit definition with the
  counter model, opt-in estimate contract, supported offline rate table, and
  forbidden 0.59 NNS/catalog estimate fields.

### 🧪 Testing

- Validated the release slice with:

  ```text
  cargo fmt --all
  cargo test -p canic-tests --test instruction_audit -- --nocapture
  cargo clippy -p canic-tests --test instruction_audit
  bash scripts/ci/instruction-audit-report.sh --help
  bash scripts/ci/instruction-audit-report.sh --estimate-execution-cycles
  bash scripts/ci/instruction-audit-report.sh --estimate-node-count 13
  bash scripts/ci/instruction-audit-report.sh --estimate-execution-cycles --estimate-node-count 0
  cargo check --workspace
  cargo test -p canic --test changelog_governance -- --nocapture
  git diff --check
  ```

  The three invalid `instruction-audit-report.sh` invocations are expected to
  exit with status `2`; they prove estimate mode rejects missing sources,
  source flags without estimate mode, and zero-valued node counts before report
  artifact creation.
