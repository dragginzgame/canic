# Canic 0.98 Changelog

## [0.98.0] - 2026-07-22 - Immutable Baseline And Dead Package Hard Cut

### 🔧 Changed

- Upgraded the repository tool contract from `ic-wasm 0.9.11` to `0.10.0`.
  Installation now consumes the platform-specific official GitHub release
  archive under an exact SHA-256 pin; the obsolete npm archive layout and its
  SHA-512 variables were removed without a fallback. Canic uses the binary CLI,
  so the release's Rust-library-only `walrus` API break does not affect Canic.

### 🩹 Fixed

- Made generated metrics cfg follow the role package's canonical normal Canic
  feature declaration rather than the intentionally feature-empty build
  dependency. Malformed or missing explicit feature declarations fail closed.
- Made protected dependency diagnostics choose the globally shortest stable
  path across every sibling subtree.
- Made ledger/CMC post-effect persistence failures mark refill replay as
  recovery-required while preserving the primary typed cause.
- Removed checkout-local paths and raw Cargo stderr from projected
  role-contract diagnostics.

### 🧹 Cleanup

- Recorded that immutable `v0.97.3` is not a passing closeout anchor and froze
  the exact source, product-tree, lockfile, and final-0.96 receipt-contract
  identities used by 0.98.
- Reran the deletion, reachability, package, stable-allocation, fixture,
  example, and documentation inventory against the released post-0.97 tree.
- Removed the consumerless public metadata-only role manifest resolver, error
  type, cache, re-exports, and tests. Configured role packages retain one
  config-driven authority with no replacement alias.
- Removed the consumerless `project-protocol-stub` package, its workspace
  member, and its lockfile row. The two live project role values remain private
  beside their sole current consumer; no shared replacement was introduced.
- Removed the explicit test for the deleted nested refill configuration path.
  The maintained root-only refill schema tests remain; no legacy alias,
  fallback, or anti-resurrection test is carried forward.

### 🧪 Testing

- All 841 host tests, 980 core tests with one ignored, and 16 Canic tests pass.
- The four-case role-attestation PocketIC suite proves generated issuer Wasm
  exposes and records metrics selected by the normal role dependency.
- The repository-pinned root PocketIC matrix passes all 28 hierarchy, auth,
  replay, placement, scaling, sharding, and upgrade journeys.
- Strict workspace all-feature Clippy, workspace check, formatting, layering,
  release/governance gates, dependency-risk inspection, and diff hygiene pass.
- Locked metadata resolves 39 workspace packages and excludes the deleted
  package.
- The project hub, project instance, and delegation root fixtures compile
  together after the deletion.
- All seven workspace-manifest tests and all 51 subnet-schema tests pass.
- Strict `canic-core` library/test Clippy passes with warnings denied.
- The Linux x86-64 `ic-wasm 0.10.0` archive installs under its exact checksum;
  version enforcement, release-integrity governance, host artifact-I/O tests,
  and the four-case role-attestation PocketIC suite pass with the new binary.
- The deterministic 28-case root PocketIC matrix passes with the upgraded
  binary across hierarchy, delegated auth, replay, placement, scaling,
  sharding, and upgrade journeys.

### 📊 Audit

- Rigorous 0.97 verdict: `NOT READY` at immutable `v0.97.3`; zero P0, five P1
  corrected in the working tree, and three documented P2 follow-ups.
- Contract impact: no Candid DTO/wire type, stable state, configuration,
  CLI/JSON, dependency, lockfile, or package-version change. Generated role
  Candid/Wasm regains the selected metrics endpoint; the accidental public
  Rust manifest resolver is hard-cut without an alias.
- Package footprint: one unpublished test package, two source files, one
  workspace-member row, and one lockfile package row removed.
- The obsolete intent fixture and unexecuted randomness contract remain
