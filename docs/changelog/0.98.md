## [0.98.14] - 2026-07-23 - Repo-Only Test Fixture Hard Cut

  `root + issuer` baseline. Its build, snapshot capture, cache restore, and

  baseline construction retains private direct lookups.
- Deleted the `MINIMAL` role constant and its self-referential test entry. All
  role constants used by current fixtures remain.

### 🧪 Testing

- The complete `canic-testing-internal` unit/integration suite passes.
- The maintained role-attestation PocketIC test target passes through the
  reduced cached issuer baseline.
- Strict all-target test-support and integration-test compilation and Clippy
  pass with warnings denied.

### 📊 Audit

- This removes 180 net Rust lines before release notes. The entire change is
  confined to unpublished workspace test support; production Wasm, Candid,
  stable state, JSON, CLI, configuration, dependencies, package versions, and
  runtime behavior are unchanged.

  added.
- Deleted the unused `BoundedString8`, `BoundedString16`, `BoundedString32`,
  and `BoundedString256` aliases. The active `64`- and `128`-byte aliases and
  the generic implementation remain.

### 🧪 Testing

  including construction, exact-bound, typed rejection, Serde, Candid, stable
  decoding, UTF-8 rejection, equality, and ordering coverage.
- Strict all-feature, all-target `canic-core` check and Clippy validate the
  surviving internal CDK substrate after the XRC hard cut.

### 📊 Audit

- This removes 109 net source lines before release notes. It changes only hidden
  `canic-core` Rust surface: Candid services, stable encodings, JSON, CLI,

## [0.98.11] - 2026-07-23 - Delegated Trust-Format Hard Cut

### 📝 Summary

- Chain-key batches are the sole delegated root-proof behavior. Configuration,
  public Rust models, verifier construction, renewal, and provisioning no
  longer carry a selector for a removed alternative.
- Root-policy and registry canonicalization retain their existing V1 domains
  and schema while removing three constant mode bytes. No V2 format, reader,
  writer, negotiation, or compatibility verification path was added.
- Proof and registry epochs are the explicit invalidation boundary. Root batch
  planning raises the durable registry counter to the configured floor, while
  batch commit continues to raise the durable proof counter before issuing
  replacement material.

### ⚠️ Breaking

- `auth.delegated_tokens.root_proof_mode` is removed. Explicit use is rejected
  by the strict TOML schema; omission selects the sole chain-key behavior.
- `RootProofMode` and the `proof_mode`/`preferred_proof_mode` fields are removed
  from `RootKeyPolicyV1`, `DelegatedAuthRegistrySnapshotV1`, and issuer-policy
  snapshots. This is a public Rust and canonical-hash hard cut; Candid, stable
  records, JSON, CLI, and the V1 domain/schema labels are unchanged.
- Before deployment, operators must set both accepted epoch floors strictly
  above the highest proof and registry epochs accepted by the pre-cut fleet.
  The root must then reprovision every issuer. Old proofs, registry hashes,
  batches, certificates, and tokens are intentionally not migrated.

### 🩹 Fixed

- Restored issuer-local active proofs are checked against the current proof
  and registry floors before status or token preparation. Pre-cut material is
  reported through the existing typed stale-material boundary and triggers
  the maintained lazy-repair path rather than minting unusable tokens.
- The checked-in disposable test fleet now uses proof and registry floors of
  2 and proves new proof creation, issuer installation, token issuance,
  subject binding, replay behavior, and verification through PocketIC.

### 📊 Audit

- Maintained source, configuration, and active protocol guidance contain no
  root-proof-mode selector or canonical encoder and introduce no delegated
  V2 domain or schema. Dependencies, package versions, Candid, stable memory,
  JSON, and CLI surfaces are unchanged.
- Excluding release notes, the slice changes 34 implementation,
  configuration, test, and active-guidance files: 218 additions and 217
  deletions, for a net increase of one line.
