# Canic 0.37

## 0.37.1 - 2026-05-16 - Layer boundary cleanup

### 🩹 Fixed

- Reran the refreshed layer boundary audit and fixed workflow-to-API drift by
  moving module-source runtime ownership into `ops::runtime::install_source`.
- Kept `api::runtime::install` as a compatibility re-export so existing
  control-plane and downstream imports do not need to change.
- Removed workflow's dependency on storage `CanisterRecord` shapes in the
  cycles authorization path by exposing minimal role/parent lookup helpers
  through ops.
- Added the workflow purity audit and moved cycles-funding policy out of
  workflow into `domain::policy::cycles_funding`.
- Moved the mutable cycles-funding grant ledger into
  `ops::runtime::cycles_funding`, leaving workflow to coordinate the read,
  policy evaluation, transfer, and grant recording.
- Moved HTTP and management-status DTO conversion helpers into ops so workflow
  no longer owns those boundary conversions.
- Added and ran the ops purity audit, then renamed delegated-auth certificate
  validation from an ops-owned policy surface to explicit certificate rules.
- Renamed internal delegated-auth TTL and certificate validation types from
  `DelegatedAuthTtlPolicy`, `CertPolicyError`, and
  `validate_cert_issuance_policy` to `DelegatedAuthTtlLimits`,
  `CertRuleError`, and `validate_cert_issuance_rules`.
- Preserved delegated-auth verifier behavior and the stable `cert_policy`
  metrics label so the cleanup does not change runtime output.
- Added and ran the access purity audit, then removed stable app-mode type
  leakage from app endpoint guards.
- Added narrow `AppStateOps` app-mode fact helpers so access can authorize
  app query/update guards without importing stable storage types.
- Added `ConfigOps::is_whitelisted` so the whitelist access predicate no
  longer reads the raw configuration model directly.

### 🔧 Changed

- Tightened `scripts/ci/run-layering-guards.sh` so CI catches workflow imports
  from the API layer and any production workflow reference to storage record
  types.
- Extended the same layering guard so production workflow code cannot define
  `*Policy` types.
- Updated the recurring layer boundary audit definition to check the current
  `domain/policy` path, the current `canic-macros` endpoint path, grouped
  imports, workflow conversion/serialization drift, DTO leakage, policy purity,
  ops-to-workflow dependencies, and endpoint macro auth/access routing.
- Extended the layering guard so production access code cannot import stable
  storage or record types directly.
- Split installation and first-fleet setup guidance into root `INSTALLING.md`
  and refreshed README examples to use the current `canic info list` command
  group.

### 📊 Audit

- Added the May 16 layer boundary report at
  `docs/audits/reports/2026-05/2026-05-16/layer-boundary.md`.
- The audit verdict is pass with drift risk after remediation. Remaining
  watchpoints are workflow RPC/cycles orchestration, workflow call/install
  Candid boundary adapters, endpoint macro auth lowering, and broad runtime
  metrics aggregation.
- Added the May 16 workflow purity report at
  `docs/audits/reports/2026-05/2026-05-16/workflow-purity.md`.
- The workflow purity audit verdict is pass with watchpoints after remediation.
  Remaining watchpoints are workflow RPC/cycles orchestration and Candid
  call/install boundary adapters.
- Added the May 16 ops purity report at
  `docs/audits/reports/2026-05/2026-05-16/ops-purity.md`.
- The ops purity audit verdict is pass with watchpoints after remediation.
  Remaining watchpoints are the expected RPC capability-envelope hotspot,
  delegated-auth token-material verification, topology policy-input mappers,
  and atomic storage state transitions.
- Added the May 16 access purity report at
  `docs/audits/reports/2026-05/2026-05-16/access-purity.md`.
- The access purity audit verdict is pass with watchpoints after remediation.
  Remaining watchpoints are delegated-token boundary decoding,
  delegated-session cleanup, access-expression predicate loops, and access
  metrics facade isolation.

### 🧪 Testing

- Ran the layering guard, focused core request-cycle tests, authenticated macro
  tests, core/control-plane/facade checks, and `canic-core` clippy with warnings
  denied.
- Ran focused cycles-funding policy and ledger tests after moving the policy
  and mutable ledger out of workflow.
- Ran focused access tests, the core check, the core clippy pass with warnings
  denied, the changelog width check, and `git diff --check`.

## 0.37.0 - 2026-05-16 - Cleanup minor

### 🩹 Fixed

- Reran the refreshed bootstrap lifecycle symmetry audit and fixed one
  post-upgrade symmetry gap: non-root runtime continuation now returns typed
  errors for config/auth continuation failures instead of panicking inside the
  workflow runtime.
- The non-root post-upgrade lifecycle adapter now records a failed runtime
  lifecycle metric and traps with the post-upgrade phase when continuation
  setup fails, matching the root path's adapter-owned error handling.

### 🔧 Changed

- Exported `DelegatedToken` from `canic::prelude` so authenticated endpoint
  handlers can use the normal `use canic::prelude::*;` import without a
  separate DTO import.

### 📊 Audit

- Added the May 16 bootstrap lifecycle symmetry report at
  `docs/audits/reports/2026-05/2026-05-16/bootstrap-lifecycle-symmetry.md`.
- The audit verdict is that the lifecycle invariant holds after remediation,
  with residual risk limited to intentional root control-plane split and
  optional init-block timer ordering hotspots.
- Refreshed and reran the canonical auth boundary invariant audit at
  `docs/audits/reports/2026-05/2026-05-16/canonical-auth-boundary.md`.
- Updated the canonical auth audit definition to current macro/core paths and
  made required-scope validation, update-token replay consumption, and private
  token-material helper limits explicit audit checks.
- The canonical auth audit found no boundary bypass. It confirmed that public
  partial verifier surfaces remain absent and that generated authenticated
  endpoints still converge on the endpoint boundary before handler execution.

### 🧪 Testing

- Ran targeted lifecycle checks across `canic-core`, `canic`,
  `canic-control-plane`, the lifecycle trap guard, and the PocketIC lifecycle
  boundary test.
- Ran focused auth boundary tests for authenticated endpoint parsing,
  delegated-token verification, delegated-session identity resolution,
  raw-caller predicate separation, required-scope rejection, update-token replay
  rejection, and subject binding rejection.
