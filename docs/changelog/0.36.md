# Canic 0.36

## 0.36.0 - 2026-05-15 - Backup/restore proofing begins

### 📚 Documentation

- Added the 0.36 Backup/Restore V1 design track, focused on proving and
  hardening the existing backup and restore execution code instead of starting
  a new design.
- Documented the target operator flow across backup create, list, inspect,
  verify, restore planning, dry-run apply, executable restore run, and status.

### 🧪 Testing

- Tightened `canic backup status --require-complete` so a completed execution
  journal is not accepted until the finalized manifest exists.
- Added backup runner coverage proving a max-step limited backup can resume
  without replaying accepted preflight work or completed mutations.
- Added backup runner coverage proving a failed snapshot operation is durably
  recorded, remains retryable, and resumes from the failed operation without
  repeating the previous stop phase.
- Added backup runner coverage proving journal locks reject concurrent
  execution before any command runs.
- Added backup runner coverage proving preflight failures leave mutating
  operations blocked and do not accept the execution preflight.
- Added backup CLI status coverage proving execution layouts render as
  `running`, `failed`, or `complete` from durable plan/journal/manifest state.
- Kept the existing finalization coverage for durable artifact layout and
  manifest integrity.
