# Canic 0.86 Changelog

## [0.86.5] - 2026-07-12 - Deploy-Plan Diagnostic Ownership

### 📝 Summary

0.86.5 gives deploy-plan blocker, warning, and unresolved-assumption
classification one focused owner. Target-resolution failures, unsupported
shapes, blocking local configuration, unverified root state, local observation
warnings, generic assumptions, stable diagnostic codes, and diagnostic-specific
next actions are constructed by one diagnostics module.

Verified evidence, comparison, aggregate status, global next actions, proposed
operations, and report assembly remain outside this owner. Classification,
diagnostic order, CLI behavior, JSON field order, text output, exit behavior,
Candid, stable state, and persisted bytes are unchanged.

### 🔧 Changed

- Deployment target name/config resolution failures move with their blocker
  construction.
- Unsupported, blocking, warning, and remaining unresolved assumptions are
  partitioned and projected by one owner.
- Local-state drift and unavailable-state warning codes remain derived from
  typed assumption kinds inside that owner.
- Diagnostic category, code normalization, source, severity, and local next
  action selection move with the classifications they describe.

### 🗑️ Removed

- The former blocker, warning, and assumption functions are deleted from the
  deploy-plan parent. No wrapper, alternate classifier, compatibility alias, or
  rule framework replaces them.

### 📊 Audit

- The deploy-plan parent falls from 1,260 lines in `0.86.4` to 1,066 lines, a
  reduction of 194 lines or 15 percent.
- Compared with the 1,915-line audit baseline, the parent boundary is 849 lines
  smaller, a 44 percent reduction.
- Diagnostic construction occupies one 219-line module. The extraction changes
  ownership rather than classification or emitted data.

### 🧪 Testing

- All 18 focused deploy-plan command/report tests pass.
- All 12 focused internal plan policy and ordering tests pass.
- Targeted `canic-cli` library Clippy passes with warnings denied.
- Changelog governance, formatting, and diff-hygiene checks pass.

## [0.86.3] - 2026-07-12 - Deploy-Plan Command and Output Ownership

### 📝 Summary

0.86.3 begins the bounded deploy-plan structural pass. Command options, local
root discovery, Clap construction, parsing, and usage now have one focused
owner. Report rendering, JSON file persistence, and command exit classification
have another. The deploy-plan parent retains orchestration, report assembly,
evidence construction, diagnostics, comparison, and proposed operations.

The workspace also advances to `ic-query 0.10.4`. Canic's maintained
subnet-catalog integration compiles and passes its cached-catalog test without a
source migration.

CLI flags, defaults, help text, errors, diagnostic ordering, JSON field order,
text output, exit behavior, Candid, stable state, and persisted bytes are
unchanged.

### 🔧 Changed

- Deploy-plan options, workspace/ICP root discovery, command parsing, Clap
  arguments, and usage text move to one command-input module.
- Text and JSON rendering, create-new JSON output, and planned/blocked exit
  classification move to one report-output module.
- The former single-file plan module becomes a normal directory module while
  retaining the existing `deploy::plan` facade used by orchestration and tests.
- The workspace `ic-query` dependency advances from `0.10.2` to `0.10.4` with
  the existing host-only feature selection.

### 🗑️ Removed

- The old `plan.rs` layout and its inline command/output implementations are
  deleted. No forwarding wrapper, compatibility alias, alternate parser, or
  second renderer remains.

### 📊 Audit

- The deploy-plan parent falls from the 1,915-line audit baseline to 1,662
  lines, a reduction of 253 lines or 13 percent.
- Command inputs occupy a 148-line module and output handling occupies a
  144-line module.
- The three files total 1,954 lines, 39 more than the single-file baseline due
  to focused module headers and imports. The gain is ownership and reduced edit
  blast radius, not code compression.

### 🧪 Testing

- All 18 focused deploy-plan library tests pass without warnings.
- Targeted `canic-cli` library Clippy passes with warnings denied.
- The focused cached subnet-catalog host test passes against `ic-query 0.10.4`.
- Changelog governance, formatting, and diff-hygiene checks pass.
