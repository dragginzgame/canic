# 0.62 Changelog

## 0.62.5 - 2026-06-08 - Release package and install validation

### Summary

- Adds the non-versioned release package/install validation checklist at:
  `docs/operations/release-package-install-validation.md`.

- Classifies existing release package, installed CLI, packaged downstream CLI,
  packaged downstream `wasm_store`, release build, local fleet install, and
  local canister validation gates.

- Adds a CI guard that checks the checklist path, required sections, package
  and install commands, artifact verification guardrails, and release-flow
  ownership wording.

### Changed

- Linked the checklist from the operations docs index, release-validation
  matrix, and diagnostic consistency audit.

- Updated the 0.62 design so `0.62.5` names the non-versioned package/install
  checklist and CI guard as concrete outputs.

- Recorded that package/install validation is RC/final-release evidence and
  does not authorize automated version bumps, tags, publishing, package
  artifact commits, dependency changes, or lockfile changes.

### Validation Commands

The checklist records these package/install RC and final-release gates:

```text
bash scripts/ci/check-release-package-install-validation.sh
make package
make test-installed-canic-cli
make test-packaged-downstream-cli
make test-packaged-downstream-wasm-store
cargo build --release --workspace --locked
make test-fleet-install
make test-canisters
```

The package/install, local ICP, and local canister gates may be assigned to CI
or a dedicated RC environment when too expensive or environment-specific for an
ordinary docs/CI slice.

## 0.62.4 - 2026-06-08 - Diagnostic consistency audit

  or invalid operation IDs, expiration, caller/shard mismatch, delegation-proof

  cost-boundary refusal, permit-boundary refusal, and durable-publication
  ambiguity.

- Adds a CI guard that checks the audit path, required sections, failure-class

### Changed

- Updated the 0.62 design so `0.62.4` names the non-versioned diagnostic audit

  future diagnostic changes must declare their output class before changing
  runtime logs, developer-tool output, CLI text, JSON schema, Candid/public API,
  metric labels, or no-public-output surfaces.
