# 0.62 Changelog

## 0.62.1 - 2026-06-08 - Release validation matrix

### Summary

- Adds the non-versioned release-validation matrix at:
  `docs/operations/release-validation-matrix.md`.

- Classifies validation into slice close-out, implementation close-out, RC
  promotion, and final release/tag checks.

- Separates required local/CI gates from focused replay/auth/cost gates,
  governance gates, package/install probes, broad workspace checks, and
  environment-specific local ICP/canister checks.

### Changed

- Updated the 0.62 design so `0.62.1` names the matrix doc as its concrete
  output.

- Linked the matrix from the general CI/deployment governance policy and the
  operations docs index so 0.62 release validation has a stable operational
  entry point.

- Added a CI guard that checks the non-versioned matrix path, required
  operational anchors, and required command entries.

- Recorded that this slice does not change runtime behavior, Candid, CLI,
  JSON/output, dependency, lockfile, fixture, snapshot, generated output, or
  package artifact surfaces.

### Validation Matrix Commands

The matrix records these required local RC gates:

```text
make fmt-check
make clippy
make test
```

It adds this CI guard:

```text
bash scripts/ci/check-release-validation-matrix.sh
```

It records these focused replay/auth/cost gates:

```text
bash scripts/ci/run-auth-trust-chain-guards.sh
cargo test --locked -p canic-core replay_policy --lib -- --nocapture
cargo test --locked -p canic-core --test cost_guard_boundary_guard -- --nocapture
cargo test --locked -p canic-core --test delegated_auth_hard_cut_guard -- --nocapture
```

It records these package/install gates for RC or final release validation:

```text
make test-installed-canic-cli
make test-packaged-downstream-cli
make test-packaged-downstream-wasm-store
make package
```

### Testing

Focused validation for this patch includes:

```text
cargo fmt --all -- --check
cargo test --locked -p canic --test changelog_governance -- --nocapture
bash scripts/ci/check-release-validation-matrix.sh
git diff --check
```
