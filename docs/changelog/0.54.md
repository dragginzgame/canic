  ```text

  canic fleet role attach <fleet> <role> --subnet <subnet> --kind <kind>
  ```

  The design no longer documents role-only package metadata, scaffold
  attachment flags, build dev flags, detach/normalize commands, or pool/max
  shard attach flags as shipped 0.49 behavior.

## 0.54.0 - 2026-05-31 - passive deployment catalog

### Added

- Added a passive deployment catalog over existing deployment-target local
  state:

  ```text
  canic deploy catalog list
  canic deploy catalog inspect <deployment>
  ```

- Added `DeploymentCatalogReportV1` as the narrow report payload for the v1
  catalog story. Catalog entries are derived from
  `.canic/<network>/deployments/<deployment>.json`, sorted by deployment name,
  and include fleet, network, root principal, root verification state, local
  state fingerprint, and warnings when available.

- Added catalog output controls:

  ```text
  canic deploy catalog list --format json
  canic deploy catalog inspect <deployment> --format json
  canic deploy catalog list --output <path>
  canic deploy catalog inspect <deployment> --output <path>
  ```

  Text remains the default output. `--output <path>` writes the selected format
  only.

### Safety

- Catalog commands are read-only local-state reports. They do not query live
  deployments, create deployment truth, infer deployments from fleet-template
  names, mutate config/topology/controllers/deployment state, install Wasm,
  register artifacts, acquire locks, sign evidence, or add deployment groups.
