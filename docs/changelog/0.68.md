# 0.68 Changelog

## 0.68.0 - 2026-06-17 - Backup cleanup closeout

### 📝 Summary

- Completes the backup cleanup closeout without changing backup layout formats,
  serialized journal or manifest shapes, backup plan construction behavior,
  selector semantics, generated operation order, runner behavior, preflight
  semantics, or public backup APIs.

- Splits broad `canic-backup` journal, manifest validation, persistence,
  runner operation, plan preflight, and plan builder internals into focused
  child modules.

### 🧹 Cleanup

- Converts the download journal module into a small facade with focused child
  modules for durable journal data shapes, resume reporting, and validation.

- Splits manifest validation into focused modules for scalar validation,
  section-local validation, and cross-section deployment/verification binding.

- Splits persistence into dedicated modules for `BackupLayout`, typed
  persistence errors, JSON atomic read/write helpers, and integrity checking.

- Splits persistence integrity into focused artifact, execution-journal,
  topology-binding, path-resolution, and report modules.

- Splits runner operation execution into focused modules for operation
  dispatch/control calls, artifact lifecycle work, download-journal helpers,
  and runner artifact path validation.

- Splits plan preflight handling into focused modules for typed request
  builders, authority receipt application, and execution preflight receipt
  validation.

- Converts the backup plan builder into a focused directory module for
  selector resolution, registry target expansion, target depth calculation, and
  operation phase generation.

- Updates persistence tests to import fixture types explicitly after the
  persistence facade stopped carrying incidental parent-module imports.

### 📊 Audit

- Keeps all extracted code inside existing `canic-backup` ownership
  boundaries; `canic_backup::journal`, `canic_backup::persistence`,
  `BackupLayout`, `PersistenceError`, `BackupPlan` preflight methods,
  `build_backup_plan`, `resolve_backup_selector`, `BackupPlanBuildInput`, and
  runner entry points remain available through the same public call sites.

- Preserves backup JSON contracts, validation error behavior, preflight receipt
  checks, execution journal integrity checks, durable artifact verification,
  finalized manifest generation, backup selector validation, and backup
  operation ordering.

### 🧪 Testing

- Revalidated focused `canic-backup` journal, manifest, persistence, runner,
  and plan coverage after the production module splits.

- Revalidated full `canic-backup` library coverage before closing out the
  cleanup line.

- Revalidated `canic-backup` compilation, `canic-backup` clippy with all
  targets and warnings denied, formatting, changelog governance, and whitespace
  checks during release preparation.
