# 0.68 Changelog

## 0.68.11 - 2026-06-17 - Root batch direct query retrieval

### ➕ Added

- Implements the MVP direct root query retrieval path for prepared root
  delegation proof batches.

- Resolves pending batch metadata by batch id, issuer principal, and
  certificate hash, then finalizes each prepared delegation certificate with
  the root canister-signature proof assembled from the direct root query
  context.

- Returns `RootDelegationProofBatchGetResponse` values with per-issuer
  `DelegationProof` payloads in request order.

### 🔧 Changed

- Changes `canic_get_delegation_proof_batch` from an unavailable root endpoint
  into the direct-query retrieval half of the 0.68 provisioning flow.

- Keeps batch install deferred in this patch; `.11` only completes prepare ->
  direct root query retrieval.

### 🧪 Testing

- Adds batch retrieval coverage for empty get requests, missing pending
  metadata, expired retrieval windows, successful proof finalization, and
  request-order preservation.

- Keeps root batch prepare/get tests running together so pending metadata
  produced by prepare remains consumable by direct root query retrieval.

## 0.68.10 - 2026-06-17 - Root batch prepare metadata

### ➕ Added

- Implements the MVP root delegation proof batch prepare response path.

- Validates batch request metadata, uses the request id as the batch id, builds
  delegation certificates, computes certificate hashes, prepares root
  canister-signature leaves, and returns per-issuer expiry and refresh
  metadata.

- Caches pending batch metadata keyed by batch id, issuer principal, and
  certificate hash so the direct root query retrieval slice can use the
  prepared material.

### 🔧 Changed

- Changes `canic_prepare_delegation_proof_batch` from an unavailable stub after
  preflight into a root-local prepare operation that returns the MVP batch
  metadata response.

- Keeps batch retrieval and batch install unavailable until the direct root
  query and signer broadcast slices land.

### 🧪 Testing

- Adds batch prepare coverage for missing metadata, empty issuer batches,
  invalid metadata TTL, registered-issuer success metadata, and persisted
  issuer policy preflight.

- Keeps the root-signature creation feature build checked so the production
  root leaf preparation path stays compilable.

## 0.68.9 - 2026-06-17 - Root issuer registry preflight

### 🔧 Changed

- Changes root delegation proof batch prepare preflight to resolve issuer
  policy from persisted root auth state instead of building a permissive
  request-scoped policy from the incoming batch entry.

- Adds root auth-state records for issuer policy and provisioner ACL state,
  with migration-safe serde defaults on the stable auth-state record.

- Adds an ops-layer mapper between root issuer storage records and the pure
  root issuer policy model so policy code remains independent of storage
  schema details.

### 🧪 Testing

- Adds batch prepare preflight coverage for registered issuers, unregistered
  issuers, disabled issuers, TTLs above issuer policy, and grants outside the
  issuer policy.

- Adds direct auth-state round-trip coverage for root issuer policy mapping.

## 0.68.8 - 2026-06-17 - Root issuer policy validation

### ➕ Added

- Adds a pure root delegation proof issuer policy validator for the 0.68 batch
  prepare path.

- Validates issuer registration, enabled state, issuer-principal binding,
  allowed audiences, allowed role grants, certificate TTL, and refresh timing.

- Derives `expires_at_ns` and `refresh_after_ns` from the validated issuer
  policy so later batch prepare metadata can reuse deterministic policy
  decisions.

### 🔧 Changed

- Wires the root batch prepare API stub through the new policy preflight before
  returning the existing unavailable error, keeping the protocol surface gated
  while policy validation lands.

### 🧪 Testing

- Adds focused unit coverage for accepted issuer policy, unregistered and
  disabled issuers, issuer mismatch, disallowed audience/grants, invalid TTL,
  and invalid refresh policy.

- Adds batch prepare preflight coverage for valid request shape and TTL
  rejection.

## 0.68.7 - 2026-06-17 - Root provisioning hard cut

### ⚠️ Breaking

- Hard-cuts issuer-initiated root delegation proof provisioning. Non-root
  callers now fail locally when using the old self-provisioning prepare helper
  and must move to the 0.68 root provisioning flow.

- Removes the stale issuer-to-root root-delegation RPC prepare helper and
  request-metadata shim that existed only to drive the unsupported nested
  retrieval path.

### ➕ Added

- Adds root delegation proof batch DTO and protocol surface for prepare, direct
  query retrieval, and install.

- Wires the new batch root endpoints to explicit unavailable API stubs until
  the 0.68 backend policy, storage, and broadcast implementation lands.

- Adds signer-local active delegation proof status with missing, valid,
  refresh-needed, and expired states plus non-secret operational metadata for
  provisioners.

### 🧪 Testing

- Adds focused auth regression coverage for the hard-cut issuer prepare path.

- Pins the new root batch provisioning and signer active-proof status protocol
  surfaces in the public protocol tests.

- Adds unit coverage for signer active-proof status lifecycle classification
  and keeps replay-policy coverage aligned with the new endpoints.

## 0.68.1 - 2026-06-17 - Delegated-token public issuance hardening

### 🩹 Fixed

- Hardens `canic_prepare_delegated_token` so the public prepare path rejects
  caller-supplied subjects that do not match the update caller before replay
  state is reserved.

- Limits open delegated-token self-issuance to login/session scopes:
  `session` and the existing `verify` scope used by current test/demo
  authenticated endpoint flows.

- Blocks public self-granting of privileged or application-specific scopes
  such as `read`, `write`, `admin`, and custom admin labels even when those
  scopes are present in the installed delegation cert.

### 🧭 Migration Notes

- Existing subnet-wide login remains supported when issuers request only
  `session`/`verify` grants across the target roles and relying canisters
  enforce resource ownership or app-specific permissions locally.

- Workflows that previously used delegated-token grants as the direct source
  of write/admin authority need a caller-authorized issuer path or verifier
  local authorization state before those privileged scopes can be issued.

### 📚 Documentation

- Documents the public delegated-token prepare endpoint as a login/session
  materialization surface rather than a general-purpose role grant issuer.

### 🧪 Testing

- Adds pure policy coverage for allowed login scopes, rejected subject
  mismatch, and rejected privileged/custom scopes.

- Adds workflow-level coverage that subject mismatch and privileged self-grant
  requests fail before replay reservation.
