# 0.10.x Detailed Changelog

Technical release notes for the `0.10.x` minor line.

## 0.10.0 - 2026-02-24 - Direct Delegated Token Model

### 📝 Summary

- `0.10.0` replaced relay-style authenticated RPC with direct delegated-token verification at endpoint boundaries.
- Auth flows now bind identity and audience more explicitly, reducing ambiguous trust paths.

### ⚠️ Breaking

- Removed relay-style authenticated RPC envelope support in favor of direct token verification at each endpoint.
- Reshaped auth DTOs to principal-based audiences with explicit shard/root binding (`root_pid`, `shard_pid`, `aud: Vec<Principal>`).

### 🔧 Changed

- Delegated token verification now enforces both subject binding (`token.sub == caller`) and audience binding (`self_pid ∈ token.aud`) before authorization.
- Certificate verification now enforces root authority binding (`cert.root_pid == env.root_pid`) in addition to signature checks.
- User shard token issuance uses a single atomic signing path and delegation requests carry shard-scoped audience principals.
- Environment import now treats `root_pid` as write-once after initialization.

### 🗑️ Removed

- Removed `AuthenticatedRequest` and authenticated relay endpoint flow (`canic_response_authenticated`) from runtime request plumbing.
- Removed local delegated-auth bypass behavior.

### 🧹 Cleanup

- Removed `SignatureInfra` compile-time source scanning from `canic-core/build.rs`; the script now validates `DFX_NETWORK` only.
- CI added grep guards for forbidden ECDSA APIs (`sign_with_ecdsa`, `verify_ecdsa`, `ecdsa_public_key`).

### 🧪 Testing

- Updated delegated-auth integration coverage for direct endpoint calls, including subject-mismatch denials.
- Updated macro validation tests for `authenticated(...)` with optional scope-compatible parsing.

```rust
#[canic_update(requires(auth::authenticated("test:verify")))]
async fn test_verify_delegated_token(token: DelegatedToken) -> Result<(), Error> {
    Ok(())
}
```

## 0.10.1 - 2026-02-24 - Authenticated Scope Syntax

### 📝 Summary

- `0.10.1` improved endpoint ergonomics by allowing scope and no-scope forms for `authenticated(...)`.

### 🔧 Changed

- `authenticated()` now accepts optional scope syntax (`authenticated()` and `authenticated("scope")`).
- Runtime auth keeps existing verification guarantees (signature, root binding, audience, expiry, and `sub == caller`).

---

## 0.10.2 - 2026-02-25 - Memory Bootstrap Ordering and Guards

### 📝 Summary

- `0.10.2` closed lifecycle ordering gaps so runtime memory bootstrap happens before env restoration and stable-memory access.

### 🩹 Fixed

- Post-upgrade lifecycle now initializes memory bootstrap before restoring env context.
- `EnvOps::restore_root()` and `EnvOps::restore_role()` now fail fast when memory registry bootstrap has not completed.
- `intent_authority` test canister now initializes memory in both `init` and `post_upgrade`, plus defensively before first update reads.

### 🔧 Changed

- Added runtime memory-bootstrap readiness tracking in `MemoryRegistryRuntime` via `is_initialized()`.
- `ic_memory!` now enforces a wasm runtime guard so stable-memory slots are accessed only during eager TLS bootstrap or after registry initialization.

```rust
init_eager_tls();
MemoryRegistryRuntime::init(...)?;
EnvOps::restore_role(...)?;
```

## 0.10.5 - 2026-02-25 - HTTP Raw Responses and Leaner Wasm Builds

### 📝 Summary

- `0.10.5` switched HTTP outcall APIs to return validated raw bytes and reduced default wasm footprint for local artifacts.

### ⚠️ Breaking

- `HttpApi::get()` and `HttpApi::get_with_label()` now return `HttpRequestResult` (raw response bytes) rather than deserializing JSON into `T`.

### 🩹 Fixed

- Fixed a deferred memory-registration race where first-touch stable-memory reads could occur before late registrations were committed.
- Endpoint dispatch now enforces runtime memory bootstrap on wasm before handler execution.

### 🔧 Changed

- HTTP ops/workflow now validate status and return raw body bytes without an extra JSON decode pass.
- `scripts/app/build.sh` now defaults to release builds for smaller wasm artifacts (set `RELEASE=0` for debug builds).
- `make test-canisters` explicitly uses `RELEASE=0 dfx build --all` so debug-gated test endpoints remain available in local smoke tests.
- Integration tests that invoke `dfx build --all` now force `RELEASE=0` for consistency with debug-gated root hierarchy/delegation test paths.
- Added a wasm-size-optimized workspace `release` profile (`opt-level = "z"`, `lto = true`, `codegen-units = 1`, `strip = "symbols"`, `panic = "abort"`).

### 🗑️ Removed

- Removed `serde_json` from the HTTP outcall decode path.
- Removed unused workspace dependencies `futures` and `ic-management-canister-types`.
