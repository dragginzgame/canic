# 0.80 Changelog

## 0.80.0 - 2026-07-03 - Packaged wasm-store bootstrap build

### 📝 Summary

- Fixes packaged downstream root `fast` builds for the implicit `wasm_store`.

- Keeps downstream projects out of Canic's internal bootstrap-store source,
  wrapper, and Cargo-profile concerns.

### 🩹 Fixed

- Generated fallback wrappers now enable the wasm-store-only Canic facade
  feature instead of the broader root control-plane feature.

- The host-side bootstrap-store builder now supplies Canic's release and fast
  profile settings when invoking Cargo for the hidden `wasm_store` build. This
  lets canonical packaged `canic-wasm-store` sources build under
  `CANIC_WASM_PROFILE=fast` without requiring downstream manifests to define
  Canic profiles.

- The packaged downstream proof now covers both bootstrap source-resolution
  paths: canonical packaged `canic-wasm-store` sibling present, and generated
  fallback wrapper when the canonical crate is absent.

### 📚 Documentation

- Active release-probe documentation now describes the special `wasm_store`
  path as Canic-owned bootstrap behavior, not a downstream wrapper
  responsibility.

### 🧪 Testing

- Focused host tests pin the generated wrapper feature and the host-owned fast
  profile config.

- The packaged downstream wasm-store proof validates both generated-wrapper and
  canonical packaged-sibling builds under `CANIC_WASM_PROFILE=fast`.

Validated with:

```bash
cargo fmt --all
bash -n scripts/ci/verify-packaged-downstream-wasm-store.sh
cargo test -p canic-host bootstrap_store
scripts/ci/verify-packaged-downstream-wasm-store.sh
git diff --check
```
