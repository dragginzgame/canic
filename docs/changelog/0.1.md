# 0.1.x Detailed Changelog

Technical release notes for the `0.1.x` minor line.

## 0.1.7 - 2025-10-08 - Root Subnet PID Capture

### 🔧 Changed

- With `dfx 0.30.2`, root can now read subnet PID metadata and persist it in `SubnetContext`.

---

## 0.1.4 - 2025-10-07 - Delegation Sync and Logging Ergonomics

### ➕ Added

- Added `ops::delegation::sync_session_with_source` to reduce repeated delegation sync flow code.
- Added a `debug!` macro that always logs with `Log::Debug` and supports an optional conditional first argument.

---

## 0.1.3 - 2025-10-05 - Documentation Refresh

### 📚 Documentation

- Refreshed logo and README content to reflect the current project layout and behavior.
- Removed outdated documentation that no longer matched code behavior.

---

## 0.1.0 - 2025-10-04 - Published

### 📝 Summary

- `0.1.0` was the first public `canic` release.

### 🔧 Changed

- Renamed from `icu` to `canic` after name conflicts on crates.io.

### ➕ Added

- Published the project to crates.io as the initial public baseline.
