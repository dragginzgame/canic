# 0.66 Changelog

## 0.66.1 - 2026-06-12 - Delegated-auth trust-anchor closeout

### Summary

- Enforces the verifier trust-anchor contract for delegated-token and
  role-attestation canister-signature verification by pairing the configured
  network label with the effective raw IC root public key.

- Requires `network = "mainnet"` to use the known mainnet raw IC root key and
  fail closed when `ic_root_public_key_raw_hex` is missing or mismatched.

- Allows the runtime root-key provider only for explicit non-mainnet verifier
  labels (`local`, `pocketic`, `testnet`) and rejects accidental use of the
  mainnet root key in those modes.

- Keeps protected endpoint verification local and self-contained; no root,
  issuer, management-canister, or threshold-ECDSA call was added to verifier
  hot paths.

- Updates the checked-in wasm-store Candid sidecar so
  `DelegatedToken.issuer_proof` uses the distinct `IssuerProof` type name.

- Refreshes closeout wording for root and issuer `"sig"` certified-data
  ownership, removes stale issuer-runtime dead-code wording, and documents the
  no-op async verifier wrappers as endpoint/runtime shape only.

## 0.66.0 - 2026-06-12 - 0.65 audit, testing, and fixing line

### Summary

- Opens 0.66 as the stabilization line for the 0.65 zero-management-ECDSA
  normal-auth epoch.

- Moves the existing endpoint perf-observability design from the 0.66 slot to
  `docs/design/0.67-perf-observability/0.67-design.md`.

- Adds `docs/design/0.66-audit-test-fix-0.65/0.66-design.md` as the active
  0.66 planning document.

- Scopes 0.66 to audit, testing, release validation, and conservative fixes for
  the 0.65 auth hard cut.

- Explicitly keeps 0.66 out of new auth protocol families, compatibility
  lanes, one-shot issuance, management-canister ECDSA in normal auth, and
  endpoint perf-observability feature work.

- Records that 0.66 may remove stale code and tests when they only preserve
  intentionally retired designs.
