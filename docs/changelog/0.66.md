# 0.66 Changelog

## 0.66.10 - 2026-06-13 - NNS topology audit commands

### Summary

- Adds `canic nns topology health` as a compact cached topology health table
  covering component registry-version alignment, subnet-catalog freshness, and
  joined node/provider/operator/data-center coverage.

- Adds `canic nns topology gaps` as the actionable companion to coverage and
  health. When cached joins are complete, it renders a small `STATUS`/`DETAIL`
  table; when joins are incomplete, it lists the exact subject, missing
  relation, and referenced id for nodes or node operators that point at missing
  providers, operators, or data centers.

- Refines `canic nns topology coverage`, `versions`, `summary`, and `health`
  text output to avoid low-signal colon headings between tables while keeping
  the existing network/source/version context available through table columns
  or the summary title line.

- Keeps all topology inspection commands cache-backed and mainnet-only. The new
  `health` and `gaps` commands use the same source endpoint fallback behavior
  as the existing topology reads when a component cache is missing.

## 0.66.9 - 2026-06-13 - Operator audit hardening

### Summary

- Adds `canic nns topology coverage` and `canic nns topology versions` so
  operators can inspect cached NNS join coverage and component registry-version
  rows without rendering the full topology summary.

- Hardens backup resume handling by rejecting persisted artifact paths that
  escape the backup root and rejecting persisted temporary download paths that
  do not match the runner's expected layout before checksum verification or
  finalization.

  current release metadata, and stale chunks from replaced or shrunken chunk
  sets no longer make a staged release appear publishable.

## 0.66.8 - 2026-06-12 - Auth proof support cleanup

### Summary

- Renames the internal verifier trust-anchor config from delegated-token-only
  wording to shared auth-proof wording, matching its use by delegated-token
  verification, role-attestation verification, and active delegation proof
  install.

- Splits non-root startup predicates so delegated-token issuers and
  role-attestation caches require root canister-signature proof verification
  support, while delegated-token endpoint verifiers additionally require issuer
  canister-signature proof verification support.

- Renames the private root delegation proof client and deletes the leftover
  endpoint-class scaffolding that made the single root proof prepare helper
  look like a broader root-auth-material client.

- Renames the internal delegated root certificate builder module and types from
  issue-oriented naming to delegation-cert preparation and proof assembly
  naming.

- Renames internal root, issuer, and role-attestation auth operation inputs from
  `Sign*Input` to `Prepare*Input` naming, so update-phase preparation code no
  longer reads like a one-shot signing path.

- Removes unused RPC workflow error variants that still described delegated
  auth delegation failures in terms of `shard_pid`.

- Cleans active delegated-auth docs, operations runbooks, and instruction-audit
  scenario text away from stale shard/mint wording in the issuer prepare/get,
  active delegation proof, and replay-diagnostic flows.

## 0.66.3 - 2026-06-12 - Strict verifier closeout

### Summary

- Requires an effective raw IC root key for every delegated-auth verifier
  network. Mainnet still requires the configured known mainnet key; local,
  PocketIC, and testnet builds may inject the IC system root key once during
  startup so protected endpoint verification never falls back to runtime
  root-key discovery.

- Enforces `delegated_token_verifier = true` at delegated-token verification
  execution time, before proof verification work, so issuer-only and
  role-attestation-only canisters cannot accidentally verify delegated tokens.

- Records ignored `.icp/local` Candid sidecars as local build artifacts that are
  excluded from release closeout evidence unless regenerated from the current
  checked-in exports.

- Pins the workspace `time` dependency at `0.3.47`, avoiding the `0.3.48`
  `ic-agent`/`tracing-subscriber` conflict while moving off the vulnerable
  `0.3.41` line.

## 0.66.1 - 2026-06-12 - Delegated-auth trust-anchor closeout

### Summary

- Enforces the verifier trust-anchor contract for delegated-token and
  role-attestation canister-signature verification by pairing the configured
  network label with the effective raw IC root public key.

- Requires `network = "mainnet"` to use the known mainnet raw IC root key and
  fail closed when `ic_root_public_key_raw_hex` is missing or mismatched.

- Requires every verifier network, including `local`, `pocketic`, and
  `testnet`, to provide an explicit `ic_root_public_key_raw_hex`; non-mainnet
  modes reject accidental use of the mainnet root key.

- Keeps protected endpoint verification local and self-contained; no root,
  issuer, management-canister, or threshold-ECDSA call was added to verifier
  hot paths.

- Updates the checked-in wasm-store Candid sidecar so
  `DelegatedToken.issuer_proof` uses the distinct `IssuerProof` type name.

- Refreshes closeout wording for root and issuer `"sig"` certified-data
  ownership, removes stale issuer-runtime dead-code wording, and documents the
  no-op async verifier wrappers as endpoint/runtime shape only.
