# 0.70 Changelog

## 0.70.15 - 2026-06-22 - Blob-storage test readability

### Changed

- Cleans up `pic_blob_storage` test readability by centralizing probe/mock
  method names as constants and adding section banners for tests, billing
  fixtures, gateway sync, endpoint guards, status/report assertions,
  Cashier/funding failure paths, install/config helpers, non-billing gateway
  lifecycle helpers, and probe upgrade helpers.

- Adds small typed helpers for repeated PocketIC funding, Cashier balance,
  mock last-top-up, and probe counter calls so billing assertions read at the
  behavior level instead of repeating low-level call boilerplate.

- Names the standard billing fixture values and adds helper wrappers for
  gateway sync, direct Cashier sync, billing status, and one-shot mock failure
  controls, keeping the PocketIC test flow in domain terms without changing
  behavior.

- Extracts the long billing wrapper scenario into named phases for initial
  balance checks, Cashier balance failure recovery, zero-cycle rejection,
  Cashier top-up error-code mapping, and final successful funding metadata.

## 0.70.14 - 2026-06-22 - Funding failure recovery coverage

### Testing

- Extends the PocketIC billing flow to prove a transient Cashier top-up failure
  through `_immutableObjectStorageFundFromProjectCycles` releases the generated
  funding path's in-flight guard, allowing an immediate subsequent valid
  top-up to succeed.

- Extends the same flow to prove malformed Cashier top-up success payloads also
  release the funding guard, and tightens reserve-skipped funding coverage so
  it must preserve the previous successful top-up metadata rather than relying
  on an empty mock history.

- Adds core guard coverage proving the transient blob-storage funding guard is
  released during unwinding as well as normal drop.

- Pins the API mapping for an already in-flight funding operation to the public
  `Conflict` error code, and documents the funding guard's drop-based cleanup
  contract at the ops boundary.

## 0.70.5 - 2026-06-21 - Reserve-blocked funding refusal

### Fixed

- Makes `_immutableObjectStorageFundFromProjectCycles` all-or-nothing with
  respect to the configured project-cycle reserve. When the requested top-up
  would violate the reserve, Canic now returns a skipped funding report with
  zero attached cycles instead of sending a partial top-up to Cashier.

- Rejects blob-storage billing configs with zero min/target upload-balance
  thresholds, and rejects gateway principal limits that cannot fit the target
  runtime before writing stable config.

- Consumes `ic-memory 0.7.1` and exposes a controller-only
  `canic_memory_ledger.memories` inventory of Canic stable memories with
  `memory_manager_id`, `stable_key`, `state`, and live backing `size` fields.
  Raw allocation records also retain optional `memory_size.wasm_pages` and
  `memory_size.bytes` diagnostics.

### Testing

- Updates focused funding-decision coverage to prove reserve-blocked requests
  attach zero cycles even when some cycles would otherwise be transferable.

- Adds unit coverage proving invalid billing config is rejected without
  replacing the previously stored valid config.

- Adds memory-ledger unit coverage for live `MemoryManager` backing sizes, DTO
  mapping, and the explicit memory inventory, plus Candid shape coverage for
  `memories` and raw record `memory_size` fields.

- Adds PocketIC coverage proving reserve-blocked funding through the generated
  endpoint does not create a successful mock Cashier top-up record.

## 0.70.2 - 2026-06-21 - Funding input hardening

### Fixed

- Rejects zero-cycle `_immutableObjectStorageFundFromProjectCycles` requests
  with `InvalidInput` before acquiring the transient funding guard. This avoids
  returning a misleading `reserve would be violated` skipped report when the
  caller requested no cycles.

- Keeps the funding guard covering the project-cycle balance observation and
  Cashier top-up decision for nonzero requests.

- Maps known Cashier `account_top_up_v1` failures to stable public error codes:
  unauthorized top-ups become `Forbidden`, account balance overflow becomes
  `ResourceExhausted`, Cashier internal errors remain `Internal`, and
  Cashier-side missing-cycle rejections become `InvalidInput`.

### Testing

  and reserve-blocked attachment decisions.

- Adds focused unit coverage for Cashier top-up error-code mapping.

- Adds PocketIC coverage proving the generated funding endpoint rejects

## 0.70.0 - 2026-06-20 - Blob storage billing MVP

### ✨ Added

- Adds the off-by-default `blob-storage-billing` feature through `canic-core`
  and the facade crate. The feature implies `blob-storage` and remains disabled
  by default.

- Adds source-backed Cashier DTOs and protocol constants for
  `account_balance_get_v1`, `account_top_up_v1`, and
  `storage_gateway_principal_list_v1` based on the maintainer-approved Toko
  contract.

- Adds a checked-in Cashier Candid fixture and protocol-surface coverage for
  the three Cashier methods, including the optional `account_top_up_v1` request
  record.

- Adds typed Cashier wrapper ops for balance reads, project-account top-ups,
  and gateway-principal list reads. The wrappers use bounded-wait calls and do
  not hardcode or default to production Cashier.

- Adds Cashier response conversion helpers that reject negative or oversized
  signed cycle balances, reject anonymous/management gateway principals,
  deduplicate returned gateway principals, and enforce a local principal-count
  bound before storage writes.

- Adds stable blob-storage billing config with explicit Cashier canister id,
  project-cycle reserve, min/target upload balances, and gateway-principal
  limit. No production Cashier id is installed by default.

- Adds configured Cashier gateway-principal sync that atomically replaces the
  local gateway-principal set and records the last successful sync timestamp.

- Adds project-as-payment-account funding through the inventoried
  `_immutableObjectStorageFundFromProjectCycles` report shape. Funding attaches
  at most the requested cycles while preserving the configured project-cycle
  reserve.

- Adds read-only backend billing status through `get_blob_storage_status`.
  Status reports billing config, project-account Cashier balance,
  gateway-principal state, readiness blockers, and explicit funding need
  without syncing gateways or transferring cycles.

- Adds `canic_emit_blob_storage_billing_endpoints!`, which emits the two 0.70
  gateway endpoints plus backend status with exact method names:
  `_immutableObjectStorageUpdateGatewayPrincipals` and
  `_immutableObjectStorageFundFromProjectCycles`, and
  `get_blob_storage_status`. The macro requires separate sync, funding, and
  status guards.

- Adds a standalone `blob_storage_cashier_mock` canister and extends
  `blob_storage_probe` with billing configuration and wrapper helpers for
  local/PocketIC validation.

### 🥾 Governance

- Marks the Cashier inventory complete for the Toko-compatible 0.70 MVP while
  keeping the actual Cashier implementation/deployed Candid as future
  superseding evidence if it contradicts the Toko-backed contract.

- Keeps production Cashier out of CI and local tests. The PocketIC flow uses an
  injected mock Cashier canister.

### 🧪 Testing

- Adds Candid round-trip and fixture parsing coverage for Cashier DTOs, status
  DTOs, and the 0.70 gateway/status method constants.

- Adds manifest coverage proving `blob-storage-billing` is opt-in and implies
  `blob-storage`.

- Adds unit coverage for Cashier signed-balance conversion, gateway-principal
  normalization, gateway-principal replacement, and malformed Cashier response
  handling.

- Adds PocketIC coverage for mock Cashier balance reads, configured
  gateway-principal sync through the macro-emitted endpoint, read-only backend
  status through the macro-emitted endpoint, and project-cycle funding through
  the macro-emitted endpoint with attached-cycle verification.
