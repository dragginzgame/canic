# 0.49 Changelog

## 0.49.4 - 2026-05-30 - declared-only canister scaffolding

### Added

- Added declared-only canister scaffolding:
  ```
  canic scaffold canister <fleet> <role>
  ```
  The command creates an ordinary canister crate under the selected fleet,
  writes `[package.metadata.canic] fleet/role`, and adds a config role
  declaration without adding topology placement. When the project root has a
  workspace `Cargo.toml`, the new crate is also added to `workspace.members`.

### Safety

- Canister scaffolding requires an existing unambiguous fleet config, rejects
  invalid names through the scaffold parser, refuses existing target
  directories, and rejects already-declared roles before writing files.

### Testing

- `cargo test -p canic-cli scaffold --lib --locked`
- `cargo test -p canic-cli --lib --locked`

## 0.49.3 - 2026-05-30 - direct role attachment

### Added

- Added direct topology attachment for declared roles:
  ```
  canic fleet role attach <fleet> <role> --subnet <subnet>
  canic fleet role attach <fleet> <role> --subnet <subnet> --kind <kind>
  ```
  The command appends a direct `[subnets."<subnet>".canisters."<role>"]`
  topology entry and defaults `--kind` to `singleton`.

### Changed

- Declared-only role inspection now points to the direct attach command as the
  next action.

### Safety

- Role attachment requires an existing role declaration, rejects `root`, rejects
  duplicate attachment, validates the kind set, and parses/validates the
  updated config before writing.

### Testing

- `cargo test -p canic-host attach_fleet_role --lib --locked`
- `cargo test -p canic-cli fleets::tests --lib --locked`

## 0.49.2 - 2026-05-29 - role declaration

### Added

- Added config-only ordinary role declaration:
  ```
  canic fleet role declare <fleet> <role> --package <path>
  ```
  The command appends a declared-only `[roles."<role>"]` entry, records the
  package path, and prints the next explicit attachment action.

### Changed

- Fleet role help now treats the namespace as role lifecycle management rather
  than read-only inspection.

### Safety

- Role declaration rejects `root`, duplicate roles, empty packages, malformed
  role names, and fleet/config mismatches before writing the updated config.
- The updated config is parsed and validated before it is written.

### Testing

- `cargo test -p canic-host declare_fleet_role --lib --locked`
- `cargo test -p canic-cli fleets::tests --lib --locked`

## 0.49.1 - 2026-05-29 - role lifecycle inspection

### Added

- Added read-only fleet role lifecycle inspection:
  ```
  canic fleet role list <fleet>
  canic fleet role inspect <fleet> <role>
  ```
  The list command renders declared fleet roles with package, lifecycle state,
  and exact topology attachment labels. The inspect command shows role
  declaration, package, declared-vs-attached state, build eligibility,
  deploy-artifact eligibility, and the next role-lifecycle action.

### Changed

- Exposed host-side role lifecycle rows so CLI surfaces use the same
  declaration and attachment derivation as build and deployment planning.

### Testing

- `cargo test -p canic-host configured_role_lifecycle --lib --locked`
- `cargo test -p canic-cli fleets::tests --lib --locked`

## 0.49.0 - 2026-05-29 - role lifecycle foundation

### Added

- Added fleet-scoped `[roles.<role>]` declarations under the required
  `[fleet] name`.
- Canister package metadata now includes both `fleet` and `role`, giving
  artifact builds an explicit `fleet.role` identity.

### Changed

- `canic::build!` validates package `fleet.role` identity against the selected
  Canic config.
- Build output now distinguishes attached roles from declared-only roles.
- `canic build` now requires visible canister artifact builds to use
  topology-attached roles, keeping declared-only roles compile-only.

### Fixed

- Generated packaged-downstream `wasm_store` bootstrap wrapper manifests now
  declare the Canic metadata required by `canic::build!`.
- CI artifact builds now default to the absolute `fleets/test/canic.toml`
  config path, keeping canister package discovery scoped to the selected test
  fleet instead of the whole workspace `fleets` tree.
- Explicit `canic build --config <path>` values are passed to canister build
  scripts as absolute paths, avoiding manifest-directory-relative config
  resolution.
