# 0.12.x Detailed Changelog

Technical release notes for the `0.12.x` minor line.

## 0.12.0 - 2026-03-07 - Root Role Attestation Framework

### 📝 Summary

- `0.12.0` introduces root-signed role attestations for root-directed authorization.
- Services can now verify caller role/subnet/audience by signed proof, while capability execution stays separately policy-gated.

### ➕ Added

- Added root issuance for `SignedRoleAttestation` with bounded lifetime (`issued_at`/`expires_at`) and explicit audience binding.
- Added root key-set publication (`canic_attestation_key_set`) so verifiers can resolve signatures by `key_id`.
- Added `canic_response_attested(request, attestation, min_accepted_epoch)` for attestation-authenticated root dispatch.
- Added verifier key-cache behavior: startup warm, periodic refresh, and one-time immediate refresh on unknown `key_id`.

### 🔧 Changed

- Root-directed RPC callers now request an attestation before privileged root operations, then dispatch through the attested path.
- Verifier checks now fail closed on signature/key errors, subject mismatch, audience mismatch, subnet mismatch, expiry, and epoch floor violations.
- Revocation now uses short TTL plus per-role `min_accepted_epoch_by_role`; rotation supports `current` and `previous` keys in a grace window.
- Added structured logs/metrics for issuance outcomes, verifier failures, unknown `key_id`, and epoch rejects.
- Legacy non-attested dispatch remains available during phased migration.

### 🧭 Migration Notes

- Services using directory-only role checks can migrate incrementally to attestation verification.
- Keep compatibility endpoints enabled until all required callers move to attested dispatch.
- Key-set certified-query delivery is still a follow-up hardening task; inter-canister fetch currently uses update-RPC.

### 🧪 Testing

- Added unit coverage for signature/key resolution, subject/audience/subnet checks, TTL/expiry, epoch floor, unknown-key refresh-once, and key-rotation acceptance.
- Added PocketIC coverage for issuance/verification happy path, unauthorized caller rejection, audience mismatch rejection, expiry rejection, and rotated-key grace handling.
