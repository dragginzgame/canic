# 0.48 Changelog

## 0.48.0 - 2026-05-28 - derived singleton topology

### Breaking

- `subnets.<name>.auto_create` is no longer accepted in `canic.toml`.
- `subnets.<name>.subnet_index` is no longer accepted in `canic.toml`.
- Existing fleet configs must remove those keys. With the hard cut, unknown
  field validation now rejects them instead of treating them as compatibility
  aliases.

### Changed

- Canic now derives root auto-create roles from configured
  `kind = "singleton"` canisters in each subnet.
- Canic now derives subnet-index roles from the same singleton role set.
- Root bootstrap, subnet-index rebuilds, host config inspection, local root
  cycle estimation, scaffold output, checked-in fleet configs, and getting
  started docs now use the derived singleton topology.

### Migration Notes

- To auto-create and index a stable subnet service, declare only the canister:

```toml
[subnets.prime.canisters.user_hub]
kind = "singleton"
```

- `kind = "shard"`, `kind = "replica"`, and `kind = "instance"` roles are not
  auto-created or subnet-indexed directly; their placement managers create
  them.
