## Result

`run_result: partial`

Application receipt admission now has one mandatory immutable absolute replay
deadline. Absent operations fail closed at or after that deadline and when the
remaining window exceeds the reviewed 24-hour signed-token ceiling. Exact
retained results remain observable before temporal or capacity rejection.

This completes the independently safe admission portion of 0.96. Terminal
eligibility and reclamation remain intentionally disabled because the
downstream action identity, recovery, observation grace, rate, backlog, and
resource-cardinality contracts are not yet frozen.

## Admission Invariant

For an application call, the maintained order is:

```text
consumer namespace check
  -> capture time once and compute pure replay-window decision
  -> validate payload and application/placement ownership
  -> exact primary lookup
  -> validate immutable binding and exact stored deadline
  -> reject orphan adjunct or placement acknowledgement
  -> apply temporal decision for an absent operation
  -> enforce receipt-record capacity
  -> enforce resource capacity
  -> insert primary and application adjunct
  -> persist resource totals
```

Computing the pure decision has no side effect. Ops does not return it until
after the exact lookup misses, so a retained pending or terminal operation
remains observable at and after its deadline. An altered deadline for the same
operation is an immutable binding conflict.

At an exact miss:

- `now >= replay_deadline_ns` returns `ReplayWindowClosed`;
- `replay_deadline_ns - now > 24 hours` returns
  `ReplayWindowTooLong`;
- exactly 24 hours remaining is admissible; and
- both rejection paths occur before capacity checks or stable mutation.

## Stable Evidence

Allocation 46 stores a schema version, complete 32-byte operation ID, and
absolute deadline. Its exact maximum encoding is 124 bytes. Missing, orphaned,
wrong-schema, wrong-identity, or Canic-owned application metadata rejects
through typed ops causes.

At the 100,000-row admission limit, the replay adjunct measures 442 ascending
or 381 deterministic-permuted Wasm pages. Primary, application replay, and
100,000 distinct maximum totals rows occupy 3,969 physical Wasm pages in the
measured ascending case after base `MemoryManager` bucket rounding. This is
the same physical subtotal as the placement tuple despite a ten-page logical
reduction.

The common primary schema, resource accounting, settlement behavior, public
snapshot surface, timer set, and placement cleanup path do not change.

## Gate Verdict

The replay-deadline admission hard cut is implemented and focused evidence is
positive. It does not authorize terminal deletion. The next bounded work is to
freeze the downstream terminal observation grace and numeric capacity/recovery
contract, then measure the smallest eligibility reservation mechanism before
adding reclamation or timer behavior.
