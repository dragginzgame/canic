# Post-0.88 Codebase Health Audit - 2026-07-13

## Scope

- Snapshot: published `v0.88.2` at `5542f4f2`.
- Reviewed: deployment-state persistence errors, installed-deployment
  observation, command projection boundaries, dependency manifests, and the
  audit-identified RPC request exports.
- Excluded: full workspace tests, PocketIC, deployment, network fetches,
  persisted-schema changes, and broad visibility churn.

## Executive Summary

- Risk score: **3 / 10**.
- All three bounded 0.88 slices are published and their acceptance conditions
  remain satisfied.
- The strongest remaining boundary issue is source loss across deployment-state
  reads and installed-deployment observation. The host owns typed I/O, JSON,
  and replica-query failures but converts them to strings before ten command
  adapters classify them.
- Cargo Machete initially reports five candidates; targeted compilation proves
  four are unused and one is a generated proc-macro consumer. The RPC request
  owner still exposes three private command adapters beyond their file.

## Findings

### Medium - Installed-deployment evidence loses typed causes

Evidence:

- The install-state owner has six boxed dynamic-error signatures covering
  validation, reads, writes, and public lookup entry points.
- `InstalledDeploymentError` stores install-state and replica-query failures as
  `String`, even though `ReplicaQueryError` already preserves I/O and JSON
  sources.
- Ten CLI conversion functions classify the same installed-deployment variants
  into command-owned errors. Ten command error enums retain install-state or
  replica-query failures as strings.
- Additional direct install-state reads convert their error to text before it
  reaches auth, backup, blob-storage, cycles, snapshot, and token boundaries.

Impact:

Commands can preserve their exit and hint policy, but callers and diagnostics
cannot inspect whether deployment evidence failed because of an invalid name,
filesystem operation, malformed JSON, replica transport, replica decoding, or
replica rejection. Source chains are lost before command classification is
complete.

Recommended hard cut:

1. Give install-state persistence one focused typed error.
2. Preserve that error and `ReplicaQueryError` directly in
   `InstalledDeploymentError`.
3. Retain the concrete causes in command errors while leaving command-specific
   no-install and lost-root wording local.
4. Delete the boxed and string variants as each owner migrates.

Do not add a global CLI error framework, public error-code schema, or generic
command-error projector.

### Low - Four dependency declarations have no consumer

Cargo Machete and source inspection initially identified five candidates:

- `sha2` in the test root canister build dependencies;
- `ic-testkit` in the runtime probe development dependencies;
- `canic-host` in the integration-test development dependencies; and
- `ic-cdk` plus `serde_json` in the facade development dependencies.

Targeted facade integration-test compilation proved that `ic-cdk` is a false
positive: the endpoint proc macros expand through that directly available
crate. Retain it with one package-local Cargo Machete annotation documenting
the generated consumer. The other four declarations have no package feature,
runtime dependency, generated API, or persisted-data effect and are removed.

### Low - RPC command adapters are wider than their owner

`CyclesRpc`, `UpgradeCanisterRpc`, and `RecycleCanisterRpc` are used only by the
request dispatch owner and its local tests. Two are re-exported from the request
module, while all three structs and their fields are public beneath a
crate-private ancestor. `RequestOps` and `RequestOpsError` need crate-wide
visibility; the concrete command adapters do not.

The correction is narrow: remove the two adapter re-exports, make all three
adapters private, and make the request namespace and error only as visible as
their actual crate consumers require. This is not authority for a broad
`unreachable_pub` rewrite.

## Lower-Priority Signals Not Selected

- `docs/status/current.md` is no longer compact. It should be archived and
  shortened as documentation maintenance, but it is not a runtime release
  slice.
- Other boxed errors remain at boundaries whose callers only render them. They
  are not selected without evidence that classification or source inspection
  occurs after erasure.
- Host, backup, scaffold, and pending-operation persistence retain their
  deliberately separate durability owners.
- The upstream unmaintained transitive dependency advisories remain outside
  local ownership.

## Recommended 0.89 Scope

1. Type install-state persistence failures.
2. Preserve install-state and replica-query causes through installed-deployment
   and command classification.
3. Remove the four proven dead dependencies, account for the one generated
   facade test dependency, and narrow the exact RPC adapter surface.

After those slices, close 0.89. Do not expand the line into global error,
visibility, dependency, or project-context frameworks.
