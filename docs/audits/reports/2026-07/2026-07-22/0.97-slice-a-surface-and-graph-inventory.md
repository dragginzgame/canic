## Purpose And Result

This report freezes the audit-only evidence required before 0.97 changes a
Cargo validator, macro expansion, public Rust path, or ICP-refill contract.
The repository was inspected directly at immutable `v0.96.8`; the design and
status documents were treated as claims rather than implementation evidence.

Slice A passes. The inventories identify one finite migration for each active
surface and expose the exact current contradictions that later slices must
hard-cut. No product source, Cargo package version, Candid, stable record, CLI,
or runtime behavior changed in this slice.

## Dependency Path Normalization

`DependencyPathEvidence` uses raw Cargo package IDs only as private visited
node identities. Its stable comparison and rendering facts are:

- dependency alias exactly as resolved on the edge;
- package name and semantic version;
- normalized source kind: `workspace_path`, `external_path`, `registry`,
  `git`, or `other`;
- a slash-normalized workspace-relative package directory only for a
  `workspace_path`; and
- no absolute manifest, registry cache, URL credential, or raw package-ID
  text.

The workspace root and manifest paths are canonicalized before containment is
tested. A contained path must resolve to `Cargo.toml`; its parent is rendered
relative to the workspace without `.` or `..`. A source-less package outside
the workspace is `external_path` and carries no local path in diagnostics.
`registry+`, `git+`, and any remaining source prefix normalize to their kind
only. Unknown or structurally incomplete evidence fails the role check rather
than borrowing checkout-specific text.

Path choice is shortest edge count, then lexical comparison of alias, package
name, version, normalized source-kind label, and optional workspace-relative
path at every step. If two internal identities normalize to identical display
facts, either path has identical output; raw package-ID spelling is never a
tie-break or a displayed contract.

### Stable error producer proof

`FabricationUnavailable`, `RateGateDenied`, and `RequestDenied` occur only in
the public/stable enum definitions, bidirectional conversion matches, and
metrics label matches. No workflow or storage mutation constructs any of the
three codes. The pure policy's `IcpRefillPolicyViolation::RateGateDenied` is a
different pre-persistence typed rejection and does not produce the stable
`IcpRefillRecordErrorCode::RateGateDenied` variant. The three stable/response
variants therefore have exact no-producer evidence and are removable together
with their conversions/metrics arms. No decode-only tombstone is retained.

## Slice B Implementation Addendum

The first real isolated role run exposed one additional Cargo 1.97 boundary:
`cargo tree --target wasm32-unknown-unknown` can print a target-inactive normal
dependency that `cargo metadata --filter-platform wasm32-unknown-unknown`
correctly excludes. The canonical producer therefore resolves tree labels
against a complete unfiltered package catalog, then admits only edges also
present in target-filtered metadata. This is a refinement of the frozen
three-way ownership, not a new validator path: unfiltered metadata owns
identity, filtered metadata owns target admission, and the package-selected
tree owns role activation.

Slice B implements that single producer and routes build, passive role
resolution, state-manifest resolution, and Medic through it. Its strict parser
is bounded by line count, line length, and depth; ambiguous package/edge
correlation fails closed. Real isolated Cargo workspaces cover the supported
shape and the frozen hard-cut rejection cases. Maintained role declarations
now state their complete feature ownership explicitly, and the one direct
protected sibling dependency identified above is removed in favor of the
semantic Canic facade.
