## Cases Executed

All six post-preflight operation variants were exercised at both sides of the
shared terminal execution-journal publication boundary:

- `Stop`;
- `CreateSnapshot`;
- `Start`;
- `DownloadSnapshot`;
- `VerifyArtifact`; and
- `FinalizeManifest`.

For each operation,
`CANIC-094-B16/<operation>/before-durable-write` kills the child after the
normal operation path has constructed and recorded its terminal result in
memory, but before the synchronized journal staging file is renamed. The
surviving canonical journal contains `Pending` and no terminal receipt.
Restart reconciles through the operation's already-proven lifecycle status,
snapshot inventory, artifact journal, checksum, or manifest authority and
publishes one normal terminal pair.

`CANIC-094-B16/<operation>/after-durable-write` kills the child only after the
rename and parent-directory synchronization return. The surviving canonical
journal contains `Completed` and exactly one matching receipt with the same
transition timestamp. Restart skips the operation, reports zero new work, and
does not append a receipt.

Result: all 12 B16 cases pass. Pre-write cases reach recovery disposition
`reconciled`; post-write cases reach `resumed` by adopting the already
terminal document. No restart repeats a stop, snapshot-create, start, or
download command.

## Finding And Implementation

No product defect was found. `BackupExecutionJournal::record_operation_receipt`
already changes the in-memory operation state and appends its receipt as one
validated transition, and the runner publishes the complete journal with one
durable write.

The runner now accepts an internal terminal-writer owner so the crash harness
can stop the exact production flow at the durable writer's named barriers.
The maintained entrypoint still selects `BackupLayout::write_execution_journal`
unconditionally. The test-only entrypoint calls the same journal validation,
serialization, synchronization, rename, and directory-sync primitive with a
barrier callback. No production failpoint, environment variable, CLI surface,
fallback writer, or alternative state transition exists.

## Boundary Proof

- Before publication, terminal state cannot survive without its receipt
  because neither has entered the canonical document.
- After publication, terminal state cannot survive without its receipt because
  document validation requires their exact identity, outcome, and timestamp
  agreement.
- Restart from pre-write `Pending` uses the specific operation authority proven
  by B05 through B15 rather than inferring failure or blindly repeating work.
- Restart from post-write `Completed` invokes no completed operation and leaves
  the one existing receipt unchanged.
- The public API, CLI, JSON, Candid, durable document schemas, and package
  versions are unchanged.
