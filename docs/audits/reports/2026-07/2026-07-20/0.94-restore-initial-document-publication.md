## Authority Trace

`RestorePlanner` derives and validates the plan from the immutable manifest.
`RestoreApplyDryRun::try_from_plan_with_artifacts` verifies every referenced
artifact before `RestoreApplyJournal::from_dry_run` constructs the pristine
journal. `create_or_adopt_restore_plan` and
`create_or_adopt_restore_apply_journal` are the only restore persistence
owners used by maintained preparation. Both use the same create-only durable
JSON primitive: write and sync sibling staging, publish without replacement,
remove staging, then sync the parent directory.

The test-only barriers wrap that primitive; they add no production failpoint,
alternate writer, JSON shape, CLI surface, or compatibility behavior. The
parent/child handshake is shared with the existing backup crash matrix rather
than duplicated for restore.
