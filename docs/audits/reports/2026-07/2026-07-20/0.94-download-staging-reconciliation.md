## Case Executed

`CANIC-094-B09/download-snapshot/effect-committed-receipt-missing` runs the
maintained backup runner in a child process after stop, snapshot creation, and
start have completed. The runner durably claims `DownloadSnapshot`, creates
its exact private staging directory, writes and synchronizes partial download
bytes, and is killed with `SIGKILL` before it can publish the `Downloaded`
artifact-journal transition or terminal execution receipt.

After command quiescence, durable authority remains the exact `Created`
artifact row. Restart validates its target, snapshot identity, and canonical
artifact path, removes only the uncommitted private staging directory, and
downloads once into a fresh private directory. It then writes one normal
execution receipt and the exact `Downloaded` row. No canonical artifact is
published, and completed replay executes no command.

Result: `B09` passes with recovery disposition `resumed`.

## Finding And Fix

### `CANIC-094-BACKUP-005` - P1 - fixed

At `v0.94.5`, a durable pending download was classified with unknown external
mutations and halted permanently. The owned private staging path supplied
enough local authority for a narrower rule: bytes not represented by the
`Downloaded` journal transition are uncommitted and may be replaced, provided
the exact `Created` target, snapshot, and path still match.

The runner now admits that retry only after validating the artifact journal.
It rejects missing or duplicate target rows, snapshot mismatch, path mismatch,
and every state other than exact `Created`. The private staging reset inspects
the entry without following it, removes only a directory, rejects symlinks,
files, and special entries, and recreates the directory with owner-only Unix
permissions.

Artifact-journal validation now gives every state an exact field set. This
prevents a weaker state from carrying a temporary path or checksum that could
compete with its canonical transition. `Downloaded` and later evidence is
preserved unchanged for `B10`; it is never erased or silently redownloaded.

No fallback reader, legacy state, duplicate download helper, compatibility
surface, journal edit, or operator escape hatch was added.

## Rejection And Boundary Proof

- A symlink at the exact staging path rejects with a typed unsafe-entry cause,

- Exact `Downloaded` evidence rejects the B09 retry path without changing the

  checksum verification.
- Contradictory `Created`, `Downloaded`, and `Durable` rows reject fields that
  belong to another artifact state.
- The all-operation pending-claim matrix now proves a durable download claim
  with exact `Created` evidence follows this canonical resume rule.
