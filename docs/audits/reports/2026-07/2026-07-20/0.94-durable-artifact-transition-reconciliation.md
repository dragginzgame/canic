## Cases Executed

`CANIC-094-B14/durable-artifact-transition/before-durable-write` starts with
an exact canonical artifact, a checksum-verified artifact row, and a pending
finalization operation. The child is killed before the durable journal rename.
Restart observes `ChecksumVerified`, adopts the already published directory by
checksum, writes `Durable`, and completes finalization.

`CANIC-094-B14/durable-artifact-transition/after-durable-write` kills the child
after the durable journal rename and parent-directory synchronization. Restart

persistence owner, and completes finalization without invoking publication.

Both cases preserve the canonical directory inode, issue no external command,
append one finalization receipt, and end with one exact durable artifact and
manifest. Result: both B14 cases pass with recovery disposition `reconciled`.

## Finding And Fix

`CANIC-094-BACKUP-009` (P1) found that finalization skipped every artifact
already marked `Durable`. If the canonical directory disappeared or changed
after the journal transition but before the manifest and execution receipt,
restart built the manifest from the journal alone and returned success.

Persistence now owns one durable-artifact verifier that requires `Durable`,
resolves the journal path beneath the backup root, requires the canonical
artifact, traverses it without following links, and verifies its checksum.
Complete-layout integrity and finalization recovery call the same helper.
Exact durable state proceeds; missing or conflicting bytes preserve their
typed cause and cannot create a manifest.

## Rejection And Boundary Proof

- Removing the canonical directory after the durable transition returns
  `PersistenceError::MissingArtifact` and leaves finalization failed.
- Changing its bytes returns the nested typed checksum mismatch with the exact
  journal hash.
- Both failures retain the durable journal claim for diagnosis, create no
  manifest, invoke no command, and do not attempt a download or publication.
- Manifest publication, terminal execution receipt boundaries, and final
  response loss remain assigned to later frozen cases.

## Surface And Scope
