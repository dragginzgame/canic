## Finding And Fix

### `CANIC-094-BACKUP-003` - P1 - fixed

At `v0.94.3`, a pending snapshot-create operation retained no authoritative
pre-effect inventory. Once its command became quiescent, restart could neither
prove absence nor identify a committed snapshot and therefore halted forever.
The artifact journal also retained only the snapshot ID, so it could not
reconstruct the complete normal execution receipt.

The execution operation now durably records all snapshot IDs observed before
the effect. Recovery computes an exact target inventory delta:

- zero new identities justifies one create;
- one new identity is adopted and persisted through the canonical artifact
  and receipt path;
- missing baseline identities, duplicate identities, or multiple new
  identities reject without mutation; and
- a complete existing artifact row reconstructs the receipt before any
  external observation.

A returned create-command failure retains that attempt's original inventory,
including across the explicit failed-to-ready retry transition. The next run
reconciles it before retrying, covering the case where the platform committed
a snapshot but command output failed.

No guessed newest snapshot, timestamp heuristic, operator-supplied identity,
manual journal edit, blind retry, alias, or fallback path was added.

## Contract And Hard-Cut Boundary

- `BackupRunnerExecutor` requires typed snapshot inventory and shares one
  `BackupRunnerSnapshot` shape between observation and creation.
- The host ICP boundary shares one `IcpSnapshot` shape between `snapshot list
  --json` and `snapshot create --json`.
- Execution-operation JSON requires explicit `snapshot_ids_before`; artifact
  rows require explicit snapshot timestamp and size fields, including `null`
  when the platform did not supply them.
- These genuine durable contracts remain version 1 under the maintainer's
  pre-1.0 hard-cut rule. Superseded shapes reject and must be regenerated.
  There is no v2 document, old reader, migration, default, or compatibility
  adapter.
- CLI syntax, ordinary CLI output, Candid, manifests, plans, and Cargo package
  versions are unchanged.
