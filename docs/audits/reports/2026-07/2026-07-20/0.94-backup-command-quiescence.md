## Cases Executed

The following frozen cases pass:

- `CANIC-094-B18/stop/owner-dead-command-in-flight`;
- `CANIC-094-B18/create-snapshot/owner-dead-command-in-flight`;
- `CANIC-094-B18/start/owner-dead-command-in-flight`; and
- `CANIC-094-B18/download-snapshot/owner-dead-command-in-flight`.

For each operation, a new runner process advances the exact operation to
`Pending` and invokes a real exec'd test command through the executor boundary.
That direct command inherits the operation's command-lifetime descriptor,
spawns an effect descendant, and reaches a named handshake. The parent
acknowledges the barrier and sends `SIGKILL` to the runner without unwinding.

The direct command then closes its own descriptor while remaining alive. A
restart still receives `BackupRunnerError::CommandInFlight`, proving the
descendant—not the dead runner or direct child—is the remaining lock owner.
The rejection occurs before any executor method and preserves the pending
journal, absent terminal receipt, and exact layout tree.

After explicit release, the descendant commits one modeled external outcome
and exits. The harness proves the command tree is quiescent before invoking
recovery. Stop and start reconcile from exact lifecycle status; snapshot
creation reconciles one exact inventory delta. None repeats its mutating
command. Download replaces the uncommitted private bytes and performs one
safe canonical download, as required by the B09 authority. Each case then
finishes the complete backup and passes integrity validation.

Result: all four B18 cases pass. Live-command rejection reaches recovery
disposition `safely_halted`; after command exit, each journey reaches
`reconciled` or `resumed` and then completes.

## Finding And Implementation

No new product defect was found. `CANIC-094-PROC-001`, fixed in `v0.94.1`,
already gives every mutating runner operation a sequence-bound kernel lock,
passes only that descriptor into the intended command tree, rejects restart
while it remains held, and requires quiescence before terminal persistence.

B18 adds only a Unix operational-readiness harness. A private test preparation
helper is shared with B16 so both cases reach each operation through the same
canonical runner sequence. The harness configures the inherited descriptor at
the executor boundary exactly as the separately proven host command adapter
does. No production process identity, PID file, polling service, daemon,
failpoint, CLI option, journal field, or fallback recovery path was added.

## Boundary Proof

- The journal is durably `Pending` before the descendant effect is released.
- Closing both runner and direct-child descriptors does not permit restart
  while the descendant retains its inherited open-file description.
- Typed in-flight rejection performs no status observation, snapshot
  inventory call, mutation, receipt append, or tracked layout write.
- Quiescence is established independently before effect reconciliation.
- Status and inventory evidence prevent duplicate stop, snapshot creation, and
  start; private uncommitted download bytes never become durable authority.
- Every journey ultimately publishes one terminal receipt for its interrupted
  operation and completes a verified backup.
- Public APIs, CLI/JSON/Candid surfaces, persisted formats, and package
  versions are unchanged.
