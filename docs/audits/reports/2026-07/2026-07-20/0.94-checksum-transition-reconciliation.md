## Finding And Fix

`CANIC-094-BACKUP-007` (P1) found that pending local replay previously called
the ordinary verification operation again. Because artifact transitions
admitted same-state advancement, restart could recompute and overwrite a
durable checksum. Changed staged bytes could therefore replace already
recorded authority rather than reject.

The runner now gives pending verification an explicit reconciliation path.
Exact `Downloaded` evidence uses the ordinary calculation path. Exact
`ChecksumVerified` evidence is adopted only after the target, snapshot,
artifact path, staging path, checksum, and actual bytes agree. A mismatch
returns the existing typed checksum cause and leaves both durable journals
unchanged.

Artifact journal advancement now permits only the immediate canonical
transitions `Created -> Downloaded -> ChecksumVerified -> Durable`. Exact
artifact-row lookup is shared by download and verification recovery and
rejects ambiguous target authority.

## Rejection And Boundary Proof

- Replacing staged bytes after the checksum-verified row is durable returns
  `ArtifactChecksumError::ChecksumMismatch` with the recorded and observed
  hashes.
- The pending execution operation remains pending, the original checksum row
  remains checksum-verified, and no canonical artifact is created.
- Recovery of the exact row requires no command-lifetime lock because checksum
  verification cannot leave an external child command alive.
- Canonical directory publication, the `Durable` transition, manifest
  publication, terminal receipt boundaries, and final response loss remain
  assigned to later frozen cases.
