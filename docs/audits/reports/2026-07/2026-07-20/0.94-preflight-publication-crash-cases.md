## Cases Executed

### `CANIC-094-B02`

The child published the preflight-applied backup plan through the maintained
durable JSON replacement primitive. The parent sent `SIGKILL` after an
acknowledged barrier on each side of the authoritative rename:

- after the temporary file was synchronized but before rename, restart found
  the original declared plan and repeated the read-only preflight; and
- after rename and parent-directory synchronization, restart adopted the
  complete applied plan and accepted the matching receipts into the journal.

Both restarts executed zero backup operations. Their only external
observations were one status read per target, and both converged on the exact
same applied plan and accepted journal.

Result: both `B02` cases pass.

### `CANIC-094-B03`

The child published preflight acceptance in the execution journal at the same
two barriers:

- before rename, restart observed the applied plan with the original
  unaccepted journal, kept mutation blocked, and repeated read-only preflight;
  and
- after rename and parent-directory synchronization, restart adopted the
  complete accepted journal without issuing any command.

Both restarts executed zero backup operations and ended with the exact applied
plan and accepted journal.

Result: both `B03` cases pass.
