## Finding And Fix

`CANIC-094-BACKUP-010` (P1) found that the canonical manifest writer used
durable replace semantics. A pending finalization restarted after manifest
publication would overwrite whatever document occupied the canonical path,
including conflicting or premature evidence, and then return success.

`BackupLayout` now owns one `publish_manifest` path. It validates the expected
manifest, atomically creates the canonical path without replacement, adopts an
existing validated manifest only when its normalized data exactly matches,
and returns `ManifestConflict` otherwise. A race that creates the destination
is reconciled through the same exact comparison.

The runner rejects any manifest present before the finalization operation is
`Pending`, retryable `Failed`, or `Completed`. This prevents a premature
document from surviving while earlier mutation continues, while still
allowing an exact published manifest to reconcile after a returned persistence
failure. No manifest content is used to invent or advance execution state.

## Rejection And Boundary Proof

- An existing valid manifest with a different creation identity returns
  `PersistenceError::ManifestConflict`; its exact bytes are unchanged and the
  finalization operation records failure.
- A manifest appearing while finalization is still `Ready` returns
  `BackupRunnerError::PrematureManifest` before any command or journal change.
- Exact post-publication recovery preserves the canonical manifest inode,
  proving that adoption is not replacement.
- Invalid existing JSON or manifest data preserves its typed parsing or
  validation cause rather than being normalized.
- Terminal execution receipt boundaries and final successful response loss
  remain assigned to later frozen cases.

## Hard Cut And Surface

- Public `BackupLayout::write_manifest` is removed;
  `BackupLayout::publish_manifest` is the sole maintained manifest publication
  API. No alias, wrapper, deprecated method, or replace fallback remains.
- Public `PersistenceError` and `BackupRunnerError` gain `ManifestConflict` and
  `PrematureManifest` respectively.
- Manifest JSON shape and version `1`, other durable documents, CLI syntax,
  JSON command output, Candid, plans, and Cargo package versions are unchanged.
