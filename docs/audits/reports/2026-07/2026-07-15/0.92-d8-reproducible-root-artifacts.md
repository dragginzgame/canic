# 0.92 D8 Reproducible Root Artifacts

## Verdict

D8 passes focused implementation and current-tree validation. Two fully
isolated offline root builds now produce byte-identical root and bootstrap
store Wasm/gzip artifacts. The complete semantic build-provenance envelopes
also match after only the frozen v2 observation-time normalization, and neither
final root Wasm contains either lane's temporary build path.

This fixes `CANIC-092-BUILD-001` and `CANIC-092-BUILD-002`. Root runtime
records retain stable role, artifact-kind, size, and content-hash evidence only.
The host builder is the sole owner of optional transform identity and records
the role, transform, mode, tool, tool version, and outcome. No path alias,
compatibility payload, deprecated field, or fallback provenance decoder is
retained.

## Authority and Hard Cut

- `EmbeddedRootBootstrapEntry` no longer carries source or generated asset
  paths. The generated root bundle uses its absolute asset path only as the
  compile-time `include_bytes!` input; that path is not emitted as runtime data.
- Root lifecycle diagnostics no longer project build-machine paths. They log
  only stable role, artifact kind, compressed/decompressed size, and hashes.
- The canonical host artifact builder records each considered shrink and
  Candid-metadata transform. Root provenance also includes the bootstrap
  store's transform decisions because those bytes are embedded into root.
- `Applied` requires a non-empty tool version.
  `ToolUnavailable` and `NotRequested` forbid a tool version. Inconsistent
  transform evidence fails provenance construction.
- An unavailable optional `ic-wasm` remains nonfatal and is recorded. A
  present tool with no version identity or a failed invoked transform fails
  the build. Failed shrink output is removed and cannot replace the original.
- `BuildProvenanceV1.transforms` is required. This is a deliberate pre-1.0
  hard cut to `canic.build_provenance.v1`; older payloads are not accepted by a
  serde default, alternate schema name, or compatibility parser.

No Candid endpoint, stable-state record, fleet configuration, deployment
truth, or install authority changes in D8.

## Rejection, Boundary, and Regression Proof

- Missing shrink tool: nonfatal and recorded as `tool_unavailable`.
- Missing metadata tool: nonfatal and recorded as `tool_unavailable`.
- Present shrink tool: output replaces the original only after success and
  records the exact version.
- Present metadata tool: success records the exact version.
- Failed shrink: rejects, preserves the original, and removes partial output.
- Failed metadata: rejects.
- Present tool with empty version identity: rejects before transform execution.
- Applied provenance without a version, or unapplied provenance with a version:
  rejects as inconsistent typed model state.
- Root/bootstrap generation: both isolated builds validate gzip, decompressed
  Wasm magic, compressed/decompressed size, and content hashes before the root
  artifact is emitted.
- Complete release-set manifest admission: positive ordering and rejection
  cases continue to pass without changing prior manifest bytes on refusal.
