# 0.92 D4 Root-Issuer Admission Ownership

## Verdict

D4 passes focused validation. Root-issuer policy and renewal-template
admission now follows one path:

```text
authenticated endpoint -> workflow -> pure policy -> ops conversion/commit
  -> model-owned state -> stable record
```

This fixes `CANIC-092-TEST-001` and removes the root-issuer portion of
`CANIC-092-LAYERING-005`. The canonical layering finding remains open because
18 other production ops-to-policy dependencies are outside D4.

No public DTO, Candid encoding, stable record, migration, alias, shim,
fallback, deprecated API, or duplicate admission path was added.

## Canonical Ownership

- The endpoint authenticates root context and delegates immediately.
- `RuntimeAuthWorkflow` maps requests through ops, invokes pure admission
  policy, and commits only after acceptance.
- Policy owns the root-policy and renewal-template decisions without storage,
  DTO, timer, or IC dependencies.
- Model owns issuer policy, template, renewal state, proof reference, outcome,
  and attempt shapes.
- Ops owns DTO/model conversion, stable-record conversion, persistence, and
  status projection. It no longer calls root-issuer admission policy.
- Stable record types and their encoded fields are unchanged.

The former policy module no longer re-exports the moved state shapes. Source
search finds no compatibility path or second root-issuer upsert authority.
