# 0.92 D7 Internal Surface Hard Cuts

## Verdict

D7 passes focused validation. The public proof-install request/outcome mirror
and the unnecessary direct `canic_core::error` root are removed completely.
Proof provisioning now follows one maintained path:

```text
root renewal or explicit root facade -> proof-install workflow
  -> internal ops batch plan -> issuer public install request/response
  -> model-owned failure classification -> persisted diagnostic
```

This fixes `CANIC-092-LAYERING-004` and `CANIC-092-SURFACE-001`. No alias,
deprecated wrapper, fallback, duplicate DTO, or compatibility path remains.
Maintained Candid endpoints and wire DTOs, stable state, installation behavior,
and stored failure labels are unchanged.

## Surface and Ownership Result

- `RootDelegationProofBatchInstallRequest` duplicated the internal batch plan
  and is deleted from the public auth DTO surface.
- `RootDelegationProofInstallOutcome` mixed one success state, one unreachable
  success state, and internal failure diagnostics. It is deleted rather than
  relocated as another public or boundary type.
- The existing internal ops plan now carries scheduled batch work directly.
  The explicit root facade passes its batch identity and proof directly to the
  private workflow.
- Issuer installation returns the existing typed
  `IssuerProofInstallError`; success is `Ok(())` rather than an internal state
  enum exposed as serialized API.
- The model owns the four persisted failure classifications. Ops maps and
  stores them; workflow owns multi-step installation orchestration.
- `canic_core::error` is no longer an external root. Direct consumer scans
  found none. The deliberate hidden `control_plane_support::error` bridge
  remains the single public sibling-support path and compiles in its consumer.
- Maintained public proof wire types remain available, including the issuer
  install request/response, batch proof, status, policy, and renewal DTOs.

Historical reports and released changelog records are retained as history.
Active Rust sources, tests, examples, and documentation contain no use of the
removed public types or direct error root.

## Failure and Diagnostic Contract

| Issuer-install result | Internal stored classification |
| --- | --- |
| request encoding, transport, or invalid response | `CallFailed` |
| `AuthProofExpired` | `ExpiredOrSuperseded` |
| `AuthMaterialStale`, `AuthProofPending`, or invalid input | `ProofMismatch` |
| other issuer rejection | `RejectedBySigner` |
| success | no failure variant; record installation success |

The exact stored labels remain `CallFailed`, `ExpiredOrSuperseded`,
`ProofMismatch`, and `RejectedBySigner`. Typed public causes remain preserved
at their existing boundary. Rejection cannot be mistaken for installation
success, and the removed `AlreadyInstalled` outcome had no reachable producer.
