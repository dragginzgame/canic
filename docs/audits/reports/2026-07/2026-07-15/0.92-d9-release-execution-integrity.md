# 0.92 D9 Release Execution Integrity

## Verdict

D9 passes focused implementation and current-tree validation. All 13 active
external GitHub Action executions are pinned to reviewed full commit
identities. CI and maintainer setup install exact executable versions; every
direct archive is verified against a repository-owned platform digest before
extraction or execution. One governance document now owns the sole
release-supported host/native/Wasm cell.

This fixes `CANIC-092-RELEASE-001`, `CANIC-092-RELEASE-002`, and
`CANIC-092-RELEASE-004`. It does not install a secret scanner or waive
`CANIC-092-RELEASE-003`. The retained release-integrity method therefore
remains blocked for final 0.92 closeout even though every D9-owned subcheck now
passes.

## Executable Tool Authority

`tool-versions.env` is the single repository-owned version/digest contract for
downloaded executable helpers and Cargo-installed maintainer/CI tools.

| Tool path | Exact identity | Verification before execution | Result |
| --- | --- | --- | --- |
| actionlint | 1.7.12 | Platform archive SHA-256; reported version | PASS |
| ShellCheck | 0.11.0 | Platform archive SHA-256; reported version | PASS |
| ICP CLI | 1.0.2 | Official platform archive SHA-256; reported version | PASS |
| `ic-wasm` | 0.9.11 | Platform npm archive SHA-512; direct binary version | PASS |
| PocketIC | 14.0.0 | Compressed and decompressed SHA-256; reported version | PASS |
| CI Cargo helpers | Exact crate versions | Cargo registry package checksum plus `--locked` | PASS |
| Maintainer Cargo helpers | Exact crate versions | Cargo registry package checksum plus `--locked` | PASS |

The contract currently contains 14 SHA-256 archive/binary pins and four
SHA-512 platform-package pins. The integrity guard discovers every maintained
version and checksum field from that contract, validates each field, and
proves that every installer-referenced checksum exists; it does not maintain a
second version-variable ledger or fixed checksum count. A shared verifier
accepts only a correctly sized lowercase digest, fails before extraction on
mismatch, and preserves expected and actual values in its deterministic
diagnostic.

The ICP CLI installer no longer downloads and executes an upstream shell
script. `ic-wasm` no longer uses npm, its JavaScript wrapper, or optional
package resolution: Canic verifies and extracts the exact platform binary.
PocketIC verifies restored cache content before use as well as both downloaded
and decompressed content on a cache miss.

Installer-supplied version arguments and environment version overrides are
removed. Install locations remain configurable, but tool identity comes only
from `tool-versions.env`. The shared IC prerequisite check loads that authority
unconditionally and rejects either `icp` or `ic-wasm` when its reported version
differs. There is no alternate unchecked installer, legacy argument form, or
download-and-execute fallback.

## Canonical Supported Matrix

| Host | Native target | Canister target | Status |

Linux AArch64 and macOS x86_64/AArch64 installer branches are explicitly
install-capable but not release-supported. Windows, other native targets,
other Wasm targets, and non-Linux-x86_64 PocketIC installation are explicitly
excluded. No installer branch can silently widen the matrix.

## Operator Contract Hard Cut

- `make install-dev` now requires existing Rustup/Cargo instead of piping the
  mutable Rust bootstrap script into a shell.
- `make update-dev` synchronizes exact repository-owned tool versions without
  updating unrelated Rust toolchains or mutating workspace dependency
  resolution.
- Version-surface mutation is transactional: a failed post-bump command
  restores every touched manifest, the lockfile, and the CLI default.
- One-shot releases execute bump, stage, commit/tag, and push sequentially;
  push requires the exact clean release commit and annotated tag, clears build
  artifacts before network mutation, then updates branch and tag refs
  atomically as the final local command.
- Current setup and ICP diagnostics direct maintainers to the checksum-bound
  repository installer rather than an unverified remote installer.
- The public host README no longer presents a remote script pipe as the normal
  setup path.
- The obsolete release-tag URL alignment test is deleted with that shortcut;
  the maintained test still binds the installed Canic CLI default to the
  workspace release version.

No runtime canister behavior, Rust/Candid API, stable state, JSON/configuration
schema, dependency lock, deployment authority, or Wasm artifact changes in D9.
Operator setup and diagnostic text change deliberately.
