## Subsystem Corrections

### Environment and cycles funding

Complete initialized environment values now belong to model. Pure policy
admits raw environment input, and ops converts the accepted model value to the
stable record.

Cycles-funding limits and ledger snapshots now belong to model. Config ops
returns limits rather than constructing a policy object, storage ops projects
ledger state, and workflow invokes pure evaluation and cooldown policy.

### Topology and index

Observed topology, registry, and index values now belong to model. Ops maps
stable records to those neutral values without importing policy-owned input
types. Topology policy consumes the model observations.

Topology policy error projection now lives with the central internal/public
error boundary. The duplicate ops conversion module is deleted; existing
public policy codes and domain fallback behavior are unchanged.

### Placement and metrics

Scaling and sharding observations and shared plan values now have model
owners. Policy computes decisions, ops maps and persists model values, and
workflow retains orchestration. Misleading policy-input mapper names and the
old scaling `upsert_from_plan` path are hard-cut rather than aliased.

Scaling and sharding metric recorders consume model-owned reason categories.
Pool policy rejection classification moves to the pool workflow, so metrics
ops no longer depend upward on policy.

### Root delegation batches

Root issuer admission now occurs in workflow before proof-epoch advancement
or insertion of a newly prepared batch. Ops returns an opaque due-template
plan, workflow validates every issuer through pure auth policy, and ops accepts
only an ordered set of issuer-bound approvals. Approval count, issuer identity,
and shared expiry are checked fail-closed before batch mutation.

The combined ops-owned get-or-create orchestration and the ops auth-policy
error mapper are removed. Explicit provisioning now performs plan, policy,
commit, exact-batch signing, and proof retrieval through the workflow. No
alias, compatibility wrapper, deprecated path, or fallback remains.
