## Canonical Authority

- `AGENTS.md` remains normative; `docs/contracts/ARCHITECTURE.md` now states
  that it mirrors that contract and must be corrected on divergence.
- The only documented dependency direction is
  `endpoints -> workflow -> policy -> ops -> model`.
- Endpoints authenticate/marshal and delegate immediately to workflow; active
  API wording no longer authorizes direct delegation to ops.
- Model owns authoritative state and storage invariants.
- Storage is a passive persisted/cached representation; ops owns deterministic
  state access, conversion, and approved single-step platform effects.
- Public core docs, crate docs, module headers, and hygiene guidance use the
  same ownership language.

The implementation still contains the separately indexed API orchestration,
DTO behavior, and 25 ops-to-policy dependency violations. Those require D4 and
later finding-backed product slices; D3 does not move or excuse them.
