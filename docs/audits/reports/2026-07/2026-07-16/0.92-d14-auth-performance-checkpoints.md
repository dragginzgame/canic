# 0.92 D14 Auth Performance Checkpoints

## Scope

The maintainer explicitly reopened P2 `CANIC-092-PERF-001` after the immutable
`v0.92.12` closeout. Released in `v0.92.13`, this bounded slice adds internal
instruction checkpoints to the two auth flows named by the finding:

- root chain-key delegation-proof creation, signing, lookup, and installation;
- delegated-token request validation, replay reservation, proof preparation,
  lazy root-proof repair, response commit, cache lookup, and full proof
  verification.

The slice changes no authorization decision, proof format, public API, stable
state, configuration, dependency, package-version, or error projection.

## Implementation

Root-proof provisioning now records stages in
`workflow/runtime/auth/provisioning`, including policy resolution, batch
preparation, chain-key signing, issuer proof lookup, and issuer installation.

Delegated-token preparation now records request/replay, proof preparation,
response encoding/commit, and lazy root-proof repair stages in
`workflow/runtime/auth/prepare`. Delegated-token verification records context
and cache-key resolution, cached verification, root policy resolution, full
embedded-proof verification, and positive-cache installation in
`ops/auth/token`.

The checkpoints use the existing crate-level `perf!` authority. No second
metrics store, timing abstraction, or flow wrapper was introduced. A source
regression test binds the maintained checkpoint-gap roster to representative
stage labels across all three owners.

## Finding Disposition

`CANIC-092-PERF-001` is fixed in `v0.92.13`. The instruction
audit can now distinguish the expensive internal stages of both named auth
flows, and its static gap detector reports those flow owners as instrumented.

The released `v0.92.12` closeout remains immutable at 40 fixed findings and
three deferred P2 watchpoints. After D14's release, the current ledger is 41
fixed findings and two deferred P2 watchpoints.
