### Accepted JSON hard cut

0.92.7 adds required `BuildProvenanceV1.transforms` evidence while retaining
the named pre-1.0 schema. An envelope generated by the old builder lacks that
field; an envelope generated by the current builder records four explicit
root/bootstrap transform decisions. Running both through the same current
conservative provenance policy produces:

```text
v0.91.6 envelope: exit 1; blocked_by_policy;
  policy.build_provenance.invalid_payload; missing field `transforms`
v0.92.11 envelope: exit 0; policy_status passed
```

This rejection is deterministic and typed at the policy boundary. The
documented migration is to rebuild affected artifacts and regenerate saved
provenance; there is intentionally no serde default, alternate schema name,
legacy DTO, or compatibility parser.
