## First Process-Death Results

### `CANIC-094-B01`

The child serialized the real execution-journal type through the maintained
durable JSON replacement primitive. At each boundary the child signalled the
parent, the parent acknowledged, the child confirmed it was armed, and the
parent sent `SIGKILL`:

- after the sibling temporary file was synchronized but before rename,
  restart found no authoritative journal and published one complete valid
  journal; and
- after rename and parent-directory synchronization, restart read the exact
  complete valid journal.

Result: both `B01` cases pass. `CANIC-094-J02` remains pending because the
other 50 backup cases have not run.

### `CANIC-094-V01` through `CANIC-094-V03`

The same acknowledged process-death protocol interrupted verification:

- before document validation;
- during the first non-empty artifact checksum read; and
- after the successful integrity result but before output.

After each death, a fresh verification recomputed the full result and passed.
A recursively sorted path, entry-type, and file-byte inventory taken before
the child ran exactly matched the inventory after restart. No plan, journal,
manifest, receipt, artifact, or pass marker changed.

Result: `CANIC-094-J03` passes with recovery disposition `resumed`.
