## Snapshot Probe

A detached canister was created on the disposable local network, installed
with an eight-byte test Wasm module, stopped, snapshotted, downloaded, and
uploaded. The original and uploaded snapshot identities were returned and
then recovered from the target snapshot inventory:

- created snapshot: `0000000000000000ffffffffffc000020101`;
- uploaded snapshot: `0000000000000001ffffffffffc000020101`;
- created snapshot inventory size: 40 bytes;
- uploaded snapshot inventory size: 8 bytes.

The stopped target was then installed with a different module, and the
uploaded snapshot was restored twice. After each restore:

- target status remained `Stopped`;
- the target module hash was the original snapshot module hash
  `93a44bbb96c751218e4c00d479e4c14358122a389acca16205b1e4d0dc5f9476`;
- the second exact restore succeeded; and
- the snapshot inventory remained exactly the same two identities.

This proves that the maintained local ICP boundary can exercise snapshot
create, upload, stopped-target restore observation, and exact repeated restore
for the executable fixture. It does not prove restored application bytes from
management status alone; the realistic journey must still start the contained
fleet and query deterministic application state before declaring success.

An uploaded snapshot is recoverable from an exclusive before/after inventory
delta, but the inventory does not itself bind that new identity to the source
artifact checksum. Upload reconciliation therefore retains the design's
exclusive-activity requirement and may not infer checksum identity from a
snapshot ID alone.

The detached canister was deleted and the local network was stopped after the
probe. No production identity, credential, canister, or network was used.

## Command-Lifetime Probe

The current host command boundary uses `std::process::Command::output()` and
does not establish a distinct owned execution identity, process group,
timeout, or restart-visible child record. A self-contained Linux probe killed
the direct parent of a running child without unwinding. The child remained
alive, was reparented to PID 1, and retained the inherited process group.

The current product therefore cannot prove that a prior external command or
its descendants are quiescent before a restarted backup or restore runner
observes state or issues another command.
