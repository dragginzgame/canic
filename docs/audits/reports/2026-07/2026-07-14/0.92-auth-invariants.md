## Verdict

- Combined run result: `fail`.
- Combined result validity: `invalid`.
- Product security verdict: no authentication bypass was found; the inspected
  and executed acceptance paths fail closed.
- Comparability: five methods are valid and comparable to their named June
  baselines. Audience v1 and replay v1 are invalid because four frozen focused
  commands select zero tests after test renames.
- Mandatory trace status: the trace blocks below are supporting evidence only,
  not completed `TRACE-AUTH-001` or `TRACE-CAPABILITY-001` runs. The separate
  mandatory-trace admission report records the missing frozen trace method.

The canonical generated endpoint path remains:

```text
transport ingress
  -> macro validation and generated access stage
  -> token material and complete proof-chain verification
  -> delegated subject binding
  -> required local-role scope validation
  -> access-expression authorization
  -> handler execution
```

Proof verification uses the configured root canister, explicit network root
key, canonical certificate and claims hashes, issuer binding, exact local
audience, signed role grants, expiry, and required scope. Positive-cache hits
rerun local semantic checks. Replay-sensitive commands use domain receipts;
delegated bearer-token verification does not introduce verifier-local
single-use state.

The combined baseline still fails because required negative integration proof
is absent, typed proof causes are collapsed into strings or broad outcomes,
and an internal proof-install plan/outcome was exposed as a public DTO with
three outcomes that current production code cannot construct.
