## Compatibility And Hard-Cut Review

The executable `v0.91.6` contract baseline is still incomplete. Before
closeout or any accepted hard cut, the line must compare:

- generated Candid/interfaces;
- JSON/configuration/provenance schemas;
- CLI command, exit-code, text, and structured output;
- stable state produced by `v0.91.6` through upgrade;
- backup/restore behavior;
- package features and public Rust surface; and
- operator/changelog migration text.

The current queue contains likely hard cuts to the public Rust/serialized
surface (`CANIC-092-LAYERING-004`, `CANIC-092-SURFACE-001`, and
`CANIC-092-RESIDUE-001`). They must remove obsolete paths completely. The
review found no authority for an alias, shim, fallback, deprecated path,
duplicate command, or old-form anti-resurrection test.

No stable-state schema change is currently proposed. If a later accepted
slice changes stable state, the full fixture, interrupted-upgrade, recovery,
and rollback boundary from the design becomes mandatory.
