The accepted design names ten mandatory trace IDs and requires every trace to
record a frozen trace method ID, version, and fingerprint. No trace method is
defined in the active catalog, fingerprint manifest, shared audit contract, or
a dedicated definition. The design's list and completion template are not a
versioned method and have no recorded fingerprint. A reviewer therefore cannot
tell which entrypoints, samples, execution fixtures, acceptance rules, or
unreviewed boundaries constitute a reproducible completed trace.

Supporting code/execution traces may still inform product findings. They do not
satisfy the mandatory trace gate and must not be reported as such.

## Admission Evidence

The accepted design at
`docs/design/0.92-holistic-audit-and-audit-system-validation/0.92-design.md:484-535`
does all of the following:

- states that every mandatory trace uses the frozen trace method;
- names `TRACE-DEPLOY-001`, `TRACE-AUTH-001`,
  `TRACE-CAPABILITY-001`, `TRACE-CYCLES-001`, `TRACE-INTENT-001`,
  `TRACE-CONTROL-001`, `TRACE-TOPOLOGY-001`, `TRACE-BLOB-001`,
  `TRACE-BACKUP-001`, and `TRACE-LIFECYCLE-001`; and
- requires `trace_method_id`, `trace_method_version`, and
  `trace_method_fingerprint` in each report.

The frozen audit system does not supply those values:

| Authority | Observed state |
| --- | --- |
| `docs/audits/METHODS.md` | Catalogs 22 active audit methods; no mandatory trace method or ten trace definitions. |
| `docs/audits/method-fingerprints-v1.md` | Fingerprints all 22 active definitions and executable composites; no trace method/template fingerprint. |
| `docs/audits/AUDIT-HOWTO.md` | Defines `code_trace` and `execution_trace` safety, but not mandatory trace sampling, completion, or verdict rules. |
| `docs/audits/META-AUDIT.md` | No mandatory trace method identity or conformance rule. |
| `0.92-method-freeze.md` | Freezes the 22 cataloged definitions and named inputs; no trace method identity. |
| Repository search | Finds the ten IDs and required fields only in the accepted design. |

Because the design itself requires identity/version/fingerprint fields, filling
them with a newly invented value during Phase C would silently change the
measurement after freeze. Using one of the topical audit-method fingerprints
would also be incorrect: none owns the common mandatory trace protocol or the
ten exact coverage contracts.

This is a post-freeze method defect. Correct closeout requires:

1. retain this invalid/blocked admission result;
2. define the exact trace entrypoints, required transitions, rejection and
   partial-failure samples, execution permissions, completion rules, and
   verdict semantics;
3. assign versioned method identities and fingerprints;
4. rerun all ten corrected trace methods against the original `v0.92.0`
   product baseline; and
5. compare only results produced by the corrected frozen method.

No product change is authorized by this finding.
