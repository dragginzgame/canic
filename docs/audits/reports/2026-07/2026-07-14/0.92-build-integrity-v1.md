# 0.92 Build Integrity And Reproducibility v1 Baseline

## Verdict

- Run result: `fail`.
- Result validity: `invalid`.
- Static build-owner review: `fail`.
- Reproducibility execution: ordinary app artifacts `pass`; root artifacts
  `fail`; raw provenance comparison `partial`.

Two isolated release-profile builds produced byte-identical ordinary app Wasm,
gzip, bootstrap-store Wasm, bootstrap-store gzip, and embedded bootstrap asset
bytes. The final root Wasm and gzip differed because generated runtime data
contains absolute build-state and Cargo `OUT_DIR` paths. Those paths are also
logged during root lifecycle initialization.

The frozen v1 method has a post-freeze comparison defect: it requires raw
provenance output comparison but does not exclude the documented observational
`generated_at` field and its derived payload digest. Canonical provenance
guidance explicitly says timestamps are not provenance. The v1 result is
therefore invalid for closeout and must be superseded by a versioned method
whose semantic comparison rule follows that active contract. The independently
confirmed shipped root-byte difference remains a product finding and must be
reproduced under the corrected method.

## Build Owner Inventory

The repository contains 21 Cargo build-script files:

- 16 narrow canister/fleet wrappers delegate to `canic::build!` with one
  configuration path;
- `crates/canic-wasm-store/build.rs` delegates to the same macro using the
  canonical host-provided configuration path when present;
- `crates/canic-core/build.rs` accepts only resolved build networks `local` and
  `ic`, defaults an absent value to local development, and rejects every other
  direct `ICP_ENVIRONMENT` value;
- `crates/canic/build.rs` declares facade cfg names and the two explicit
  internal-test/metrics environment inputs;
- `crates/canic-control-plane/build.rs` declares its test-only cfg; and
- `canisters/test/delegation_root_stub/build.rs` embeds three explicitly
  prepared test Wasm modules, rejects missing/malformed Wasm for Wasm builds,
  uses inert host placeholders only for host checking, and declares its source
  and prepared-artifact rerun inputs.

Files named `build.rs` below `src/` are ordinary Rust modules, not Cargo build
scripts, and were classified by their manifest relationship rather than file
name alone.

The host owner is `canic-host::canister_build`: it admits a declared attached
role, applies the exact resolved build network/config/ICP root to Cargo, builds
the bootstrap store before root, copies the raw artifact, optionally shrinks,
exports and embeds local Candid metadata, writes deterministic gzip, and then
lets `canic-cli` optionally emit stable build provenance. A failed installed
transform tool stops the build and preserves the original artifact; a missing
tool is intentionally nonfatal under the accepted 0.91.4 contract.

authority: a named environment resolves through `icp.yaml` to build network

build state even when the resolved build network is `ic`. The root builder
always refreshes its bootstrap store in the same context before root
compilation, so the `.icp/local` storage name is not evidence of local auth
configuration being embedded into an IC artifact.
