## Endpoint Macro Sequencing

Macro validation requires explicit public/auth semantics and a first
`DelegatedToken` argument for authenticated endpoints. Expansion reads the raw
transport caller, resolves the authenticated subject, builds `AccessContext`,
and awaits `eval_access` before dispatching handler code. Caller/topology
predicates use the transport lane; authenticated predicates use the delegated
subject lane.

The private token-material helper used by delegated-session bootstrap does not
claim to be endpoint authorization. The bootstrap validates its requested
subject against the verified token before replay binding or session mutation.
Executable bootstrap mismatch/replay proof remains finding
`CANIC-092-AUTH-001`, but the code ordering itself is correct.

## Root Proof and Issuer Sequencing

Root issuer policy/template APIs require root authority before mutation.
Chain-key prepare validates mode, registry/policy, TTL, audience, grants,
capacity, and canonical batch material before persistence. Signing covers one
canonical batch header and reuses signed state. Install planning derives each
issuer witness without re-signing, and the issuer verifies root trust plus its
local issuer binding before storing active proof.

Normal issuer token prepare starts from local active proof and does not call the
root on success. Lazy repair runs only for classified stale/missing proof
errors, makes at most one root provision attempt, and retries local prepare once.
Public prepare rejects subject mismatch and privileged self-grants before replay
reservation.

## Replay Sequencing Analysis

Root RPC supports command-specific preflight modes. Where replay reserves
before authorization, policy denial aborts the unused reservation. Successful
authorized execution is committed afterward. A failure after an external
effect retains recovery-required state instead of reopening the effect.
Cached response decode failure does not return partial success.

The endpoint replay-policy catalog is classification metadata, not itself an
execution guard. This method's primary RPC paths have real replay integration.
The separately traced durable-publication paths do not have their declared cost
integration and remain `CANIC-092-COST-001`; that is not converted into an auth
ordering failure here because the frozen scope explicitly names auth/RPC paths.
