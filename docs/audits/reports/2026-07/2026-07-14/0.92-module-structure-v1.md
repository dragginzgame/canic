### Upward dependency details

The 25 production files span auth delegation, config, conversion, placement
mappers, environment, metrics, cycles, index storage, auth storage, and
topology input mapping. This is not a grep-only conclusion:

- `ops/auth/delegation/chain_key_batch/mod.rs` imports and calls
  `validate_root_delegation_proof_prepare_policy`;
- `ops/auth/delegation/root_issuer_renewal/mod.rs` imports and calls
  `validate_root_issuer_renewal_template_policy`;
- `ops/auth/delegation/root_issuer_policy.rs` defines policy validation and
  constructs `RootIssuerPolicy` before persistence;
- `ops/storage/auth/{mod.rs,mapper.rs}` round-trip renewal policy types through
  stable records; and
- `ops/storage/cycles.rs` snapshots and restores `FundingLedgerSnapshot`, a
  state-transfer type owned under policy.

The edge is one-way, so it is not a cycle. It is nevertheless a confirmed
lower-to-higher dependency and competing state/decision owner.
