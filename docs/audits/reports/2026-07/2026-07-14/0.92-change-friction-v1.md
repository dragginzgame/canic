## Post-Freeze Method Defect

Canonical finding: `CANIC-092-AUDIT-014` (`audit_method_defect`, P1,
confirmed, open).

### Incomplete and contradictory scope

The method says every file must be assigned to exactly one of eight
subsystems, but the current core tree has materially larger active scope:

| Coverage | Files | Logical LOC | Share |
| --- | ---: | ---: | ---: |
| eight mapped subsystems | 383 | 50,549 | 70.1% files / 68.1% LOC |
| unmapped active scopes | 163 | 23,654 | 29.9% files / 31.9% LOC |
| total core | 546 | 74,203 | 100% |

Specific contradictions:

- `domain/policy/pure/**` contains current pure policy but is outside the
  listed `policy/**` path. The map instead treats `access/**`, which owns
  runtime authentication boundaries, as policy.
- `api/**` is one of the eight subsystems but does not appear in the five-layer
  CAF model, so API feature changes have no deterministic layer attribution.
- The 0.91.0 role-admission sample changes only `bootstrap/**`, `config/**`,
  and a root support file. All seven core files are unmapped, producing no
  valid subsystem or CAF value for a selected functional feature.
- Current `role_contract/**`, `replay_policy/**`, `infra/**`, lifecycle, root
  contracts, and other active scopes are likewise absent.

### Competing result algorithms

Step 11 defines a five-bucket weighted index and permits a one-tier escalation
only when at least two tracked metrics increase week over week. The later
`Risk Score` section instead adds independent hub, CAF, enum, cross-layer,
growth, capability-surface, and fan-in points to the rounded index. It provides
no precedence or evidence-deduplication rule. On the same supporting data the
weighted base rounds to 6 while the trailing modifiers clamp to 10.

### Unfrozen measurement identities

- The selection rule says to prefer 3-5 recent slices but does not freeze a
  release window, complete-commit versus core-only scope, merge handling, or a
  deterministic selection order.
- `p95`, switch sites, internal/external imports, primary module, fan-in, and
  decision sites do not have exact counter/parser definitions. Grouped Rust
  imports make line-based results especially unstable.
- The first frozen run cannot compute LOC/fan-in deltas required by the
  gravity-well condition, yet the trailing score can still add a hub-growth
  modifier without a comparable prior measurement.

Impact: independent auditors cannot reproduce the selected population,
classify all files, calculate all CAF layer counts, or choose one final score.

Required correction:

1. Preserve this result as invalid supporting evidence; do not edit v1.
2. Define the complete product scope and an exhaustive subsystem-to-layer
   map, including API and current policy ownership.
3. Freeze slice selection, percentile, import/counter, fan-in, and module
   locality rules.
4. Retain one risk algorithm with overlap and first-baseline behavior.
5. Version/fingerprint v2 and rerun it against the immutable `v0.92.0`
   product snapshot.
