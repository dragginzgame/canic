## Semantic Provenance Rule

Both JSON envelopes are parsed and normalized with exactly:

```text
del(.generated_at, .payload.generated_at, .payload_sha256)
```

`payload_sha256` is excluded only because it is calculated over the typed
payload containing the observational payload timestamp. The complete nested
payload remains and is compared after removal of that one timestamp. No source,
Cargo, configuration, environment, feature, status, artifact, path-display,
size, or artifact-hash field is removed.

Raw app JSON differs at exactly outer `generated_at`, payload `generated_at`,
and `payload_sha256`. Its normalized envelope hash is identical across lanes.
Root normalized JSON differs only on the final root raw/gzip artifact hashes;
sizes and all other normalized fields match.

## Result

preserving the decision-bearing root difference. The eventual product slice
must remove runtime absolute paths and record transform identity/status; it
must not normalize root artifact hashes, hide differing bytes, or add a
compatibility build path.
