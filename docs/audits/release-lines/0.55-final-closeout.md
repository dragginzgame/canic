# Canic 0.55.x Final Closeout Audit

Date: 2026-06-01

## Executive Summary

Verdict: **PASS**.

0.55.x is release-ready as a stabilization/readiness line. It proves the
compact v1 operator story without adding another product layer or expanding
mutation authority. The maintained local smoke and heavier operator proof now
cover the fast setup/catalog/evidence subset and the build-provenance to
deployment-check evidence path.

No blocker or high-severity finding was found. The 0.55.3 candidate audit is
not stale, but it is superseded by this final post-0.55.4 audit because 0.55.4
added the operator proof after the candidate audit landed.

0.55 avoided deployment groups, signing, locks, registry import, teardown,
controller mutation, active adoption/import, broad live verification, provider
wrappers, new stable DTO families, and one-command deployment pipelines.

create fleet -> scaffold role -> attach role -> build with provenance ->

## Script Audit

### `scripts/ci/v1-readiness-smoke.sh`

Verified:

- uses `#!/usr/bin/env bash` and `set -euo pipefail`;
- requires an executable local Canic binary or `CANIC_BIN`;
- creates a temporary root with `mktemp -d`;
- installs a cleanup trap that removes the temp root;
- writes only temp workspace files and explicit output artifacts;
- runs the intended command sequence:
  ```text
  canic fleet create demo --yes
  canic scaffold canister demo store
  canic fleet role inspect demo store
  canic fleet role attach demo store --subnet prime
  canic fleet role inspect demo store
  canic deploy catalog list --format json --output catalog.json
  canic evidence gate --policy policy.toml --envelope envelope.json --format json --output gate.json
  ```
- asserts declared-only, attached, empty catalog, and policy gate output;
- exits non-zero on failed commands or failed assertions;
- does not use broad `|| true`;
- does not require network/mainnet or live replica state.

### `scripts/ci/v1-operator-proof.sh`

Verified:

- uses `#!/usr/bin/env bash` and `set -euo pipefail`;
- requires an executable local Canic binary or `CANIC_BIN`;
- creates a unique proof root by default;
- rejects an explicitly supplied non-empty proof directory;
- copies only `fleets/demo/canic.toml` into the proof root;
- runs the intended command sequence:
  ```text
  canic build demo app --profile fast --icp-root <proof-root> --provenance <proof-root>/build-provenance.json
  canic deploy register demo-local --fleet-template demo --root uxrrr-q7777-77774-qaaaq-cai --allow-unverified
  canic deploy check demo-local --format envelope-json --build-provenance <proof-root>/build-provenance.json
  ```
- captures the expected non-zero deployment-check status and then verifies the
  saved envelope has `exit_class = "blocked_by_policy"`;
- does not install the fleet, verify a live root, query or mutate a replica,
  change controllers, register artifacts in `wasm_store`, sign evidence,
  acquire locks, create groups, perform teardown, or import brownfield state;
- intentionally leaves proof artifacts under the proof root for inspection;
- writes persistent files only under the proof root and normal Cargo build
  outputs under `target/`.

## Proof Artifact Audit

The local smoke generated and checked:

- temp fleet config and scaffolded canister files;
- declared-only role inspection containing `state: declared` and deploy
  artifact blocking;
- attached role inspection containing `state: attached` and deploy artifact
  eligibility;
- `catalog.json` with `"entries": []` plus `catalog.no_deployment_state`;
- `gate.json` with `"policy_status": "passed"` and
  `"gate_exit_class": "success"`.

The heavier operator proof generated:

```text
/tmp/canic-v1-operator-proof.7ttoIP/build-provenance.json
/tmp/canic-v1-operator-proof.7ttoIP/deployment-check-envelope.json
```

Inspected evidence showed:

- build provenance payload schema `canic.build_provenance.v1`;
- build target identity with `fleet = "demo"` and `role = "app"`;
- build payload `build_status = "success"`;
- build artifact kind `wasm_gzip`;
- deployment-check target kind `deployment`;
- deployment target `demo-local`;
- deployment-check input of kind `build_provenance`;
- input schema `canic.build_provenance.v1`;
- deployment-check payload schema `canic.deployment_check.v1`;
- deployment-check envelope `exit_class = "blocked_by_policy"`;
- path displays are relative or redacted where expected.

The `blocked_by_policy` result is correct. The proof intentionally registers
local deployment-target state without installing the fleet, verifying a live
root, or building every fleet artifact.

## Passive / Active Boundary Audit

0.55 did not introduce or imply:

- deployment groups or promotion lanes;
- saved-evidence catalogs beyond existing policy manifests;
- locks;
- signing or attestations;
- provider wrappers;
- artifact registry import;
- `wasm_store` retention or garbage collection;
- active adoption/import;
- controller mutation;
- topology mutation beyond explicit `canic fleet role attach`;
- install or upgrade authority;
- teardown;
- broad live verification;
- one-command deployment pipelines.

Additional boundary checks:

- `canic build <fleet> <role> --provenance <path>` remains attached-role
  strict, covered by build tests and the v1 checklist.
- `canic deploy check <deployment> --format envelope-json` emits saved
  evidence and does not install or live-verify the registered root in the
  operator proof.
- `canic evidence gate` evaluates saved evidence and does not turn policy
  success into deployment truth.
- `canic deploy catalog list/inspect` reads local deployment-target state only
  and does not refresh live state.
