# 0.65 Zero-ECDSA Auth Cleanup Report

Status: not closeout-clean.

This report is the release-blocking cleanup artifact for the 0.65
zero-management-ECDSA normal-auth hard cut. It records the current state after
the design tightening pass, the standalone delegated-grant capability proof
hard cut, the bounded positive delegated-token verifier cache, signed opaque
delegated-token `ext` claims, and the issuer-proof DTO/canonical foundation.
It is not the final cleanup proof: active source scans still show normal-auth
ECDSA and old shard-signature surfaces that must be removed, converted, or
reduced to typed rejection paths before closeout.

## Summary

- Deleted files/modules in this cleanup pass:
  `crates/canic-core/src/workflow/rpc/capability/grant.rs`.
- Removed DTOs and stable fields in this tightening pass: none.
- Candid regeneration in this tightening pass: not run.
- Implementation cleanup status: incomplete.
- Positive verifier cache status: implemented for the current token verifier
  shape using proof hash, claims hash, `shard_sig` hash, and caller. The final
  zero-ECDSA issuer-proof cut must replace the `shard_sig` hash component with
  `issuer_proof_hash`.
- Delegated-token ext status: implemented for the current token issue and
  verify path as opaque signed `ext: Option<Vec<u8>>` inside
  `DelegatedTokenClaims`, with canonical hash coverage and a 4096-byte bound.
- Issuer-proof foundation status: DTOs, canonical `IssuerProof` hashing,
  `issuer_proof_binding_hash`, issuer canister-signature verifier-message
  helpers, and the future issuer-proof cache-key helper are implemented. The
  runtime token shape still has to replace `shard_sig` with `issuer_proof`.
- Closeout status: blocked until source, feature, Candid, docs, and test
  evidence are clean.

## Removed DTOs And Stable Fields

No production DTOs or stable fields were removed in this cleanup pass.
`CapabilityProof::DelegatedGrant` and `DelegatedGrantProof` remain as legacy
wire-decodable DTOs, but root capability RPC rejects standalone delegated-grant
capability proof envelopes before payload decode, hash checks, secp256k1
verification, replay, or execution.

Required removal or migration remains for:

- `DelegationProof.root_sig` old wire success paths.
- `shard_sig` success-path token fields.
- `shard_public_key_sec1`.
- `IcThresholdEcdsaSecp256k1` auth DTO variants.
- threshold-ECDSA key-name and derivation-path hashes.
- delegated root public-key records used only by normal auth verification.
- old replay/cost fields that classify normal auth as `ThresholdEcdsaSign`.

## Stable-State Migration Decisions

Final migration decisions are still required per deleted auth field. Each field
must end in exactly one bucket:

- safely ignored during decode,
- explicit one-time migration drops it, or
- runtime fails closed with a controlled incompatible-state error.

Do not silently preserve old stable auth authority fields.
