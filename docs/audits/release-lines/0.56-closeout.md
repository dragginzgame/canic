# Canic 0.56.x Closeout Audit

## Executive Summary

Verdict: PASS

0.56 completed its intended release-confidence role. It did not add product
surface; it tightened the installed and packaged proof paths around the compact
v1 story. The retained proof set now answers three clear release questions:

- can an installed `canic` binary run the maintained v1 readiness smoke from a
  temp project;
- can packaged CLI/support crates inspect a downstream fleet without repository
  crate paths or stale command shapes;
- can the special packaged downstream `wasm_store` bootstrap wrapper build from
  packaged Canic sources.

No blocker or high findings were found. The only notable caveat is that the
MSRV baseline is now Rust `1.91.0`; an attempted `+1.88.0` check failed before
compilation because the workspace declares `rust-version = "1.91.0"`, which is
expected.

## Proof Behavior

### Installed CLI

Verified `scripts/ci/verify-installed-canic-cli.sh`:

- installs `canic-cli` into a temporary `install-root`;
- asserts the binary is exactly `<temp>/install-root/bin/canic`;
- rejects repository `target/*` binaries;
- isolates `HOME`, `CARGO_HOME`, `CARGO_TARGET_DIR`, and `TMPDIR`;
- runs `scripts/ci/v1-readiness-smoke.sh` with `CANIC_BIN` set to the
  installed binary;
- passes with `installed canic CLI probe passed`.

### Packaged Downstream CLI

Verified `scripts/ci/verify-packaged-downstream-cli.sh`:

- creates package archives before downstream use;
- unpacks packages under a temporary package root;
- patches `canic`, `canic-backup`, `canic-control-plane`, `canic-core`,
  `canic-host`, and `canic-macros` to unpacked packages;
- rejects repository crate paths and `target/debug/canic` in the packaged tool
  root;
- creates a downstream fleet with explicit `[roles.root]` and `[roles.app]`
  package declarations;
- runs current read-only v1 commands:

  ```text
  canic fleet list
  canic fleet role list downstream
  canic fleet role inspect downstream app
  canic deploy catalog list --format json --output <path>
  ```

- passes with `packaged downstream CLI probe passed`.

### Packaged Wasm Store

Verified `scripts/ci/verify-packaged-downstream-wasm-store.sh`:

- packages same-version Canic sibling crates explicitly;
- unpacks packages under a temporary package root;
- excludes `canic-wasm-store` from the package root so the generated wrapper
  path is exercised;
- rejects repository crate paths and `target/debug/canic`;
- patches downstream `canic` dependencies to packaged sibling crates;
- runs the special `canic-host` `build_artifact` example for `wasm_store`;
- verifies generated wrapper and artifacts:

  ```text
  .icp/local/generated/canic-wasm-store/Cargo.toml
  .icp/local/canisters/wasm_store/wasm_store.wasm
  .icp/local/canisters/wasm_store/wasm_store.wasm.gz
  .icp/local/canisters/wasm_store/wasm_store.did
  ```

- verifies generated wrapper paths point at packaged `canic`,
  `canic-control-plane`, `canic-core`, and `canic-macros` sources;
- passes with `packaged downstream wasm_store probe passed`.

## Boundary Audit

0.56 did not add:

- deployment groups or promotion lanes;
- signing, attestations, or locks;
- provider-specific GitOps integration;
- artifact registry import or `wasm_store` retention/GC;
- active adoption/import;
- controller mutation;
- topology mutation beyond existing explicit role lifecycle commands;
- install/upgrade authority;
- teardown;
- broad live verification;
- new stable public DTO families;
- one-command deployment pipelines.

The proof scripts preserve repository fixtures. Temporary state is created under
`/tmp` via `mktemp` and cleaned up by traps, except the existing operator proof
prints its proof root for inspection by design.
