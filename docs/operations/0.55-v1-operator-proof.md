# 0.55 V1 Operator Proof

This proof is the heavier release-readiness rehearsal for the compact v1
operator story. It covers the part intentionally skipped by the fast local
smoke:

```text
canic build <fleet> <role> --provenance <path>
canic deploy check <deployment> --evidence-envelope
```

It still avoids live install/controller work. The script builds the checked-in
`demo.app` canister, writes stable build provenance, registers one explicit
local deployment target in a temporary proof root, and emits a deployment-check
evidence envelope that fingerprints the build provenance.

## Run It

From the repository root:

```bash
cargo build -p canic-cli
scripts/ci/v1-operator-proof.sh
```

To choose the output directory:

```bash
scripts/ci/v1-operator-proof.sh /tmp/canic-v1-operator-proof
```

To use a different `canic` binary:

```bash
CANIC_BIN=/path/to/canic scripts/ci/v1-operator-proof.sh
```

The output directory must be empty when supplied explicitly.

## What It Runs

The script writes proof artifacts under its output directory and runs:

```text
canic build demo app --profile fast --icp-root <proof-root> --provenance <proof-root>/build-provenance.json
canic deploy register demo-local --fleet-template demo --root uxrrr-q7777-77774-qaaaq-cai --allow-unverified
canic deploy check demo-local --evidence-envelope --build-provenance <proof-root>/build-provenance.json
```

Expected outputs:

```text
<proof-root>/build-provenance.json
<proof-root>/deployment-check-envelope.json
```

## Expected Result

Successful output ends with:

```text
v1 operator proof passed
```

The deployment-check envelope is expected to have:

```text
exit_class = "blocked_by_policy"
```

That is intentional. The proof registers a local deployment target, but it
does not install the fleet, verify a live root, or build every fleet artifact.
The raw `canic deploy check` command therefore returns a blocked status; the
script treats that as success only after verifying the saved envelope contains
the expected `blocked_by_policy` exit class.
The point is to prove that the v1 evidence chain is wired honestly:

- `canic build` emits stable `canic.build_provenance.v1`;
- `canic deploy register` creates explicit deployment-target local state;
- `canic deploy check --evidence-envelope` emits stable envelope output for
  the known deployment target;
- the deployment-check envelope fingerprints the supplied build provenance.

## What It Does Not Do

This proof does not:

- install or upgrade Wasm;
- query or mutate a live replica;
- verify the registered root;
- mutate repository `.canic` or `.icp` state;
- change controllers;
- register artifacts in `wasm_store`;
- sign evidence;
- acquire locks;
- create deployment groups;
- perform teardown;
- import brownfield deployments.

The only persistent writes are under the proof output directory and normal
Cargo build outputs under `target/`.
