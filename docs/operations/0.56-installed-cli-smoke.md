# 0.56 Installed CLI Smoke

This proof verifies the installed-binary path for the compact v1 operator
story. It is intentionally smaller than a deployment proof: it checks that an
installed `canic` binary can run the maintained v1 readiness smoke from a temp
project without falling back to the repository debug binary or repository
state.

Run it from the repository root:

```bash
make test-installed-canic-cli
```

or directly:

```bash
scripts/ci/verify-installed-canic-cli.sh
```

## What It Proves

The script:

1. installs `canic-cli` into a temporary install root;
2. asserts the proof binary is the temporary installed binary, not
   `target/debug/canic`;
3. isolates `HOME`, `CARGO_HOME`, `CARGO_TARGET_DIR`, and `TMPDIR` under the
   proof root;
4. runs the maintained v1 readiness smoke through that installed binary:

   ```text
   scripts/ci/v1-readiness-smoke.sh
   ```

The smoke covers:

```text
canic fleet create demo --yes
canic scaffold canister demo store
canic fleet role inspect demo store
canic fleet role attach demo store --subnet prime
canic deploy inspect catalog list --json --output catalog.json
canic evidence gate --policy policy.toml --envelope envelope.json --json --output gate.json
```

## What It Does Not Do

This proof does not:

- build Wasm artifacts;
- install or upgrade canisters;
- query or mutate a live replica;
- mutate repository `.canic` or `.icp` state;
- depend on mainnet;
- use repository `target/debug/canic`;
- add deployment groups, locks, signing, registry import, teardown, controller
  mutation, or active adoption/import.

Successful output ends with:

```text
installed canic CLI probe passed
```
