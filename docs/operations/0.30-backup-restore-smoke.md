# 0.30 Backup / Restore Smoke Checklist

> Historical note: This document predates the 0.33 ICP CLI hard cut and may
> show DFX-era commands. Use the current README and 0.33 changelog for live
> operator commands.

Use this checklist before closing a 0.30 backup/restore release. It exercises
the intended operator path without requiring manual JSON inspection between
steps.

## Inputs

Prepare these values first:

* `<source-canister>`: the project or fleet root canister to snapshot
* `<root-canister>`: the Canic root canister used for topology discovery
* `<run-id>`: a unique local backup run directory name

## Capture

```bash
canic snapshot download \
  --canister <source-canister> \
  --root <root-canister> \
  --recursive \
  --out backups/<run-id> \
  --stop-before-snapshot \
  --resume-after-snapshot \
  --network local
```

Expected result:

* `backups/<run-id>/manifest.json` exists
* `backups/<run-id>/download-journal.json` exists
* all downloaded artifacts are durable
* discovery and pre-snapshot topology hashes match

## Canonical Smoke

```bash
canic backup smoke \
  --dir backups/<run-id> \
  --out-dir smoke/<run-id> \
  --require-design \
  --require-restore-ready
```

Expected result:

* `smoke/<run-id>/preflight/preflight-summary.json` exists
* `smoke/<run-id>/restore-apply-dry-run.json` exists
* `smoke/<run-id>/restore-apply-journal.json` exists
* `smoke/<run-id>/restore-run-dry-run.json` exists
* `smoke/<run-id>/smoke-summary.json` reports `restore_ready=true`

Use the remaining sections when you need to inspect or rerun one phase
individually.

## Validate

```bash
canic manifest validate \
  --manifest backups/<run-id>/manifest.json \
  --out smoke/<run-id>/manifest-validation.json \
  --require-design
```

Expected result:

* `design_conformance.design_v1_ready` is `true`

## Preflight

```bash
canic backup preflight \
  --dir backups/<run-id> \
  --out-dir smoke/<run-id>/preflight \
  --require-design \
  --require-restore-ready
```

Expected result:

* `preflight-summary.json` has `manifest_design_v1_ready=true`
* `preflight-summary.json` has `restore_ready=true`
* `restore-plan.json` contains `design_conformance.design_v1_ready=true`

## Plan

```bash
canic restore plan \
  --backup-dir backups/<run-id> \
  --out smoke/<run-id>/restore-plan.json \
  --require-verified \
  --require-design \
  --require-restore-ready
```

Expected result:

* restore plan generation succeeds
* `readiness_summary.ready` is `true`
* `design_conformance.design_v1_ready` is `true`

## Dry Run

```bash
canic restore apply \
  --plan smoke/<run-id>/restore-plan.json \
  --backup-dir backups/<run-id> \
  --dry-run \
  --out smoke/<run-id>/restore-apply-dry-run.json \
  --journal-out smoke/<run-id>/restore-apply-journal.json
```

Expected result:

* every expected upload, load, reinstall, and verification operation is present
* blocked operations are explainable before execution
* operation-kind counts match the plan

## Runner Preview

```bash
canic restore run \
  --journal smoke/<run-id>/restore-apply-journal.json \
  --dry-run \
  --out smoke/<run-id>/restore-run-dry-run.json \
  --require-run-mode dry-run \
  --require-next-action rerun
```

Expected result:

* runner preview succeeds
* `batch_summary.complete` matches the journal state
* no operation is executed during preview

## Optional Guarded Step

Use a harmless command adapter such as `/bin/true` only with a fixture journal or
other non-production restore target:

```bash
canic restore run \
  --journal smoke/<run-id>/restore-apply-journal.json \
  --execute \
  --dfx /bin/true \
  --max-steps 1 \
  --updated-at 2026-05-05T00:00:00Z \
  --out smoke/<run-id>/restore-run-step.json \
  --require-run-mode execute \
  --require-stopped-reason max-steps-reached \
  --require-next-action rerun \
  --require-executed-count 1
```

Expected result:

* exactly one operation is marked completed
* the journal remains resumable
* the runner writes operation receipts and batch counters before returning
