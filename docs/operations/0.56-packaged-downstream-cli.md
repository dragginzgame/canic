# 0.56 Packaged Downstream CLI Proof

This proof verifies that the packaged `canic-cli` crate and its packaged
support crates can run current v1 read-only commands against a clean downstream
project.

Run it from the repository root:

```bash
make test-packaged-downstream-cli
```

or directly:

```bash
scripts/ci/verify-packaged-downstream-cli.sh
```

## What It Proves

The script:

1. creates package archives for the CLI and required support crates;
2. unpacks those archives into a temporary package root;
3. creates a temporary tool workspace that patches crates to the unpacked
   package root, not repository crate paths;
4. creates a downstream project with current fleet-scoped role declarations:

   ```text
   fleets/downstream/canic.toml
   [roles.root]
   [roles.app]
   ```

5. runs the packaged CLI crate against the downstream project:

   ```text
   canic fleet list
   canic fleet role list downstream
   canic fleet role inspect downstream app
   canic deploy inspect catalog list --format json --output <path>
   ```

6. asserts that the downstream app role is attached and deploy-artifact
   eligible, and that catalog output remains empty with a no-state warning
   when no deployment-target state exists.

## Guardrails

The proof fails if the packaged tool root contains repository crate paths such
as:

```text
<repo>/crates
target/debug/canic
```

It isolates `HOME`, `CARGO_TARGET_DIR`, and `TMPDIR` under the proof root for
the packaged CLI execution. It intentionally reuses the caller's `CARGO_HOME`
and `RUSTUP_HOME` because the packaged proof runs Cargo offline and must use
the existing local Cargo registry/cache and configured Rust toolchain.

## What It Does Not Do

This proof does not:

- install or upgrade canisters;
- build Wasm artifacts;
- query or mutate a live replica;
- mutate repository `.canic` or `.icp` state;
- use role-only package metadata;
- use topology-only package-backed roles;
- add deployment groups, locks, signing, registry import, teardown, controller
  mutation, or active adoption/import.

Successful output ends with:

```text
packaged downstream CLI probe passed
```
