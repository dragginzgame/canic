# 0.56 Packaged Wasm Store Bootstrap Proof

This proof verifies the special packaged downstream `wasm_store` bootstrap
paths. It is a release probe for Canic's internal bootstrap build path, not
ordinary downstream canister dependency guidance.

Run it from the repository root:

```bash
make test-packaged-downstream-wasm-store
```

or directly:

```bash
scripts/ci/verify-packaged-downstream-wasm-store.sh
```

## What It Proves

The script:

1. creates package archives for `canic-host`, `canic`, and required support
   crates;
2. unpacks those archives into temporary package roots;
3. creates temporary tool workspaces patched only to unpacked package contents;
4. creates downstream projects that depend on packaged `canic`;
5. runs the host `build_artifact` example for the special `wasm_store` role
   under `CANIC_WASM_PROFILE=fast`;
6. verifies that the generated-wrapper fallback builds when
   `canic-wasm-store` is absent:

   ```text
   .icp/local/generated/canic-wasm-store/Cargo.toml
   .icp/local/canisters/wasm_store/wasm_store.wasm
   .icp/local/canisters/wasm_store/wasm_store.wasm.gz
   .icp/local/canisters/wasm_store/wasm_store.did
   ```

7. verifies that the generated wrapper uses `wasm-store-canister`, defines the
   Canic profiles, and patches packaged Canic sibling crates instead of
   repository crate paths;
8. verifies that the canonical packaged `canic-wasm-store` sibling builds
   under `fast` when present, without generating a wrapper.

## Guardrails

The proof fails if the packaged tool roots or generated wrapper use repository
crate paths such as:

```text
<repo>/crates
target/debug/canic
```

It isolates `HOME`, `CARGO_TARGET_DIR`, and `TMPDIR` under the proof root for
the packaged build execution. It intentionally reuses the caller's
`CARGO_HOME` and `RUSTUP_HOME` because the packaged proof runs Cargo offline
after package resolution and must use the existing local Cargo cache and
configured Rust toolchain.

## Special Scope

The first scenario intentionally excludes `canic-wasm-store` from the temporary
package root so the generated wrapper path is exercised. The second scenario
includes the canonical packaged `canic-wasm-store` sibling so Canic's preferred
source-resolution path is covered too.

This does not imply that ordinary users should import `canic-wasm-store` as an
rlib, carry their own wrapper crate, or depend on Canic-managed canister crates
directly.

## What It Does Not Do

This proof does not:

- install or upgrade canisters;
- query or mutate a live replica;
- mutate repository `.canic` or `.icp` state;
- use role-only package metadata;
- use topology-only package-backed roles;
- add deployment groups, locks, signing, registry import, teardown, controller
  mutation, or active adoption/import.

Successful output ends with:

```text
packaged downstream wasm_store probe passed
```
