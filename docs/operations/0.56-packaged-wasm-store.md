# 0.56 Packaged Wasm Store Bootstrap Proof

This proof verifies the special packaged downstream `wasm_store` bootstrap
wrapper path. It is a release probe for Canic's internal bootstrap build path,
not ordinary downstream canister dependency guidance.

Run it from the repository root:

```bash
make test-packaged-downstream-wasm-store
```

or directly:

```bash
scripts/ci/verify-packaged-downstream-wasm-store.sh
```

## What It Proves

The script:

1. creates package archives for `canic-host`, `canic`, and required support
   crates;
2. unpacks those archives into a temporary package root;
3. creates a temporary tool workspace patched only to unpacked package
   contents;
4. creates a downstream project that depends on packaged `canic`;
5. runs the host `build_artifact` example for the special `wasm_store` role;
6. verifies that the generated wrapper builds:

   ```text
   .icp/local/generated/canic-wasm-store/Cargo.toml
   .icp/local/canisters/wasm_store/wasm_store.wasm
   .icp/local/canisters/wasm_store/wasm_store.wasm.gz
   .icp/local/canisters/wasm_store/wasm_store.did
   ```

7. verifies that the generated wrapper patches packaged Canic sibling crates
   instead of repository crate paths.

## Guardrails

The proof fails if the packaged tool root or generated wrapper uses repository
crate paths such as:

```text
<repo>/crates
target/debug/canic
```

It isolates `HOME`, `CARGO_TARGET_DIR`, and `TMPDIR` under the proof root for
the packaged build execution. It intentionally reuses the caller's
`CARGO_HOME` and `RUSTUP_HOME` because the packaged proof runs Cargo offline
after package resolution and must use the existing local Cargo cache and
configured Rust toolchain.

## Special Scope

This proof intentionally excludes `canic-wasm-store` from the temporary package
root so the generated wrapper path is exercised. That is the point of the
probe: downstreams should not need to carry their own wasm-store wrapper crate.

It does not imply that ordinary users should import `canic-wasm-store` as an
rlib or depend on Canic-managed canister crates directly.

## What It Does Not Do

This proof does not:

- install or upgrade canisters;
- query or mutate a live replica;
- mutate repository `.canic` or `.icp` state;
- use role-only package metadata;
- use topology-only package-backed roles;
- add deployment groups, locks, signing, registry import, teardown, controller
  mutation, or active adoption/import.

Successful output ends with:

```text
packaged downstream wasm_store probe passed
```
