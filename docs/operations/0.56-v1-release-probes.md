# 0.56 V1 Release Probe Inventory

0.56 keeps only release probes with a clear packaged or installed v1 question.
This is an inventory, not a new product surface.

## Retained Probes

| --- | --- | --- | --- | --- | --- |

## Guardrails

Retained probes must not use old v1 command shapes:

```text
canic build <role>
canic build --dev
canic scaffold canister <fleet> <role> --attach
canic fleet role attach <fleet> <role> --pool <pool>
canic fleet role attach <fleet> <role> --max-shards <n>
```

The installed CLI proof must exercise the installed binary by default, not
`target/debug/canic`.

It also isolates `HOME`, `CARGO_HOME`, `CARGO_TARGET_DIR`, and `TMPDIR` under
the proof root before running the readiness smoke.

The packaged downstream proofs may use the repository to create package
archives, but downstream execution must resolve through the temporary package
root or installed binary path, not repository-local crate paths.

The packaged downstream CLI proof also checks current fleet-scoped role
declarations, role inspection, and passive catalog output from a downstream
project root.

The `wasm_store` probe is intentionally special. It protects the generated
bootstrap wrapper path and is not ordinary guidance for downstream canister
crate dependencies.
