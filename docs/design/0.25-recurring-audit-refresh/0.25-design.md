# 0.25 Recurring Audit Refresh

## Intent

`0.25` is a narrow follow-through line.

It is not a new architecture line, a new topology line, or a broad cleanup lane.
Its purpose is to refresh the current recurring audits against the codebase as it stands after `0.24`, record one retained summary per audit, and land only the fixes that those fresh audits directly justify.

## Scope

Primary recurring audit passes:

- layering
- capability surface
- wasm footprint
- instruction footprint

Secondary recurring audit passes may be included only if they are already part of the active audit cadence and produce actionable results without widening the line.

Current completed retained passes:

- `layer-violations`
- `capability-surface`
- `wasm-footprint`
- `instruction-footprint`
- `bootstrap-lifecycle-symmetry`
- `change-friction`
- `complexity-accretion`
- `subject-caller-binding`
- `canonical-auth-boundary`
- `capability-scope-enforcement`
- `expiry-replay-single-use`
- `token-trust-chain`
- `audience-target-binding`
- `auth-abstraction-equivalence`

## Rules

1. Run each recurring audit once for the line unless a fix requires a confirming rerun.
2. Keep one retained summary per audit instead of committing large generated artifact trees by default.
3. Prefer audit-backed fixes over speculative cleanup.
4. Do not widen `0.25` into unrelated runtime refactors or new feature work.
5. If an audit surfaces a larger redesign issue, record it and defer it rather than forcing it into this line.

## Desired Outcome

By the end of `0.25`, we should have:

- one fresh retained summary for each recurring audit
- a small number of targeted fixes tied directly to those audits
- a clearer current picture of runtime layering, exported surface, wasm size, and sampled instruction hotspots

Current readout:

- the refreshed auth and capability invariants still hold
- no broad boundary break was found in the current `0.24` to `0.25` state
- the strongest remaining pressure is complexity concentration in the auth/runtime seams, especially:
  - `crates/canic-core/src/access/expr.rs`
  - `crates/canic-core/src/access/auth.rs`
  - `crates/canic-core/src/storage/stable/auth.rs`
  - `crates/canic-core/src/ops/runtime/metrics/auth.rs`

## Non-Goals

- broad feature additions
- new public API themes
- another major test-harness redesign
- speculative cleanup not connected to a fresh audit result
