## Findings

### CANIC-082-CONTROL-PLANE-TEMPLATE-GC-001

Severity: P3
Category: ops_ownership
Owner: `canic-control-plane::support`
Current location: `crates/canic-control-plane/src/support.rs`
Intended location: `crates/canic-control-plane/src/ops/storage/template/mod.rs`

Evidence:
The crate-private support module re-exported `WasmStoreGcExecutionStats`, and
the template API imported the type through support even though the type is
defined and populated by template storage ops.

Risk:
The support module looked like the owner of a storage-ops result type, blurring
the boundary between helper façade code and the storage operation that produces
the stats.

Decision:
Fixed with a no-behavior-change import cleanup. The template API imports
`WasmStoreGcExecutionStats` from template storage ops directly, while support
continues to return the same type from its helper function.

Regression test:
Control-plane all-target compile coverage verifies the template API and support
function signatures after the import cleanup.

## Fixes

- Removed the crate-private `support::WasmStoreGcExecutionStats` re-export.
- Updated the template API to import `WasmStoreGcExecutionStats` from template
  storage ops.
- Kept support helper behavior and return type unchanged.

## Deferred

- Wasm-store GC behavior remains unchanged.
- Template storage stats shape remains unchanged.
