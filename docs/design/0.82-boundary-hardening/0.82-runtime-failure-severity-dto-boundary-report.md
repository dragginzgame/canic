## Findings

### CANIC-082-RUNTIME-DTO-001

Severity: P2
Category: dto_boundary
Owner: `canic-core::dto::runtime`
Current location: `crates/canic-core/src/dto/runtime.rs`
Intended location: `crates/canic-core/src/domain/runtime.rs`

Evidence:
`ops::runtime::recent_failure`, `ops::runtime::bootstrap`, and
`api::runtime` used `dto::runtime::FailureSeverity` when recording and
projecting heap-only recent-failure metadata.

Risk:
The DTO module was acting as the semantic owner for runtime failure severity
metadata used below the endpoint boundary.

Decision:
Fixed with a no-behavior-change ownership move. The enum now lives in
`domain::runtime`, while `dto::runtime` publicly re-exports it so existing API
paths and serialized shapes remain stable.

Regression test:
`dto::runtime::tests::runtime_enums_roundtrip_candid_with_canonical_variant_labels`
continues to prove the public DTO path round-trips `FailureSeverity` through
Candid.

## Fixes

- Added `domain::runtime` as the owner for `FailureSeverity`.
- Re-exported the same value from `dto::runtime`.
- Updated runtime API and runtime ops code to import the domain owner.

## Deferred

- Other runtime status/check/visibility enums remain DTO-owned.
- Runtime status DTO structs remain unchanged.
