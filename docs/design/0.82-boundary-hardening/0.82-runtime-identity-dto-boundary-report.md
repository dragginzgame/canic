## Findings

### CANIC-082-RUNTIME-ID-001

Severity: P2
Category: dto_boundary
Owner: `canic-core::dto::subnet`
Current location: `crates/canic-core/src/dto/subnet.rs`
Intended location: `crates/canic-core/src/domain/subnet.rs`

Evidence:
`workflow::runtime::root` consumed `dto::subnet::SubnetIdentity` directly even
though the type represents runtime identity rather than a passive transport-only
response shape.

Risk:
The DTO module was acting as the semantic owner for a runtime workflow input,
making it less clear where lifecycle/runtime identity decisions belong.

Decision:
Fixed with a no-behavior-change ownership move. Runtime identity values now
live in `domain::subnet`, while `dto::subnet` publicly re-exports them so the
existing macro/init path and serialized Candid shape remain stable.

Regression test:
`dto::subnet::tests::reexported_subnet_identity_roundtrips_through_candid`
proves the public DTO path still round-trips the re-owned value through Candid.

## Fixes

- Added `domain::subnet` as the owner for `SubnetIdentity` and
  `SubnetContextParams`.
- Re-exported the same values from `dto::subnet`.
- Updated runtime root workflow to import `SubnetIdentity` from the domain
  owner.
- Added a Candid roundtrip regression test through the DTO path.

## Deferred

- Lifecycle API and macro-generated init signatures intentionally remain on the
  DTO path because they are boundary surfaces.
