## Slice Contract

Boundary:
`canic info metrics` and `canic info cycles` response parsing for metrics,
cycle tracker, and cycle top-up report pages.

Current owner:
`canic-cli::metrics::parse`, `canic-cli::cycles::parse`, and
`canic-cli::cycles::transport`, with fallback parsing from text
`response_candid` payloads.

Intended owner:
The CLI report parsers consume structured JSON values returned by the ICP
query wrapper. Candid text is not a fallback report surface for these operator
reports.

Files/modules in scope:

- `crates/canic-cli/src/metrics/parse.rs`
- `crates/canic-cli/src/metrics/tests.rs`
- `crates/canic-cli/src/cycles/parse.rs`
- `crates/canic-cli/src/cycles/transport.rs`
- `crates/canic-cli/src/cycles/tests.rs`

Files/modules explicitly out of scope:

- Endpoint method names and endpoint DTO shapes.
- Candid method signatures for metrics, cycle tracker, or cycle top-ups.
- Canister-side metrics/cycles behavior.
- Host cycle-balance, metadata, and readiness parsers.
- Inspect runtime-status parsing, which was hard-cut separately.
- CLI command names, options, help text, or output rendering.

Public surfaces affected:
`canic info metrics` and `canic info cycles` no longer parse text-only
`response_candid` fallback output for metrics, tracker, or top-up report pages.

Serialized surfaces affected:
No endpoint, Candid, JSON, deployment-truth, evidence/report, or stable-state
schema changes. The CLI accepted input wrapper shape is intentionally narrowed
to structured JSON values for these report pages.

Commands/endpoints affected:
No command or endpoint name changes.

Expected behavior change:
ICP query output for these report pages must include structured JSON entries.
Malformed structured entries are rejected even when a `response_candid` text
payload is present.

Expected non-changes:

- CLI command surfaces remain unchanged.
- Successful metrics/cycles report output remains unchanged.
- Canister endpoint Candid signatures remain unchanged.
- Host-side non-report fallback parsers remain out of scope.
- No compatibility fallback, alias, or shim is retained for these report pages.

## Findings

### CANIC-082-CLI-RESPONSE-CANDID-001

Severity: P2
Category: cli_surface
Owner: `canic-cli::metrics` and `canic-cli::cycles`
Current location: `crates/canic-cli/src/metrics/parse.rs`,
`crates/canic-cli/src/cycles/parse.rs`,
`crates/canic-cli/src/cycles/transport.rs`
Intended location: structured JSON parsing in the CLI report parsers

Evidence:
Metrics and cycles report parsers accepted `response_candid` text payloads when
structured JSON fields were missing or malformed, and cycles transport retried
the raw command output with Candid text parsers after structured parsing failed.

Risk:
The CLI carried a second untyped report input surface after the recent typed
report hard cuts. That fallback made malformed structured data recoverable via
legacy text parsing and kept compatibility behavior alive before 1.0.

Decision:
Fixed with a hard cut. Metrics and cycles report pages now require structured
JSON parsing, and `response_candid` text is ignored/rejected for these report
parsers.

Regression test:
Parser tests now assert that text-only `response_candid` payloads are rejected
and that malformed structured entries remain rejected even when
`response_candid` is present.

## Fixes

- Removed metrics Candid text parsing helpers and `response_candid` fallback.
- Removed cycle tracker and cycle top-up Candid text parsing helpers.
- Removed transport-level fallback parsing of raw Candid text output.
- Updated metrics and cycles tests to pin rejection of text-only
  `response_candid` report payloads.

## Deferred

- Host-side non-report parsers that still inspect `response_candid` remain out
  of scope for this CLI metrics/cycles hard cut.
- No old-fallback compatibility tests were added; this is an intentional hard
  cut, not a compatibility surface.
