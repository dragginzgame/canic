## Findings

### CANIC-082-TIMER-MODE-OPS-001

Severity: P2
Category: ops_ownership
Owner: `canic-core::ops::runtime::metrics::timer`
Current location: `crates/canic-core/src/ops/runtime/metrics/timer.rs`
Intended location: `crates/canic-core/src/domain/runtime.rs`

Evidence:
`TimerMode` was defined in the runtime metrics module while timer scheduling
ops, runtime metrics projection, and runtime status tests consumed it as a
shared runtime scheduling-mode value.

Risk:
Timer scheduling-mode semantics could drift between timer ops and runtime
metric projection if the value remains owned by the metrics recorder.

Decision:
Fixed with a no-behavior-change ownership move. `TimerMode` now lives in
`domain::runtime`, while `ops::runtime::metrics::timer` publicly re-exports it
so existing imports remain stable.

Regression test:
Runtime metric tests continue to prove timer mode labels and counter keys are
unchanged.

## Fixes

- Added `TimerMode` to `domain::runtime`.
- Re-exported `TimerMode` from the existing timer metrics path.
- Updated timer scheduling ops and metrics projection to import the domain
  owner directly.

## Deferred

- Timer metric storage and public metric projection labels remain unchanged.
- Timer scheduling behavior remains unchanged.
