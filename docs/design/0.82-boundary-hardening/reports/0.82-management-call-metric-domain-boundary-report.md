## Findings

### CANIC-082-MANAGEMENT-CALL-METRIC-OPS-001

Severity: P2
Category: ops_ownership
Owner: `canic-core::ops::runtime::metrics::management_call`
Current location: `crates/canic-core/src/ops/runtime/metrics/management_call.rs`
Intended location: `crates/canic-core/src/domain/metrics.rs`

Evidence:
Management-call operation, outcome, and reason values were defined in the
management-call metrics recorder while IC management ops consumed them as
shared metric classification values.

Risk:
Management-call metric semantics could drift between operation owners and
runtime metric recording if the values remain owned by the metrics recorder.

Decision:
Fixed with a no-behavior-change ownership move. The management-call metric
dimension values now live in `domain::metrics`, while
`ops::runtime::metrics::management_call` publicly re-exports them so existing
imports remain stable.

Regression test:
Runtime metric tests continue to prove management-call counter keys and reset
behavior are unchanged.

## Fixes

- Added management-call metric dimension values to `domain::metrics`.
- Re-exported the same values from the existing management-call metrics path.
- Updated management ops to import the domain owner directly.

## Deferred

- Management-call metric storage remains in runtime metrics ownership.
- Management canister operation behavior remains unchanged.
