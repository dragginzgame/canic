## Findings

### CANIC-082-BLOB-STORAGE-INTEROP-WORDING-001

Severity: P3
Category: diagnostic_owner
Owner: blob-storage protocol inventory gate
Current location: `docs/contracts/BLOB_STORAGE_INVENTORY.md`,
`scripts/ci/check-blob-storage-inventory-gate.sh`,
`crates/canic/tests/protocol_inventory_gate.rs`
Intended location: active inventory evidence uses interoperability wording

Evidence:
The active blob-storage inventory contract and gate used "Compatibility Notes"
and "Toko compatibility" for the current Toko source evidence and
blob-root mapping.

Risk:
The wording suggested a backwards-compatibility or shim policy even though the
active contract requires current source-backed interoperability evidence.

Decision:
Renamed the active Toko evidence wording to interoperability while preserving
the same section marker depth, required fields, and gate checks.

Regression test:
`cargo test --locked -p canic --test protocol_inventory_gate`
