## Slice Contract

Boundary:
Lifecycle `init` and `post_upgrade` before-bootstrap adapters restore runtime
state synchronously and must not inline async work or schedule timers. Async
bootstrap work is allowed only in explicit schedule helpers that install
zero-delay lifecycle timers.

Current owner:
The 0.82 design and `AGENTS.md` define the lifecycle rule, and PocketIC tests
cover observable lifecycle behavior, but there was no focused source-boundary
guard protecting the synchronous adapter shape.

Intended owner:
`canic-core` integration boundary tests guard lifecycle adapter source shape
across `canic-core`, `canic-control-plane`, and the public `canic` macro
facade.

Files/modules in scope:

- `crates/canic-core/tests/lifecycle_boundary_guard.rs`
- `crates/canic-core/src/lifecycle/init/**`
- `crates/canic-core/src/lifecycle/upgrade/**`
- `crates/canic-control-plane/src/api/lifecycle.rs`

Files/modules explicitly out of scope:

- Lifecycle runtime behavior changes.
- Timer implementation changes.
- User hook semantics.
- PocketIC lifecycle fixture behavior.
- Public endpoint, CLI, Candid, JSON, deployment-truth, or evidence surfaces.

Public surfaces affected:
None.

Serialized surfaces affected:
None.

Commands/endpoints affected:
None.

Expected behavior change:
None.

Expected non-changes:

- `canic::start!` remains the public lifecycle macro facade.
- Before-bootstrap adapters still perform the same runtime/config/env/memory
  restore work.
- Root and non-root async bootstrap still runs from zero-delay lifecycle
  timers.
- User async hooks remain scheduled by lifecycle timers, not awaited inline.

Validation:

- `cargo fmt --all -- --check` passed.
- `cargo test --locked -p canic-core --test lifecycle_boundary_guard` passed.

Rollback plan:
Remove `crates/canic-core/tests/lifecycle_boundary_guard.rs`. No runtime code
rollback is required because this slice does not change runtime code.

Follow-ups:

- Keep PocketIC lifecycle coverage as the authority for behavior-level install
  and upgrade guarantees.
- If lifecycle behavior changes later, add or update PocketIC coverage rather
  than relying on source-shape checks alone.

## Findings

### CANIC-082-LIFECYCLE-GUARD-001

Severity: P2
Category: lifecycle
Owner: lifecycle before-bootstrap adapters
Current location: `crates/canic-core/src/lifecycle/**` and
`crates/canic-control-plane/src/api/lifecycle.rs`
Intended location: `crates/canic-core/tests/lifecycle_boundary_guard.rs`

Evidence:
The 0.82 design requires lifecycle adapters to restore synchronously and
schedule async work only after invariants are restored. Existing code matched
the rule, but it was not protected by a focused source-boundary guard.

Risk:
Future lifecycle cleanup could accidentally add `.await`, timer scheduling, or
bootstrap workflow calls to before-bootstrap adapters, weakening the lifecycle
ordering boundary before PocketIC tests catch a broader symptom.

Decision:
Fixed by adding a focused boundary guard for before-bootstrap lifecycle adapter
bodies. The guard rejects inline async fragments, timer scheduling, and direct
bootstrap workflow scheduling inside those adapters.

Regression test:
`cargo test --locked -p canic-core --test lifecycle_boundary_guard`

### CANIC-082-LIFECYCLE-GUARD-002

Severity: P2
Category: lifecycle
Owner: lifecycle async bootstrap schedule helpers
Current location: `crates/canic-core/src/lifecycle/**` and
`crates/canic-control-plane/src/api/lifecycle.rs`
Intended location: `crates/canic-core/tests/lifecycle_boundary_guard.rs`

Evidence:
Async bootstrap is intentionally present in root and non-root schedule helpers.
Those helpers must remain zero-delay lifecycle-timer scheduling points rather
than inline lifecycle hook work.

Risk:
Future cleanup could move async bootstrap out of explicit schedule helpers or
drop the zero-delay timer shape, making lifecycle ordering less auditable.

Decision:
Fixed by adding a focused guard that checks the four root/non-root schedule
helpers continue to use zero-delay lifecycle scheduling for async bootstrap.

Regression test:
`cargo test --locked -p canic-core --test lifecycle_boundary_guard`

## Fixes

- Added a lifecycle source-boundary guard for synchronous before-bootstrap

- Added schedule-helper checks for root and non-root async bootstrap timer
