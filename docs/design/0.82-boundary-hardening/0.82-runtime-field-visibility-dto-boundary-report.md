## Findings

### CANIC-082-RUNTIME-DTO-002

Severity: P2
Category: dto_boundary
Owner: `canic-core::dto::runtime`
Current location: `crates/canic-core/src/dto/runtime.rs`
Intended location: `crates/canic-core/src/domain/runtime.rs`

Evidence:
`api::runtime` owned runtime field classification rows but imported
`dto::runtime::RuntimeFieldVisibility` to assign visibility semantics such as
`PublicSafe`, `OperatorOnly`, `ControllerOnly`, and `FeatureGated`.

Risk:
The DTO module was acting as the semantic owner for field-level runtime
visibility metadata used by runtime internals before the endpoint boundary.

Decision:
Fixed with a no-behavior-change ownership move. The enum now lives in
`domain::runtime`, while `dto::runtime` publicly re-exports it so existing API
paths and serialized shapes remain stable.

Regression test:
`dto::runtime::tests::runtime_enums_roundtrip_candid_with_runtime_variant_labels`
continues to prove the public DTO path round-trips `RuntimeFieldVisibility`
through Candid.

## Fixes

- Added `RuntimeFieldVisibility` to `domain::runtime`.
- Re-exported the same value from `dto::runtime`.
- Updated runtime API code to import the domain owner.

## Deferred

- Other runtime status/check enums remain DTO-owned.
- Runtime status DTO structs remain unchanged.
