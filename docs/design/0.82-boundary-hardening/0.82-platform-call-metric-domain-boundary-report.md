## Findings

### CANIC-082-PLATFORM-CALL-METRIC-OPS-001

Severity: P2
Category: ops_ownership
Owner: `canic-core::ops::runtime::metrics::platform_call`
Current location: `crates/canic-core/src/ops/runtime/metrics/platform_call.rs`
Intended location: `crates/canic-core/src/domain/metrics.rs`

Evidence:
Platform-call surface, mode, outcome, and reason values were defined in the
platform-call metrics recorder while IC call, HTTP, ledger, and management ops
consumed them as shared metric classification values.

Risk:
Platform-call metric semantics could drift between operation owners and public
metric projection if the values remain owned by the metrics recorder.

Decision:
Fixed with a no-behavior-change ownership move. The platform-call metric
dimension values now live in `domain::metrics`, while
`ops::runtime::metrics::platform_call` publicly re-exports them so existing
imports remain stable.

Regression test:
Runtime metric tests continue to prove platform-call labels and counter keys
are unchanged.

## Fixes

- Added platform-call metric dimension values to `domain::metrics`.
- Re-exported the same values from the existing platform-call metrics path.
- Updated IC call, HTTP, ledger, and management ops to import the domain owner
  directly.

## Deferred

- Platform-call metric storage and public metric labels remain unchanged.
- IC platform call behavior remains unchanged.
