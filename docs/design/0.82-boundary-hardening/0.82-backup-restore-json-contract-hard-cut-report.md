## Slice Contract

Boundary:
Backup and restore durable JSON contracts should reject unknown fields instead
of ignoring stale or misspelled data. Backup execution journals own current
per-operation backup receipts; unused plan-facing receipt types should not
remain in the plan model.

Current owner:
Backup manifest, backup plan, backup download journal, backup execution
journal, restore plan, restore apply dry-run, restore apply journal, restore
receipt, command preview, and related report structs accepted unknown JSON
fields through serde defaults. Unused `BackupReceipt` and
`BackupReceiptStatus` types remained in backup plan types after execution
operation receipts became the maintained durable receipt path.

Intended owner:
Current backup/restore JSON structs are strict schemas. Execution journals own
per-operation backup receipts.

## Findings

### CANIC-082-BACKUP-RESTORE-JSON-HARDCUT-001

Severity: P2
Category: legacy_surface
Owner: `canic-backup::manifest`
Current location: `crates/canic-backup/src/manifest/types.rs`
Intended location: strict current backup manifest schema

Evidence:
Backup manifest data structs used serde's default unknown-field behavior, and
active tests asserted that an old removed source-snapshot field was ignored.

Risk:
Unknown backup manifest fields could hide stale, misspelled, or retired data in
a durable artifact consumed by restore planning and verification.

Decision:
Added `deny_unknown_fields` to backup manifest structs and replaced the stale
ignored-field test with current strict-manifest rejection coverage.

Regression test:
`cargo test --locked -p canic-backup manifest --lib`

### CANIC-082-BACKUP-RESTORE-JSON-HARDCUT-002

Severity: P2
Category: legacy_surface
Owner: `canic-backup::plan`
Current location: `crates/canic-backup/src/plan/types.rs`
Intended location: strict current backup plan and preflight schemas

Evidence:
Backup plan and preflight data structs used serde's default unknown-field
behavior, and unused plan-facing `BackupReceipt` and `BackupReceiptStatus`
types remained after execution operation receipts became the maintained durable
receipt path.

Risk:
Unknown backup plan/preflight fields could hide stale or misspelled data before
mutation, and unused receipt types made receipt ownership look split between
plan and execution layers.

Decision:
Added `deny_unknown_fields` to backup plan and preflight structs, added
strict-plan rejection coverage, and removed the unused plan-facing receipt
types.

Regression test:
`cargo test --locked -p canic-backup plan --lib`

### CANIC-082-BACKUP-RESTORE-JSON-HARDCUT-003

Severity: P2
Category: legacy_surface
Owner: `canic-backup::journal`, `canic-backup::execution`, and `canic-backup::restore`
Current location: backup/restore journal and report types
Intended location: strict current backup/restore JSON contracts

Evidence:
Download journal, backup execution journal, restore plan, restore apply dry-run,
restore apply journal, command preview, receipt, and report structs used
serde's default unknown-field behavior.

Risk:
Unknown fields in persisted backup/restore JSON could hide stale, misspelled,
or retired data in artifacts used for resume, integrity checks, restore
planning, or restore execution.

Decision:
Added `deny_unknown_fields` to the current backup/restore journal, restore
plan, restore dry-run, restore apply journal, receipt, command preview, and
report structs that deserialize JSON.

Regression test:
`cargo test --locked -p canic-backup journal --lib` and
`cargo test --locked -p canic-backup restore --lib`

## Fixes

- Backup manifests, backup plans, and backup preflight contracts now reject
  unknown fields.
- Backup download journals, backup execution journals, restore plans, restore
  apply dry-runs, restore apply journals, receipts, command previews, and
  related reports now reject unknown fields.
- Removed unused plan-facing backup receipt types after execution journals
  became the maintained receipt owner.

## Deferred

- Backup download-journal cleanup remains a separate slice because the restore
  and artifact execution paths still rely on current journal behavior.
- Restore apply journal receipt field names remain unchanged; this slice only
  tightens unknown-field rejection.
