## Findings

### CANIC-082-APP-STATUS-DTO-001

Severity: P2
Category: dto_boundary
Owner: `canic-core::dto::state`
Current location: `crates/canic-core/src/dto/state.rs`
Intended location: `crates/canic-core/src/domain/state.rs`

Evidence:
`AppStatus` was defined in the DTO module while storage ops consumed it to map
command status values into the domain-owned `AppMode`.

Risk:
App command status semantics could drift between boundary DTO ownership and the
state domain owner that already owns the persisted app-mode semantics.

Decision:
Fixed with a no-behavior-change ownership move. `AppStatus` now lives in
`domain::state`, while `dto::state` publicly re-exports it so existing API paths
and serialized shapes remain stable.

Regression test:
`dto::state::tests::app_status_roundtrips_candid_through_dto_path` proves the
DTO re-export path preserves the Candid contract.

## Fixes

- Added `AppStatus` to `domain::state`.
- Re-exported `AppStatus` from the existing state DTO path.
- Updated app-state storage ops to import `AppStatus` from the domain owner.

## Deferred

- App command and response DTO shapes remain in `dto::state`.
- App-state mutation and cascade behavior remain unchanged.
