## Slice Contract

Boundary:
Deployment-truth local artifact observation for role artifacts and release-set
manifest digests.

Current owner:
`canic-host::deployment_truth::observe::artifacts`, with a cross-network
fallback from `.icp/<network>/canisters` to `.icp/local/canisters`.

Intended owner:
`canic-host::deployment_truth::observe::artifacts` observes only the artifact
root for the selected network. Missing selected-network artifacts are reported
as unresolved observations.

Files/modules in scope:

- `crates/canic-host/src/deployment_truth/observe/artifacts/mod.rs`
- `crates/canic-host/src/deployment_truth/tests/local_observation_plan/artifacts/mod.rs`

Files/modules explicitly out of scope:

- Deployment-truth schemas.
- Deployment-plan schemas.
- CLI command names, options, or output modes.
- Artifact build/stage behavior.
- Local replica status probing.
- Evidence envelopes, deployment truth storage, stable-state layout, or Candid
  DTOs.

Public surfaces affected:
Deployment-truth and deploy-plan observation reports for non-local networks no
longer use `.icp/local/canisters` when `.icp/<network>/canisters` is missing.

Serialized surfaces affected:
Report schemas are unchanged. Report contents change for this edge case:
`local_artifacts.network_fallback` is no longer emitted, `artifact_root` remains
unresolved, and no role artifacts are observed from the local fallback root.

Commands/endpoints affected:
No command or endpoint name changes.

Expected behavior change:

- Artifact observation for a selected non-local network requires
  `.icp/<network>/canisters`.
- Existing `.icp/local/canisters` artifacts are ignored when the selected
  network is not `local`.
- Missing selected-network artifacts are reported through the existing
  `local_artifacts.root` observation gap.

Expected non-changes:

- Local network artifact observation still reads `.icp/local/canisters`.
- Existing artifact digest and release-set manifest parsing are unchanged when
  the selected network artifact root exists.
- Deployment truth, deploy plan, evidence, Candid, and stable-state schemas are
  unchanged.
- No compatibility fallback, alias, or shim is retained.

Validation:

- `cargo fmt --all` passed.
- `cargo test --locked -p canic-host local_artifact_manifest --lib` passed.
- `cargo test --locked -p canic-host deployment_truth --lib` passed.
- `cargo clippy --locked -p canic-host --tests -- -D warnings` passed.
- `cargo fmt --all -- --check` passed.
- `rg -n "local_artifacts\.network_fallback|observing fallback" crates/canic-host/src/deployment_truth/observe/artifacts/mod.rs` returned no production matches.
- `git diff --check` passed.

Rollback plan:
Restore the `.icp/local/canisters` fallback in
`resolve_artifact_root_for_observation` and emit
`local_artifacts.network_fallback`. This is not recommended unless Canic
intentionally reintroduces cross-network artifact-root fallback semantics.

Follow-ups:

- If operators need explicit artifact root overrides for non-local networks,
  design a named configuration or CLI surface instead of restoring implicit
  local fallback.
- Keep local replica status HTTP fallback out of this slice; it is status
  probing, not artifact provenance.

## Findings

### CANIC-082-ARTIFACT-ROOT-NETWORK-FALLBACK-001

Severity: P2
Category: deployment_truth
Owner: `canic-host::deployment_truth::observe::artifacts`
Current location: `crates/canic-host/src/deployment_truth/observe/artifacts/mod.rs`
Intended location: selected-network artifact root observation

Evidence:
`resolve_artifact_root_for_observation` accepted `.icp/local/canisters` for a
non-local selected network when `.icp/<network>/canisters` was missing, while
recording `local_artifacts.network_fallback`.

Risk:
Deployment truth and deploy-plan observations could attribute local artifacts
to a non-local network scope. The warning preserved visibility, but it still
mixed artifact provenance across network boundaries.

Decision:
Fixed with a hard cut. Artifact observation now requires the selected network
artifact root and reports `local_artifacts.root` when it is missing.

Regression test:
The local artifact manifest test now verifies that selecting `ic` does not
observe `.icp/local/canisters` artifacts and reports the missing selected
network artifact root.

## Fixes

- Removed cross-network `.icp/local/canisters` fallback from artifact-root
  resolution.
- Removed `local_artifacts.network_fallback` emission from production artifact
  observation.
- Updated local artifact manifest coverage to expect an unresolved selected
  network artifact root instead of fallback observation.

## Deferred

- No artifact-root override surface was added. If needed, that should be a
  separate explicit design rather than an implicit fallback.
