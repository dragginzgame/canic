## Findings

### CANIC-082-INSTALL-ROOT-LEGACY-FLEET-STATE-001

Severity: P2
Category: legacy_surface
Owner: `canic-host::install_root::state`
Current location: `crates/canic-host/src/install_root/state.rs`
Intended location: current deployment-target install state only

Evidence:
`read_deployment_install_state` called `reject_legacy_fleet_state` when current
deployment-target state was missing. That helper inspected
`.canic/<network>/fleets/<deployment>.json` and returned a legacy-specific
error.

Risk:
The install-state reader retained active knowledge of a removed local state
layout. Even though it failed closed, it kept old path semantics alive in a
core read path and in operator-facing help text.

Decision:
Fixed with a hard cut. The install-state reader now reads only current
deployment-target state and returns `None` when it is absent.

Regression test:
The legacy rejection test was removed with the production legacy probe. Current
deployment-target state coverage remains under the renamed `targets` test
module.

## Fixes

- Removed the legacy fleet-state path helper and rejection path from
  install-state reads.
- Removed the legacy-specific fail-closed test.
- Renamed the remaining state-root verification test module from `legacy` to
  `targets`.
- Removed 0.46 legacy recovery language from deployment registration docs/help.

## Deferred

- No stale-state cleanup command was added. That should be a separate explicit
  maintenance design if needed.
