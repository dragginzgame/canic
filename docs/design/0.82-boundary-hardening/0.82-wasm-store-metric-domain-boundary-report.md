## Findings

### CANIC-082-WASM-STORE-METRIC-OPS-001

Severity: P2
Category: ops_ownership
Owner: `canic-core::ops::runtime::metrics::wasm_store`
Current location: `crates/canic-core/src/ops/runtime/metrics/wasm_store.rs`
Intended location: `crates/canic-core/src/domain/metrics.rs`

Evidence:
Wasm-store operation, source, outcome, and reason values were defined in the
wasm-store metrics recorder while install-source resolution and the lifecycle
facade consumed them as shared metric classification values.

Risk:
Wasm-store metric semantics could drift between install-source resolution,
facades, and public metric projection if the values remain owned by the metrics
recorder.

Decision:
Fixed with a no-behavior-change ownership move. The wasm-store metric
dimension values now live in `domain::metrics`, while
`ops::runtime::metrics::wasm_store` and `api::lifecycle::metrics` publicly
re-export them so existing imports remain stable.

Regression test:
Runtime and lifecycle facade metric tests continue to prove wasm-store labels
and metric rows are unchanged.

## Fixes

- Added wasm-store metric dimension values to `domain::metrics`.
- Re-exported the same values from the existing wasm-store metrics path and
  lifecycle metrics facade.
- Updated install-source resolution to import the domain owner directly.

## Deferred

- Wasm-store metric storage remains in runtime metrics ownership.
- Install-source resolution behavior remains unchanged.
