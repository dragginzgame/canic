## Slice Contract

Boundary:
The backup snapshot driver trait should expose the concrete boxed driver error
object it accepts. A public alias for `Box<dyn Error + Send + Sync + 'static>`
does not own a distinct backup protocol concept before 1.0.

Current owner:
`canic-backup::snapshot` defined `SnapshotDriverError` as a public alias for a
boxed dynamic error and re-exported it through the snapshot module.

Intended owner:
`SnapshotDriver` owns the driver boundary and uses
`Box<dyn StdError + Send + Sync + 'static>` directly for driver failures.
`SnapshotDownloadError::Driver` stores the same concrete boxed source error
directly.

Files/modules in scope:

- `crates/canic-backup/src/snapshot/types.rs`
- `crates/canic-backup/src/snapshot/tests/mod.rs`
- `crates/canic-cli/src/snapshot/download/mod.rs`
- `docs/status/current.md`

Files/modules explicitly out of scope:

- Backup manifest, journal, plan, receipt, and restore schemas.
- Snapshot command names, flags, and help output.
- Snapshot capture, restore, and topology behavior.
- ICP CLI command execution behavior.
- Deployment truth and evidence/report schemas.
- Stable-state layout.

Public surfaces affected:
Rust namespace only:

- `canic_backup::snapshot::SnapshotDriverError` is removed.

Serialized surfaces affected:
None.

Commands/endpoints affected:
None.

Expected behavior change:
None at runtime. External Rust code implementing `SnapshotDriver` must use
`Box<dyn StdError + Send + Sync + 'static>` directly for driver failures.

Expected non-changes:

- `SnapshotDriver` method behavior is unchanged.
- `SnapshotDownloadError::Driver` still preserves the boxed driver failure as
  the source error.
- Snapshot download CLI behavior is unchanged.
- Backup manifest, journal, plan, receipt, JSON, deployment truth,
  evidence/report, and stable-state surfaces are unchanged.

Validation:

- `cargo test --locked -p canic-backup snapshot --lib`
- `cargo test --locked -p canic-cli snapshot`
- `cargo clippy --locked -p canic-backup -p canic-cli --tests -- -D warnings`
- `cargo fmt --all -- --check`
- `cargo test --locked -p canic --test changelog_governance`
- `git diff --check`
- `rg -n "SnapshotDriverError" crates/canic-backup/src crates/canic-cli/src -S`

Rollback plan:
Restore the alias only if the maintainer intentionally chooses to keep that
Rust namespace before 1.0. Do not add compatibility aliases by default.

Follow-ups:

- Continue auditing public Rust type aliases and keep only aliases with a clear
  semantic owner or protocol value.
- Keep backup manifest/journal/receipt types as named structs because those are
  real serialized contracts.

## Findings

### CANIC-082-SNAPSHOT-DRIVER-ERROR-ALIAS-HARDCUT-001

Severity: P3
Category: api_boundary
Owner: `canic-backup::snapshot`
Current location: `crates/canic-backup/src/snapshot/types.rs`
Intended location: `SnapshotDriver` uses the concrete boxed driver error shape

Evidence:
`SnapshotDriverError` was a public alias for
`Box<dyn StdError + Send + Sync + 'static>` and was used by the backup snapshot
driver trait, tests, and CLI driver adapter.

Risk:
The alias added a public Rust namespace for a general dynamic error object
without owning a distinct backup data or protocol concept.

Decision:
Removed `SnapshotDriverError` and used
`Box<dyn StdError + Send + Sync + 'static>` directly across the snapshot driver
boundary.

Regression test:
`cargo test --locked -p canic-backup snapshot --lib`
