## Slice Contract

Boundary:
Pure policy modules should not import `view/` projection modules. Policy input
and decision shapes belong with the policy producer; ops may map storage records
into those policy shapes and map policy decisions into DTO responses.

Current owner:
Policy input and decision structs were split between
`canic-core::view::{env,topology,placement}` and
`canic-core::domain::policy::pure`.

Intended owner:
`canic-core::domain::policy::pure`.

Files/modules in scope:

- `crates/canic-core/src/domain/policy/pure/**`
- `crates/canic-core/src/ops/runtime/env/**`
- `crates/canic-core/src/ops/topology/input/mapper.rs`
- `crates/canic-core/src/ops/placement/{scaling,sharding}/**`
- Internal workflow call sites that consume scaling and sharding policy shapes.
- Obsolete `crates/canic-core/src/view/{env,topology,placement}/**` policy-shape
  modules.

Files/modules explicitly out of scope:

- Public DTO response shapes.
- Stable storage records.
- Sharding/scaling policy behavior.
- Runtime env validation behavior.
- Host deployment-truth policy helper layout.
- Automated dependency-boundary release gates.

Public surfaces affected:
None. `view` and `domain` are crate-private in `canic-core`.

Serialized surfaces affected:
None.

Commands/endpoints affected:
None.

Expected behavior change:
None.

Expected non-changes:

- Env validation returns the same validated fields.
- Topology policy consumes the same registry input data.
- Scaling and sharding policies compute the same plan states and blockers.
- Public DTO responses remain mapped in ops.
- Stable records and storage writes remain unchanged.

Validation:

- `cargo fmt --all -- --check` passed.

- `cargo clippy --locked -p canic-core --lib -- -D warnings` passed.
- `cargo test --locked -p canic-core policy --lib` passed.
- `cargo test --locked -p canic-core registry_policy --lib` passed.
- `cargo test --locked -p canic-core --lib` passed.

Rollback plan:
Restore the deleted view modules, restore policy re-exports from `view`, and
switch ops/workflow imports back to the previous view paths.

Follow-ups:

- Keep `view/` for projections that are actually derived read-only views rather
  than policy input or decision structs.
- Add a robust dependency-boundary helper before enforcing this as a release
  gate.

## Findings

### CANIC-082-POLICY-VIEW-001

Severity: P2
Category: policy_purity
Owner: `canic-core::domain::policy::pure`
Current location: `crates/canic-core/src/view/{env,topology,placement}/**`
Intended location: `crates/canic-core/src/domain/policy/pure/**`

Evidence:
Pure policy modules imported validated env, topology policy input, scaling
worker plan, and sharding policy input/decision types from `view/`.

Risk:
This made policy decisions depend on an upper-layer projection namespace and
blurred whether those structs were policy contracts or presentation/read-model
views.

Decision:
Fixed with a no-behavior-change ownership move. Policy input and decision
structs now live in `domain::policy::pure`; ops mappers convert storage records
to policy inputs and policy decisions to DTO responses.

Regression test:
Covered by `cargo test --locked -p canic-core policy --lib`,
`cargo test --locked -p canic-core registry_policy --lib`, and
`cargo test --locked -p canic-core --lib`.

## Fixes

- Moved `ValidatedEnv` into `domain::policy::pure::env`.
- Moved topology policy input structs into `domain::policy::pure::topology`.
- Moved scaling worker plan entries into
  `domain::policy::pure::placement::scaling`.
- Moved sharding placement, assignment, plan-state, and blocker-reason types
  into `domain::policy::pure::placement::sharding`.
- Deleted obsolete `view::env`, `view::topology`, and `view::placement` modules.

## Deferred

- Remaining `view/` modules for blob-storage and ICP refill projections are not
  part of this slice.
- Automated boundary tests are still deferred until a robust dependency checker
  exists.
