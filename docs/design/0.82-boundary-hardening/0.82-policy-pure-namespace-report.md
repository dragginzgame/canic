## Findings

### CANIC-082-POLICY-NS-001

Severity: P3
Category: policy_purity
Owner: `canic-core::domain::policy`
Current location: `crates/canic-core/src/domain/policy/**`
Intended location: `crates/canic-core/src/domain/policy/pure/**`

Evidence:
Core policy modules were already intended to be pure, but their module path did
not make that boundary explicit.

Risk:
Future cleanup or feature work could read `domain::policy` as a generic policy
area rather than a side-effect-free decision namespace, making import-boundary
audits harder to read.

Decision:
Fixed with a no-behavior-change namespace move. Core pure policy modules now
live under `domain::policy::pure`, and internal call sites use the explicit
namespace.

Regression test:
Covered by `cargo test --locked -p canic-core policy --lib`.

## Fixes

- Moved core pure policy modules under `domain::policy::pure`.
- Updated internal `canic-core` call sites from direct `domain::policy::*`
  module paths to `domain::policy::pure::*`.
- Kept `domain::policy` as the namespace shell and did not add compatibility
  aliases for the old direct module paths.

## Deferred

- Host deployment-truth policy helper layout remains unchanged.
- Automated policy import gates remain deferred until a robust dependency
  boundary helper exists.
