## Slice Contract

Boundary:
Top-level and command-family help/version rendering should be reached through
the canonical flag forms `--help`, `-h`, `--version`, and `-V`. Bare `help` and
`version` word forms are aliases and should not remain active pre-1.0 command
surface.

Current owner:
`canic-cli::cli::help` accepted bare `help` and `version` as shared aliases.
`canic-cli::inspect` and `canic-cli::medic` also had local word-help handling
for special nested command paths. Global forwarding tests still carried
removed-command fixtures for retired info/auth leaves.

Intended owner:
Canonical flag handling is owned by the CLI help helper and any focused leaf
helper that must intercept leaf flags before option parsing. Maintained global
forwarding tests cover current command leaves only.

Files/modules in scope:

- `crates/canic-cli/src/cli/help.rs`
- `crates/canic-cli/src/inspect/mod.rs`
- `crates/canic-cli/src/medic/mod.rs`
- `crates/canic-cli/src/medic/tests.rs`
- `crates/canic-cli/src/tests.rs`
- `crates/canic-cli/src/deploy/command.rs`
- `docs/status/current.md`

Files/modules explicitly out of scope:

- Command-specific option behavior other than help/version flag preflight.
- Operator command names other than removing bare word help/version aliases.
- Archived changelogs and archived design/status docs.
- Endpoint method names.
- Candid, JSON report, deployment truth, evidence/report, and stable-state
  schemas.

Public surfaces affected:
CLI behavior changes: bare word forms such as `canic help`, `canic version`,
`canic <command> help`, and nested special-case help word forms are no longer
the maintained help/version surface. Operators should use flag forms.

Serialized surfaces affected:
None.

Commands/endpoints affected:
CLI help/version preflight only. No endpoint names changed.

Expected behavior change:

- `--help` and `-h` remain the maintained help forms.
- `--version` and `-V` remain the maintained version forms.
- Bare `help` and `version` word aliases are no longer accepted by shared CLI
  preflight helpers.
- Inspect and medic local help interceptors use flag forms only.
- Stale global-forwarding fixtures for removed info/auth command leaves were
  removed instead of retained as anti-resurrection tests.

Expected non-changes:

- Top-level help content remains grouped by current command families.
- Command-family and leaf help remains available through `--help` / `-h`.
- Version output remains available through `--version` / `-V`.
- Global `--icp` and `--network` forwarding for current command leaves is
  unchanged.
- No JSON, Candid, evidence, deployment truth, or stable-state contract changed.

Rollback plan:
Restore bare word help/version handling only if the maintainer intentionally
chooses a command-word help surface before 1.0. Do not add compatibility
aliases, shims, or anti-resurrection tests by default.

Follow-ups:

  removed-command fixtures.
- Keep archived changelogs/history historical; update active docs and help only.

## Findings

### CANIC-082-CLI-HELP-ALIAS-HARDCUT-001

Severity: P2
Category: legacy_surface
Owner: `canic-cli::cli::help`
Current location: `crates/canic-cli/src/cli/help.rs`
Intended location: canonical flag-only help/version preflight

Evidence:
The shared CLI helper accepted `help` and `version` as bare first-argument
aliases alongside `--help`, `-h`, `--version`, and `-V`.

Risk:
Bare word aliases expand the public command surface and conflict with the
pre-1.0 hard-cut policy against aliases and compatibility routes.

Decision:
Removed bare word handling from shared help/version detection and updated
top-level help copy to advertise `canic <command> --help`.

Regression test:
`cargo test --locked -p canic-cli cli --lib`

### CANIC-082-CLI-HELP-ALIAS-HARDCUT-002

Severity: P2
Category: legacy_surface
Owner: `canic-cli::inspect`, `canic-cli::medic`
Current location: `crates/canic-cli/src/inspect/mod.rs`, `crates/canic-cli/src/medic/mod.rs`
Intended location: focused leaf helpers accept canonical flags only

Evidence:
Inspect and medic had focused local interceptors that still accepted bare
`help` and `version` words for nested leaf help/version paths.

Risk:
Local helper exceptions could keep command-word aliases alive after the shared
helper was hard-cut.

Decision:
Updated focused local interceptors and tests to use `--help`, `-h`,
`--version`, and `-V` only.

Regression test:
`cargo test --locked -p canic-cli inspect --lib`
`cargo test --locked -p canic-cli medic --lib`

### CANIC-082-CLI-HELP-ALIAS-HARDCUT-003

Severity: P3
Category: test_ownership
Owner: `canic-cli::tests`
Current location: `crates/canic-cli/src/tests.rs`
Intended location: maintained global-forwarding tests cover current leaves only

Evidence:
Global forwarding tests still carried fixtures for removed `info medic` and
auth renewal/provisioner command leaves.

Risk:
Anti-resurrection fixtures keep retired command names active in tests and make
the maintained command surface harder to audit.

Decision:
Removed removed-command fixtures and kept positive coverage for current global
forwarding leaves and canonical `--help` tails.

Regression test:
`cargo test --locked -p canic-cli cli --lib`
