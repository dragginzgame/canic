## Slice Contract

Boundary:
The wasm-store template APIs own their endpoint-facing public methods and may
use private helper functions for local glue. Private facades and crate-private
support modules should only exist when they add a real boundary. A private
facade or support module that only forwards template API calls does not own
separate semantics.

Current owner:
`LocalWasmStoreApi` was a private pass-through facade between
`WasmStoreCanisterApi` and local wasm-store support helpers. The crate-private
`support` module also held the remaining root bootstrap, publication, local
wasm-store, and GC helper logic even though only `api::template` called it.

Intended owner:
`WasmStoreCanisterApi`, `WasmStoreBootstrapApi`, and
`WasmStorePublicationApi` remain the public endpoint-facing facades. Their
private glue lives in `api::template`; storage and workflow owners still
perform the underlying operations.

Files/modules in scope:

- `crates/canic-control-plane/src/api/template/mod.rs`
- `crates/canic-control-plane/src/lib.rs`
- `crates/canic-control-plane/src/support.rs`

- Root bootstrap endpoint surfaces.
- Root publication endpoint surfaces.

## Findings

### CANIC-082-WASM-STORE-TEMPLATE-SUPPORT-001

Severity: P3
Category: ops_ownership
Owner: `canic-control-plane::api::template`
Current location: `crates/canic-control-plane/src/api/template/mod.rs`
Intended location: direct `WasmStoreCanisterApi` to private template helper calls

Evidence:
After the public wasm-store facade hard cut, `LocalWasmStoreApi` was private
and every method only forwarded to the corresponding local support helper.

Risk:
The extra private facade created another apparent owner for local wasm-store
semantics without adding validation, policy, or endpoint behavior.

Decision:
Removed `LocalWasmStoreApi` and routed `WasmStoreCanisterApi` directly to
private template helpers.

Regression test:
Control-plane and `canic` facade tests compile the wasm-store endpoint-facing
API after the private helper removal.

### CANIC-082-WASM-STORE-TEMPLATE-SUPPORT-002

Severity: P3
Category: ops_ownership
Owner: `canic-control-plane::support`
Current location: `crates/canic-control-plane/src/support.rs`
Intended location: private helpers in `crates/canic-control-plane/src/api/template/mod.rs`

Evidence:
Repo-wide source search showed the crate-private `support` module was called
only from `api::template`. Its functions mediated template API calls to
template storage ops, subnet storage ops, and the wasm-store publication
workflow without adding an independent boundary.

Risk:
The hidden support module looked like another owner for template API behavior
and preserved a parallel helper namespace after the public and private API
facades were narrowed.

Decision:
Removed the crate-private `support` module and moved its helper logic into
private functions in the template API module. Storage ops and workflow owners
remain unchanged.

Regression test:
Control-plane, wasm-store canister facade, and root control-plane facade tests
compile the moved helpers through the same public API methods.

## Fixes

- Removed private `LocalWasmStoreApi`.
- Removed `crates/canic-control-plane/src/support.rs`.
- Removed the matching crate module declaration.
- Updated `WasmStoreCanisterApi` to call private template helpers directly.
- Moved root bootstrap, publication overview/admin, local wasm-store status,
  staging, chunk, and GC helper logic into private template API helpers.
