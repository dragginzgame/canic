## Slice Contract

Boundary:
Root bootstrap subnet identity resolution during init and post-upgrade.

Current owner:
`canic-control-plane::workflow::bootstrap::root`, with a non-IC fallback that
used the root canister principal as the subnet principal when registry
discovery was unavailable.

Intended owner:
Root lifecycle initialization owns explicit local/test subnet identity.
Root bootstrap owns only registration of the resolved identity and IC registry
discovery for IC builds.

Files/modules in scope:

No command, endpoint, Candid, JSON, deployment-truth, evidence/report, or

Serialized surfaces affected:
None.

Commands/endpoints affected:
None.

Expected behavior change:
Root bootstrap no longer falls back to `canister_self()` when subnet discovery
is unavailable. Local/test builds use the subnet identity already seeded by
lifecycle init. IC builds fail the bootstrap phase if the NNS registry does not
return a subnet.

Expected non-changes:

- `SubnetIdentity::Manual` remains the explicit test/support lifecycle identity

- Local/test root setup can still use explicit lifecycle-seeded subnet identity.
- IC builds still resolve subnet identity from the NNS registry.
- Pool import, canister creation, wasm-store reconcile, and index rebuild
  behavior are unchanged after subnet identity succeeds.
- No compatibility fallback, alias, or replacement shim is added.

Rollback plan:
Restore the non-IC branch that set both `Env.subnet_pid` and app registry
subnet identity to `IcOps::canister_self()` when registry discovery was
unavailable. This is not recommended because it reintroduces implicit topology
identity.

Follow-ups:

- Keep root lifecycle init as the sole owner of explicit local/test subnet
  identity.
- If future local bootstrap modes need a different subnet identity, add an
  explicit lifecycle/init contract rather than a bootstrap fallback.

## Findings

### CANIC-082-ROOT-BOOTSTRAP-SUBNET-001

Severity: P2
Category: lifecycle_bootstrap
Owner: `canic-control-plane::workflow::bootstrap::root`
Current location: `crates/canic-control-plane/src/workflow/bootstrap/root.rs`
Intended location: root lifecycle init owns explicit local/test identity; root
bootstrap registers or discovers identity without fallback.

Evidence:
`root_set_subnet_id` used `IcOps::canister_self()` as the subnet principal
whenever `try_get_current_subnet_pid` did not produce an IC registry subnet and
the build network was not `ic`.

Risk:
Bootstrap could silently invent topology identity instead of requiring the
runtime identity to come from either lifecycle init or IC registry discovery.
That kept a compatibility-style fallback alive in lifecycle bootstrap before
1.0.

Decision:
Fixed with a hard cut. Local/test builds now read the lifecycle-seeded subnet
principal and fail if it is missing. IC builds fail the subnet identity phase
when registry discovery returns `None` or errors.

Regression test:
Focused bootstrap/control-plane tests and clippy validation cover the new
fallible bootstrap path. No anti-resurrection test was added for the removed
fallback.

## Fixes

- Made `root_set_subnet_id` return `Result<(), InternalError>`.
- Made init and post-upgrade bootstrap stop and mark bootstrap failed when
  subnet identity resolution fails.
- Removed the non-IC `canister_self()` subnet fallback branch.
- Preserved explicit lifecycle-seeded local/test subnet identity.

## Deferred

- No broad lifecycle rewrite was attempted.
- No root init argument or `SubnetIdentity` Candid shape changes were made.
