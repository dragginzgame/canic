## Findings

### CANIC-082-RUNTIME-DTO-003

Severity: P2
Category: dto_boundary
Owner: `canic-core::dto::runtime`
Current location: `crates/canic-core/src/dto/runtime.rs`
Intended location: `crates/canic-core/src/domain/runtime.rs`

Evidence:
`api::runtime` produced runtime check statuses, diagnostic severities, and
state-domain statuses while importing their semantic values from
`dto::runtime`.

Risk:
The DTO module was acting as the semantic owner for runtime diagnostic status
values used by runtime internals before the endpoint boundary.

Decision:
Fixed with a no-behavior-change ownership move. The enums now live in
`domain::runtime`, while `dto::runtime` publicly re-exports them so existing
API paths and serialized shapes remain stable.

Regression test:
`dto::runtime::tests::runtime_enums_roundtrip_candid_with_runtime_variant_labels`
continues to prove the public DTO path round-trips the moved status enums
through Candid.

## Fixes

- Added `RuntimeCheckStatus`, `RuntimeDiagnosticSeverity`, and
  `RuntimeStateDomainStatus` to `domain::runtime`.
- Re-exported the same values from `dto::runtime`.
- Updated runtime API code to import the domain owners.

## Deferred

- Top-level health, readiness, runtime, and timer status enums remain
  DTO-owned.
- Runtime status DTO structs remain unchanged.
