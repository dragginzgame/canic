## Findings

### CANIC-082-RUNTIME-ENUM-LABEL-001

Severity: P2
Category: serialized_surface
Owner: `canic-core::domain::runtime`
Current location: `crates/canic-core/src/domain/runtime.rs`
Intended location: `crates/canic-core/src/domain/runtime.rs`

Evidence:
Candid's Rust crate documents support for explicit `#[serde(rename = "...")]`
attributes when deriving `CandidType`, and explicitly states that
`rename_all` is not supported. The runtime enums used `rename_all` for Serde
JSON labels plus per-variant PascalCase aliases to keep Candid decoding working.

Risk:
The alias workaround made Candid accept a second runtime enum label surface and
hid the mismatch between Candid-supported attributes and Serde-only
`rename_all` behavior.

Decision:
Hard-cut runtime introspection enum labels to explicit per-variant
`serde(rename = "...")` attributes. Remove all runtime enum `serde(alias)`
attributes and avoid `rename_all` for these Candid-bearing enums.

Regression test:
`dto::runtime::tests::runtime_enums_serialize_canonical_snake_case_labels`
pins the Serde labels, and
`protocol_surface::runtime_introspection_dto_candid_shapes_are_named` pins
snake_case labels in the generated Candid type environment.

## Fixes

- Replaced runtime enum `serde(rename_all = "snake_case")` plus
  `serde(alias = "...")` with explicit per-variant `serde(rename = "...")`.
- Kept JSON and Candid labels aligned on the same canonical snake_case values.
- Updated runtime enum tests and protocol-surface assertions to pin the hard
  cut.

## Deferred

- No old-label compatibility tests were added; this is an intentional hard cut,
  not a compatibility surface.
