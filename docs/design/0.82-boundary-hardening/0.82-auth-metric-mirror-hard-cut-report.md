## Slice Contract

Boundary:
Runtime auth metric recorders for session lifecycle, session bootstrap,
identity fallback, and role-attestation verification outcomes.

Current owner:
`canic-core::ops::runtime::metrics::auth`, with compatibility mirroring into
`canic-core::ops::runtime::metrics::access`.

Intended owner:
`canic-core::ops::runtime::metrics::auth` owns auth event dimensions and
projection. Access metrics remain dedicated to endpoint access-expression
evaluation.

Files/modules in scope:

- `crates/canic-core/src/ops/runtime/metrics/auth`
- `crates/canic-core/src/access/auth/mod.rs`

Files/modules explicitly out of scope:

- Auth identity resolution behavior.
- Delegated-token verification.
- Access-expression metrics for endpoint guard evaluation.
- Metrics query pagination and sorting semantics.
- CLI command surfaces, endpoint routes, Candid, JSON, deployment truth,
  evidence/report schemas, or stable-state layout.

Public surfaces affected:
Auth session and attestation events no longer create compatibility rows in the
older `access` metric family. The canonical `auth` metric family remains.

Serialized surfaces affected:
Public metric rows no longer include mirrored Access-family rows for auth
session and attestation events. Canonical Auth-family metric labels are
unchanged.

Commands/endpoints affected:
No command or endpoint name changes. Metrics query surfaces continue to expose
the security metrics tier, but without the duplicated Access-family auth rows.

Expected behavior change:
Operators consuming old `access/auth_session/*` or
`access/auth_attestation_verifier/*` metric rows must switch to the canonical
`auth/*` metric family.

Expected non-changes:

- Auth identity resolution behavior is unchanged.
- Auth session and attestation event recording still happens.
- Canonical Auth-family metric dimensions and labels are unchanged.
- Access metrics still record endpoint access-expression guard outcomes.
- No compatibility mirror, alias, or replacement shim is added.

## Findings

### CANIC-082-AUTH-METRIC-MIRROR-001

Severity: P2
Category: metrics_surface
Owner: `canic-core::ops::runtime::metrics::auth`
Current location: `crates/canic-core/src/ops/runtime/metrics/auth`
Intended location: `crates/canic-core::ops::runtime::metrics::auth`

Evidence:
Auth session and attestation recorders wrote both canonical Auth-family
metrics and older Access-family rows using private endpoint/predicate labels.

Risk:
The metrics surface carried duplicate auth semantics in two metric families,
which made ownership unclear and left old observability labels alive before
1.0.

Decision:
Fixed with a hard cut. Auth recorders now write only the canonical Auth metric
family, and the private Access-row label vocabulary was removed.

Regression test:
Auth recorder tests now assert canonical Auth-family dimensions directly, and
access identity tests assert identity-fallback metrics through `AuthMetrics`.

## Fixes

- Removed auth session and attestation mirroring into Access metrics.
- Deleted the private auth endpoint/predicate label module used only for the
  Access mirror.
- Updated auth metric tests to assert canonical Auth-family dimensions.
- Updated auth identity tests to assert fallback metrics through `AuthMetrics`.

## Deferred

- No old Access-row compatibility tests were added; this is an intentional hard
  cut, not a compatibility surface.
- Root bootstrap fallback was handled in a separate lifecycle/bootstrap
  hard-cut slice.
