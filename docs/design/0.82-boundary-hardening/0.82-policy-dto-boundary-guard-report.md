## Slice Contract

Boundary:
Pure policy modules must not import side-effect layers or perform side effects.
Passive DTO modules must not import internal behavior layers or perform side
effects. Public error DTO mapping remains a documented boundary-adapter
exception.

Current owner:
The 0.82 design documented these boundaries and previous slices audited them,
but there was no focused test guard for the full pure-policy namespace or the
non-error DTO trees.

Intended owner:
`canic-core` integration boundary tests guard the pure-policy and passive-DTO
import/effect rules.

Files/modules in scope:

- `crates/canic-core/tests/policy_purity_boundary_guard.rs`
- `crates/canic-core/tests/passive_dto_boundary_guard.rs`
- `crates/canic-core/src/domain/policy/pure/**`
- `crates/canic-core/src/dto/**`
- `crates/canic-control-plane/src/dto/**`

Files/modules explicitly out of scope:

- Public DTO schemas and Candid shapes.
- Policy behavior.
- Error-to-public-error mapping behavior.
- Runtime code, storage code, ops code, and workflow code.
- Public endpoint, CLI, JSON, deployment-truth, or evidence surfaces.

Public surfaces affected:
None.

Serialized surfaces affected:
None.

Commands/endpoints affected:
None.

Expected behavior change:
None.

Expected non-changes:

- Pure policy decisions remain unchanged.
- DTO definitions and serialized shapes remain unchanged.
- `dto::error` remains the public error boundary adapter and is explicitly
  excluded from the passive DTO import guard.
- Blob-storage wire error variant names remain unchanged.

## Findings

### CANIC-082-POLICY-DTO-GUARD-001

Severity: P2
Category: policy_purity
Owner: `canic-core::domain::policy::pure`
Current location: `crates/canic-core/src/domain/policy/pure/**`
Intended location: `crates/canic-core/tests/policy_purity_boundary_guard.rs`

Evidence:
The 0.82 design declares pure policy must not import storage, ops, workflow,
DTO, CDK/runtime, timer, or serialization boundaries. The existing code
matched that rule, but it was not enforced by a focused guard.

Risk:
Future cleanup or feature work could reintroduce side-effect imports into pure
policy modules without tripping focused tests.

Decision:
Fixed by adding a focused source-boundary guard that scans pure policy modules
for forbidden side-effect imports, async/timer/IC call fragments, and wire
serialization derives/usages.

Regression test:
`cargo test --locked -p canic-core --test policy_purity_boundary_guard`

### CANIC-082-POLICY-DTO-GUARD-002

Severity: P2
Category: dto_boundary
Owner: `canic-core::dto` and `canic-control-plane::dto`
Current location: DTO source trees
Intended location: `crates/canic-core/tests/passive_dto_boundary_guard.rs`

Evidence:
The 0.82 design declares DTOs are passive boundary data. The existing
non-error DTO trees matched that rule, but it was not enforced across the full
Canic core/control-plane DTO surface.

Risk:
Future DTO additions could import internal behavior layers or side-effect
helpers and blur DTO/passive boundary ownership.

Decision:
Fixed by adding a focused passive DTO guard for non-error DTO trees. The guard
explicitly excludes `dto::error`, which is a public error boundary adapter
rather than a passive data-only module.

Regression test:
`cargo test --locked -p canic-core --test passive_dto_boundary_guard`

## Fixes

- Added a pure-policy boundary guard for forbidden imports, side effects, IC
  calls, timers, async work, and wire serialization fragments.
- Added a passive DTO boundary guard for forbidden internal behavior-layer
  imports and side-effect fragments in non-error DTO modules.
- Documented `dto::error` as the explicit public error boundary-adapter
  exception.

## Deferred

- Behavior-level lifecycle changes remain deferred to focused PocketIC upgrade
  coverage. The source-shape lifecycle guard is tracked separately in
  `0.82-lifecycle-boundary-guard-report.md`.
