# Canic 0.87 Design: Operator Boundary Hygiene

Date: 2026-07-12

## Status

- Status: active; Slice A release-prepared as `0.87.0`, Slices B-C pending.
- Audit source:
  `docs/audits/reports/2026-07/2026-07-12/codebase-health.md`.
- Implementation tracker: [status.md](status.md).

## Problem

The 0.86 structural pass closed its three selected production hubs, but a fresh
flow audit found smaller duplication at operator boundaries:

1. scaffold commands could return failure after leaving partial local state;
2. commands repeatedly reconstruct the same host ICP and
   installed-deployment failures;
3. path-resolution tests mutate process-global environment under unrelated
   locks.

These are one bounded line because each concerns authority at a host/operator
boundary: local files, external ICP diagnostics, or process inputs. The line is
not a general cleanup program.

## Decisions

### Slice A - Failure-atomic scaffold mutation

Scaffold commands must either complete their declared local mutations or
restore the pre-command state before returning an error.

- Use the existing host durable byte-replacement function for modified project
  files.
- Capture only the existing workspace and fleet bytes the command can change.
- Route failures through one private rollback function.
- Remove only a scaffold directory that preflight proved did not exist.
- Retain both the operation and rollback failure when cleanup fails.

### Slice B - One typed ICP failure boundary

`canic-host::icp` owns process execution, raw output, version compatibility,
and the interpretation of external ICP CLI diagnostics. Command modules consume
that typed result and own only command-specific response policy.

- Add one typed ICP diagnostic classification for the exact external cases
  Canic acts upon.
- Keep unclassified output available as an external command failure.
- Stop rebuilding `IcpCommandError` fields in each CLI command.
- Stop rebuilding installed-deployment transport failures in each CLI command.
- Preserve command-specific exit codes, hints, and user-facing context at the
  command boundary.
- Delete consumer-local ICP string classifiers and replaced exhaustive
  translators in the same slice.

The host adapter may expose a focused projection or source-preserving variant.
It must not expose a generic error graph or move CLI presentation policy into
the host.

### Slice C - Pure path precedence inputs

Production wrappers may read the current directory and `CANIC_*` environment,
but their precedence decisions must be testable without mutating global state.

- Keep the existing public path and config-selection entry points.
- Extract only the precedence logic needed by those entry points into pure
  functions receiving explicit override values and a start/root path.
- Replace environment-mutating tests with direct pure-input tests.
- Delete the separate environment locks and restore helpers when no test uses
  them.
- Preserve all current path normalization, discovery, and fallback behavior.

## Hard Cut

- Replaced truncating writes, error translators, diagnostic string classifiers,
  environment mutation helpers, and module-local environment locks are removed
  in their implementation slice.
- No aliases, forwarding wrappers, deprecated variants, fallback classifiers,
  or compatibility behavior remain.
- Report and CLI behavior is preserved unless a duplicated adapter currently
  produces contradictory output; any such correction is explicit and tested.

## Deliberate Non-Abstractions

0.87 does not add:

- a filesystem transaction framework, journal, or recovery command;
- a generalized project-context or path service;
- a global application error framework;
- a parser for arbitrary third-party diagnostic text;
- a generic thread pool or query fan-out framework;
- another role-feature, memory-allocation, or descriptor authority;
- module splitting based only on file length;
- deployment, stable-memory, Candid, or persisted-schema changes.

Host and backup durable writers remain separate because they sit in different
crate ownership boundaries. List, metrics, and cycles fan-out remains local
because their small loops have different keys and failure semantics.

## Acceptance

### Slice A

1. Successful scaffold output is byte-for-byte unchanged.
2. Failure after an earlier write removes the new scaffold directory.
3. Changed pre-existing workspace or fleet documents are restored exactly.
4. Existing paths are never recursively removed.
5. Workspace replacement is durable and never truncates in place.
6. Tests assert filesystem state and typed errors.

### Slice B

1. External ICP wording acted upon by Canic is classified only in
   `canic-host::icp`.
2. Command modules no longer exhaustively copy every `IcpCommandError` variant.
3. Repeated installed-deployment transport reconstruction is removed.
4. Raw unclassified diagnostics and underlying I/O/JSON sources are retained.
5. Existing exit behavior and actionable command hints have focused tests.

### Slice C

1. Host path/config precedence is covered through explicit inputs.
2. No host test calls `set_var` or `remove_var`.
3. No test-only environment lock or restoration helper remains.
4. Existing environment variables, precedence, normalization, discovery, and
   public entry points are unchanged.

## Sequence

1. Slice A: scaffold failure atomicity - release-prepared as `0.87.0`.
2. Slice B: ICP and installed-deployment error convergence.
3. Slice C: environment-free path precedence tests.
4. Close 0.87 and run a fresh audit; do not extend this design by accretion.
