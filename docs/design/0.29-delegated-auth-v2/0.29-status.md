# Canic Delegated Auth V2 0.29 Status

- **Design:** [`0.29-design.md`](0.29-design.md)
- **Scope:** implementation state, verification, and known blockers for the `0.29`
  Delegated Auth V2 hard-cut design

## Completed

- Added V2 wire/data structures for `DelegationCertV2`, `DelegationProofV2`,
  `DelegatedTokenClaimsV2`, and `DelegatedTokenV2`.
- Added supporting V2 wire/data structures for root key authority, root key
  certificates, root key sets, role/principal audiences, and shard key binding.
- Added `CanicAuthCanonicalV2` hashing for delegation certificates, token
  claims, root-key certificate payloads, and role hashes.
- Added canonical encoding tests for deterministic sorting/deduplication,
  noncanonical role/scope rejection, domain separation, and root-key certificate
  payload signing boundaries.
- Added V2 audience policy helpers for shape validation, role/principal subset
  checks, verifier membership checks, and single-role hash binding.
- Added root-side V2 certificate policy validation for version, root principal,
  certificate TTL, token TTL, role hash, shard public-key hash, and expected
  shard derivation hash.
- Added V2 root-key trust-anchor resolution for trusted local root keys and
  embedded root-key certificate fallback through `RootKeyAuthorityV2`.
- Added a pure V2 verifier path that validates token and certificate structure,
  cert policy, root-key trust, root and shard signatures, time bounds, audience
  membership, role hash binding, and scope attenuation without local proof
  lookup.
- Added pure root-side V2 proof issuance with certificate construction,
  root/shard key hash binding, verifier role-hash derivation, TTL policy
  enforcement, canonical cert-hash signing, root key embedding, and optional
  root-key certificate preservation.
- Added the internal runtime root signing hook that resolves local root and
  shard threshold-ECDSA public keys, binds key-name and shard derivation-path
  hashes into `ShardKeyBindingV2`, signs the prepared cert hash, and returns a
  self-contained `DelegationProofV2` without verifier fanout.
- Added pure shard-side V2 token minting with cert-hash binding, audience and
  scope attenuation, TTL enforcement, nonce carry-through, embedded proof
  preservation, and injected shard signing.
- Added the internal runtime shard signing hook that prepares canonical V2 token
  claims, signs the prepared claims hash with the local shard threshold-ECDSA
  derivation path, and returns a self-contained `DelegatedTokenV2`.
- Added the internal runtime verifier hook that checks configured shard key
  binding, resolves the local root trust anchor from the root key cache, applies
  local verifier role membership, and calls the pure V2 verifier without
  proof-cache lookup.
- Added explicit `DelegationApi` V2 helpers for local root proof issuance,
  signer token issuance from an explicit V2 proof, and verifier validation.
- Added the explicit root `canic_request_delegation_v2` endpoint and matching
  protocol constant for requesting self-contained `DelegationProofV2` values.
- Added the remote `DelegationApi::request_delegation_v2` helper for
  signer-to-root V2 proof requests.
- Added `DelegatedTokenMintRequestV2` and `DelegationApi::mint_token_v2`,
  which request a V2 root proof and immediately mint a self-contained V2 token
  without using local proof-cache state.
- Added local/dev V2 signer endpoints on the user-shard demo canister and
  delegation signer stub for exercising the one-call V2 mint path.
- Added V2 verifier root-key pull-on-verify. When a V2 token reaches a guarded
  endpoint and the verifier has no matching root key cached, Canic fetches the
  expected root threshold-ECDSA public key by configured key name and root
  derivation path, then rejects the token if the fetched key hash does not
  match the certificate.
- Changed endpoint guards to V2-only delegated-token decoding. Guard validation
  uses the self-contained verifier path and does not look up local delegation
  proofs.
- Added macro support for `DelegatedTokenV2` and Candid `Reserved` as
  authenticated endpoint first-argument types.
- Migrated repo-local protected token endpoint surfaces to Candid `Reserved` so
  guarded methods do not expose token wire details while the guard performs raw
  first-argument V2 token verification.
- Changed signer lifecycle prewarm to warm local shard signing key material
  only. Startup no longer requests or stores delegation proofs.
- Tightened root-side V2 TTL policy ownership so `DelegationProofIssueRequestV2`
  carries only requested certificate TTL; root derives certificate and token TTL
  ceilings from `auth.delegated_tokens.max_ttl_secs`.
- Removed topology-time verifier proof catch-up. Canister placement and state
  propagation no longer push root-cached V1 proofs into newly created verifier
  canisters.
- Removed legacy V1 signer issuance APIs, V1 root `IssueDelegation` routing,
  verifier proof-install endpoints, proof-fanout/admin repair DTOs, proof-cache
  stable storage, proof-cache config, proof-sync metrics, and V1-only test
  suites.

## Known Boundaries

- V2 is a bearer-token system. Replay protection is bounded by token TTL only.
- Verifier role configuration is trusted. A verifier with the wrong local role is
  compromised for delegated-auth purposes.
- `RootKeyAuthorityV2` is a critical trust anchor. Compromise is equivalent to
  compromise of the root key hierarchy.
- V2 has no core revocation model. Certificates and tokens remain valid until
  expiry unless a future explicit revocation mechanism is designed.
