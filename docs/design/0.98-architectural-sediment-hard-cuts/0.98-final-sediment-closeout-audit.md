## Scope

In scope:

- all product source under `crates/`;
- facade, macro, generated-boundary, control-plane, host, CLI, backup, test
  support, canister, and fleet consumers;
- Cargo features and dependencies;
- stable records, state descriptors, replay encodings, Candid DTOs, JSON
  evidence, CLI proof fixtures, and current documentation;
- public, restricted, hidden, feature-gated, test-only, and single-variant
  items; and
- representative end-to-end ownership for config, auth, replay, topology,
  deployment evidence, backup/restore, and operator parsing.

Excluded:

- immutable historical reports, archived designs, released changelog entries,
  and generated `target/` or `.icp/` output;
- sibling repositories, including the stale local Toko checkout; and
- new 0.99 App/Fleet identity or 0.100 multi-subnet Fleet Registry behavior.

Tests and generated consumers were included wherever they were the only
consumer or proved a macro, Candid, persistence, or release boundary.

## Findings and corrections

### P1

| ID | Evidence and failure scenario | Correction |
| --- | --- | --- |
| `CANIC-098-FINAL-APP-REGISTRY-001` | Root bootstrap wrote a stable Subnet-to-root App Registry and exposed a controller query, but no runtime decision consumed the map. The endpoint comment reserved future behavior and the path would compete with 0.100's real Fleet Registry. | Deleted DTO/API/workflow/ops/storage/bootstrap/protocol/test/state-contract surface; retired memory ID 14 without reuse. |
| `CANIC-098-FINAL-STABLE-COMPAT-002` | App, auth, directory, sharding, and replay records explicitly defaulted fields missing from old bytes. Replay also omitted absent current fields, preserving old map shapes despite the hard-cut policy. | Removed explicit old-shape defaults and omission rules; current replay encoding writes every field; exact bytes updated and tested. |
| `CANIC-098-FINAL-PROOF-FIXTURE-003` | Installed and packaged CLI proof fixtures wrote removed `network` rather than strict current `environment`; their fake ICP process also emitted the obsolete decoded-value JSON instead of ICP 1.1's canonical `response_bytes` envelope. The release proof therefore failed before proving the intended command surface. | Replaced the stale install-state key, reset the fixture to the current pre-1.0 schema version, and made every fake query/update emit exact typed Candid response bytes through the maintained ICP envelope. |
