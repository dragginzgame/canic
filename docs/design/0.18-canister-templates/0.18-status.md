## Core 0.18 Direction

- [x] Establish root-owned template manifest authority as a first-class storage boundary.
- [x] Introduce canonical template identifiers, version identifiers, and manifest state/value types.
- [x] Seed approved manifests from the current embedded release table so provisioning resolves role -> manifest before it resolves bytes.
- [x] Replace role-keyed embedded wasm lookup with template-id keyed retrieval.
- [x] Introduce template canister retrieval APIs and deterministic chunk retrieval.
- [x] Move ordinary deployable payload ownership out of root.
- [x] Reduce root to the irreducible bootstrap set.
