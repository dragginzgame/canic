## Verdict

**NOT READY.** The immutable `v0.97.3` release does not satisfy its own
closeout contract. Five P1 defects were confirmed by tracing the tag rather
than trusting the prior closeout claim. All five have the smallest in-scope
correction in the current working tree, but those corrections are not part of
the immutable release anchor.

The exact release blocker is to publish the reviewed corrections and close
against that new immutable commit. The corrected working tree passes the
cumulative 28-case root PocketIC matrix with repository-pinned
`ic-wasm 0.10.0`; the final release gate must bind that evidence to the new
anchor. No 0.97 package version, stable schema, JSON, configuration, or CLI
change was made by this audit. Generated role Candid/Wasm regains the
`canic_metrics` method already required when the normal role dependency
selects `metrics`.

## Reconstructed Scope And Baseline

The design, Slice A inventory, release history, detailed changelog, status
tracker, and baseline-to-tag diff agree on three bounded product changes:

1. **Role-owned runtime dependencies.** Cargo resolver 2, a feature-empty
   workspace Canic dependency with defaults disabled, one canonical direct
   normal `canic` edge per role with explicit features, and a separately
   purpose-bound feature-empty build edge. Build, deploy, Medic, state,
   internal PocketIC, and release paths must use one target-correct
   `RoleCargoGraphEvidence` producer and the core role-contract checker.
2. **Public CDK hard cut.** Human-facing `canic::cdk` is removed. Declarative
   and procedural macros use only the exact hidden `canic::__internal::cdk`
   boundary. Cycle and ICRC-21 DTOs live at their semantic owners, with no
   alias, forwarding module, fallback package, or deprecated path.
3. **Root-owned manual ICP refill.** One controller-guarded root endpoint owns
   manual root self-refill. Configuration is root-only and presence-enabled;
   memory ID 33, replay, cost, stable operation, and recovery state remain
   root-owned. There is no generic capability, nonroot emitter, automatic
   threshold, timer, arbitrary source/target, fabrication path, or cross-shape
   recovery adapter.

The immutable baseline is released `v0.96.8`; it is an ancestor of the
immutable `v0.97.3` anchor. The audit inspected the complete baseline-to-tag
diff and searched the repository outside that diff for consumers and obsolete
assumptions. A separate clean detached worktree was used to distinguish tag
behavior from the shared working tree, which already contained unrelated open
0.98 work.

## Findings And Corrections

### P1 — CANIC-097-CLOSE-001 — Protected path selection was not globally shortest

**Evidence:** `crates/canic-host/src/role_contract/package/mod.rs`,
`validate_runtime_graph` and `shortest_protected_path`.

At the tag, each sibling direct dependency was searched independently and the
first protected result was returned. Cargo edge order could therefore select a
longer path before a later one-edge violation, contradicting the frozen
shortest-edge-count rule. Existing tests covered a path and deterministic
ordering inside one subtree, not competition across sibling roots.

**Correction:** collect all sibling candidates, sort by edge count and the
existing normalized edge key, then render only the global winner.

**Regression:**
`protected_dependency_diagnostic_selects_the_globally_shortest_path` proves a
lexically earlier three-edge path loses to a later one-edge path.

### P1 — CANIC-097-CLOSE-002 — Refill lost explicit recovery after an external effect

**Evidence:** `crates/canic-core/src/workflow/ic/icp_refill/execution.rs`,
`transfer_operation` and `notify_operation`; and
`crates/canic-core/src/workflow/ic/icp_refill/replay.rs`,
`reserve_icp_refill_replay`.

The workflow marked the replay receipt `ExternalEffectInFlight` before the
ledger or CMC call. Transport uncertainty correctly became
`RecoveryRequired`, but a failure while persisting the returned ledger/CMC
result escaped directly. The receipt remained in-flight, and exact retry
returned only `OperationInProgress`; no deterministic recovery disposition
was visible even though the external call had already run.

**Correction:** every result-state write after either external call now passes
through `preserve_icp_refill_post_effect_result`, which marks
`ExternalEffectStatusUnknown` while retaining the primary typed cause.

**Regression:**
`refill_post_effect_storage_failure_requires_recovery_and_preserves_effect_receipt`
proves both the typed storage cause and the recovery-required receipt state.

### P1 — CANIC-097-CLOSE-003 — Obsolete manifest lookup competed with role config

**Evidence at the tag:**
`crates/canic-host/src/release_set/paths/manifests.rs`,
`root_manifest_path` and `canister_manifest_path`; and
`crates/canic-host/src/workspace_discovery/mod.rs`,
`resolve_canister_manifest_from_metadata_under`.

These public helpers selected a package by `[package.metadata.canic].role`
under `fleets/` without consulting the canonical
`[roles.<role>].package` declaration. They had no production caller, but they
were an accidental public second authority whose tests explicitly maintained
the obsolete flow.

**Correction:** delete the helpers, their error DTO, cache, re-exports, and
tests. No alias or replacement was added; current config-driven callers remain
unchanged.

**Regression evidence:** the final stale-symbol search finds no definition,
re-export, or call to the deleted surface, while all 841 host tests pass.

### P1 — CANIC-097-CLOSE-004 — Role diagnostics exposed checkout-local data

**Evidence at the tag:** `crates/canic-host/src/role_contract/mod.rs`,
`resolve_declared_role_package_contract`;
`crates/canic-host/src/role_contract/package/mod.rs`, role/config/metadata/tree
error construction; and
`crates/canic-host/src/state_manifest/resolution.rs`.

Several stable role-contract reasons included absolute config/manifest paths
or raw Cargo error output. Those reasons reach Medic/state diagnostics, in
conflict with the explicit rule that local paths and possible registry
credentials never enter diagnostics. Existing tests proved normalized
dependency-path rendering, not failures before graph correlation.

**Correction:** retain typed finding ownership but use stable action-level
reasons for config, manifest, metadata, and tree failures. Filesystem/TOML
causes are preserved where safe without embedding the selected absolute path;
raw Cargo stderr is not projected.

**Regression:** `role_contract_rejection_does_not_expose_local_or_source_data`
proves neither the absolute root nor a secret-like malformed source value is
projected; the final structural search covers the other reported leak paths.

### P1 — CANIC-097-CLOSE-005 — Generated metrics used the build dependency's features

**Evidence at the tag:** `crates/canic/src/build_support/metrics.rs`,
`metrics_feature_enabled`; and `crates/canic/src/macros/build.rs`,
`__canic_build_internal`.

Resolver 2 intentionally isolates the feature-empty build dependency from the
role's normal dependency. The build macro nevertheless evaluated
`cfg!(feature = "metrics")` inside that build-dependency instance. It therefore
emitted `canic_disable_bundle_metrics` even when the role's canonical normal
edge explicitly enabled `metrics`. The issuer Wasm then lacked
`canic_metrics`. Host graph tests proved declaration shape but never queried
the built endpoint.

**Correction:** the build macro reads the canonical normal
`[dependencies].canic.features` array from the role manifest. Missing or
malformed explicit features fail closed. Build-dependency features have no
effect on endpoint selection.

**Regressions:** three parser tests prove normal/build separation and malformed
rejection. The four-case `pic_role_attestation` suite proves the actual issuer
Wasm exposes metrics and records both denied and successful guarded calls.

## Documented P2 Debt

### P2 — CANIC-097-CLOSE-006 — No disposable-environment ICP refill effect journey

Refill replay, policy, state transitions, cost permits, overflow, and recovery
are extensively unit tested, but no maintained PocketIC or disposable local
ICP test executes the real ledger transfer and CMC notification adapters end
to end. This is a bounded evidence gap, not a confirmed product defect. Adding
a fake production path would be worse; a future operational test should use
real disposable system-canister behavior.

### P2 — CANIC-097-CLOSE-007 — Build-purpose validation is lexical

`validate_build_script_purpose` counts trimmed source lines matching one exact
`canic::build!(...);` spelling. A matching line inside a raw string can satisfy
the check, while an equivalent multiline invocation is rejected. Ordinary
bypass still fails later because required generated config environment is
missing, so no concrete released artifact bypass was found. Replace this only
with a small syntax-aware check when build-script admission is next changed.

### P2 — CANIC-097-CLOSE-008 — Passive resolution rereads mutable config

`resolve_project_state_manifest` parses each config, then
`validate_declared_role_package` and
`resolve_declared_role_package_contract` read it again. A concurrent local edit
can combine graph evidence and contract requirements from different snapshots.
Authoritative build uses the existing from-config path and is not affected;
the passive path fails closed or reports inconsistent diagnostics. A future
bounded change can pass one parsed snapshot through the passive call chain.

## Hard-Cut And Stale-Path Search

The final active-source search found:

- no human-facing `canic::cdk` use or re-export;
- only the intended hidden `canic::__internal::cdk` macro boundary and
  internal `canic_core::cdk` implementation imports;
- no generic refill endpoint emitter, feature, role capability, arbitrary
  target, automatic refill timer, or legacy nested/enabled config reader;
- no deleted manifest resolver, error DTO, cache, re-export, or caller;
- no old `metrics_feature_enabled` build-dependency projection; and
- one canonical `RoleCargoGraphEvidence` producer consumed by the maintained
  build, Medic, state, internal-test, and release paths.

Occurrences of `root_manifest_path` that remain are local variable/parameter
names for an already selected workspace manifest, not the deleted role
resolver. `icp-refill:v1` remains only in durable operation identity strings;
it is not a feature or compatibility path.

### Incomplete Or Environment-Blocked

`cargo test --workspace` reaches one unrelated pre-existing guard failure:
`canic-core/tests/stable_memory_abi_guard.rs` flags raw
`MemoryManager::init` calls in the test-only
`storage/stable/receipt_capacity_tests.rs`. Both the guard and flagged lines
are unchanged from `v0.96.8`; this audit did not broaden 0.97 to repair old
test infrastructure.

The first `cargo test --workspace --exclude canic-core` attempt reached
`canic-tests/tests/root_suite.rs` with the host's unsupported
`ic-wasm 0.10.0`. The audit installed the repository checksum-bound 0.9.11
binary under `/tmp` without replacing the host tool, then reran the suite
outside the network sandbox so PocketIC could bind loopback. All 28 root
journeys passed in 404.45 seconds.

Follow-up on 2026-07-22 upgraded the repository contract itself to
`ic-wasm 0.10.0` using DFINITY's checksum-bound official release archives.
Canic invokes only the maintained CLI commands, so the upstream breaking
change to the Rust library's `walrus` re-export does not apply. The installer,
exact-version gate, release-integrity guard, artifact-I/O tests, and the
four-case role-attestation PocketIC suite pass with the upgraded binary. A
clean parallel root-matrix run passed 27 of 28 cases; the sole delegated-auth
case passed immediately in isolation with the identical transformed artifacts.
The deterministic single-thread matrix then passed all 28 cases in 392.69
seconds, so no `ic-wasm` command or output incompatibility remains.

## Scope And Contract Impact

The audit corrections remain inside 0.97 authority, recovery, diagnostics,
and generated-surface requirements. They add no speculative framework or
compatibility layer.

- **Stable schema/state:** unchanged.
- **Candid:** DTO and wire types are unchanged. Generated role service surface
  regains `canic_metrics` where the selected `metrics` feature requires it.
- **JSON/configuration/CLI:** unchanged.
- **Dependencies and `Cargo.lock`:** unchanged by audit corrections.
- **Package versions:** unchanged.
- **Public Rust API:** the consumerless manifest lookup helpers are removed as
  the required hard cut of an accidental competing authority. No replacement
  or alias exists.

## Closeout Recommendation

Do not describe immutable `v0.97.3` as closed. Keep 0.98 product cleanup
separate from these correction findings even if the next release transaction
contains both sets of commits.

0.97 can close only after:

1. the five corrected P1s are reviewed and included in an immutable release;
2. strict formatting, workspace check/Clippy, layering/governance, focused
   host/core/Canic tests, `pic_role_attestation`, and the 28-case root
   PocketIC suite pass at that anchor; and
3. status and release-line evidence identify the corrected immutable commit
   rather than retroactively claiming that `v0.97.3` passed.

The three P2s are documented follow-ups and do not require compatibility code
or a general cleanup extension.
