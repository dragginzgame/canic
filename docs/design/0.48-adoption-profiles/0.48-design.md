# 0.48 Design: Adoption Profiles

## Status

TBD.

This is a placeholder for post-0.46 design work. It captures adoption paths
that are important for real projects but should not expand the 0.41 through
0.46 deployment truth roadmap.

---

## Tentative Scope

Potential adoption profiles:

- brownfield IC project adoption;
- partial fleet adoption;
- standalone or leaf-only single-canister mode;
- hybrid Rust/non-Rust fleets;
- external wasm roles with digest-pinned artifacts;
- split root/fleet adoption for existing canisters;
- non-destructive pool registration;
- destructive pool import dry-run and confirmation;
- feature matrix for root, hub, leaf, and standalone crates;
- migration guide from legacy `dfx` or `icp` projects.

## Boundary

0.48 should not weaken 0.41's truth model.

Brownfield adoption starts with inventory and safety reports. Mutation remains
blocked until authority reconciliation proves it is safe.
