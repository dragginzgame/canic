## Core Architecture

- [x] Add hierarchical funding path (`child -> parent -> deposit_cycles(child)`).
- [x] Enforce parent-only funding invariant for all child top-ups.
- [x] Remove root fallback paths from child top-up execution.
- [x] Add `APP_STATE.cycles_funding_enabled` global kill switch.

- [ ] Ensure top-up orchestration stays in workflow (no policy/ops layering violations).
- [ ] Keep deterministic single-step side effects in ops only.

- [ ] Remove funding-source policy toggles from top-up config.
- [ ] Keep top-up config focused on limits (`threshold`, `amount`, caps, cooldown).

- [x] Enforce config invariant: `topup.amount * 2 < topup.threshold`.
- [x] Add policy limits (`max_per_request`, `max_per_child`, `cooldown_secs`).

## Parent Funding Endpoint

- [ ] Define parent funding endpoint contract (`canic_request_cycles_v1` or equivalent).
- [x] Validate caller is a direct child.
- [ ] Validate replay metadata (`request_id`, `issued_at`, `ttl`).
- [x] Enforce order: replay guard -> auth -> kill switch -> policy -> execution.
- [x] Return `Unavailable` with message `cycles funding disabled` when switch is off.
- [x] Validate requested amount against policy limits before execution.

## Replay and Idempotency

- [x] Ensure duplicate-same requests are idempotent for funding operations.
- [x] Ensure duplicate-different requests fail closed.
- [x] Ensure replay state does not commit on policy-denied requests.
- [x] Ensure replay persistence behavior cannot cause false-negative execution failures.
- [x] Ensure replay guard still runs when global kill switch is disabled.
