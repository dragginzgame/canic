# Canic Delegated Auth Lifecycle 0.28 Status

- **Design:** [`0.28-design.md`](0.28-design.md)
- **Scope:** implementation state, verification, and known blockers for the `0.28`
  delegated auth lifecycle design

## Completed

- signer prewarm no longer returns early for reusable local proof
- non-root init and post-upgrade both run signer auth prewarm before ready
- PocketIC regression coverage has been added for the lifecycle gap: a verifier
  is reinstalled to lose its local proof cache, signer post-upgrade prewarm runs,
  and the verifier must accept tokens again after root fanout
- demo canister hooks are not part of the repair model
- `DelegationRequest` no longer carries caller-controlled verifier targets
- root derives verifier targets from `DelegationAudience` and registry
- replay hashing no longer includes caller-supplied verifier targets
- root execution fails closed when required verifier fanout is missing or failed
- signer-side verifier target derivation/validation has been removed
- `include_root_verifier` has been removed from canonical `DelegationRequest`
- root always caches its own verifier proof for issued delegation proofs
- unused `DelegationProvisionRequest` has been removed
- signer-target fanout has been removed from canonical `DelegationWorkflow::provision`
- obsolete root-to-signer proof push endpoint `canic_delegation_set_signer_proof`
  has been removed
- delegation fanout responses and metrics are verifier-only; the dead signer
  target kind and signer push metric branches have been removed
- explicit admin verifier fanout is repair-only; the unused admin prewarm branch
  has been removed now that lifecycle prewarm uses canonical root issuance
- verifier-target derivation now returns a typed error instead of a sentinel role
- stale RPC errors for caller-supplied verifier targets have been removed

## Remaining Work

No remaining cleanup items are known for the signed-off `0.28` delegated auth
lifecycle design.

- verifier proof cache loss after reinstall produces a local proof miss
