# 0.47 Design: Wasm Store Artifact Registry And Retention

## Status

TBD.

This is a placeholder for post-0.46 design work. It is intentionally not part
of the 0.41 through 0.46 deployment truth roadmap.

0.41 through 0.46 make `wasm_store` observable deployment infrastructure:

- visible in plan, inventory, artifact manifest, and diff;
- typed as both `ArtifactRole::WasmStore` and possible
  `ArtifactTransport::WasmStore`;
- digest-checked;
- receipted through staging;
- comparable as artifact availability evidence.

0.47 is the possible line where `wasm_store` becomes a provenance-rich artifact
registry.

---

## Tentative Scope

Potential capabilities:

- content-addressed artifact catalog;
- artifact provenance metadata;
- source/build identity storage;
- canonical embedded config metadata stored with artifacts;
- receipt and deployment-group pinning;
- rollback artifact selection;
- cross-deployment cache reuse;
- target trust-domain artifact availability reports;
- deployment-aware retention and GC;
- migration from template/version addressing to digest-addressed artifacts.

---

## Boundary

`wasm_store` availability remains artifact evidence.

Live inventory remains deployment truth.

0.47 must not make the store a deployment authority source.
