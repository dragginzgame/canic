# 0.60 Design: NNS Subnet Inspection

## Status

- Status: Tentative
- Line: `0.60.x`
- Theme: host-side IC NNS subnet catalog, inspection CLI, and optional estimate
  resolver source

0.60 is the NNS inspection line. It adds operator-visible IC network subnet
metadata without changing canister runtime perf accounting, cycle funding, or
deployment mutation semantics.

0.59 owns the instruction/cycle terminology hard cut and offline estimate
math:

```text
docs/design/0.59-instruction-accounting-cost-estimates/0.59-design.md
```

0.60 owns the work intentionally deferred from 0.59:

- local NNS subnet catalog;
- cached canister-ID-to-IC-subnet resolution;
- `canic subnet network list`;
- `canic subnet network info`;
- `canic subnet network refresh`;
- stale-cache policy and registry provenance;
- optional use of the catalog as an estimate source.

## Problem

Operators need a local, scriptable way to inspect where deployed IC canisters
live in the IC subnet topology and what subnet metadata affects pricing
interpretation. That is a real operator feature, not a prerequisite for the
0.59 instruction audit.

The feature must keep three boundaries clear:

- IC network subnets from the NNS registry are not Canic logical fleet subnets.
- Cached NNS metadata is operator observation state, not deployment-owned
  truth.
- NNS metadata can decorate cost estimates, but it must not change measured
  instruction rows or runtime perf counters.

## Relationship To 0.59

0.59 estimates require explicit offline inputs:

```text
--estimate-node-count <n>
--cycles-per-billion-instructions <cycles>
```

0.60 may add a third source:

```text
subnet_source = "nns-registry-cache"
```

That source is optional. Instruction audits without estimates remain
network-free. Estimate generation must not perform live NNS lookups by default;
it may read a fresh local catalog, refresh only when explicitly requested, or
fall back to explicit flags.

## In Scope

- Host-side local catalog of IC NNS subnet metadata.
- Cached routing-range resolver from canister principal to IC subnet principal.
- Subnet metadata records with node count, subnet type, geographic scope, and
  charge applicability.
- Read-only `canic subnet network list` and
  `canic subnet network info <x>` commands.
- Explicit `canic subnet network refresh` command for live mainnet registry
  reads.
- Atomic catalog writes with previous-cache preservation on refresh failure.
- Deterministic stale-cache policy.
- JSON and text output suitable for operator scripts.
- Optional integration with instruction-audit `execution_cycle_estimate`
  artifacts.

## Out Of Scope

- Canister runtime calls to the NNS registry for estimate decoration.
- Exact cycle-bill reconstruction.
- Deployment, controller, canister, cycle-balance, or Canic runtime mutation.
- A new `canic cycles ...` profiling namespace.
- Custom-network registry refresh in the first 0.60 release. Custom networks
  continue to use explicit rate inputs until a separate custom-registry design
  exists.
- Treating cached NNS metadata as authority for deployment truth.

## Catalog Location

The local catalog belongs under the current ICP root and selected IC network,
but outside deployment-owned state:

```text
<icp_root>/.canic/ic/network-subnet-catalog.json
```

It must not live under:

```text
<icp_root>/.canic/ic/deployments/
<icp_root>/.canic/ic/fleets/
```

The catalog is host/operator observation state. Backup/restore and deployment
truth flows may read it in future slices, but they must not require it as
deployment authority.

## Catalog Shape

Catalog records must include a schema version. Canic must reject unknown future
catalog schema versions unless a migration exists.

```text
catalog_schema_version
network = "ic"
registry_canister_id
registry_version
fetched_at
fetched_by
source_endpoint
resolver_backend
subnets[] = {
  subnet_principal
  subnet_type
  geographic_scope
  subnet_label
  node_count
  charges_apply_by_default
}
routing_ranges[] = {
  start_canister_id
  end_canister_id
  subnet_principal
}
```

`subnet_type` values:

```text
application
fiduciary
system
unknown
```

`geographic_scope` values:

```text
global
europe
unknown
```

European subnets are application-style charged subnets with
`geographic_scope = europe`, not a separate billing type. This preserves their
application pricing behavior while still exposing the data-residency property
operators care about.

## Stale Cache Policy

Default freshness:

```text
default_stale_after = 7d
```

Commands may override this with:

```text
--stale-after <duration>
--allow-stale-subnet-catalog
```

Behavior:

- `list` and `info` show stale status and can display stale cached data.
- Estimate-producing commands omit catalog-derived estimates by default when
  the catalog is stale.
- `--allow-stale-subnet-catalog` permits estimates from stale data and records
  `catalog_stale = true`.
- `--refresh-subnet-catalog` or `list --refresh` refreshes before resolving.
- A future strict flag may fail instead of omitting estimates, but omission is
  the default safe behavior.

## Resolver

The cached resolver maps:

```text
deployment target -> canister principal -> cached routing range -> IC subnet
principal -> cached node count/type -> rate table
```

`canic subnet network info <x>` accepts either an IC subnet principal or a
canister principal. Resolution is deterministic:

1. if `<x>` exactly matches a `subnet_principal` in the local catalog, resolve
   it as a subnet;
2. else, if `<x>` falls within a cached routing range, resolve it as a
   canister principal;
3. else, return `NotFound` and suggest `canic subnet network refresh`.

The command may accept:

```text
--as subnet
--as canister
```

to force interpretation in tests and in ambiguous synthetic-principal cases.

System subnet results must not blindly apply application-subnet pricing. Record
`subnet_type = "system"` and mark charge applicability as not applicable where
cycles charges do not apply.

## Refresh

`canic subnet network refresh` performs the live mainnet NNS registry fetch and
rewrites the catalog atomically. Network failure leaves the previous catalog
intact and reports the failure clearly.

Refresh steps:

1. query the NNS registry canister;
2. fetch the current routing table and subnet records;
3. read subnet membership;
4. count active nodes;
5. read subnet type when available;
6. decode into an in-memory catalog;
7. validate non-empty subnet list and routing table;
8. write to a temp file;
9. fsync the temp file when supported;
10. atomically rename over the old catalog;
11. keep the previous catalog on any failure.

Useful flags:

```text
--format text|json
--output <path>
--dry-run
```

0.60 supports mainnet NNS refresh first. Custom networks require explicit rate
or explicit node count until a later custom-registry slice.

## CLI Surface

Do not add a public `canic cycles ...` profiling command for this line. That
namespace means real cycle balances, minting, transfer, top-up, and ICP
conversion.

Add a read-only subnet catalog namespace:

```text
canic subnet network list
canic subnet network info <subnet-principal-or-canister-principal>
canic subnet network refresh
```

The `network` segment is intentional:

```text
canic subnet network ...   # IC network subnets from the NNS registry
canic subnet fleet ...     # reserved for possible future Canic fleet topology
```

This namespace reads and refreshes IC network subnet metadata from the local
NNS subnet catalog. It never mutates deployments, canisters, controllers, cycle
balances, role attachments, or Canic runtime state.

`canic subnet network list` should default to cached catalog data and show:

```text
SUBNET       TYPE         GEO     NODES  CHARGES  RANGES  VERSION  FETCHED_AT  STALE
<principal>  application  global  13     yes      4       123456   2026-...    no
<principal>  application  europe  13     yes      2       123456   2026-...    no
<principal>  fiduciary    global  34     yes      1       123456   2026-...    no
<principal>  system       global  13     no       2       123456   2026-...    no
```

Useful list flags:

```text
--refresh
--type application|fiduciary|system|unknown
--geo global|europe|unknown
--format text|json
--show-ranges
--range-limit <n>
--range-offset <n>
--stale-after <duration>
--allow-stale-subnet-catalog
```

Default list output must show only a routing-range count. `--show-ranges`
enables range rows, capped by `--range-limit` in text mode. If more ranges are
available, text output should say:

```text
showing 50 of 312 ranges; use --range-limit or --format json
```

`canic subnet network info <x>` should show:

```text
input_principal
resolved_as
subnet_principal
subnet_type
geographic_scope
node_count
charges_apply_to_subject
charge_applicability_reason
registry_canister_id
registry_version
catalog_schema_version
fetched_at
catalog_stale
resolver_backend
matched_canister_principal
matched_routing_range
cycles_per_billion_instructions
rate_source
formula_version
```

## Estimate Integration

When the catalog is used as an estimate source, reports record:

```text
subnet_source = "nns-registry-cache"
registry_canister_id = <nns-registry-principal>
registry_version = <version>
subnet_principal = <resolved-subnet-principal>
subnet_node_count = <cached-node-count>
subnet_type = <cached-subnet-type>
geographic_scope = <cached-geographic-scope>
catalog_schema_version = <version>
catalog_stale = false
resolver_backend = "local-nns-subnet-catalog"
```

If the catalog is stale, missing, or cannot map the canister ID to a subnet,
the estimate should be omitted unless the caller supplies an explicit fallback
or asks Canic to refresh the catalog. The instruction report itself should
still complete.

Explicit rate and explicit node count remain valid. Explicit rate wins over
catalog-derived rate.

## Layering

- `canic-core::perf` remains instruction-only instrumentation.
- Runtime metric DTOs remain instruction DTOs.
- Catalog DTOs and resolver output live in host/operator code, not canister
  runtime code.
- Live NNS calls live behind host-side infra/ops boundaries.
- CLI adapters parse commands and delegate immediately.
- Estimate decoration may read the catalog, but measured instruction rows do
  not depend on the catalog.

This keeps runtime layering intact and prevents a reporting feature from
creating canister-side registry dependencies.

## Build Order

1. Add catalog data shapes, parser/validator, schema-version rejection, and
   fixture tests.
2. Add cached resolver over routing ranges and subnet records.
3. Add `canic subnet network list` and `info` over cached data.
4. Add stale-cache evaluation and output fields.
5. Add live mainnet NNS refresh with atomic write behavior.
6. Wire catalog-derived estimates as an optional source for instruction-audit
   reports.
7. Run an operator proof that refreshes, lists, resolves a known canister, and
   produces an estimate artifact without changing instruction rows.

## Proposed Release Shape

### 0.60.0 - NNS Subnet Inspection

Ship the smallest useful operator surface:

- local catalog schema and cache path;
- live mainnet refresh;
- cached `list` and `info`;
- stale-cache status;
- JSON/text output;
- fixture and atomic-write tests.

### 0.60.1 - Estimate Catalog Source

Wire the catalog into instruction-audit estimates only after the inspection
surface is stable. This keeps the operator catalog product from blocking 0.59
offline estimates.

## Acceptance Criteria

- `canic subnet network list`, `info`, and `refresh` are read-only with respect
  to deployments, canisters, controllers, cycle balances, and runtime state.
- The catalog lives under `<icp_root>/.canic/ic/network-subnet-catalog.json`
  and not under deployment-owned state.
- Catalog records include `catalog_schema_version` and reject unknown future
  versions unless migrated.
- `info <canister-principal>` resolves through cached routing ranges to the
  same subnet metadata the estimate resolver would use.
- Stale status is deterministic and visible in text and JSON output.
- Refresh failure keeps the previous catalog intact.
- Range output is capped by default in text mode.
- European subnets preserve application-pricing behavior while exposing
  `geographic_scope = europe`.
- System subnet results mark charge applicability as not applicable.
- Instruction reports remain complete when catalog lookup fails.
- Instruction reports without estimates do not read the catalog.
- Catalog-derived estimate artifacts record subnet, registry, catalog, and
  resolver provenance.

## Required Tests

- Catalog schema round-trips through stable JSON.
- Unknown future catalog schema version is rejected.
- Empty subnet list or empty routing table is rejected on refresh.
- Cached resolver maps a canister principal through routing ranges to subnet
  principal, node count, subnet type, and geographic scope.
- `info <x>` resolves `<x>` deterministically as subnet, canister, or unknown.
- `list --format json` includes registry version, fetched-at, subnet
  principal, subnet type, geographic scope, node count, charge applicability,
  stale status, and rate metadata.
- `list` text output caps routing ranges unless `--show-ranges` is supplied.
- `refresh` writes a temp file and preserves the previous catalog on failure.
- Missing or stale catalog omits estimates unless explicit fallback or stale
  allowance is supplied.
- Successful catalog-derived estimate records registry version, subnet
  principal, catalog schema version, and resolver backend.
- Failed live lookup leaves the instruction report complete.
- No `canic cycles ...` command is added for profiling or inspection.
