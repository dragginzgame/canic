Date: 2026-07-06

```text
schema_version: 1
```

## Slice Contract

Boundary:
Wasm-store endpoint macros use `WasmStoreCanisterApi` as the canonical
endpoint-facing facade for local store operations, and
`WasmStorePublicationApi::admin` as the canonical root publication mutation
facade. Broader local-store helpers and direct publication helper methods
should not be exported as public Rust APIs.

Current owner:
`canic::api::canister::template` re-exported
`WasmStoreApi` and the alias `EmbeddedTemplateApi`, and
`canic-control-plane::api::template::WasmStoreApi` was public even though it was
only used inside the same module. `WasmStorePublicationApi` also exposed direct
helper methods that duplicated the typed admin command path.

Intended owner:
`WasmStoreCanisterApi` owns the public wasm-store canister API surface used by
endpoint macros. `WasmStorePublicationApi::admin` owns root publication
mutations through `WasmStoreAdminCommand`. Local and direct helper facades are
private implementation details.

Files/modules in scope:

- `crates/canic/src/api/mod.rs`
- `crates/canic-control-plane/src/api/template/mod.rs`

Files/modules explicitly out of scope:

- Wasm-store endpoint method names.
- Wasm-store Candid request/response shapes.
- Wasm-store storage behavior.
- Wasm-store GC behavior.
- Root bootstrap APIs.
- Root publication admin/overview endpoint behavior.
- CLI, JSON, deployment-truth, stable-state, or evidence/report schemas.

Public surfaces affected:
Rust API paths `canic::api::canister::template::EmbeddedTemplateApi`,
`canic::api::canister::template::WasmStoreApi`, and
`canic_control_plane::api::template::WasmStoreApi` are removed. Direct
`WasmStorePublicationApi` helper methods for individual publication actions are
also removed in favor of `WasmStorePublicationApi::admin`. This is an
intentional pre-1.0 hard cut for unused compatibility/helper surfaces.

Serialized surfaces affected:
None.

Commands/endpoints affected:
None.

Expected behavior change:
None for operator commands, endpoint behavior, Candid, JSON, stable state,
deployment truth, or evidence/report schemas.

Expected non-changes:

- Wasm-store endpoints continue to call `WasmStoreCanisterApi`.
- Root bootstrap/publication endpoints keep using their canonical APIs.
- Root publication mutations still flow through `WasmStoreAdminCommand`.
- Local template catalog, status, chunk staging, chunk retrieval, and GC
  behavior remain unchanged.

Validation:

- `cargo fmt --all` passed.
- `cargo test --locked -p canic-control-plane --lib` passed.
- `cargo test --locked -p canic --lib` passed.
- `cargo test --locked -p canic --features wasm-store-canister --lib` passed.
- `cargo test --locked -p canic --features control-plane --lib` passed.
- `cargo clippy --locked -p canic-control-plane --tests -- -D warnings`
  passed.
- `cargo clippy --locked -p canic --tests -- -D warnings` passed.
- Removed-facade `rg` checks over active code returned no matches.
- `cargo fmt --all -- --check` passed.
- `cargo test --locked -p canic --test changelog_governance` passed.
- `git diff --check` passed.

Rollback plan:
Restore the `WasmStoreApi` public struct name and the `canic` facade
re-exports, then move `WasmStoreCanisterApi` calls back through that facade.
Restore direct `WasmStorePublicationApi` helper methods and their support
wrappers if the Rust facade needs those methods again.

Follow-ups:

- Keep endpoint-facing API names specific to their endpoint role.
- Keep local helper facades private unless they are intentionally part of the
  public runtime API.
- Keep publication mutations on the typed admin command unless a separate Rust
  API surface is intentionally designed.

## Findings

### CANIC-082-WASM-STORE-API-FACADE-001

Severity: P3
Category: legacy_surface
Owner: `canic::api::canister::template`
Current location: `crates/canic/src/api/mod.rs`
Intended location: `WasmStoreCanisterApi`

Evidence:
`EmbeddedTemplateApi` and the public `WasmStoreApi` re-export were not used by
endpoint macros, tests, docs, or canisters. Endpoint macros use
`WasmStoreCanisterApi`.

Risk:
The alias and broad helper facade made two public Rust names appear to own the
same wasm-store canister surface, increasing pre-1.0 API confusion.

Decision:
Removed both public re-exports from the `canic` facade. The endpoint-facing
public name remains `WasmStoreCanisterApi`.

Regression test:
`canic` library tests and clippy compile the endpoint macro facade after the
hard cut.

### CANIC-082-WASM-STORE-API-FACADE-002

Severity: P3
Category: visibility
Owner: `canic-control-plane::api::template`
Current location: `crates/canic-control-plane/src/api/template/mod.rs`
Intended location: private local helper

Evidence:
`WasmStoreApi` was public, but only `WasmStoreCanisterApi` called it inside the
same module.

Risk:
Public helper visibility made implementation-sharing code look like a supported
runtime API.

Decision:
Renamed the helper to private `LocalWasmStoreApi` and narrowed its methods to
module-private functions.

Regression test:
`canic-control-plane` library tests and clippy validate the private helper and
public endpoint facade after the visibility tightening.

### CANIC-082-WASM-STORE-API-FACADE-003

Severity: P3
Category: legacy_surface
Owner: `canic-control-plane::api::template::WasmStorePublicationApi`
Current location: `crates/canic-control-plane/src/api/template/mod.rs`
Intended location: `WasmStorePublicationApi::admin`

Evidence:
`WasmStorePublicationApi` exposed direct Rust helpers for individual
publication and retired-store GC actions, but the endpoint macro exposes the
typed `canic_wasm_store_admin` command and `canic_wasm_store_overview` query.
No active code called the direct helper methods.

Risk:
The extra Rust helper surface duplicated the admin command vocabulary and made
there appear to be multiple supported root publication mutation facades.

Decision:
Removed the direct publication helper methods from `WasmStorePublicationApi`
and removed the now-unused crate-private support wrappers. The typed
`WasmStoreAdminCommand` path remains the canonical mutation surface.

Regression test:
Control-plane and `canic` facade tests compile the endpoint-facing admin and
overview paths after the direct helper removal.

## Fixes

- Removed `EmbeddedTemplateApi` from `canic::api::canister::template`.
- Removed public `WasmStoreApi` re-export from `canic::api::canister::template`.
- Renamed the control-plane helper to private `LocalWasmStoreApi`.
- Narrowed helper methods to module-private functions.
- Kept `WasmStoreCanisterApi` as the endpoint-facing public wasm-store facade.
- Removed direct root publication helper methods from `WasmStorePublicationApi`
  and kept the typed admin command as the canonical mutation facade.
- Removed crate-private support wrappers that were only used by those direct
  publication helper methods.

## Deferred

- No endpoint, Candid, storage, or GC behavior changes were made.
- Root bootstrap APIs were not changed.
- Root publication command behavior remains owned by `WasmStoreAdminCommand`.
