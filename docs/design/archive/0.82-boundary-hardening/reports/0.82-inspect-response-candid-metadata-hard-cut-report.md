## Slice Contract

Boundary:
`canic inspect` runtime-status report wrapper metadata for decoded
`canic_runtime_status` responses.

Current owner:
`canic-cli::inspect`, with typed `response_bytes` decoding plus redundant
response-wrapper presence metadata in text and JSON reports.

Intended owner:
`canic-cli::inspect` requires typed `response_bytes` for decoding and reports
only the stable response format needed to explain the decoded runtime payload.

Files/modules in scope:

- `crates/canic-cli/src/inspect/mod.rs`

Files/modules explicitly out of scope:

- `canic_runtime_status` endpoint method name and Candid signature.
- Runtime introspection DTOs.
- Inspect command names, options, target resolution, or endpoint guards.
- Deployment truth, evidence, stable-state layout, or generated artifacts.

Public surfaces affected:
`canic inspect` text and JSON reports no longer include
`response_candid_present` or `response_bytes_present` under `runtime_status`.

Serialized surfaces affected:
The inspect report wrapper JSON shape intentionally removes the
`runtime_status.response_candid_present` and
`runtime_status.response_bytes_present` fields. Runtime endpoint DTOs and
Candid shapes are unchanged.

Commands/endpoints affected:
No command or endpoint name changes.

Expected behavior change:
Inspect reports no longer expose metadata about whether an old
`response_candid` wrapper field was present, or redundant metadata stating that
`response_bytes` was present after successful decoding. The report continues to
require typed `response_bytes` and still exposes `response_format`.

Expected non-changes:

- `canic inspect` still decodes `canic_runtime_status` from typed Candid
  response bytes.
- Missing or invalid `response_bytes` remains an invalid response.
- Source attribution remains `runtime_observed`.
- Runtime status text/JSON sections remain otherwise unchanged.
- No compatibility fallback, alias, or shim is retained.

Validation:

- `cargo fmt --all` passed.
- `cargo test --locked -p canic-cli inspect` passed.
- `cargo clippy --locked -p canic-cli --tests -- -D warnings` passed.
- `cargo fmt --all -- --check` passed.
- `rg -n "response_candid_present|response_bytes_present|response_candid|response_text|fallback" crates/canic-cli/src/inspect/mod.rs` returned no matches.
- `git diff --check` passed.

Rollback plan:
Restore `RuntimeStatusPayload::response_candid_present` and
`RuntimeStatusPayload::response_bytes_present`, then render them in text and
JSON reports. This is not recommended unless Canic intentionally reintroduces
response-wrapper presence metadata as a supported inspect report field.

Follow-ups:

- Keep `canic inspect` runtime decoding on typed response bytes.
- Do not add anti-resurrection tests for old fallback metadata unless the
  maintainer explicitly requests them.

## Findings

### CANIC-082-INSPECT-RESPONSE-CANDID-METADATA-001

Severity: P2
Category: cli_surface
Owner: `canic-cli::inspect`
Current location: `crates/canic-cli/src/inspect/mod.rs`
Intended location: stable response-format attribution in the inspect report wrapper

Evidence:
`canic inspect` already rejected runtime-status responses without
`response_bytes`, but its text and JSON report wrapper still exposed
`response_candid_present` and `response_bytes_present` metadata.

Risk:
The report carried response-wrapper presence vocabulary after old text fallback
parsing had been hard-cut, leaving old response-shape semantics and redundant
decode-state metadata visible in an operator-facing automation surface.

Decision:
Fixed with a hard cut. Inspect reports now expose only typed-byte response
format attribution for runtime-status decoding.

Regression test:
Inspect report text and JSON tests now assert `response_format` without
response-wrapper presence fields, and missing `response_bytes` still fails.

## Fixes

- Removed `response_candid_present` from `RuntimeStatusPayload`.
- Removed `response_bytes_present` from `RuntimeStatusPayload`.
- Removed response-Candid presence detection from inspect response parsing.
- Removed response-wrapper presence fields from text and JSON inspect reports.
- Updated inspect tests and samples to avoid old fallback vocabulary.

## Deferred

- No old-field compatibility tests were added; this is an intentional hard cut,
  not a compatibility surface.
