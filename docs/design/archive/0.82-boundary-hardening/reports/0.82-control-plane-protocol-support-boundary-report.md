## Findings

### CANIC-082-CONTROL-PLANE-PROTOCOL-SUPPORT-001

Severity: P3
Category: support_boundary
Owner: `canic_core::protocol`
Current location: `control_plane_support::protocol`
Intended location: direct public protocol owner imports

Evidence:
`control_plane_support::protocol` re-exported public wasm-store endpoint names
and method lists from `canic_core::protocol`. Control-plane users only needed
those already-public constants for the typed wasm-store client and its protocol
manifest tests.

Risk:
A hidden support mirror makes `control_plane_support` look like an alternate
protocol API and can grow into a compatibility layer for endpoint names that
should be hard-cut at the canonical protocol owner.

Decision:
Fixed with a no-behavior-change support-boundary cleanup. Control-plane code
now imports protocol constants directly from `canic_core::protocol`, and the
hidden protocol support mirror was removed.

Regression test:
Covered by the wasm-store client protocol manifest tests, control-plane tests,
and clippy. No behavior-specific regression is required because protocol names
and endpoint classifications are unchanged.
