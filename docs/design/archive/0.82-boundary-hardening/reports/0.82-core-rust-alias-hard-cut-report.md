Date: 2026-07-07

schema_version: 1

## Slice Contract

Boundary:
Core helper and infra modules should expose named protocol DTOs and concrete
result/byte shapes. Local public aliases that are only used inside their
defining modules should not remain as pre-1.0 Rust namespace.

Current owner:
`canic-core::cdk::spec::standards::icrc::icrc21` defined `ConsentResult` as a
public alias for `Result<ConsentInfo, Icrc21Error>`.
`canic-core::cdk::utils::hash` defined `HashBytes` as a public alias for
`Vec<u8>`. `canic-core::infra::ic::nns::registry` defined
`GetSubnetForCanisterResponse` as a public alias for
`Result<GetSubnetForCanisterPayload, String>`.
`canic-core::infra::ic::icp_refill` defined `Icrc1TransferResult` and
`NotifyTopUpResult` as public aliases for raw ledger/CMC result shapes that
were imported by ops.

Intended owner:
`ConsentMessageResponse` remains the named ICRC-21 response DTO. Hash helpers
return `Vec<u8>` directly. NNS registry decoding uses the concrete Candid
`Result<GetSubnetForCanisterPayload, String>` shape directly. ICP refill infra
and ops use the concrete raw ledger/CMC result shapes directly.

Files/modules in scope:

Rollback plan:
Restore an alias only if the maintainer intentionally chooses to keep that Rust
namespace before 1.0. Do not add compatibility aliases by default.

Follow-ups:

- Continue auditing public Rust type aliases and keep only aliases with a clear
  semantic owner or ergonomics value.
- Do not collapse named Candid DTOs into aliases.

## Findings

### CANIC-082-CORE-ALIAS-HARDCUT-001

Severity: P3
Category: dto_boundary
Owner: `canic-core::cdk::spec::standards::icrc::icrc21`
Current location: `crates/canic-core/src/cdk/spec/standards/icrc/icrc21/mod.rs`
Intended location: named ICRC-21 DTOs and concrete result conversion

Evidence:
`ConsentResult` was a public alias for `Result<ConsentInfo, Icrc21Error>` and
was only used by the local `From` implementation for `ConsentMessageResponse`.

Risk:
The alias added a public Rust namespace without owning a distinct protocol
concept, making the ICRC-21 boundary noisier before 1.0.

Decision:
Removed `ConsentResult` and implemented conversion from
`Result<ConsentInfo, Icrc21Error>` directly.

Regression test:
`cargo test --locked -p canic-core icrc21 --lib`

### CANIC-082-CORE-ALIAS-HARDCUT-002

Severity: P3
Category: utility_boundary
Owner: `canic-core::cdk::utils::hash`
Current location: `crates/canic-core/src/cdk/utils/hash.rs`
Intended location: hash helpers expose concrete byte buffers

Evidence:
`HashBytes` was a public alias for `Vec<u8>` and was only used by hash helper
return types in the defining module.

Risk:
The alias added a public Rust namespace without owning a distinct hash type or
protocol concept.

Decision:
Removed `HashBytes` and returned `Vec<u8>` directly from byte-producing hash
helpers.

Regression test:
`cargo test --locked -p canic-core hash --lib`

### CANIC-082-CORE-ALIAS-HARDCUT-003

Severity: P3
Category: infra_boundary
Owner: `canic-core::infra::ic::nns::registry`
Current location: `crates/canic-core/src/infra/ic/nns/registry.rs`
Intended location: NNS registry adapter decodes the concrete Candid result shape

Evidence:
`GetSubnetForCanisterResponse` was a public alias for
`Result<GetSubnetForCanisterPayload, String>` and was only used by the local
Candid decode call.

Risk:
The alias added a public Rust namespace for an internal infra adapter response
without owning a stable external DTO.

Decision:
Removed the alias and decoded `Result<GetSubnetForCanisterPayload, String>`
directly.

Regression test:
`cargo test --locked -p canic-core nns --lib`

### CANIC-082-CORE-ALIAS-HARDCUT-004

Severity: P3
Category: infra_boundary
Owner: `canic-core::infra::ic::icp_refill`, `canic-core::ops::ic::icp_refill`
Current location: `crates/canic-core/src/infra/ic/icp_refill.rs`,
`crates/canic-core/src/ops/ic/icp_refill.rs`
Intended location: ICP refill infra and ops expose concrete raw result shapes

Evidence:
`Icrc1TransferResult` and `NotifyTopUpResult` were public aliases for
`Result<Nat, TransferError>` and `Result<Nat, NotifyTopUpError>` and were
imported by ops only to preserve shorthand return types.

Risk:
The aliases added public Rust namespace for raw protocol result shapes without
owning distinct named DTOs.

Decision:
Removed the aliases and returned the concrete raw result shapes directly from
infra and ops.

Regression test:
`cargo test --locked -p canic-core icp_refill --lib`
