## Slice Contract

Boundary:
Public `ErrorCode` variants emitted for registry policy errors when replica,
shard, or instance roles require service-owned parent capabilities.

Current owner:
`canic-core::dto::error::ErrorCode`, with policy-to-public-error mapping in
`canic-core::api::error` and direct local-replica decoding in
`canic-host::replica_query::wire`.

Intended owner:
`canic-core::dto::error::ErrorCode`, using current service-owned topology
vocabulary that matches `domain::policy::pure::topology::registry`.

Files/modules in scope:

- `crates/canic-core/src/dto/error.rs`
- `crates/canic-core/src/api/error.rs`
- `crates/canic-core/src/test/seams/registry_policy_seam.rs`
- `crates/canic-host/src/replica_query/wire/mod.rs`
- `crates/canic-wasm-store/wasm_store.did`

Files/modules explicitly out of scope:

- Endpoint routes.

- Deployment truth, evidence/report schemas, or stable-state layout.
- Historical changelog and archived design references.

Public surfaces affected:
The public `ErrorCode` variants for service-parent topology failures are
renamed:

- `PolicyReplicaRequiresSingletonWithScaling` ->
  `PolicyReplicaRequiresServiceWithScaling`
- `PolicyShardRequiresSingletonWithSharding` ->
  `PolicyShardRequiresServiceWithSharding`
- `PolicyInstanceRequiresSingletonWithDirectory` ->
  `PolicyInstanceRequiresServiceWithDirectory`

Serialized surfaces affected:
The Candid `ErrorCode` variant labels in the checked-in wasm-store DID and
runtime public error payloads now use the service-owned names. The old
singleton-named variants are not retained as aliases.

Commands/endpoints affected:
No command or endpoint name changes. Endpoints that return these policy errors
now return the renamed `ErrorCode` variants.

Expected behavior change:
Clients matching policy error codes must use the service-owned variant names.
Old singleton-named policy error codes are removed.

Expected non-changes:

- Registry policy decisions are unchanged.
- Policy error messages are unchanged.
- Direct local-replica query decoding still supports the current public
  `ErrorCode` shape.
- No compatibility aliases, dual decoders, or old-code forwarding paths are
  added.

## Findings

### CANIC-082-POLICY-ERROR-CODE-001

Severity: P2
Category: api_wire_codes
Owner: `canic-core::dto::error`
Current location: `crates/canic-core/src/dto/error.rs`
Intended location: `crates/canic-core::dto::error`

Evidence:
The public `ErrorCode` enum still exposed
`PolicyReplicaRequiresSingletonWithScaling`,
`PolicyShardRequiresSingletonWithSharding`, and
`PolicyInstanceRequiresSingletonWithDirectory`, while the pure policy owner and
public messages use service-parent vocabulary.

Risk:
The public API carried stale topology terms after the model moved to
service-owned manager pools. Keeping those variants would make the pre-1.0
surface harder to cleanly document and support.

Decision:
Fixed with a hard cut. Public policy error codes now use the service-owned
names, and old singleton-named variants are not retained.

Regression test:
`api::error::tests::registry_policy_service_parent_errors_use_service_owned_public_codes`
pins the new public error-code mapping.

## Fixes

- Renamed the three stale public `ErrorCode` variants to service-owned names.
- Updated policy-to-public-error mapping to return the new names.
- Updated host direct local-replica error-code decoding to the new names.
- Updated the checked-in wasm-store DID error-code labels.
- Updated focused registry policy seam expectations.

## Deferred

- No old-code anti-resurrection tests were added; this is an intentional hard
  cut, not a compatibility surface.
- Auth metric mirroring into older Access rows remains for a separate metrics
  hard-cut slice.
- Non-IC root bootstrap fallback was handled in a separate
  lifecycle/bootstrap hard-cut slice.
