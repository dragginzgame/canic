## Findings

### CANIC-082-LIFECYCLE-METRIC-OPS-001

Severity: P2
Category: ops_ownership
Owner: `canic-core::ops::runtime::metrics::lifecycle`
Current location: `crates/canic-core/src/ops/runtime/metrics/lifecycle.rs`
Intended location: `crates/canic-core/src/domain/metrics.rs`

Evidence:
Lifecycle phase, role, stage, and outcome values were defined in the lifecycle
metrics recorder while lifecycle adapters and the lifecycle facade consumed
them as shared metric classification values.

Risk:
Lifecycle metric semantics could drift between lifecycle adapters, facades,
and public metric projection if the values remain owned by the metrics
recorder.

Decision:
Fixed with a no-behavior-change ownership move. The lifecycle metric dimension
values now live in `domain::metrics`, while
`ops::runtime::metrics::lifecycle` and `api::lifecycle::metrics` publicly
re-export them so existing imports remain stable.

Regression test:
Runtime and lifecycle facade metric tests continue to prove lifecycle labels
and metric rows are unchanged.

## Fixes

- Added lifecycle metric dimension values to `domain::metrics`.
- Re-exported the same values from the existing lifecycle metrics path and
  lifecycle metrics facade.

## Deferred

- Lifecycle metric storage remains in runtime metrics ownership.
- Lifecycle init/post-upgrade behavior remains unchanged.
