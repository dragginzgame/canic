## Slice Contract

Boundary:
Compatibility, alias, fallback, and legacy paths discovered by targeted
repo-wide scans after the 0.82 DTO/domain boundary hardening work.

Current owner:
Mixed. The sweep inspected live CLI inspect parsing, replay stable-storage
tests, public error-code mapping, auth runtime metrics, access-expression DSL
helpers, root bootstrap subnet-identity fallback, and wasm-store pruning labels.

Intended owner:
Each compatibility-looking path must either be removed in an owned hard-cut
slice or classified as active behavior that needs a separate explicit
source-breaking decision.

Files/modules in scope:

- `crates/canic-cli/src/inspect/mod.rs`
- `crates/canic-core/src/storage/stable/replay.rs`
- `crates/canic-core/src/state_contract.rs`
- `crates/canic-core/src/api/error.rs`
- `crates/canic-core/src/ops/runtime/metrics/auth`
- `crates/canic-core/src/access/expr/mod.rs`
- `crates/canic-control-plane/src/workflow/bootstrap/root.rs`

Files/modules explicitly out of scope:

- Broad command-surface deletion not owned by this slice.
- Public error-code renaming.
- Public metric-label removal.
- Root bootstrap subnet-identity behavior changes for non-IC environments.
- Wasm-store publication pruning behavior.
- New anti-resurrection tests for old commands.

Public surfaces affected:
`canic inspect` no longer accepts an ICP JSON response that only contains
`response_candid` for `canic_runtime_status`. The command now requires typed
`response_bytes` so the runtime status is decoded from the canonical Candid
bytes.

Serialized surfaces affected:
Valid `canic inspect --json` reports still contain a typed runtime status
object. A response without `response_bytes` now fails, and a later inspect
metadata hard cut removes response-wrapper presence fields from the report
wrapper. The state manifest removed state test declaration now points at the
active shared replay receipt round-trip test rather than a legacy manual root
replay record test.

Commands/endpoints affected:
Only CLI inspect response parsing is affected. Runtime endpoint names, guards,
Candid DTOs, deployment truth, evidence/report schemas, and stable-state active
layout are unchanged.

Expected behavior change:

- Text-only `response_candid` fallback output from `icp canister query` is no
  longer accepted by `canic inspect`.
- The legacy test-only root replay record encoder/decoder is removed.

Expected non-changes:

- `canic inspect` still queries only `canic_runtime_status`.
- Inspect still rejects ambiguous targets, broad deployment fanout, endpoint
  mode flags, and `--format json`.
- Runtime DTO Candid shapes remain unchanged.
- Active replay receipt CBOR storage remains unchanged.
- Removed root replay state remains declared as removed in the state manifest.

Validation:

- `cargo fmt --all` passed.
- `cargo fmt --all -- --check` passed.
- `cargo test --locked -p canic-cli inspect` passed.
- `cargo test --locked -p canic-core storage::stable::replay --lib` passed.
- `cargo test --locked -p canic-core state_contract --lib` passed.
- `cargo test --locked -p canic-core --test candid_serde_boundary_guard` passed.
- `cargo clippy --locked -p canic-cli --tests -- -D warnings` passed.
- `cargo clippy --locked -p canic-core --tests -- -D warnings` passed.

Rollback plan:
Restore the `response_candid` fallback handling in `canic-cli::inspect` and
restore the test-only `RootReplayRecord` encoder/decoder plus its state
manifest test declaration. This is not recommended unless Canic intentionally
reintroduces these compatibility paths.

Follow-ups:

- Public pre-0.64 policy error wire codes were hard-cut in
  `0.82-policy-error-code-hard-cut-report.md`.
- Auth metric mirroring into the older Access metrics family was hard-cut in
  `0.82-auth-metric-mirror-hard-cut-report.md`.
- The non-IC root bootstrap subnet fallback was hard-cut in
  `0.82-root-bootstrap-subnet-identity-hard-cut-report.md`.
- Keep the endpoint DSL `requires(...)` helper; it is an active DSL function,
  not a compatibility shim.

## Findings

### CANIC-082-HARD-CUT-SWEEP-001

Severity: P2
Category: cli_surface
Owner: `canic-cli::inspect`
Current location: `crates/canic-cli/src/inspect/mod.rs`
Intended location: `canic-cli::inspect`

Evidence:
`runtime_response_payload` accepted an ICP JSON object containing only
`response_candid`, produced a `RuntimeStatusPayload` with no typed
`CanicRuntimeStatus`, and added warning/next-action guidance. A later inspect
metadata hard cut removed the remaining response-wrapper presence fields from
valid inspect reports.

Risk:
The inspect command kept an untyped compatibility response shape alive after
0.81 made typed runtime status the canonical inspection surface. That allowed a
successful inspect report without decoding the runtime endpoint DTO.

Decision:
Fixed with a hard cut. `canic inspect` now requires `response_bytes`, decodes
the typed Candid `Result<CanicRuntimeStatus, Error>`, and rejects text-only
`response_candid` fallback output.

Regression test:
`inspect::tests::response_without_response_bytes_is_rejected` pins the typed
response-byte requirement, and
`inspect::tests::decodes_runtime_status_from_response_bytes` continues to cover
the canonical typed path.

### CANIC-082-HARD-CUT-SWEEP-002

Severity: P2
Category: stable_state_tests
Owner: `canic-core::storage::stable::replay`
Current location: `crates/canic-core/src/storage/stable/replay.rs`
Intended location: `canic-core::storage::stable::replay`

Evidence:
`RootReplayRecord` existed only under `cfg(test)` as a "Legacy test-only root
replay record shape" with manual byte encoding/decoding tests. The root state
contract removed-state declaration still referenced one of those legacy tests.

Risk:
The test tree retained a stale stable-state layout after root replay moved to
the shared replay receipt store, which made the removed-state declaration look
as if old manual encoding was still part of current coverage.

Decision:
Fixed with a hard cut. Removed the test-only legacy `RootReplayRecord`
encoder/decoder and pointed the removed root replay state declaration at the
current shared replay receipt CBOR round-trip test.

Regression test:
`storage::stable::replay::tests::replay_receipt_record_round_trips_through_cbor_storable`
covers the active replay receipt storage shape, and
`state_contract::tests::root_manifest_tracks_reserved_core_memory_ids` keeps
the removed-state declaration visible in the manifest.

### CANIC-082-HARD-CUT-SWEEP-003

Severity: P2
Category: api_wire_codes
Owner: `canic-core::api::error`
Current location: `crates/canic-core/src/api/error.rs`
Intended location: separate source-breaking API error-code slice

Evidence:
`registry_policy_error_code` documents that several public wire codes keep
pre-0.64 names for compatibility even though messages now describe
service-owned manager pools.

Risk:
The public error code surface still carries historical naming. Removing it in
this sweep would be a source-breaking API decision with external client impact.

Decision:
Fixed in a later hard-cut slice. See
`0.82-policy-error-code-hard-cut-report.md`.

Regression test:
`api::error::tests::registry_policy_service_parent_errors_use_service_owned_public_codes`
pins the new canonical error-code names.

### CANIC-082-HARD-CUT-SWEEP-004

Severity: P2
Category: metrics_surface
Owner: `canic-core::ops::runtime::metrics::auth`
Current location: `crates/canic-core/src/ops/runtime/metrics/auth`

CLI/stable-test hard-cut sweep.

Decision:
Fixed in a later hard-cut slice. See
`0.82-auth-metric-mirror-hard-cut-report.md`.

Regression test:
Auth metric recorder tests assert the remaining canonical Auth-family
dimensions directly.

### CANIC-082-HARD-CUT-SWEEP-005

Severity: P3
Category: dsl_surface
Owner: `canic-core::access::expr`
Current location: `crates/canic-core/src/access/expr/mod.rs`
Intended location: `canic-core::access::expr`

Evidence:
`requires(...)` is documented as an alias for `all(...)` used by endpoint
declarations.

Risk:
The word "alias" can look like a compatibility shim in broad scans, but the
function is part of the endpoint access-expression DSL and is documented by the
macro README.

Decision:
Keep. This is an active DSL helper, not a legacy compatibility route.

Regression test:
Existing macro/access-expression tests cover the endpoint DSL shape.
