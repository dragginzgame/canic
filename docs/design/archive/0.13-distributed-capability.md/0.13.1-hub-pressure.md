## Objective

Reduce control-plane coupling by decomposing three high-pressure hubs without changing external behavior:

- `crates/canic-core/src/api/rpc.rs`
- `crates/canic-core/src/workflow/rpc/request/handler.rs`
- `crates/canic-core/src/ops/auth.rs`

## Baseline (2026-03-07)

- Complexity Risk Index: `6.55/10`
- Velocity Risk Index: `5.80/10`
- Hub LOC:
  - `workflow/rpc/request/handler.rs`: `1581`
  - `ops/auth.rs`: `1253`
  - `api/rpc.rs`: `900`

## Scope (0.13.1)

1. API boundary thinning
- Keep `RpcApi` as boundary wiring + public error mapping only.
- Move capability envelope/proof/replay decision logic to workflow/ops-owned units.

2. Root request workflow split
- Split handler responsibilities into dedicated units:
  - context extraction/mapping
  - authorization decisions
  - replay checks/commit
  - capability execution dispatch

3. Auth ops split
- Separate concerns into distinct units:
  - signing paths
  - delegated-token verification
  - role-attestation verification
  - key-material/cache helpers

## Out of Scope (0.13.1)

- Public DTO/API contract changes.
- Replay model redesign.
- Policy semantics redesign.

## Delivery Phases

1. PR1: API thinning
- Extract proof/envelope/replay metadata validation from `api/rpc.rs`.
- Keep behavior identical (no contract changes).

2. PR2: Workflow replay/authorization split
- Extract replay and authorization from `workflow/rpc/request/handler.rs`.
- Preserve replay ordering and compatibility path behavior.

3. PR3: Auth ops decomposition
- Split `ops/auth.rs` into signing/verification/key-cache units behind existing facade.
- Preserve error mapping and verifier behavior.

## Acceptance Criteria

1. Behavior parity
- Existing root capability and delegated-auth tests pass unchanged.
- PocketIC capability flows remain green.

2. Structural outcomes
- No single control-plane module above `~1000` LOC target trend in follow-up line.
- Hotspot domain mixing reduced per module (target: `<=2` dominant domains per module).

3. Risk trend
- Complexity and velocity indices do not regress above current baseline in next recurring audit.
