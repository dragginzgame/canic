- **Scope:** implementation state, verification, and known blockers for the `0.29`

## Completed

- Added wire/data structures for `DelegationCert`, `DelegationProof`,
  `DelegatedTokenClaims`, and `DelegatedToken`.
- Added supporting wire/data structures for explicit root public keys,
  role/principal audiences, and shard key binding.
- Added `CanicAuthCanonical` hashing for delegation certificates, token
  claims, and role hashes.
- Added canonical encoding tests for strict sorted/unique vector enforcement,
  noncanonical role/scope/principal rejection, and domain separation.
- Added delegated auth audience policy helpers for shape validation, role/principal subset
  checks, verifier membership checks, and single-role hash binding.
- Added root-side certificate policy validation for version, root principal,
  certificate TTL, token TTL, role hash, shard public-key hash, and deterministic
  shard key binding.
- Added root-key trust-anchor resolution for one explicit trusted local root key.
- Added a pure delegated verifier path that validates token and certificate structure,
  cert policy, root-key trust, root and shard signatures, time bounds, audience
  membership, role hash binding, and scope attenuation without local proof
  lookup.
- Added pure root-side proof issuance with certificate construction,
  root/shard key hash binding, verifier role-hash derivation, TTL policy
  enforcement, and canonical cert-hash signing.
- Added the internal runtime root signing hook that resolves local root and
  shard threshold-ECDSA public keys, binds key-name and shard derivation-path
  hashes into `ShardKeyBinding`, signs the prepared cert hash, and returns a
  self-contained `DelegationProof` without verifier fanout.
- Added pure shard-side delegated token minting with cert-hash binding, audience and
  scope attenuation, TTL enforcement, nonce carry-through, embedded proof
  preservation, and injected shard signing.
- Added the internal runtime shard signing hook that prepares canonical delegated token
  claims, signs the prepared claims hash with the local shard threshold-ECDSA
  derivation path, and returns a self-contained `DelegatedToken`.
- Added the internal runtime verifier hook that checks configured shard key
  binding, resolves the local root trust anchor from `SubnetState`, applies
  local verifier role membership, and calls the pure delegated verifier without
  proof-cache lookup.
- Renamed the per-canister verifier-style config to
  `delegated_auth.attestation_cache` and kept the generated lifecycle cfg as
  `canic_auth_attestation_cache`, matching the attestation-cache behavior it
  actually gates.
- Added explicit `DelegationApi` helpers for local root proof issuance,
  signer token issuance from an explicit proof, and verifier validation.
- Added the explicit root `canic_request_delegation` endpoint and matching
  protocol constant for requesting self-contained `DelegationProof` values.
- Added the remote `DelegationApi::request_delegation` helper for
  signer-to-root proof requests.
- Added `DelegatedTokenMintRequest` and `DelegationApi::mint_token`,
  which request a root proof and immediately mint a self-contained delegated token
  without using local proof-cache state.
- Added local/dev delegated signer endpoints on the user-shard demo canister and
  delegation signer stub for exercising the one-call delegated auth mint path.
- Replaced delegated verifier root-key pull-on-verify with `SubnetState` root
  trust-anchor cascade. Authenticated access guards no longer fetch root key
  material; query, composite-query, and update guards verify from local state
  populated by cascaded `SubnetState`.
- Added identity-bound stable key caches for shard and role-attestation public
  keys. Cache reads now require the current configured key name and stored
  public-key hash to match before returning key material. Delegated root trust
  material now lives in cascaded `SubnetState`, not the auth key cache.
- Changed endpoint guards to current-shape delegated-token decoding. Guard validation
  uses the self-contained verifier path and does not look up local delegation
  proofs.
- Authenticated endpoint guards now require `DelegatedToken` as the first
  argument type. Candid `Reserved` placeholder acceptance has been removed.
- Changed signer lifecycle prewarm to warm local shard signing key material
  only. Startup no longer requests or stores delegation proofs.
- Tightened root-side delegated auth TTL policy ownership so `DelegationProofIssueRequest`
  carries only requested certificate TTL; root derives certificate and token TTL
  ceilings from `auth.delegated_tokens.max_ttl_secs`.
- Removed topology-time verifier proof catch-up. Canister placement and state
  propagation no longer push root-cached V1 proofs into newly created verifier
  canisters.
- Removed V1 signer issuance APIs, V1 root `IssueDelegation` routing,
  verifier proof-install endpoints, proof-fanout/admin repair DTOs, proof-cache
  stable storage, proof-cache config, proof-sync metrics, and V1-only test
  suites.

## Known Boundaries

- delegated auth is a bearer-token system. Replay protection is bounded by token TTL only.
- Verifier role configuration is trusted. A verifier with the wrong local role is
  compromised for delegated-auth purposes.
- Root public-key configuration is the delegated-auth trust anchor.
- Stable key caches are correctness-preserving caches only. Presence is not a
  trust condition; a cache hit requires configured key-name and key-hash
  identity to match. The delegated root public key is subnet state, not a cache.
- Cold authenticated queries can fail before `SubnetState` has populated the
  delegated root public key. The behavior is a clean Canic auth denial, never a
  query-time IC key-fetch execution error.
- delegated auth has no core revocation model. Certificates and tokens remain valid until
  expiry unless a future explicit revocation mechanism is designed.
- Removed the non-current Candid `Reserved` authenticated argument path,
  exported old endpoint macro names, and `__canic_load_config`.
