# 0.49 Design: CI, GitOps, And Provenance

## Status

TBD.

This is a placeholder for post-0.46 design work. It captures CI/CD, GitOps, and
supply-chain needs that should be designed after the deployment truth,
authority, execution, promotion, lifecycle, and comparison foundations exist.

---

## Tentative Scope

Potential capabilities:

- stable `--json` outputs across deployment commands;
- stable machine-readable exit codes by safety-report category;
- optional signed plans and receipts;
- CI deployment locks or leases;
- GitHub and GitLab action wrappers;
- build provenance fields on `RoleArtifact`;
- source revision and dirty-worktree recording;
- reproducible build checks;
- policy gates for safety reports;
- project manifest for split monorepos and CI roots;
- CLI/library API boundary for automation consumers.

---

---

## Boundary

0.49 should not make signed receipts a substitute for live inventory.

Receipts remain evidence. Current observed deployment state remains truth.
