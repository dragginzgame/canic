Progress note (2026-03-07):
- Added DTO primitives in `dto::capability` (`CapabilityService`, `CapabilityProof`,
  `RoleAttestationProof`, `DelegatedGrantProof`, `CapabilityRequestMetadata`) and
  concrete root wire shapes (`RootCapabilityEnvelopeV1`, `RootCapabilityResponseV1`).
- Added root endpoint/API wiring for `canic_response_capability_v1` with
  deny-by-default service/version/proof-version checks and role-attestation
  capability-hash binding enforcement.
- Updated non-root root-RPC call path to send `RootCapabilityEnvelopeV1` over
  `canic_response_capability_v1`, including role-attestation proof and bound
  capability hash.
- Envelope metadata is now validated (`issued_at`/`ttl_seconds`/clock skew), and
  replay input is projected from `(request_id, nonce)` into root replay metadata
  before dispatch.
- Root endpoint now verifies `Structural` proof mode for local topology
  predicates and verifies delegated-grant capability hash consistency before
  delegated-grant claim/signature checks.
- Added explicit envelope header validation with unit coverage for service,
  capability-version, and proof-version fail-closed dispatch.
- Root capability endpoint now uses a replay-first workflow entrypoint so the
  pilot authorization order is `proof -> replay -> policy -> execute -> persist replay`.
- Added PocketIC integration coverage for `canic_response_capability_v1`
  role-attestation flows (success + audience-mismatch + expiry failures).
- Added capability-endpoint policy-denial coverage for `IssueRoleAttestation`
  (`subject` mismatch and missing `audience`) to verify fail-closed behavior.
- Added proof-failure coverage for epoch floor rejection and tampered-signature
  rejection in role-attestation verification and capability invocation paths.
- Added fail-closed coverage for structural scope denial and role-attestation
  `capability_hash` mismatch rejections.
- Root capability request dispatch remains enum-based and exact (`Request` ->
  `RootCapabilityRequest` -> `RootCapability`) with no string/reflection routing.
- Added delegated-grant proof verification for root capability execution,
  including issuer/subject/audience/scope/time checks and domain-separated
  signature verification (`CANIC_DELEGATED_GRANT_V1`) against cached root keys.
- Updated PocketIC delegated-grant coverage to assert scope fail-closed behavior
  instead of unsupported-proof-mode rejection.
- Added structured endpoint logs and root capability metrics for envelope
  validation/rejection and proof verification/rejection outcomes.
- Updated the 0.13 design doc with implementation deltas for structural and
  delegated-grant verification behavior, replay ordering, and metric coverage.
- Removed legacy `canic_response` root endpoint wiring; root request dispatch now
  relies on proof-backed endpoints (`canic_response_attested`,
  `canic_response_capability_v1`) only.
